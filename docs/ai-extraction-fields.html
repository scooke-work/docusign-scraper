UI field types: Text · Number · Date · Currency · Dropdown  |  API data types: string · number · datetime · duration · enum · boolean. Custom fields support Text / Number / Date / Dropdown; plus custom_provisions for user-defined terms.
Enum values are the documented value sets (the pre-populated options used for filtering) from Docusign's Extracted Agreement Insights and Definitions. Other is the universal catch-all returned when the AI cannot locate the value in the agreement. The API returns equivalent UPPER_SNAKE_CASE codes (e.g. Auto-Renew→AUTO_RENEW, Yes, with conditions→YES_WITH_CONDITIONS, 30 days→THIRTY_DAYS; SILENT appears for unspecified). Value sets are English (U.S.) and vary slightly by language. Type-specific enum fields also surface for certain agreement types: NDA Type (Mutual; One Way), Employment Type (Fixed Term; Full Time; Intern; Part Time), Party Role (Buyer; Seller), Late Fees Apply (Yes; No).
⚡ Auto-calculated fields are the only ones Docusign documents as set from other fields. Sources: Agreement Manager API schema & reference; Agreement Manager support (Fields Manager, Update Agreement Status, Create an Obligation); openapi-navigator schema definitions.
