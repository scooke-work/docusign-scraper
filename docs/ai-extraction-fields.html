These dropdown values are not published in Docusign's help docs or API — captured from the product UI: Status = Active / Fulfilled / Unfulfilled / Dependent; Frequency = One Time / Recurring; Payment value = amount (number) + currency (ISO 4217). No obligations API: the Agreement Manager API exposes only getAgreement / getAgreementsList — obligations are a UI/product feature. Discounts appears as an AI-extracted attribute but is not a field in the Edit Obligation form; its type is undocumented.
Field types (standard + custom, shown in Fields Manager): Text · Number · Date · Dropdown · Currency  |  API data types: string · number · datetime · duration · enum · boolean. Creating a custom field (Add a field) offers all five types incl. Currency (verified from the live UI; the published support docs are stale and list only Text/Number/Date/Dropdown). Custom-field Category choices: General · Termination · Renewal · Payment · Legal and Compliance · Other. custom_provisions hold user-defined terms.
Enum values are the documented value sets (the pre-populated options used for filtering) from Docusign's Extracted Agreement Insights and Definitions. Other is the universal catch-all returned when the AI cannot locate the value in the agreement. The API returns equivalent UPPER_SNAKE_CASE codes (e.g. Auto-Renew→AUTO_RENEW, Yes, with conditions→YES_WITH_CONDITIONS, 30 days→THIRTY_DAYS; SILENT appears for unspecified). Value sets are English (U.S.) and vary slightly by language. Type-specific enum fields also surface for certain agreement types: NDA Type (Mutual; One Way), Employment Type (Fixed Term; Full Time; Intern; Part Time), Party Role (Buyer; Seller), Late Fees Apply (Yes; No).
