# How to create a new elastic template version

This topic demonstrates how to use the Click API to create a new version of an elastic template. You can specify whether you require users who have previously accepted an agreement created from that elastic template to accept the new version when they return to your website.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {API\_USERNAME} | A [GUID](http://guid.one/) value that identifies a specific Docusign user within a Docusign account. Copy the value from the **User ID** field revealed when you select the user on the [Users](https://admindemo.docusign.com/authenticate?goTo=users) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/clickapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {CLICKWRAP\_ID} | A [GUID](http://guid.one/) value that identifies an individual elastic template. You can obtain this value by following the steps in [How to get a list of elastic templates](https://developers.docusign.com/docs/click-api/how-to/retrieve-elastic-templates/). |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Authentication](https://developers.docusign.com/docs/click-api/click101/auth/) for guidance on your options to get an OAuth token.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid OAuth access token for {ACCESS\_TOKEN}.

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}"

'--header' "Accept: application/json"

'--header' "Content-Type: application/json")

## Step 3. Construct the request body

Next, create the request body. To update a specific elastic template to a new version using the Click API, you need to construct an elastic template body in JSON format. This is similar to constructing a JSON body when you created the original elastic template using the Click API, but requires additional parameters (see **Required data** above). The following example constructs a JSON body for a Terms of Service document constructed from a base64-encoded PDF file. **Tip**: The example base64-encoded document shown below is a generic sample that you can copy/paste into your own code. There are many free online base64 encoders/decoders, such as [FreeFormatter.com](https://www.freeformatter.com/base64-encoder.html), that you can use to encode the **documentBase64** parameter.

## Step 4. Call the Click API

Now that you’ve constructed your API headers and elastic template body, make a request to the [Clickwraps:createClickwrapVersion](https://developers.docusign.com/docs/click-api/reference/accounts/clickwraps/createclickwrapversion/) endpoint to update an elastic template in your account with a new version. Here’s how you make the call, but make sure to substitute your API account ID for {ACCOUNT\_ID} and a specific elastic template ID for {CLICKWRAP\_ID}.

**Note:** For compatibility with previous versions of the Click API, elastic templates use clickwrap endpoints.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/Click/eg003CreateNewClickwrapVersion.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/Click/Examples/CreateNewClickwrapVersion.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/click/services/CreateNewVersionClickwrapService.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/click/examples/createNewClickwrapVersion.js), [PHP](https://github.com/docusign/code-examples-php/blob/master/src/Services/Examples/Click/CreateNewClickwrapVersionService.php), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/main/examples/Click/eg003CreateNewClickwrapVersion.ps1), [Python](https://github.com/docusign/code-examples-python/blob/master/app/click/examples/eg003_create_new_clickwrap_version.py), or [Ruby](https://github.com/docusign/code-examples-ruby/blob/main/app/services/clickwrap/eg003_create_new_clickwrap_version_service.rb).
