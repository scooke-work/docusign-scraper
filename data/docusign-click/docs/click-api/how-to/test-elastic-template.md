# How to test an elastic template

Before you embed an elastic template in your website or app, you should preview it to make sure it appears and behaves the way you want. However, the web page in which you test your elastic template cannot be a local file that you open in a browser. The page must be hosted on a web server. The Docusign elastic template test page takes care of this for you, making it easy to preview the behavior and appearance of your elastic template.

## Required data

Running the code in this how-to requires this data, if you choose to open the test page directly. If you use the Elastic Template Tester, you don’t need to get this data prior to testing. See [Step 1. Open the elastic template test page](https://developers.docusign.com/docs/click-api/how-to/test-elastic-template/#step-1-open-the-elastic-template-test-page) for information about the two methods of accessing the test page.

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {CLICKWRAP\_ID} | A [GUID](http://guid.one/) value that identifies an individual elastic template. You can obtain this value by following the steps in [How to get a list of elastic templates](https://developers.docusign.com/docs/click-api/how-to/retrieve-elastic-templates/). |
| {ENVIRONMENT} | A deployment environment to which you intend to make an API request. If you want to make an API request to a production server, specify `prod`. If you’re unsure, specify `demo`. |

## Before you begin

To follow these steps, you must have an active elastic template. To learn how to create one, see [How to create an elastic template](https://developers.docusign.com/docs/click-api/how-to/create-elastic-templates/).

By default, new elastic templates are inactive, but you can only test elastic templates that have a `status` of `active`. See [How to activate an elastic template](https://developers.docusign.com/docs/click-api/how-to/activate-elastic-template/).

For compatibility with previous versions of the Click API, elastic templates use clickwrap endpoints.

## Step 1. Open the elastic template test page

To open the elastic template test page, you can either use the Elastic Template Tester or open the test page directly. Using the Elastic Template Tester is easier because it doesn’t require you to obtain your account ID and an elastic template ID to run the test.

To access the Elastic Template Tester:

1. Select **Templates** from the [Docusign web app](https://apps-d.docusign.com/send/templates/) top nav, then select **Elastic Templates**.
2. Select **Test Template** from the drop-down menu next to the elastic template you want to test.
3. On the **Test Elastic Template** window, select **TEST TEMPLATE**.

Alternatively, you can paste the URL for your elastic template test directly into your browser’s navigation bar. You will need to know your API account ID and the ID of the elastic template to test.

To open the test page directly:

1. Copy the value from the **API Account ID** field in the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page.
2. Find the elastic template ID by doing either of the following:
   - Select **Templates** from the [Docusign web app](https://apps-d.docusign.com/send/templates/) top nav, then select **Elastic Templates**. Select **COPY CODE** to display a window that includes the elastic template ID in the `clickwrapId` property.
   - Use the Click API and follow the steps in [How to get a list of elastic templates](https://developers.docusign.com/docs/click-api/how-to/retrieve-elastic-templates/).
3. Enter the following URL in your browser, substituting your API account ID for {API\_ACCOUNT\_ID}, the elastic template's ID for {CLICKWRAP\_ID}, and the environment you are using for {ENVIRONMENT}. The valid values for {ENVIRONMENT} are `demo` and `prod` (for production): `https://developers.docusign.com/click-api/test-clickwrap?a={API_ACCOUNT_ID}&cw={CLICKWRAP_ID}&eh={ENVIRONMENT}`

## Step 2. Identify the user

In the **Unique ID** text box, enter any unique value that could be used to identify a user, such as an email address or employee ID. If your elastic template includes [dynamic content fields](https://developers.docusign.com/docs/click-api/click101/customize-elastic-template-fields/), supply values for those fields in the remaining textboxes. Then select **Test**.

**Note:** If a dynamic content field receives incompatible values or no value, its entire block will not be rendered. If you are having trouble displaying dynamic content in your document, make sure that you:

- Only enter allowed values for each field type
- Provide values for all of your dynamic content fields, even if they are empty strings

Your elastic template displays, similar to this example.

Accept or decline the terms by selecting the appropriate button as well as any checkboxes that your elastic template might include. Afterwards, a confirmation page appears.

To preview the elastic template again, select **Test again.** You can also share a link to the elastic template preview with others by selecting the copy icon next to the link.

A user does not have to authenticate with Docusign to use the elastic template test page or copy the embed code. Only share the link with appropriate users within your organization.

(Optional) If you want to make changes to your elastic template, select **Templates** from the [Docusign web app](https://apps-d.docusign.com/send/templates/) top nav, select **Elastic Templates**, then select **Edit** from the drop-down menu next to the elastic template; or use the [ClickWraps:updateClickwrapVersion](https://developers.docusign.com/docs/click-api/reference/accounts/clickwraps/updateclickwrapversion/) endpoint or an equivalent SDK method.

## Step 3. Copy the code to embed

If your elastic template looks and behaves the way you expect, copy the code that you will embed in your website or app. To copy the code:

1. Select **Templates** from the [Docusign web app](https://apps-d.docusign.com/send/templates/) top nav, then select **Elastic Templates**.
2. Locate your elastic template and select **COPY CODE** to display the JavaScript code that you need to embed into your website or app.
3. Select **COPY** to copy the JavaScript code to your clipboard.
