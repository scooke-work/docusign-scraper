# How to embed an elastic signing agreement

This topic demonstrates how to embed an agreement using an elastic template with a unique agreement link and dynamic content using the [ClickWraps:createHasAgreed](https://developers.docusign.com/docs/click-api/reference/accounts/clickwraps/createhasagreed/) method.

> **Important:** You must [create](https://developers.docusign.com/docs/click-api/how-to/create-elastic-templates/), [activate](https://developers.docusign.com/docs/click-api/how-to/activate-elastic-template/), and [add dynamic content](https://developers.docusign.com/docs/click-api/click101/customize-elastic-template-fields/#add-dynamic-content-to-your-elastic-template) to a elastic template to complete this how-to.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/clickapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {CLICKWRAP\_ID} | A [GUID](http://guid.one/) value that identifies an individual elastic template. You can obtain this value by following the steps in [How to get a list of elastic templates](https://developers.docusign.com/docs/click-api/how-to/retrieve-elastic-templates/). |
| Elastic template dynamic content | Elastic template [dynamic content](https://developers.docusign.com/docs/click-api/click101/customize-elastic-template-fields/) enables you to add customized field values to each instance of a elastic template agreement. This is the set of fields that appears on a given elastic template document within the **INSERT DATA** editor. |
| {CLIENT\_USER\_ID} | A unique identifier for a user that should originate from your own system. Docusign uses this value to identify each user. Note that:  - This value can be anything your backend system will use to track individual users. Examples include employee IDs, email addresses, and surrogate key values. - The Click API uses the client user ID, API account ID, and clickwrap ID to track which users have already accepted an elastic signing agreement. - An elastic template agreement will not appear again after it has been accepted with a given client user ID. The client user ID needs to be updated to a new, unique value each time the elastic template is embedded. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Authentication](https://developers.docusign.com/docs/click-api/click101/auth/) for guidance on your options to get an OAuth token.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid OAuth access token for {ACCESS\_TOKEN}.

1

2

3

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}"

'--header' "Content-Type: application/json"

'--header' "Accept: application/json")

## Step 4. Call the Click API

Now that you have constructed your API headers and request body, make a POST request to the Click API and include the request body to populate the dynamic fields within your elastic signing agreement. Here’s how you make the call, but make sure to substitute your API account ID for {account\_id} and elastic template ID for {clickwrap\_id}.
> **Note:** For compatibility with previous versions of the Click API, elastic templates use clickwrap endpoints.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/Click/eg006EmbedClickwrap.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/Click/Examples/EmbedClickwrap.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/click/services/EmbedClickwrapService.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/click/examples/embedClickwrap.js), [PHP](https://github.com/docusign/code-examples-php/blob/master/src/Services/Examples/Click/EmbedClickwrapService.php/), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/Click/eg006EmbedClickwrap.ps1), [Python](https://github.com/docusign/code-examples-python/blob/master/app/click/examples/eg006_embed_clickwrap.py), or [Ruby](https://github.com/docusign/code-examples-ruby/blob/master/app/services/clickwrap/eg006_embed_clickwrap_service.rb).

## Step 5. Write your HTML for embedding the elastic template

Write the code you need to embed your elastic template on your web page. This includes three HTML tags:

- A `<div>` that identifies where on your web page the elastic template should render
- Two `<script>` tags:
  - One that calls the Docusign elastic template JavaScript library
  - Another that calls the Docusign `Clickwrap.render` method from the JavaScript library
    - Paste the `agreementUrl` that is generated in the prior step

1

2

3

4

5

6

7

8

9

<div id="ds-terms-of-service"></div>

<script src="https://demo.docusign.net/clickapi/sdk/

latest/docusign-click.js"></script>

<script>docuSignClick.Clickwrap.render({

agreementUrl: "https://demo.docusign.net/clickapi/

v1/redeem?agreementToken=AcTZT8…P5gvB0",

onAgreed: function() {

// Triggered if the user has just agreed

}

}, "#ds-terms-of-service");

</script>
