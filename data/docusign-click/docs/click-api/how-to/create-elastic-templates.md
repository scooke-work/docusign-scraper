# How to create an elastic template

This topic describes how to use the Click API to create an elastic template that you can embed in your website or app.

To learn how to generate a elastic template by using the UI, see [Create an Elastic Template](https://support.docusign.com/s/document-item?bundleId=igo1666647424726&topicId=vxi1572482718750.html) in the user guide.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/clickapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Authentication](https://developers.docusign.com/docs/click-api/click101/auth/) for guidance on your options to get an OAuth token.

## Step 4. Call the Click API

Now that you have constructed your API headers and elastic template body, make a request to the [ClickWraps:createClickwrap](https://developers.docusign.com/docs/click-api/reference/accounts/clickwraps/createclickwrap/) endpoint to create a new elastic template for your account. Make sure to substitute your API account ID for {ACCOUNT\_ID}.

**Note:** For compatibility with previous versions of the Click API, elastic templates use clickwrap endpoints.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/Click/eg001CreateClickwraps.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/Click/Examples/CreateClickwrap.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/click/examples/C001ControllerCreateClickwrap.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/click/examples/createClickwrap.js), [PHP](https://github.com/docusign/code-examples-php/blob/master/src/Services/Examples/Click/CreateClickwrapService.php), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/main/examples/Click/eg001CreateClickwrap.ps1), [Python](https://github.com/docusign/code-examples-python/blob/master/app/click/examples/eg001_create_clickwrap.py), or [Ruby](https://github.com/docusign/code-examples-ruby/blob/master/app/services/clickwrap/eg001_create_clickwrap_service.rb).

## Step 5. Add dynamic content (optional)

Once you have created your elastic template with the API, you can optionally use the **Agreement Cloud Editor** to add [dynamic content](https://developers.docusign.com/docs/click-api/click101/customize-elastic-template-fields/) to your document. Dynamic content enables you to add customized field values to each instance of an elastic signing agreement. Dynamic content cannot be created using the API.

To add dynamic content to your elastic template:

1. Open the[Templates](https://apps-d.docusign.com/send/templates) panel in the Docusign web app.
2. Select the **Elastic Template** tab.
3. Open the menu to the right of your chosen elastic template's **COPY CODE** button, then choose **Edit**.
4. Select **Edit** right to **Documents** to edit the document.
5. Choose **Edit** to open the **Agreement Cloud Editor**.
6. Drag data placeholders onto your document from the **Insert Data** menu on the left. The placeholders will be filled with data that you specify when embedding the elastic template in your site. For more information, see [How to embed an elastic signing agreement](https://developers.docusign.com/docs/click-api/how-to/embed-elastic-templates/).

The image shows a document opened in the **Agreement Cloud Editor**. The document has five data placeholders (highlighted in blue) to represent the dynamic content. For more information on how to test and pass in values for dynamic content, see [Customize elastic template fields](https://developers.docusign.com/docs/click-api/click101/customize-elastic-template-fields/).
