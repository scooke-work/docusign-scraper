# How to grant office access to a form group

Docusign Rooms for Real Estate only

This topic demonstrates how to assign an office to a [form group](https://developers.docusign.com/docs/rooms-api/rooms101/forms/managing-forms-rooms/) for your Docusign Rooms for Real Estate account. As a prerequisite, make sure that you have [created a form group](https://developers.docusign.com/docs/rooms-api/how-to/create-form-group/). Assigning an office to a form group associates that group with that office, granting the group access to the office. Granting office access to a form group will enable you to filter which form groups are available based on that office.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi/v2  Where <domain> is demo.rooms.docusign.com for the developer environment and rooms.docusign.com for production. |
| {FORM\_GROUP\_ID} | A [GUID](http://guid.one/) value that identifies your form group. This value is created by Docusign when you [create a form group](https://developers.docusign.com/docs/rooms-api/how-to/create-form-group/). |
| Forms enabled on your Rooms account | The user role needs the **Manage form groups** permission assigned. These permissions are only available to accounts with Forms enabled. To enable this feature, contact your Account Manager or Business Development representative. |
| {OFFICE\_ID} | An identifier for the office location that is responsible for the room itself. Retrieve this value with the [Offices:GetOffices](https://developers.docusign.com/docs/rooms-api/reference/offices/offices/getoffices/)  method. |
| Rooms enabled for your account | You will need access enabled to use the Rooms API. See [creating a Rooms developer account](https://developers.docusign.com/docs/rooms-api/rooms101/create-account/). |

## Step 1. Obtain your OAuth token

To make the API calls shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Authentication](https://developers.docusign.com/docs/rooms-api/rooms101/auth/) for guidance on your options to get an OAuth token.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid OAuth access token for {ACCESS\_TOKEN}.

1

2

3

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}"

'--header' "Accept: application/json"

'--header' "Content-Type: application/json")

Next, make a GET request to your Rooms account by using the [Offices:GetOffices](https://developers.docusign.com/docs/rooms-api/reference/offices/offices/getoffices/) method or supporting SDK method to get your office ID. Here’s how you make the call, but make sure to substitute your base path for {BASE\_PATH} and your API account ID for {API\_ACCOUNT\_ID}. Select one of the returned office IDs to use in the next step.

printf \

'

{

"name": "Sample Room Form Group",

}' >$request\_data

## Step 4. Select a form group ID

To find the form group that you want to assign the office to, you need to make a GET request to obtain an available [form group ID](https://developers.docusign.com/docs/rooms-api/reference/forms/formgroups/getformgroups/). The Rooms API will return a list of available form group IDs; select the one you want to continue through the example. Here’s how you make the call but make sure to substitute your API account ID for {API\_ACCOUNT\_ID}.

1

2

3

4

Status=$(curl -w '%{http\_code}' --request POST $

{base\_path}/restapi/v2/accounts/${API\_ACCOUNT\_ID}/

form\_groups \

"${Headers[@]}" \

--data-binary @${request\_data} \

--output ${response})

## Step 5. Call the Rooms API

Now that you have constructed your API headers, make a POST request to the [FormGroups:GrantOfficeAccessToFormGroup](https://developers.docusign.com/docs/rooms-api/reference/forms/formgroups/grantofficeaccesstoformgroup/) method to grant office access to your form group for your Rooms account. Here’s how you make the call, but make sure to substitute your base path for {BASE\_PATH}, API account ID for {API\_ACCOUNT\_ID}, form group ID for {FORM\_GROUP\_ID}, and office ID for {OFFICE\_ID}.

**Note:** This Rooms API call does not require a request body, but still needs to be sent as a POST request.

## Expected response

You will receive a 204 Status code response with an empty response body.
