# How to create a room with a template

This topic demonstrates how to create a Docusign room using a predefined template. If you've created a template for your organization in either Rooms for Mortgage or Rooms for Real Estate by following the steps in [Using room templates and task lists](https://developers.docusign.com/docs/rooms-api/rooms101/templates/), you can create rooms based on this template using the Rooms API.

To complete this how-to, the Rooms API must be enabled for your account. For more information, see [creating a Rooms developer account](https://developers.docusign.com/docs/rooms-api/rooms101/create-account/).

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi/v2  Where <domain> is demo.rooms.docusign.com for the developer environment and rooms.docusign.com for production. |
| {ROLE\_ID} | A room-specific ID that can be allocated to a given user within a room. The role ID is necessary to set permissions on rooms for individuals within your Docusign organization. Retrieve this value with the [Roles:GetRoles](https://developers.docusign.com/docs/rooms-api/reference/roles/roles/getroles/) method. |
| {TEMPLATE\_ID} | A [GUID](http://guid.one/) value that identifies a Docusign [template](https://developers.docusign.com/docs/esign-rest-api/esign101/concepts/templates/) that is used to create a Docusign [room](https://developers.docusign.com/docs/rooms-api/rooms101/concepts#rooms). This value is automatically generated by Docusign when the template is created. |

## Step 1. Obtain your OAuth token

To make the API calls shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Authentication](https://developers.docusign.com/docs/rooms-api/rooms101/auth/) for guidance on your options to get an OAuth token.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid OAuth access token for {ACCESS\_TOKEN}.

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}"

'--header' "Accept: application/json"

'--header' "Content-Type: application/json")

## Step 4. Construct the request body

Next, create the request body. To create a room, you need to specify some room attributes.

| Attribute | Description |
| --- | --- |
| `name` | A string value that identifies the room. |
| `roleId` | A string value that specifies which group of users can control the room. |
| `transactionSideId` | A string value that defines the type of transaction that is occurring (such as `buy`, `list`, or `dual`). You can update this up until the transaction is completed. |
