# Rooms API 101

*Docusign Rooms* simplifies complex agreements by bringing together each person involved with a transaction in a digital space called a *room* where every detail, task, and transaction document is centrally located and managed.

In Docusign Rooms, you set the requirements for each party by assigning *roles* to the people involved in the transaction. Roles can be assigned tasks to complete (typically viewing, uploading, or signing documents) and can have different levels of access to the room’s data and functionality; for example, a seller or agent would typically have greater visibility and access than a buyer. The room records the complete history of each action taken within the room and, when the transaction is complete, archives this information for your records.

Each room is integrated with [Docusign eSignature functionality](https://developers.docusign.com/docs/esign-rest-api/esign101/) and can be created easily and simply using  [templated task information](https://developers.docusign.com/docs/rooms-api/rooms101/templates/). Different templates, task lists, and roles can be created for different regions and offices within your company, enabling you to comply easily with different regional transaction processes and requirements.

## Rooms for Real Estate

Docusign Rooms for Real Estate is built on the Rooms API, which enables you to perform some actions programmatically within your own apps, such as importing data to create a room. See [How to create a room with data](https://developers.docusign.com/docs/rooms-api/how-to/create-room-with-data/) for details.

**Note:** A given Docusign account may be linked with a Rooms for Real Estate account.

The advantages of using Rooms to manage real estate transactions include:

- Standardizing your procedures by creating templates and task lists that you can use over and over.
- Defining simple, clear responsibilities for each room participant by assigning lists of specific tasks.
- Integrating with Docusign eSignature functionality.
- Controlling company standards and real estate procedures across many different regions and offices.
- Manage your user permissions and responsibilities with custom roles and permission sets.

## Rooms API vs UI functionality

The Rooms API enables you to perform most Rooms actions programmatically, rather than manually through the UI. This is useful for automating and integrating Rooms functionality within your own apps and reducing data entry.

- Use the Rooms API for automated tasks and data entry (such as importing or exporting data to or from a room).
- Use the UI for management tasks, administration scenarios, and to perform actions that are not yet supported by the Rooms API.

**Note:** The Rooms API will be expanded in upcoming releases to support additional functionality, including management and administration tasks.
