# Task Components in Docusign Rooms

## Details

On the task DETAILS tab, add the task name, due date, description,
and choose settings.

![rooms-task-details](https://docusign-be-prod.zoominsoftware.io/api/bundle/clr1643042075783/page/images/roomstaskdetails.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjE0NDQsInNoZWFmIjoiY2xyMTY0MzA0MjA3NTc4MyJ9.CNmSTSqK6eV33VXKDqnNdO_k-sVL83n6adjNDDbbtXo&_LANG=enus)

| Field | Description |
| --- | --- |
| Task name | Enter a task name. |
| Due date | Optionally select a due date. The dates in the drop-down list are specific dates configured in the room's Details tab. For example, if you choose the transaction close date from the list, the task looks to the room's close date on the Details tab to determine the due date.  If you select a due date, a numerical field displays. This sets the number of days from the due date that the task must be completed. For example, if the due date is closing day on 1/15 and the date offset is -2 days, the task is due on 1/13. |
| Description | Enter a task description. If the task doesn't require a document, use the description to communicate what needs to be done. |
| Required Task | Select if the task must be completed to submit the task list and close the room. Required tasks must be reviewed. If you select Required Task, the Requires Review check box is selected and disabled. |
| Requires Review | Select if the reviewer(s) must approve the task. If selected, you need to choose at least one reviewer on the PEOPLE tab. Note: Optional tasks can require review. For example, a task to upload a document might require review, but if the document doesn't apply to the transaction, the member can skip it. |
| Task can be edited | Select to allow edits to the task. To edit the task, members must be on the task (as a watcher, assignee, or reviewer) and have a room role with permission to edit tasks. |
| Task can be deleted | Select to allow members to delete the task. To delete the task, members must be on the task (as a watcher, assignee, or reviewer) and have a room role with permission to delete tasks. |

## Documents

If the task requires someone to complete a document, type the document name in the
DOCUMENTS tab.

![rooms-task-document](https://docusign-be-prod.zoominsoftware.io/api/bundle/clr1643042075783/page/images/roomstaskdocument.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjE0NDQsInNoZWFmIjoiY2xyMTY0MzA0MjA3NTc4MyJ9.CNmSTSqK6eV33VXKDqnNdO_k-sVL83n6adjNDDbbtXo&_LANG=enus)

Adding a document name creates a placeholder on the DOCUMENTS tab,
so users know which document to provide.

![rooms-task-placeholder-appraisal](https://docusign-be-prod.zoominsoftware.io/api/bundle/clr1643042075783/page/images/roomstaskplaceholderappraisal.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjE0NDQsInNoZWFmIjoiY2xyMTY0MzA0MjA3NTc4MyJ9.CNmSTSqK6eV33VXKDqnNdO_k-sVL83n6adjNDDbbtXo&_LANG=enus)

Users can apply more than one document to a placeholder. Therefore, you can create one task
to upload multiple related documents.

## Responsibilities

Task responsibilities are: assignee, reviewer, and watcher. Assign role(s) to each of these
responsibilities. You can add multiple roles with the same responsibility if members with
any of those roles own that responsibility.

Select ADD under each responsibility (ADD ASSIGNEE/ADD REVIEWER/ADD
WATCHER) to assign roles. See the table below for an overview of task responsibilities.

Note: The person's room role also determines what they can do
with tasks. For example, a member with any of these responsibilities can edit the task if
their room role has Edit tasks permission (for editable tasks) or
Edit any task permission (for all tasks).

| Responsibility | Description |
| --- | --- |
| Assignee | Responsible for completing the task.  The assignee receives notifications when they are initially added to a task, task reminders, due dates, and action is taken on the task. |
| Reviewer | Responsible for approving or declining the task.  If you designate a reviewer, the task is set to Requires Review. Reviewers in the first review level receive a notification when the assignee completes the task.  Only the reviewer roles you specify on the task can review it. If some roles in your company should review any task, give those roles the Review any task permission.  To create an approval chain, where one level must approve the task before it moves to the next level, add review levels. |
| Watcher | Receives notifications when the task is completed or approved. |

Tip: You can assign a role as both the assignee and
the reviewer. For example, you might want a transaction coordinator to complete a task on
behalf of an agent, and be the reviewer on the task.

## Review levels

You can add review levels to a task. Review levels are beneficial if your company needs to
approve tasks or documents through an approval chain. For example, when a member uploads a
document, you might want an office manager to approve the document first, then send it to
legal.

![rooms-task-review-levels](https://docusign-be-prod.zoominsoftware.io/api/bundle/clr1643042075783/page/images/roomstaskreviewlevels.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjE0NDQsInNoZWFmIjoiY2xyMTY0MzA0MjA3NTc4MyJ9.CNmSTSqK6eV33VXKDqnNdO_k-sVL83n6adjNDDbbtXo&_LANG=enus)

In the Reviewer(s) section, select ADD REVIEW LEVEL to add up to
five review levels. At each level, someone with that room role must approve the task. Adding
multiple roles allows people with any of those roles to approve. A reviewer in level 1 must
approve the task before it can move to level 2, and so on.

When a task has multiple review levels, the current review level displays in the status
label (for example, In Review (level 2 of 2)).

![tasks-review-levels-2](https://docusign-be-prod.zoominsoftware.io/api/bundle/clr1643042075783/page/images/tasksreviewlevels2.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjE0NDQsInNoZWFmIjoiY2xyMTY0MzA0MjA3NTc4MyJ9.CNmSTSqK6eV33VXKDqnNdO_k-sVL83n6adjNDDbbtXo&_LANG=enus)

Reviewers see the current review level of the task in the Status
section.

![rooms-task-review-full-pane-status](https://docusign-be-prod.zoominsoftware.io/api/bundle/clr1643042075783/page/images/roomstaskreviewfullpanestatus.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjE0NDQsInNoZWFmIjoiY2xyMTY0MzA0MjA3NTc4MyJ9.CNmSTSqK6eV33VXKDqnNdO_k-sVL83n6adjNDDbbtXo&_LANG=enus)

When a reviewer declines the task, they choose whether to send it back to the assignee or a
previous review level. When a reviewer approves the task, the task is sent to the next
review level, or it is marked Approved if it is the last review level.

## Reminders

Select ADD REMINDER  in the task REMINDERS
tab to add email reminders for the assignee(s).

![rooms-task-reminders](https://docusign-be-prod.zoominsoftware.io/api/bundle/clr1643042075783/page/images/roomstaskreminders.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjE0NDQsInNoZWFmIjoiY2xyMTY0MzA0MjA3NTc4MyJ9.CNmSTSqK6eV33VXKDqnNdO_k-sVL83n6adjNDDbbtXo&_LANG=enus)

For each reminder, select the date the reminder is associated with. The Actual
Due Date is the due date that is set up on the task. Other dates in the
drop-down list are transaction milestone dates in the room Details tab.

A numerical field displays next to each reminder date. Leave this value at
0 to send the reminder on that date. Otherwise, use the arrows to
set the number of days from the due date that the reminder should be sent. For example,
-3  sends a reminder three days before the date, and
+3 sends a reminder three days after the date.
