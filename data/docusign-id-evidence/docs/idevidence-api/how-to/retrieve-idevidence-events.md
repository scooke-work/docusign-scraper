# How to retrieve ID Evidence events

This topic demonstrates how to retrieve ID Evidence event data in JSON format.

## Prerequisites

- If you do not see a sidebar link named **Identity Verification** under eSignature Settings in your Docusign account, you do not have ID Verification (IDV) enabled. See [ID verification](https://developers.docusign.com/docs/esign-rest-api/esign101/concepts/recipients/auth/#idv) for information on enabling IDV in your account.
- You must [enable ID Evidence storage](https://developers.docusign.com/docs/idevidence-api/idevidence101/enabling-idevidence-storage/).
- In your Docusign account, you must have a completed IDV envelope: you must have [sent an IDV envelope](https://developers.docusign.com/docs/esign-rest-api/how-to/id-verification/), the recipient must have [completed the IDV process](https://support.docusign.com/s/document-item?bundleId=ced1643229641057&topicId=iww1578456547699.html), and they must have signed the envelope.
- In your locally downloaded Quickstart or launcher `/``config` folder, you must save your IDV envelope ID GUID in a file named `IDV_ENVELOPE_ID`.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [PowerShell](https://github.com/docusign/code-examples-powershell).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {IDV\_ENVELOPE\_ID} | A [GUID](http://guid.one/) value that identifies the [Docusign envelope completed using ID Verification](https://developers.docusign.com/docs/esign-rest-api/how-to/id-verification/). |
| {PROOF\_BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/api  Where <domain> is proof-d.docusign.net for the developer environment and {server}.proof.docusign.net for production. |
| {RECIPIENT\_ID\_GUID} | A [GUID](http://guid.one/) value that Docusign assigns to identify each recipient in an envelope. This value is globally unique for all recipients, not just those in your account. |

## Step 1. Obtain your OAuth token

To make the API calls shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

## Step 2. Retrieve your recipient ID GUID

To obtain your resource token in the next step, you must provide your recipient ID GUID in the request body. Call the [EnvelopeRecipients:list](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/enveloperecipients/list/) endpoint with your {API\_ACCOUNT\_ID} and {IDV\_ENVELOPE\_ID} to retrieve your recipient ID GUID.

## Step 3. Obtain your resource token

To call the ID Evidence API, you need a valid [resource token](https://developers.docusign.com/docs/idevidence-api/idevidence101/resource-tokens/). To obtain your resource token, call the [EnvelopeRecipients:createRecipientProofFileResourceToken](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/enveloperecipients/createrecipientprooffileresourcetoken/) endpoint with your {API\_ACCOUNT\_ID}, {IDV\_ENVELOPE\_ID}, and {RECIPIENT\_ID\_GUID}.

${account\_id}/envelopes/${idv\_envelope\_id}/

recipients/${recipientIdGuid}/identity\_proof\_token"

"${Headers[@]}" \

--output ${response})

## Step 4. Construct your API headers

After you have a valid resource token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid resource token for {RESOURCE\_TOKEN}.

1

2

3

declare -a Headers=('--header' "Authorization:

Bearer ${resourceToken}" \

'--header' "Accept: application/json, text/

plain, \*/\*" \

'--header' "Content-Type: application/json;

charset=UTF-8")

## Step 5. Call the ID Evidence API to retrieve event data

Now that you have constructed your API headers, call the [Events:list](https://developers.docusign.com/docs/idevidence-api/reference/idevidence/events/list/) endpoint with your {PROOF\_BASE\_PATH} and [RECIPIENT\_ID\_GUID} to retrieve ID Evidence event data for a recipient.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/IdEvidence/eg001RetrieveEvents.sh), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/ID_Evidence/retrieveEvents.ps1).
