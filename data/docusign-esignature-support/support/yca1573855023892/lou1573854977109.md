# Docusign eSignature Signing Process

Learn about the Docusign eSignature signing process and eSignature signing terminology.

Here is an overview of the basic Docusign signing process with links to more detailed topics.

For signing device requirements, see the [System Requirements for Signing](https://support.docusign.com/s/document-item?bundleId=yca1573855023892&topicId=thx1573854978578.html&_LANG=enus "Review these minimum signing system requirements. Docusign signing is also compatible with assistive technology, as
described in the [Docusign Accessibility Hub](https://www.docusign.com/accessibility?postSignIn=true) article.

## Signing overview

The signing process begins when you receive an email notification from Docusign informing you that you have a document to sign. The document is contained in a Docusign envelope. Docusign sends the email notification on behalf of the Sender. The notification contains a message from the Sender and a link to open the documents. Alternately, you can access the documents by going to the Docusign website and entering the document security code from the email.

Note: If the email message has a small shield and the text Part 11 Module Enabled, there are special requirements for signing these documents. See the [Sign Part 11 Module Enabled Documents](https://support.docusign.com/s/document-item?bundleId=yca1573855023892&topicId=yzu1573855023673.html&_LANG=enus "Read about special requirements, including U.S. Food and Drug Administration (FDA) regulations, for signing Part 11 Module-enabled documents.") article for more information.

To begin signing, select CONTINUE.

Note: The sender may optionally include an Electronic Record and Signature disclosure. Selecting this option indicates that you agree to the disclosure.

## Docusign signing terminology

Email notification
:   The means by which signers are informed that they have an electronic document to sign. The email notification includes a link that takes the signer directly to the document..

Envelope
:   The container that holds one or more documents. The envelope also holds the recipient information and the fields placed on the document.

Sender
:   Persons who can create, send, and manage Docusign envelopes.

Signer
:   An envelope recipient who must act on an envelope or documents within an envelope.

Fields
:   Indicate where a Signer must act on the document. For example, completing data, selecting, or applying a signature.

## Signer authentication

Senders can ask you to verify your identity before signing. This option is set by the person or business that is sending the documents.

Identity authentication methods include:

Phone
:   You answer a phone call and receive a verbal authentication code. You enter the code to view your documents.

SMS
:   You receive an SMS text message at a specified phone number. The text message contains an access code that you must enter to view your documents.

Knowledge-based ID check
:   You answer detailed questions about yourself. The questions are based on data available in public records (such as current and former addresses).
