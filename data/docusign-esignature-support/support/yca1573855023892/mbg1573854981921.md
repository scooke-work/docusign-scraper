# Verify Your Identity

Learn about using the different authentication methods to verify your identity for
Docusign eSignature.

When signing a document, senders might request information to verify who you are.

Senders ask you to verify your identity through:

- Access code
- SMS Authentication
- Phone authentication
- Knowledge-based authentication
- ID Verification
- Account Logon: Only for Part 11 Module Enabled documents

Note: If you change signing responsibility as described in
the [Change Signing Responsibility](https://support.docusign.com/s/document-item?bundleId=yca1573855023892&topicId=uin1573855010064.html&_LANG=enus "Learn how to assign a document to another person if you're not the right person to sign the document.") article, SMS and phone authentication are not
supported for the new signer.

## Access code

The person sending the envelope sends you an access code. Enter the access code
before viewing the documents. This option can be combined with other authentication
options.

1. Enter the code and select Validate to complete the
   security request. You have three attempts to enter the code correctly. After
   entering the access code, you are taken through the normal signing process.
2. Select I never received an access code if you do not
   know the access code . Doing so notifies the sender to request the code.

## SMS Authentication

You receive a security request page and must select a phone number for text
authentication. Select Send SMS to continue or select
Cancel to postpone the text message and return later.

1. Select Send SMS to send an SMS text message with a
   passcode to the selected phone number.
2. Enter the passcode and select Confirm Code to complete
   the security request. After the passcode is confirmed, you are taken through the
   normal signing process.

## Phone authentication

You receive a security request page and must answer a phone call at a number you
provide to receive an authentication code that you enter on page. Once you
successfully pass the authentication challenge, you can access and sign your
documents as usual.

1. Select Send Code.
2. The Voice Authentication dialog box updates to show the call is in process and
   provides a place for you to enter the verification code.
3. Once the call connects, you hear the authentication code. Enter the code in the
   Voice Authentication dialog box and click Confirm Code.
4. When the authentication is successful, you continue with the normal signing
   process.

## Knowledge-based authentication

You are prompted to enter some personal information and then you are presented with
questions about yourself based on data available in public records.

1. After entering the information, select Next to
   continue.
2. You are presented with a set of questions based on publicly available information. Answer the
   questions to complete the ID verification. After the ID verification is
   successfully completed, you are taken through the normal signing process.

## ID verification

Select a country and choose one of the available options for the ID to use to verify
your identity.

Most countries accept a passport or photo ID, but also may allow other options, such
as Bank ID, eID, or smartcards.

1. Select Next at the Let's Verify your
   Identity prompt.
2. Select the country.
3. Select an available ID option.
4. Follow the prompts for the ID you selected to complete the authorization.
5. When the verification is complete, you are taken through the normal signing
   process.

If the list of supported countries does not include your country, follow these steps:

1. At the Let's Verify your Identity prompt, select any
   country from the list.
2. Select an available ID option.
3. Follow the prompts for the ID you selected to complete the authorization.
4. The ID verification fails, and you receive a notification that the sender must
   manually review the ID you submitted.

   The sender sends you a notification when
   the ID is approved.
5. Upon ID approval, you can resume and complete the signing.

## Account logon-based authentication for Part 11 Module Enabled documents

If you receive an email that has the notation Part 11 Module Enabled, there are some
special requirements. You might need to authenticate a few times. How and where you
authenticate depend on what the sender required.

For added security, senders might require account logon authentication for Part 11
Module Enabled documents.

![Screenshot of Part 11 Module enabled document notation.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yca1573855023892/page/images/signpart11.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjQzNjgsInNoZWFmIjoieWNhMTU3Mzg1NTAyMzg5MiJ9.tlPP0aynA9HkLzdc2W4KsfeCd6lhefoVWpjuBSVDGH0&_LANG=enus "Screenshot of Part 11 Module enabled document notation.")

You may need to authenticate through:

- Account logon: You enter the username and password for your Docusign account to
  access and sign the documents. If you don’t have an account, you must first
  create an account and then enter your username and password.
- Account logon and another method: You authenticate through Account logon and one
  of the other identity verification methods (Access Code, SMS Authentication,
  Phone Authentication or ID Check).

With Account logon, you have a limited number of chances to correctly enter
your password. If you don’t enter the correct password, you are locked out of
Docusign and can’t finish signing.

For detailed information, see the [Sign Part 11 Module Enabled Documents](https://support.docusign.com/s/document-item?bundleId=yca1573855023892&topicId=yzu1573855023673.html&_LANG=enus "Read about special requirements, including U.S. Food and Drug Administration (FDA) regulations, for signing Part 11 Module-enabled documents.")
article.
