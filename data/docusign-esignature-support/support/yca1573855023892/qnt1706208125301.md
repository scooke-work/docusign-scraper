# Data Verification Issues for Signing

Learn about issues that can occur when signing an agreement with Docusign Data Verification fields.

Here are some issues that can occur when signing agreements with Data Verification fields.

| Issue | Context | Solution |
| --- | --- | --- |
| You entered information in a Data Verification field, but you're not prompted to verify the information. | You may have quickly moved through multiple fields and the system is catching up. | Refresh the page.  Select Finish Later from the Finish menu [v] and try again. |
| You're confident that the information in the field is correct, but verification attempts fail. | The databases for the verification providers may have limitations. Or there may be formatting errors in your input. | Check the format of the data that you entered to ensure it meets field input requirements. For example:   - The phone number must start with a country code. - The email address must include proper email syntax, with an   email prefix and an email domain. |
| You're confident that the information in the field is incorrect, but verification is passed. | There may be verification provider's database limitations. | To learn more about the reason for each verification result, contact the sender. During Beta, sending accounts must contact [Docusign Support](https://support.docusign.com/s/contactSupport) for more information than is available in the envelope history. |
| A `Verification Failed` message displays with no explanation | Verification results may not be compatible with the signing process. | Ask the sender to void and resend the envelope. |
| You cannot finish signing the agreement because a Data Verification field failed validation. | The sender blocks signers from finishing the envelope unless they pass verification. | Ask the sender to void or resend the envelope with verification turned off. |
