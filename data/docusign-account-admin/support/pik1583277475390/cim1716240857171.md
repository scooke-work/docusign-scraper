# User Invitation and Activation Process

Read about how administrators initiate the user invitation process and the steps users
must complete to activate their membership and join an account.

Adding a user to an account requires actions from an administrator and the new user. As an
administrator, you can add a user and send an invitation to their email address. To
complete the process, the user must complete the activation process.

Optionally, you can include an access code as part of the activation process. The activation
process also has an option for the user to decline the invitation.

The invitation and activation process includes these steps:

1. Log in to your Docusign account and open the Users
   page.
2. Create a user by adding their email address, profile information, and optional access code.
   An invitation is sent to the email address of the new user. ![The screenshot shows the optional access code field when you are creating a user.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/Add-user.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjUxMjQsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.r6ECj0fRrkG6cePO_hjpeEWmSaOd9-ApxWii44BFQUM&_LANG=enus "The screenshot shows the optional access code field when you are creating a user.")
3. The user receives an activation email and selects the Activate button.
   ![The screenshot shows an example of the user activation email.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/User-activation-email.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjUxMjQsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.r6ECj0fRrkG6cePO_hjpeEWmSaOd9-ApxWii44BFQUM&_LANG=enus "The screenshot shows an example of the user activation email.")

   The Create Account or Decline Invitation page opens. ![The screenshot shows the Create Account or Decline Invitation page.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/User-accept.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjUxMjQsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.r6ECj0fRrkG6cePO_hjpeEWmSaOd9-ApxWii44BFQUM&_LANG=enus "The screenshot shows the Create Account or Decline Invitation page.")
4. If the user selects Create Account and the administrator has required
   an access code, the user views the Access Code page. If an access code is not
   required, the user is redirected to the Create Your Password page. If the user
   declines the invitation, the invite is invalidated and a message page informs the
   user that the invitation is no longer valid. ![The screenshot shows an example of the Access Code page.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/user-access-code.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjUxMjQsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.r6ECj0fRrkG6cePO_hjpeEWmSaOd9-ApxWii44BFQUM&_LANG=enus "The screenshot shows an example of the Access Code page.")
5. If the user enters the access code correctly, they are redirected to the Create Your Password
   page. If the user does not enter the access code correctly, the invitation is
   invalidated. If you want to re-invite the user, you can resend an invitation on the
   Users page. ![The screenshot shows an example of the Create Your Password page.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/User-password.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjUxMjQsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.r6ECj0fRrkG6cePO_hjpeEWmSaOd9-ApxWii44BFQUM&_LANG=enus "The screenshot shows an example of the Create Your Password page.")
6. After the user sets a password, the user is activated and added to the account. The user is
   logged in to the account and views the Docusign account Home page.
