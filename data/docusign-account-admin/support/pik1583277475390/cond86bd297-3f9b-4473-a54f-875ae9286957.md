# Responsive Signing Optimize View Setting for Administrators

## What is the difference between AI-assisted mode and Basic mode for the mobile responsive Optimize View setting?

When using mobile responsive signing with Optimize View for User's Device selected as the default view, Docusign automatically determines whether a recipient should open a document in the mobile-friendly (HTML) view or the desktop (PDF) view on a mobile device. This decision is made to ensure the best possible reading experience and is powered by one of two modes: AI-assisted mode (default for US and EU accounts) or Basic mode (non-AI, default for Canada, Japan, Australia, and USFed accounts).

The key difference between these modes is how the decision is made. AI-assisted mode evaluates the actual mobile rendering (the HTML output) to determine readability and formatting quality, providing a more accurate and user-friendly experience. In contrast, Basic mode analyzes only the original document's structure and complexity, such as layout, tables, and formatting, to predict whether it will render well on mobile, without evaluating the final HTML output. In other words, AI-assisted mode makes decisions based on the final user experience, while Basic mode relies only on characteristics of the source document.

For more information on mobile responsive signing, see the [Responsive Signing](https://support.docusign.com/s/document-item?bundleId=gbo1643332197980&topicId=cbv1578456473989.html&_LANG=enus) topic.

## Is responsive signing AI-assisted? What options are available to opt out?

Responsive signing is AI-assisted in all regions except for Canada, Japan, Australia, and USFed accounts. To disable AI-assisted mode and switch to the Basic mode of responsive signing, administrators can disable Enable AI tools in signing for sent agreements setting on the Signing Settings page.
