# Add Identity Verification Configurations

Learn how account administrators add configurations for identity verification.

## Before you begin

Japanese customers, note that you can set up this configuration to store the back of IDs, including Japan's National Identity Card (My Number Card). If you implement this setup, ID Evidence will store the 12-digit individual number, which is personally identifiable information (PII). Instead, you may use the Docusign ID Verification with Japanese IDs configuration. This ID Verification method respects data privacy by only allowing storage of the front of Japanese Individual Number Cards.

As an account administrator, you can add a configuration to accounts to verify the identity of recipients. By default, each account must have at least one configuration for identity verification. You can add configurations as needed.

1. From the Accounts view, select Identity Verification.

   The Identity Verification screen opens.
2. Select Add Configuration.

   The Add Configuration screen opens.
3. Choose an Identity Verification method and select Next.

   Note: The default verification method is ID Verification. To add other methods, contact Docusign customer support.

   The Add ID Verification page opens.![The screenshot shows the ID Verification configuration screen](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/idverification.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjU3MDcsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.ebFOZuGw5M5xc2Ph3BFIUx-O8FNBwwamTnvq5EZxsms&_LANG=enus "The screenshot shows the ID Verification configuration screen")
4. Enter a name for your configuration. Optionally, add a description.

   Note: The name and description are not translated. Senders see the text in the language it is written.
5. Select the maximum number of verification attempts. This number represents how many times recipients can attempt to verify before the envelope is locked. If recipients are unsuccessful in verifying their identity, by default, the envelope is locked and users must contact the sender to unlock it.
6. Select one of the levels of variation for name acceptance standards when matching the name of the recipient with the name in the ID:

   - Maximum variation: Allows additional variations of middle names, suffixes, equivalent names and tolerates minor typing mistakes. This is the default selection. The Maximum variation option is not compatible with the IDV - DS EU Advanced signature type.
   - Moderate variation: Allows additional variations of middle names and suffixes.
   - Minimal variation: Allows minor variations of first and last names only.

   Note: For more information about this setting, see [ID Name Matching](https://support.docusign.com/s/document-item?bundleId=ced1643229641057&topicId=rhb1657542911402.html&_LANG=enus).
7. Select how often recipients must verify when accessing an envelope.

   If you select Every access or Every access until role is completed, you can turn on Allow SMS authentication after one successful Photo ID or Passport verification. The recipient provides a mobile phone number to complete this type of authentication. This option is required if you want to use manual review and the IDV - DS EU Advanced signature type in the same configuration.
8. (Optional) In the Accepted IDs section, uncheck the Accept all recommended ID types, issuing countries and regions and select Manage if you want to customize the list of countries and regions from which IDs are accepted. Select Save in the Manage accepted IDs screen to update your list.
9. (Optional) In the Stored data from IDs section, select the additional data that you want ID Evidence to store for each type of ID verification and transaction. Depending on your configuration, types of identity verification may include Passport and Photo IDs, Electronic ID, and Knowledge-Based Authentication (KBA). By default, only non-personally identifiable information (non-PII) is stored. A copy of the ID itself is not stored. For more information about how and where data is stored, see the  [ID Evidence](https://support.docusign.com/s/document-item?bundleId=ced1643229641057&topicId=mrc1578885288021.html&_LANG=enus)  page.

   Attention: If your configuration authorizes the use of Japanese Individual Number Cards and allows storage of data on the back of IDs, ID Evidence will store the 12-digit individual number showing on the back of Japanese Individual Number Cards. This data is personally identifiable information (PII) and considered sensitive. To avoid this situation, either unselect storage of the back of IDs or remove the Japanese Individual Number Card from the list of Accepted IDs for your configuration. Alternatively, use the Docusign ID Verification with Japanese IDs configuration. This ID Verification method respects data privacy by only allowing storage of the front of Japanese Individual Number Cards.
10. In the Other settings section, select the action to take when recipients are unable to verify their identity after the maximum automatic verification attempts.

    Note: You can choose what happens when the maximum number of verification attempts are reached. The setting Manually review the ID document allows senders to examine the ID document and either accept or reject it instead of locking the envelope. In most cases, the sender completes the manual ID review. An administrator can also navigate to the envelope and select Review ID to complete the review if necessary.
11. Select how recipients submit their IDs, either by photo capture or file upload or by photo capture only.
12. Select Add.

The configuration appears in the configurations list and is available for senders.
