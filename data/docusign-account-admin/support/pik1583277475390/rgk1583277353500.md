# Signing Experience Settings

Read about how administrators use the In-session Landing Pages and Watermark Configuration settings to specify two key behaviors of the signing experience. These settings define where signers are routed in different situations and they ensure that in-process transactions clearly reflect an incomplete signing status.

These settings determine how the signing experience is shown to signers:

- In-session Landing Pages: Determine the web pages to which PowerForm (in-session) signers are routed for a variety of situations. These users will be sent to the specified destination URLs for each situation. If no URL is specified, users will be sent to a Docusign default destination URL.
- Watermark Configuration: The Watermark feature is designed to ensure that in-process transactions cannot be printed and signed on paper and that any saved or printed in-process documents display a watermark.

  When the Watermark feature is enabled and document signing is not complete, a watermark is displayed on each page of the documents when a printable version of the document is downloaded. If the document contains digital signatures, those signatures are stripped out.

  When all recipients have signed the document, the watermark is removed from the printable version of the document and digital signatures are visible.

  Important: If the Watermark feature is active and the Allow recipient to sign on paper option is enabled, the watermark does not display when viewing or downloading documents to sign.

- Auto-Navigation: This selects the document navigation option signing recipients see for their initial review of documents during a signing session. The selected option controls the navigation behavior as the recipient steps through the fields.

  If you use Radio Buttons on your documents, the options to navigate blank required fields are recommended for a smoother signing experience.

  The available auto-navigation options are as follows:

  - Page Only: With this option, there is no automatic navigation to the next field. The signer must manually navigate through the documents. The signer can scroll through pages, select Next, or use the document map on the right side of the browser to go to specific pages.
  - Navigate required fields: The initial navigation moves to the next empty or filled-in required field. Optional fields are skipped unless they contain invalid data. For example, if a zip code field is optional and contains an invalid entry, this field is required until corrected.
  - Navigate blank required fields: The initial navigation moves to the next empty required field. Optional fields are skipped unless they contain invalid data. For example, if a zip code field is optional and contains an invalid entry, this field is required until corrected.
  - Navigate all fields: The initial and subsequent navigation moves to the next required or optional field.
  - Page then Navigate required fields: The initial navigation goes to the top of the next page and then moves to the next empty or filled-in required field on the page. If there are no required fields on the page, the navigation stops at the top of the page and then goes to the next page when the signer selects Next.
  - Page then Navigate blank required fields: The initial navigation goes to the top of the next page and then moves to the next empty required field on the page. If there are no blank required fields on the page, the navigation stops at the top of the page and then goes to the next page when the signer selects Next.
  - Page then Navigate all fields: The initial navigation goes to the top of the next page and then moves to the next required or optional field on the page. If there are no required or optional fields on the page, the navigation stops at the top of the page and then goes to the next page when the signer selects Next.
- Allow senders to override: When selected, this option allows senders to disable the signing auto-navigation option for envelopes they send.
- Enable responsive signing: When selected, account administrators have the following default view options for responsive signing on mobile devices:

  - Optimize View for User’s Device: This is the default setting for most accounts. Docusign dynamically determines whether an agreement should open in the mobile-friendly (HTML) view or the desktop (PDF) view for mobile recipients, ensuring the best possible reading experience. Optimize View for User's Device supports two internal modes:

    - AI-assisted mode (default for most accounts): Leverages AI to evaluate document readability and automatically select the best experience for mobile recipients. If the document scores well for mobile readability, it opens in the mobile-friendly (HTML) view. If not, it opens in the desktop (PDF) view to preserve formatting and readability.
      - Accounts currently using Optimize View for User’s Device will automatically use the AI-assisted mode. Accounts that have opted out of AI features will continue using Basic (non-AI) mode.
      - The existing Enable AI tools in signing for sent agreements setting on the Signing Settings page determines whether Optimize View for User’s Device runs in AI-assisted or Basic mode.     - Basic (non-AI) mode: Analyzes the structure and complexity of the uploaded source document to predict whether it is likely to render well in mobile-friendly HTML. It does not evaluate the generated HTML output. Instead, the optimized view is determined solely based on the characteristics of the original document, unlike AI-assisted mode, which evaluates the rendered HTML directly to assess readability and quality. This mode remains available for accounts that prefer not to use AI features.
  - Open Only in Mobile-Friendly View: All documents will open with the mobile-friendly toggle set to ON, regardless of predicted quality.

    Note: In September 2025, accounts that have the Enable mobile-friendly as the default view setting enabled will be automatically updated to the Open Only in Mobile-Friendly View setting.
  - Open Only in Desktop View: All documents will open with the mobile-friendly toggle set to OFF (PDF view), regardless of predicted quality.
