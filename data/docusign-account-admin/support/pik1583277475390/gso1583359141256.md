# Claim a Domain

Learn how to prove ownership of a domain as part of claiming the domain for an
organization.

1. From the Organization view, select Domains.
2. Select Add Domain.
3. Enter the domain name, for example, fontara.com.

   If another organization has claimed this domain, an error message displays.
   ![The screenshot shows the Add Domain page and the Domain Name field.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/adddomainfont.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjU5NDgsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.AHierVRQMTarmQm8yLmVAU2fi1THmHBG-T-WP6e19nI&_LANG=enus "The screenshot shows the Add Domain page and the Domain Name field.")
4. Select Claim.

   The Verify Domain information page opens.
   ![add-domain-font2](https://docusign-be-prod.zoominsoftware.io/api/bundle/pik1583277475390/page/images/adddomainfont2.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MjU5NDgsInNoZWFmIjoicGlrMTU4MzI3NzQ3NTM5MCJ9.AHierVRQMTarmQm8yLmVAU2fi1THmHBG-T-WP6e19nI&_LANG=enus "add-domain-font2")
5. Copy either the CNAME record or TXT record to your domain name server (DNS). You might
   need to work with your organization's IT administrator to add this to your DNS.
6. Outside of Docusign Admin, update your domain’s DNS entry with either a CNAME record or
   a TXT record:

   **To create a CNAME record in your DNS (recommended)**

   1. Navigate to your domain's DNS record management page.
   2. Add a new CNAME record.

      Note: This step is not to modify your
      corporate domain’s apex. This step is to add a subdomain using a CNAME resource
      record pointing to verifydomain.docusign.net. Most DNS vendors ask for the value
      below.
   3. **Host Name/Alias:** Input your subdomain value using the 32-digit GUID. Example:
      2cacbbaa-e1b9-4b82-a86e-68d539d865da

      Note: If you are using the
      Fully Qualified Domain Name (FQDN), add the domain at the end of the GUID. Example:
      2cacbbaa-e1b9-4b82-a86e-68d539d865da.**fontara.com**
   4. **Destination Data/Target/Alias To/Canonical Name:** Input where traffic should
      be directed. Example: verifydomain.docusign.net.

      Note: The
      trailing "." at the end of the domain is required.
   5. **TTL:** Default or 1 hour / 3600 seconds

   **To create a TXT record in your DNS**

   1. Navigate to your domain's DNS record management page.
   2. Add a new TXT record.
   3. **Name:** @ or \*
   4. **Text:** TXT token from step 5 - Example:
      docusign=2cacbbaa-e1b9-4b82-a86e-68d539d865da
   5. **TTL:** Default or 1 hour / 3600 seconds

   Note: The process of
   updating DNS entries varies by vendor. You might need to coordinate with your network
   administrator to make this change. It may take up to 72 hours for DNS changes to
   propagate. Coordinating ahead of time will ensure timely deployment of Single Sign-On.

   You can confirm that your changes are active with the steps
   outlined in [Additional Information for Claiming Domains](https://support.docusign.com/s/document-item?bundleId=pik1583277475390&topicId=uqb1583359141519.html&_LANG=enus "Read about additional information for organization administrators who are claiming domains.").
7. After the DNS changes are active, return to your organization, and select
   Domains.
8. In the same row for the domain, select Actions > Verify DNS.

   Docusign checks if the generated tokens are part of the DNS record.
   If successful, the domain status changes to "Active".

Your domain ownership is proven.
