# Reminders and Expirations - Account Defaults

Read about the settings for the default reminder and expiration behavior for documents
sent from an account. Account administrators can determine the settings to be used on all
envelopes or allow users to specify the settings on the envelopes they send.

These settings are the default behavior for all envelopes sent from your account. As an
account administrator, you can choose to enforce these settings for all envelopes, or allow
users to modify the values for each envelope they send.

Note: When sending using the eSignature API, the API doesn't default to
these settings unless you explicitly set useAccountDefaults to True in your API call. If you
are sending using the API, refer to the [From the Trenches: Reminders and Expiration](https://www.docusign.com/blog/dsdev-trenches-reminders-expirations) article for more
information.

Note: Users can always use the [resend envelope](https://support.docusign.com/s/document-item?topicId=hqt1578456337100.html&bundleId=oeq1643226594604&_LANG=enus) feature to send a manual reminder to
recipients whose turn it is to sign.

You must be an account administrator with All Administration Capabilities to manage these
settings. See [Permission Profiles](https://support.docusign.com/s/document-item?bundleId=pik1583277475390&topicId=pof1583277362435.html&_LANG=enus "Read about how administrators use permission profiles to manage levels of access for a number of users. Permission profiles are sets of permissions that administrators assign to individual users. By managing the permission profiles, administrators can make changes to multiple users simultaneously.") for details.

For information on how senders can modify reminder and expiration settings on each envelope
they send, see the [Set Advanced Options for Your Signers, Envelopes, and Templates](https://support.docusign.com/s/document-item?topicId=tfm1578456367923.html&bundleId=gbo1643332197980&_LANG=enus) topic.
