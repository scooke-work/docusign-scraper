# An "INVALID\_SESSION\_ID" Error Displays When I Invoke Salesforce APIs

I see a web service error: INVALID\_SESSION\_ID when listing layouts, publishing buttons,
or fetching documents from Salesforce when sending for signature.

I see a web service error: INVALID\_SESSION\_ID when I invoke a Salesforce API
command. The error displays when I execute the following commands:

- List layouts
- Publish buttons
- Fetch documents from Salesforce when sending for signature

## Resolution 1: Disable "Lock sessions to the IP address from which they originated"

1. Access the Session Settings page as a Salesforce
   administrator:

   1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg3MzQsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.p-qghixd9A2RgazKSO_tRoeyxxTENQj1BWvUBHh3-6w&_LANG=enus) in the upper right
      of the page.
   2. Select Setup.
   3. Enter Session Settings in the Quick Find
      field.
   4. Select the Session Settings link in the
      left navigation bar.
2. Uncheck the Lock sessions to the IP address from which they
   originated box under the Session
   Settings header.
3. Select the Save button.
4. Log out of Salesforce and log back in.
5. Retry your API transaction.

   You have completed the process. You have disabled the Lock sessions to the IP
   address from which they originated function in your
   Salesforce organization.

## Resolution 2: Add Docusign IP ranges to your allowlist

1. [Add Docusign IP
   ranges to your Salesforce allowlist](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=lip1592574174158.html).
2. Add any other restricted IP addresses shown in the user’s login
   history.

   You have completed the process.
   INVALID\_SESSION\_ID errors no longer
   display.
