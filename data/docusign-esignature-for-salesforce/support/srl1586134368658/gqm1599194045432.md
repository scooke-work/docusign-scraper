# Enable Standard Docusign CLM Federated Security

Learn how to enable standard Federated Security in your Docusign CLM
account.

Federated Security uses a standard mapping between Salesforce and Docusign security settings:

Table 1. Standard Federated Security Mapping

| Salesforce Security Access | CLM Security Access |
| --- | --- |
| Read | View |
| Edit | View and Edit |
| Delete | View, Edit, and Delete |

Users with Read access on a Salesforce record are given
View access to the CLM content on that page.

Users with Salesforce Edit access are given View
and Edit access in CLM.

Users with Salesforce Delete access are given View,
Edit and Delete access in CLM.

Important:

- Having No Access rights in Docusign CLM prevents
  Salesforce users from seeing CLM content
- Salesforce users must have at least View rights to
  all Salesforce folders in the CLM account
- Docusign recommends that all Salesforce users are given **View, Edit, and Delete**
  access to CLM folders. Federated security defaults to the most restrictive
  access. So a user with **Delete** access in Salesforce and **View**
  access in CLM would only have View access in CLM. Giving all Salesforce
  users **View, Edit, and Delete** access lets administrators manage all
  security through Salesforce.
- If a user logs in to CLM with the CLM username linked to their Salesforce
  user, Federated Security is respected.
- Creating separate CLM credentials (not tied to a Salesforce user) is not recommended. It
  circumvents the Federated Security model.

Use this procedure to enable standard Federated Security mapping in your Docusign CLM
account:

1. Log in to your Docusign CLM account as an administrator.
2. Select the Admin tab.
3. Select Connected Apps at the bottom of the left
   navigation pane.
4. Select the Manage link in the Salesforce tile.
5. Check the Enable federated security box under the
   Security header.

   ![Federated Security enabled.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/Federated-Security-enabled.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg5MDYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.OiRSgDjL_mr-9CSszIJTl-8AcWXjTfR-7yrCxO5mSXw&_LANG=enus)
6. (Optional):

   - New object mapping: Set custom security
     mappings.
   - Cache time: Change how often Docusign CLM syncs
     with Salesforce.
7. Select the Save button on the bottom of the page.

The process is complete. Salesforce Federated Security is enabled
on your CLM account. Salesforce users have access to Docusign CLM content security based
on their Salesforce Profile or Role security. No separate security model is needed for
Salesforce users in your CLM account.
