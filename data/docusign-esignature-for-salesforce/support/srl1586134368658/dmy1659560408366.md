# Step 1: Prepare Your Salesforce Organization for Experience Cloud Envelope Templates

Learn how to prepare your Salesforce organization before creating a Docusign Experience Cloud envelope template.

## Before you begin

Complete the following tasks before building an Experience
Cloud envelope template:

- Purchase a Docusign Business Professional or higher
  license.
- Turn on Enable Community Cloud in your Docusign account. Only your Docusign account
  representative can activate this feature.

Use this procedure to prepare your Salesforce
organization for creating Experience Cloud envelope templates:

1. [Enable a system sender in your Docusign for Salesforce app.](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=jxz1611642321527.html&_LANG=enus "Learn to enable a System Sender so that non-Docusign users can send and sign envelopes.")
2. Change the Docusign Envelope Template Sharing Settings in
   Salesforce to Public Read Only. This grants Experience
   Cloud users basic access to Docusign envelope template custom
   objects:
   1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY4NDYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.r8VMlSRd1VU28aq7bzUijDAaPRDdqKMyg8tsjDZCttE&_LANG=enus) and choose
      Setup.
   2. Enter and select Sharing Settings in the Quick
      Find field.
   3. Select Edit.
   4. Select Public Read Only from the Docusign Envelope Template dropdown.

      ![A screenshot displays the Public Read Only setting for the Docusign Envelope Template object.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/Public-Read-Only-setting-for-DocuSign-Envelope-Template-object.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY4NDYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.r8VMlSRd1VU28aq7bzUijDAaPRDdqKMyg8tsjDZCttE&_LANG=enus "A screenshot displays the Public Read Only setting for the Docusign Envelope Template object.")
   5. Select Save.

   The Sharing Settings page refreshes, displaying your changes.
3. Add Docusign Administrator status to your delivered by user.
   Messages sent from this template display the delivered by
   user as the sender.

   Note: Add the delivered by user to the
   Salesforce System Administrator permission
   set.
4. Set the delivered by user as an organization-wide email
   address:
   1. Enter and select Organization-Wide Addresses in
      the Quick Find field.
   2. Display Name: Add a name for the
      delivered by user.
   3. Email Address: Add the email address of the
      delivered by user.
   4. Profiles: Select your Experience Cloud user
      profiles.
   5. Select Save.

   The Organization-Wide Email Addresses page
   refreshes, displaying your changes.
5. Enable Digital Experiences in Salesforce:
   1. Enter Digital Experiences in the Quick Find field
      and select the Settings link that displays under
      Digital Experiences.
   2. Select the Enable Digital Experiences check
      box.
   3. Select Save.![A screenshot displays the Enable Digital Experiences check box on the Experiences page.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/Enable-Digital-Experiences-check-box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY4NDYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.r8VMlSRd1VU28aq7bzUijDAaPRDdqKMyg8tsjDZCttE&_LANG=enus "A screenshot displays the Enable Digital Experiences check box on the Experiences page.")
   4. Select OK in the Enable digital
      experiences dialog box that displays.

   The Settings page closes. The All Sites page displays.
6. [Create and publish a new Experience Cloud
   site.](https://help.salesforce.com/s/articleView?id=sf.networks_creating.htm&type=5)
7. Turn on the Allow using standard external profiles...
   option:
   1. Go to Setup >  Digital Experiences >  Settings.
   2. Select Allow using standard external profiles for
      self-registration, user creation, and login. ![A screenshot displays the Allow using standard external profiles for self-registration, user creation, and login setting highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/Allow-using-standard-external-profiles-for-selfregistration-user-creation-and-login.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY4NDYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.r8VMlSRd1VU28aq7bzUijDAaPRDdqKMyg8tsjDZCttE&_LANG=enus "A screenshot displays the Allow using standard external profiles for self-registration, user creation, and login setting highlighted.")
   3. Select OK and choose Save
      on the dialog box that displays.

   The page refreshes. The Setup Home page
   displays.
8. Activate your Experience Cloud site:
   1. Enter Digital Experiences in the Quick Find field
      and select All Sites.
   2. Select Workspaces next to the Experience Cloud
      site that you want to configure.
   3. Select Administration.
   4. Select Activate.
   5. Select OK on the dialog box that displays.
   6. Select Publish.
   7. Select Publish on the Publish your
      site? dialog box.
9. Grant access to the Experience Cloud page:
   1. Select Members on the Administration page.
   2. Select Profiles:

      1. Select a profile from the Search dropdown
         menu.
      2. Select profiles from the Available Profiles
         list and choose Add.
   3. Select Docusign Experience Cloud User and
      Docusign Sender from the Available
      Permission Sets list and choose
      Add.
   4. Select Save.

      ![A screenshot displays Salesforce Experience Cloud members settings with the Save button highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/Experience-Cloud-members-settings.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY4NDYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.r8VMlSRd1VU28aq7bzUijDAaPRDdqKMyg8tsjDZCttE&_LANG=enus "A screenshot displays Salesforce Experience Cloud members settings with the Save button highlighted.")
   5. Select Publish.
   6. Select Publish on the Publish your
      site? dialog box.
10. Add a Docusign custom button to your Experience Cloud
    site:
    1. Enter Digital Experiences in the Quick Find field
       and select All Sites.
    2. Select Builder next to the Experience Cloud
       site.
    3. Select the Settings
       ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/Salesforce-Experience-Cloud-settings-gear-icon.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY4NDYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.r8VMlSRd1VU28aq7bzUijDAaPRDdqKMyg8tsjDZCttE&_LANG=enus) icon and select Navigation.
    4. Select the Default Navigation dropdown arrow and
       choose Edit.
    5. Select + Add Menu Item.
    6. Create a new menu item:

       1. Name: Enter the name of a Salesforce
          object.
       2. Type: Select Salesforce
          Object.
       3. Object Type: Select a Salesforce
          object.
       4. Default List View: Keep
          Default.
       5. Publicly available: Select the check
          box.
    7. Select Save Menu.
    8. Select Publish.
    9. Select Publish on the Publish your
       site? dialog box.

## Next steps

You have completed step 1 of the process. You have enabled all
prerequisites for your Salesforce organization.
