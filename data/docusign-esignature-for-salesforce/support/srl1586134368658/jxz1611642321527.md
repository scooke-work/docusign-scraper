# Enable a System Sender to Let Non-Docusign Users Send and Sign Envelopes

Learn to enable a System Sender so that non-Docusign users can send and sign envelopes.

When you enable a System Sender, Salesforce users without a Docusign license can send and sign envelopes. When non-Docusign users send envelopes, the System Sender user displays as the sender.

Envelopes sent using a System Sender are owned by the System Sender user. Users sending through the System Sender cannot view, correct, resend, or void envelopes. Non-Docusign users can track envelopes sent by a System Sender in the [Docusign Envelope Status component](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=all1586134265721.html&_LANG=enus).

Important:

- To use a System Sender, you must write Salesforce Apex code for the [Docusign Apex Toolkit for Salesforce](https://developers.docusign.com/docs/salesforce/salesforce101/apex-toolkit/). The Apex Toolkit only works with eSignature for Salesforce. You cannot use it with Docusign Gen.
- To send envelopes, non-Docusign users must be active Salesforce users with Docusign Sender permission set membership.
