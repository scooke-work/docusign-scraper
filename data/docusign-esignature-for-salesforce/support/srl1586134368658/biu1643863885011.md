# I See an “Object Reference Is Not Set to an Instance of an Object” Error When Invoking Salesforce APIs

I see a web service error: "Object Reference is not set to an instance of an object"
when listing layouts, publishing buttons, or fetching documents from Salesforce when sending
for signature.

I see a web service error: "Object Reference is not set to an instance of an object"
when I invoke a Salesforce API command.

**Steps to Reproduce**:

1. Invoke one of the following Salesforce API commands:
   - List layouts
   - Publish buttons
   - Fetch documents from Salesforce when sending for signature

   **Result**: I see an error message: "Object Reference is
   not set to an instance of an object". I am unable to execute
   my API command.

## Resolution: Disable the "Lock sessions to the IP address from which they originated" function in Salesforce

1. Access the Session Settings page as a Salesforce
   administrator:

   1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg3NDEsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.4LR905BM2Xnt3_0dRswZ4MR3jrq0esG93Bd5LSDo3_E&_LANG=enus) in the upper-right of
      the page.
   2. Select Setup.
   3. Enter Session Settings in the Quick Find
      field.
   4. Select the Session Settings link in the
      left navigation bar.
2. Uncheck the Lock sessions to the IP address from which they
   originated box under the Session
   Settings header.
3. Select the Save button.
4. Log out of Salesforce and log back in.
5. Retry your API transaction.

   You have completed the process. You have disabled
   theLock sessions to the IP address from which they
   originated function in your Salesforce
   organization.
