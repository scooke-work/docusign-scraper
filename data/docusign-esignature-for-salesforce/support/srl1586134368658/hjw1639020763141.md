# Learn About Using Envelope Events To Automatically Trigger Updates to Salesforce Fields

You can set up your envelope template to automatically update up to 25 Salesforce
fields when envelope events occur. This is useful when you want changes that occur in
the signing process to automatically update your Salesforce records.

There are five
envelope events that can trigger Salesforce field changes:

Table 1. Envelope Event Types

| Envelope Event | Action |
| --- | --- |
| Envelope Sent | A Salesforce field updates as soon as an envelope is sent. |
| Envelope Delivered | A Salesforce field updates as soon as an envelope is received. |
| Envelope Completed | A Salesforce field updates only when all signatures are completed. |
| Envelope Declined | A Salesforce field updates if a recipient declines to sign. |
| Envelope Voided | A Salesforce field updates if an envelope is voided. |

Note:

- You can add up to **five events** for each envelope event type. So you
  can add up to five Envelope Sent events, five
  Envelope Delivered events, and so on.
- You can add a maximum of **25 envelope events** to an envelope template.
- Envelope events of the same type can not have the same Field
  Name value.

  For example, two Envelope
  Completed events can not use
  Description as the Field Name.

Important:
Docusign can not check these fields for validation rules.
If user input violates validation rules, the field
update fails. Test your template before using it to ensure it does not contain errors.
