# User Management for Docusign Gen, CLM, Negotiate, and eSignature for Salesforce

Learn to add Docusign Gen, CLM, Negotiate, and eSignature users and manage permissions
on the user management page.

Docusign Apps Launcher is a package of Docusign applications you can use in Salesforce. The
Apps Launcher package contains the Docusign
eSignature,
CLM, Gen, and
Negotiate applications. Each Salesforce user needing to use
these apps must have a Docusign user. Administrators create Docusign users and grant
permissions on the Docusign user management page.

The following is a list of apps and permission types administrators can assign:

Docusign Gen
:   Generator: Generators can generate contracts and send
    them for signature.

Docusign CLM
:   Administrator: Administrators can access everything in
    CLM and manage connections with Salesforce. They can't manage Docusign users in
    Salesforce.
:   User: Users can access workflows and view, edit, and
    upload documents.
:   Viewer: Viewers can only preview and upload
    documents.

Docusign eSignature
:   Administrator: Administrators create templates and manage
    all Docusign users in Salesforce.
:   User: Users can send documents for signature.

Docusign Negotiate
:   Negotiator: Negotiators can send content for review and
    signature.
