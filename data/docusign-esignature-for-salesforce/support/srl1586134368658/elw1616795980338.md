# Docusign Envelope Template Merge Field Types

Here are the options available on the Add Merge Fields dialog box:

Here are the options available on the Add Merge Fields dialog box:

Table 1. FIELD PROPERTIES

| Field Properties Options | Description |
| --- | --- |
| Allow Writeback to Salesforce | When a signer changes Merge Field data, those changes are written back to the Salesforce record. This is triggered by an envelope event (Envelope Sent, Envelope Completed, and so on.) A maximum of 255 characters per field may be written back. |
| Allow Sender to Edit | Allow a sender to edit Salesforce merge field data. |
| Required Field | Check this box to prevent signers from leaving a field blank. Allow signed envelope content to update fields on a Salesforce record. |
| Read Only (for signer) | Prevent signers from editing a field. |

Table 2. MORE OPTIONS FIELDS

| Option Type | Description |
| --- | --- |
| Tooltip | Enter written guidance for the sender (ex: "Adds the customer's full billing address".) |
| Anchor Text | Automatically place merge fields on key words in a document. |
| Validation dropdown menu | Set rules for text that users can and can not enter in a field. |
| - No Validation | Any text can be entered in the field. |
| - Email | Text entered must be in the form of an email address. |
| - Number | Text entered can only be numbers. |
| - Date | Text entered must be in date format. |
| - SSN | Text entered must be numbers in a xxx-xx-xxxx format. |
| Error Message | (Only if Validation is selected) Enter a custom error message if illegal characters are entered. |

Table 3. MORE OPTIONS CONDITIONAL FIELDS

| Merge Field Type Selected | More Options Items Available |
| --- | --- |
| Picklist | Anchor Text |
| Boolean | Tooltip, Anchor Text |
| String | Tooltip, Anchor Text, Validation Allow Writeback: Maximum of 100 characters allowed |
| Textarea | Tooltip, Anchor Text, Validation Allow Writeback: Maximum of 255 characters allowed |
