# Learn About Using AutoPlace Fields in Envelope Templates

Learn how to build AutoPlace fields in template documents to specify where signer fields
appear.

AutoPlace fields are a feature you can use when building a Docusign envelope
template. AutoPlace automatically adds Docusign fields to specified places in a
document.

AutoPlace fields save time and reduce errors because senders do not have to manually place
fields for recipients.

- You can add AutoPlace fields throughout your documents for each recipient role.
- You can add automatic AutoPlace fields for up to 99 recipients
  (Signer 1, Signer 2, and so
  on.)
- You can build custom fields to add Salesforce fields and additional recipients
  to your document template.

Table 1. AutoPlace terms

| Term | Description |
| --- | --- |
| Anchor text | Text that tells Docusign what field to add to a document that is sent for e-signature. For example, if you add  \s1\ to a document, Docusign automatically adds a signature field to that location. This text is typically white, so it blends into the background of the page, and is not visible to the signer. |
| AutoPlace | A type of field that Docusign places over anchor text. Docusign automatically places these fields so that senders do not have to manually add them. |
| Envelope template recipient roles | Placeholder text that tells Docusign the role and signing order of recipients. |
| Merge field | A Docusign custom field that is connected to a Salesforce field. These fields are connected by the Relate to Salesforce property. |
| Writeback | A Docusign field that updates a Salesforce field when the signature process is complete. |
