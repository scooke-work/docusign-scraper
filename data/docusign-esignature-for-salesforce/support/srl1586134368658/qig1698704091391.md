# Make a Standard Salesforce User Your Integration User

Learn to make a standard Salesforce user your Docusign integration
user.

Connect your Salesforce organization to your Docusign account to use Docusign Apps Launcher in Salesforce.

To create the connection, you must enter Docusign and Salesforce
credentials on the Docusign Setup page. The connecting
Salesforce user is called the integration user.

If you don't want to use a Salesforce administrator as your integration user, you can use
standard credentials with some modifications.

Use this procedure to make a standard Salesforce user your integration user:

1. Clone a standard Salesforce user profile and name it Docusign
   Integration User.
2. Add permissions to the Integration User profile:
   1. Open the new profile and select Edit.
   2. Administrative Permissions: Select the following
      options:

      - Manage Users
      - API Enabled
      - View Setup and Configuration
      - Modify Metadata Through Metadata API Functions
      - Customize Application
      - Manage Custom Permissions
   3. Standard Object Permissions and Custom
      Object Permissions:

      - View All: Select for each object Apps
        Launcher must read.
      - Modify All: Select for each object Apps
        Launcher must update.

      Note: These settings grant access to your Salesforce organization's
      ownership hierarchy. These permissions do not grant unrestricted
      access to all data on these objects. You can also restrict access to
      specific data points using [Field Level
      Security](https://help.salesforce.com/s/articleView?id=ind.fsc_admin_configure_field_level_security.htm&type=5) or [Sharing
      Settings](https://help.salesforce.com/s/articleView?id=ind.cg_task_update_sharing_settings.htm&type=5).
   4. Open Object Manager and set Field Level
      Security for all fields Apps Launcher must read or
      update. These settings grant access to features like document generation
      or workflow automation:

      - Visible: Select for each object Apps
        Launcher must read.
      - Read-Only: Deselect for each object Apps
        Launcher must update, including all Docusign
        objects.![A screenshot displays field-level security settings for the Docusign Integration User profile.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/A-screenshot-displays-fieldlevel-security-settings-for-the-Docusign-Integration-User-profile.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY2MzYsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.xQ3I0vmksUjObyEhZ64bPfF2xqODAem9ovc5_qdFNas&_LANG=enus "A screenshot displays field-level security settings for the Docusign Integration User profile.")
   5. Select Save.
3. Create a standard Salesforce user to act as your integration user.

   Note: Create generic credentials not tied to a person, like
   docusign-integration-user@yourorg.com. View this
   topic for more information about [integration user best practices](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=rhi1657226496755.html&_LANG=enus "Learn how best to manage the integration user that connects your Salesforce and Docusign accounts.").
4. Set Docusign access for your Salesforce integration
   user:

   1. Log on to your Docusign account as an
      administrator.
   2. Create a Docusign administrator using your Salesforce integration user
      credentials.
5. Add your integration user to the Docusign Integration User
   profile.
6. Add your integration user to the Docusign Administrator
   permission set.
7. [Connect your Docusign
   account to your Salesforce organization using the Docusign
   and Salesforce integration user credentials](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=zfp1586134208988.html&_LANG=enus "Learn to connect your Salesforce organization to your Docusign account.").

You have completed the process. You made a standard Salesforce
user your integration user and connected Docusign and Salesforce. This
integration user maintains the connection between accounts without requiring
unrestricted administrator permissions.
