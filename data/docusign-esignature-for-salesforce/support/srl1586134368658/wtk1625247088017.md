# Use Merge Field Syntax to Add By Role Recipients to an Envelope Template

Learn to use merge field syntax to add By Role recipients to an envelope template.

## Using By Role recipients

Using the By Role recipient type, you can dynamically add recipients to
a Docusign envelope template. By Role allows you to add a placeholder recipient like
Signer 1. The sender can complete the recipient's details
when creating an envelope.

## Using merge field syntax for By Role recipients

You can use merge field syntax to add By Role recipients and recipient details.

Example: You want a template to add a
recipient using any record's Contact field. You add a By Role
recipient with {!Contact.Name} and
{!Contact.Email} as the recipient details. A sender selects
the envelope template button, and Docusign adds the record's contact as a recipient.

## Common objects used with merge field syntax

Administrators commonly use the following objects when creating merge field syntax:

- Users
- Contacts
- Leads
- Person Account
- Account owner when sending from Opportunities

## Using Lookup Fields

You can use the Lookup Fields option to add recipients from
Users, Contacts, and Leads without using merge field syntax. You must use merge
field syntax when adding recipients from the same object as the custom button.

Example: The envelope template button
is on a Contact layout and is importing the name and email from a
contact.

## Finding merge field syntax

You can copy Salesforce merge field syntax from the following locations:

- Object syntax: Setup >  Object Manager > API NAME
- Field syntax: Setup >  Object Manager >  (select object) >  Fields & Relationships  > FIELD NAME

## Using custom merge field syntax

You can also use merge field syntax to add a recipient from any custom field on a
parent or lookup object. When creating merge field syntax for Salesforce custom
objects, you must use \_c in your syntax. The following are
examples of custom merge field syntax:

- {!Applicant\_c.Name\_c}
- {!Applicant\_c.Email\_c}

## User, Contact, and Lead examples

| Merge field syntax | Action |
| --- | --- |
| `{!$User.FirstName}` | Specify a Salesforce user's first name |
| `{!$User.LastName}` | Specify a Salesforce user's last name |
| `{!$User.Email}` | Specify a Salesforce user's email address |
| `{!Contact.Name}` | Specify a Salesforce contact name |
| `{!Contact.Email}` | Specify a Salesforce contact email address |
| `{!Lead.Name}` | Specify a Salesforce lead name |
| `{!Lead.Email}` | Specify a Salesforce lead email address |

## Lookup Relationship examples

| Merge field syntax | Action |
| --- | --- |
| `{!Account.Owner.Name}` | Specify a Salesforce Account owner name |
| `{!Account.Owner.Email}` | Specify a Salesforce Account owner email address |
| `{!Opportunity.Owner.Name}` | Specify a Salesforce Opportunity owner name |
| `{!Opportunity.Owner.Email}` | Specify a Salesforce Opportunity owner email address |
| `{!Opportunity.LastModifiedBy.Name}` | Specify the name of the Salesforce user who last modified a record |
| `{!Opportunity.LastModifiedBy.Email}` | Specify the email address of the Salesforce user who last modified a record |
| `{!Opportunity.Account.Owner.Name}` | Specify the name of an Opportunity account owner |
| `{!Opportunity.Account.Owner.Email}` | Specify the email address of an Opportunity account owner |
| `{!Opportunity.OpportunityContactRoles.Role}` | Specify the role of an Opportunity contact |

## Salesforce Person Account examples

You can add Salesforce Person Account merge fields if you have any of these
Salesforce editions: Professional, Enterprise, Performance, Unlimited, and
Developer.

| Merge field syntax | Action |
| --- | --- |
| `{!Account.Name}` | Specify a Salesforce person account name |
| `{!Account.PersonEmail}` | Specify a Salesforce person account email address |
| `{!Account.PersonBirthdate}` | Specify a Salesforce person account birthday |
| `{!Account.PersonTitle}` | Specify a Salesforce person account title |
| `{!Account.IsPersonAccount}` | Determine whether you are working with a Salesforce person account or a business account |

## Access Code syntax

You can add merge text to the recipient Access Code field to
create an access code using Salesforce data.

Example: You want to secure an envelope but
avoid the added risk of messaging an access code to a recipient. You add
`{!Contact.Last4SSN__c}` to the recipient's Access
Code field. When sending, Docusign creates an access code using the last
four digits of the recipient's Social Security number. You inform the recipient that the
access code is the last four digits of their Social Security number.
