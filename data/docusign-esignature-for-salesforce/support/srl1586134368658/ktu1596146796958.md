# Step 6: Add Options Like Quick Send and Writeback

How to create reminders for recipients, set an expiration date, enable the Quick Send
experience, have a Salesforce status changed when a signature process is complete, and
more.

Use this procedure to add options to your Envelope Template:

Tip:

- [This video](https://players.brightcove.net/6118377982001/default_default/index.html?videoId=6171919622001) walks you through the process.
- [This topic](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=hwi1611095724834.html&_LANG=enus) shows you all of the
  advanced options available on this step.

1. Configure any of the following options:

- Reminders: Add a reminder to your recipients from the dropdown menu.
  Select Custom from the dropdown menu to add a custom
  reminder of up to 999 days.
- Expiration: Set a date or a number of days (up to
  999) after which the envelope expires.
- Sending Experience: Choose the options available for senders:
  - Default: Senders can add documents, choose
    recipients, and add fields before sending
  - Quick Send: Senders can add non-signing recipients, edit
    reminders, expiration, and the email subject and message fields
  - Place Fields: Senders are taken to the Place Fields step, skipping
    the documents and recipients steps.
  - Send Now: Skip all controls and send the envelope immediately
- Document Writeback: Select the check box for any
  option:
  - Save completed documents back to the Opportunity
    Record
  - Save certificate of completion (COC) document back to the
    Opportunity Record

  Important: Document, merge field, and data writeback will not work if the user has
  Multi-Factor Authentication for API Logins turned on in their Salesforce
  profile.
- Field Updates: Automatically update Salesforce fields when envelope
  events occur:
  1. Select the + Add Update button. A row of fields displays.
  2. Envelope Event: Select an event
     (Envelope Completed, Envelope
     Voided and so on) that triggers a Salesforce field
     update.
  3. Field Name: Enter or select the Salesforce Field Name
     (Amount, Name, and
     so on) that is updated when the envelope event occurs.
  4. Field Update: Enter or select a value that will update the
     Salesforce field name when the envelope event occurs.
     ![Field Updates events.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/bkw1639001451979.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg5MDksInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.XSGjuIm-Uf7aFqWJ-zJFY4qbsiOoiJlwB-ceWYjMXRs&_LANG=enus)

     Note:
     - You can add up to **five events** for each envelope
       event type. So you can add up to five
       Envelope Completed events,
       five Envelope Sent events, and so
       on.
     - Five field updates per envelope event means that you can
       add a maximum of **25 envelope events**.
     - Envelope events of the same type cannot have the same
       Field Name value. For
       example, two Envelope Completed
       events cannot use Stage as the
       Field Name.

2. Select Next.

You have completed step six.
