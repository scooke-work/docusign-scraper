# Use Manage App Policies to Allow or Prevent Docusign User Access

Learn how to use Manage App Policies to allow or prevent users
from authorizing their Docusign access.

New Docusign app users see an Authorize button when
they access Docusign content on a record. New users must enter
their Salesforce and Docusign credentials to access Docusign apps. Administrators can access the Manage App
Policies page to allow or prevent users from self-authorizing.

Use this procedure to manage your Docusign app policies:

1. Select Manage App Policies.

   ![A screenshot displays the Connected Apps OAuth Usage page with the Docusign Manage App Policies link highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/srl1586134368658/page/images/Salesforce-Manage-App-Policies-link.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY3ODUsInNoZWFmIjoic3JsMTU4NjEzNDM2ODY1OCJ9.t2CK3Y-t9poTG5JvvlWFy36SIDicpZ62rQNbCq_0tak&_LANG=enus "A screenshot displays the Connected Apps OAuth Usage page with the Docusign Manage App Policies link highlighted.")
2. Select Edit Policies.
3. Permitted Users: Select an option from the dropdown
   menu:

   - All users may self-authorize: Users may select the
     Authorize button to use Docusign apps.
   - Admin approved users are pre-authorized: Administrators assign Docusign app access to specific groups.

     Note: Enabling the Admin approved users are
     pre-authorized option denies access to all current
     users. Administrators use Salesforce profiles to assign
     access.
4. Select Save.

   The page closes. The Docusign Connected App Detail
   page displays.

You have completed the process. You have allowed or prevented
users from authorizing their Docusign access in your Salesforce
organization.
