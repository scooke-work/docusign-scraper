# Document Preview Error: "There Were Some Problems With Your Document Generation"

If administrator credentials from a Docusign Demo account are used to connect Docusign and Salesforce, you get the error: There were some problems with your document
generation when reaching the Preview Documents step of the template creation
process.

Identified an intermittent issue with creating an envelope template when a Docusign Demo
account is connected to Salesforce. If administrator credentials from a Docusign Demo account are used to connect Docusign and Salesforce, you get the error:
There were some problems with your document generation
when reaching the Preview Documents step of the template creation process. This
results in the following issues:

- Recipients' Docusign Roles are replaced with Salesforce Roles
- Docusign signature fields for those recipients are deleted from the Envelope
  Template

**Salesforce Roles Delete Docusign Role and Signature Fields from Envelope
Template**

**Steps to Reproduce**:

1. Add a Docusign recipient to an Envelope Template.
2. Add a Salesforce recipient from a Related List (like
   OpportunityContactRole).
3. Save the Envelope Template.
4. Start the envelope creation process with eSignature for Salesforce.
5. Advance to the Recipients step. Recipients have their Docusign Roles replaced with
   Salesforce Roles.

   Note: This only applies to recipients with Salesforce Roles
   (Decision Maker, Evaluator, etc.).
6. Advance to the Prepare and Send step. Docusign recipients' signature
   fields are missing.

   **Result**: Both the recipient's Docusign Role and
   eSignature tags are missing from the template.

## **Resolution 1: Remove the Role field from the Salesforce Contact**

1. Visit the Salesforce record page containing the Contact Role.
2. Find the Contact Roles section.
3. Select the arrow ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/iqc1594156517921/page/images/Arrowhead.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc0ODgsInNoZWFmIjoiaXFjMTU5NDE1NjUxNzkyMSJ9.bczH8uENBaAZi5pzhZY_lKtYiv5I3Wg_d0bsb-Thx-w&_LANG=enus) to the left of the Contact to be
   edited.
4. Select Edit from the drop-down menu. The Edit
   Contact Role dialog box opens.

   ![Edit Salesforce Role](https://docusign-be-prod.zoominsoftware.io/api/bundle/iqc1594156517921/page/images/Edit-Salesforce-Role.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc0ODgsInNoZWFmIjoiaXFjMTU5NDE1NjUxNzkyMSJ9.bczH8uENBaAZi5pzhZY_lKtYiv5I3Wg_d0bsb-Thx-w&_LANG=enus "Edit Salesforce Role")
5. Select the Role drop-down menu.
6. Select --None-- from the list.
7. Select the Save button.

   ![Clear Salesforce Role](https://docusign-be-prod.zoominsoftware.io/api/bundle/iqc1594156517921/page/images/Clear-Salesforce-Role.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc0ODgsInNoZWFmIjoiaXFjMTU5NDE1NjUxNzkyMSJ9.bczH8uENBaAZi5pzhZY_lKtYiv5I3Wg_d0bsb-Thx-w&_LANG=enus "Clear Salesforce Role")You are returned to the Contact Roles section. The
   Role field is now empty.

## **Resolution 2: Disconnect Docusign from Salesforce, clear cache and reconnect**

1. Select the Docusign Setup tab.
2. Select the Disconnect link on the Docusign Setup tab in the upper-right
   of the page.
3. Log out of Salesforce.
4. Clear your browser cache.
5. Log back in to Salesforce.
6. Open Docusign Apps Launcher as an administrator. The Connect Docusign to Salesforce page opens.
7. Follow the instructions to [Connect Salesforce and
   Docusign](https://support.docusign.com/s/document-item?topicId=zwk1586134361708.html&bundleId=ygy1586134394050&_LANG=enus).

   Note: Connect using the same Docusign Demo administrator
   credentials.

   The process is complete. The recipient's Salesforce Role no
   longer conflicts. Docusign Roles and signature fields remain as expected
   when using an Envelope Template to send for signature.
