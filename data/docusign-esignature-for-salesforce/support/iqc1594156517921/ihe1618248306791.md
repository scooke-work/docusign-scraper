# Docusign-Salesforce Relationships Prevent Gen for Billing Administrators From Uninstalling Docusign Apps Launcher

As a Gen for Billing administrator, you see an Unable to uninstall
package error when uninstalling Docusign Apps Launcher.
You must first remove extra relationships, permission sets, and Apex class components from
your organization.

As a Gen for Billing administrator, you see an Unable to uninstall
package error when uninstalling Docusign Apps
Launcher. You must first remove extra relationships, permission sets, and Apex class
components from your organization.

Steps to Reproduce:

1. You remove the standard Docusign-Salesforce dependencies (Docusign components, permission set assignments, and so
   on.)
2. You navigate to Setup > Installed
   Packages in your Salesforce organization.
3. You select the Uninstall link next to the Docusign
   Apps Launcher package of apps.
4. You follow the Salesforce prompts to finish the uninstallation process.

**Result**: You see an Unable to uninstall package error
message and an extra list of Gen for Billing dependencies to be removed.

## Resolution: Remove all Gen for Billing dependencies before uninstalling the package

1. Access the Installed Packages page as a
   Salesforce administrator:

   1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/iqc1594156517921/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc1MDksInNoZWFmIjoiaXFjMTU5NDE1NjUxNzkyMSJ9.soPXzAEN3nbEXl92QKSXk6Fw3rdpJARGs9exlZnfWqI&_LANG=enus) in the upper-right of
      the page.
   2. Select Setup.
   3. Enter Installed Packages in the Quick
      Find field.
   4. Select the Installed Packages link in the
      left column.
2. Select the Uninstall link next to the Docusign Apps Launcher package of apps.
3. Follow the Salesforce prompts to finish the uninstallation process. The
   Unable to uninstall package error message
   appears, with a list of Docusign-Salesforce dependencies to be removed.
4. Remove all standard Docusign-Salesforce dependencies
   (Docusign components, permission set assignments,
   and so on.)
5. Remove the additional Gen for Billing
   dependencies:

   - InvoiceTriggerGenBilling: Delete this
     trigger from the Apex Trigger Detail page.
   - Docusign Log: Delete this custom object
     from the Fields & Relationships page
   - Docusign Billing Admin: Remove all users
     from this permission set
6. Follow the Salesforce prompts to finish the Docusign
   Apps Launcher uninstallation process.

   You have completed the process. You have removed all Docusign-Salesforce dependencies and uninstalled the
   Docusign Apps Launcher package of apps.
