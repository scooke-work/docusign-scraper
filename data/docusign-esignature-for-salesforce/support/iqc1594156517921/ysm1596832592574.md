# Envelope Template Recipient’s Docusign Role and eSignature Tags are Missing

When adding recipients to an Envelope Template in Docusign eSignature for Salesforce,
conflicts between Docusign Role and Salesforce Roles can cause problems.

When adding recipients to an Envelope Template in Docusign eSignature for Salesforce ,
conflicts between Docusign Role and Salesforce Roles can cause problems.

When a user prepares a document for signature using an Envelope Template that
contains recipients with both Salesforce Roles **and**
Docusign Roles, the
following can occur:

- Recipients' Docusign Roles are replaced with Salesforce Roles
- Docusign signature fields for those recipients are deleted from the Envelope
  Template

**Steps to Reproduce**:

1. Add a Docusign recipient to an Envelope Template.
2. Add a Salesforce recipient from a Related List (like
   OpportunityContactRole).
3. Save the Envelope Template.
4. Start a signature process with eSignature for Salesforce.
5. Advance to the Recipients step. Recipients have their Docusign Roles replaced with
   Salesforce Roles.

   Note: This only applies to recipients with Salesforce Roles
   (Decision Maker, Evaluator, etc.).
6. Advance to the Prepare and Send step. Docusign recipients' signature
   fields are missing.

   **Result**: Both the recipient's Docusign Role and
   eSignature tags are missing from the template.

## Resolution: Remove the Role field from the Salesforce Contact

1. Visit the Salesforce record page containing the Contact Role.
2. Find the Contact Roles section.
3.    edited.
4. Select Edit from the drop-down menu. The Edit
   Contact Role dialog box opens.

   ![Edit Salesforce Role](https://docusign-be-prod.zoominsoftware.io/api/bundle/iqc1594156517921/page/images/Edit-Salesforce-Role.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc0ODEsInNoZWFmIjoiaXFjMTU5NDE1NjUxNzkyMSJ9.c_dS2gQlm8WwSwLxFran7VVkuAosAsAEEJpCKEPiMA4&_LANG=enus "Edit Salesforce Role")
5. Select the Role drop-down menu.
6. Select --None-- from the list.
7. Select the Save button.

   ![Clear Salesforce Role](https://docusign-be-prod.zoominsoftware.io/api/bundle/iqc1594156517921/page/images/Clear-Salesforce-Role.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc0ODEsInNoZWFmIjoiaXFjMTU5NDE1NjUxNzkyMSJ9.c_dS2gQlm8WwSwLxFran7VVkuAosAsAEEJpCKEPiMA4&_LANG=enus "Clear Salesforce Role")You are returned to the Contact Roles section. The
   Role field is now empty.

   The process is complete. The recipient's Salesforce Role no longer conflicts.
   Docusign Roles and signature fields remain as expected when using an
   Envelope Template to send for signature.
