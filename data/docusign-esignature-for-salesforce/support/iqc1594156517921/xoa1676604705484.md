# Host an In Person Signing Session on the Salesforce Mobile App

You can host an In Person signing session from the
Salesforce mobile app.

You can host an In Person signing session from the
Salesforce mobile app.

With In Person signing, a host guides a Docusign signing session. The host must be a
Docusign user. Signing recipients are required to be in the room with the host
during the session. The host launches the signing session from their device and
guides other signing recipients through the process.

Use this procedure to host an In Person signing session from the Salesforce mobile
app:

1. Open a Salesforce record.
2. Select the In Person signer envelope template button.

   Note: The button may be hidden in the ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/iqc1594156517921/page/images/Salesforce-More-button.jpg?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTcyNDEsInNoZWFmIjoiaXFjMTU5NDE1NjUxNzkyMSJ9.bNMWV1_TDlbbM6-QsD7ZrLdKyNACv7WNnuRAoQSppTY&_LANG=enus)
   More menu.
3. Complete all required fields and select Send.

   A message opens: Sent for Signature. The screen
   refreshes. The Security Requests from Sender page
   opens.
4. Sign the document as the host:

   1. Select Start on the Security Requests
      from Sender page.
   2. Follow the prompts to sign the document as the host.
   3. Select CONTINUE on the Request a
      Copy page.
   4. Select CONTINUE on the Pass
      Control page.
5. Pass the device to the first recipient and guide them through the signing
   process:

   1. Instruct the recipient to add or complete signature fields.
   2. Instruct the recipient to select FINISH.
   3. Instruct the recipient to select CONTINUE on the
      Pass Control page.
6. Repeat the previous step for each additional signing recipient.

   The page refreshes. You see the Salesforce record.

You have completed the process. You have hosted an In Person
signing session on the Salesforce mobile app.
