# Disable Salesforce Lightning Web Security to Prevent Docusign App Errors

Learn how to prevent errors when using Docusign apps in your Salesforce
organization.

Docusign Apps Launcher uses Salesforce Lightning Aura components and Lightning Web components.
Customers who have Lightning Web Security enabled can see an
error message when opening Docusign apps.

The error message reads: This page has an error. You might just need to refresh it.
Error in $A.getCallback() [TypeError: d is not a function] Failing descriptor:
{ltng:outApp}

Important: This error applies to Docusign Apps
Launcher version 5.4 and lower. For Docusign Apps Launcher version 5.5 and higher,
you must enable Lightning Web Security. Visit Setup >  Installed Packages to check your version of Docusign Apps Launcher.

Use this procedure to prevent Lightning web components errors when opening Docusign apps:

1. Access the Session Settings page as a Salesforce
   administrator:
   1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/gab1637077302276/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc3MzgsInNoZWFmIjoiZ2FiMTYzNzA3NzMwMjI3NiJ9.wFvOPtPGc9xG7ehsVEW6GJ2l9BV5WQlP_5RqrMWX0NU&_LANG=enus) icon and
      choose Setup.
   2. Enter and select Session Settings in the Quick
      Find field.
2. Uncheck the Use Lightning Web Security for Lightning web
   components box.

   ![A screenshot of the Session Settings page displays Use Lightning Web Security for Lightning web components unchecked.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gab1637077302276/page/images/Disable-the-Use-Lightning-Web-Security-for-Lightning-Web-Components-check-box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTc3MzgsInNoZWFmIjoiZ2FiMTYzNzA3NzMwMjI3NiJ9.wFvOPtPGc9xG7ehsVEW6GJ2l9BV5WQlP_5RqrMWX0NU&_LANG=enus "A screenshot of the Session Settings page displays Use Lightning Web Security for Lightning web components unchecked.")
3. Select Save.
4. Empty your browser cache.

You have completed the process. You have disabled the
Use Lightning Web Security for Lightning web components
setting. Your users do not see an error when they use Docusign apps in your Salesforce
organization.
