# Use Filter and Conditional Tags Together in a Word-Based Gen Template

Learn to build tables to import conditional content from Salesforce
records.

You can build tables to import content from Salesforce records.

Filter tags only import certain fields from a Salesforce record.

Conditional tags display or hide blocks of text depending on
Salesforce field data.

You can use these two types of tags together to create custom Gen templates.

Example
: You want to create tables that add
products by category on your generated agreements. You create specific tables for each
category. Each table has tags that filter all line items in the product table, returning
only the ones you need. You add the tables to your Gen template.

Use this procedure to add a table with Filter and Conditional tags:

1. Select a document name.

   The document preview page opens in a new tab or window.
2. Add a table with custom XML to your Word template:
   1. Select Download.
   2. Open the Word document and go to where you want the table.
   3. Insert a table with a header, at least one row, and four columns.
   4. In the top row, enter a name for each column.
   5. Enter `<# <TableRow Select="/` in the first cell of the first
      row.

      |  |  |  |
      | --- | --- | --- |
      | Product Name | Product Code | Price |
      | **`<# <TableRow Select="/`** |  |  |
   6. Create a tag that imports Product Name content and
      suppresses content that does not match Services. Place this
      text directly after the <# <TableRow Select="/ tag.

      |  |  |  |
      | --- | --- | --- |
      | Product Name | Product Code | Price |
      | **`<# <TableRow Select="./Quote_Line_Items"/>#><#<SuppressTableRow Select="./Product_Category" NotMatch='Services'/>#><#<Content Select="./Product_Name" Optional="true />#>`** |  |  |
   7. Create a tag in the Product Code column that imports
      Product Code data from the Product
      Name field.

      |  |  |  |
      | --- | --- | --- |
      | Product Name | Product Code | Price |
      | **`<# <TableRow Select="./Quote_Line_Items"/>#><#<SuppressTableRow Select="./Product_Category" NotMatch='Services'/>#><#<Content Select="./Product_Name" Optional="true />#>`** | **`<#<Content Select="./Product_Code" Optional="true />#>`** |  |
   8. Create a tag for the Price column that imports items from
      the Line Net Total  column. Add the /mo
      tag at the end for services that match the Monthly Billing Unit
      code.

      |  |  |  |
      | --- | --- | --- |
      | Product Name | Product Code | Price |
      | **`<# <TableRow Select="./Quote_Line_Items"/>#><#<SuppressTableRow Select="./Product_Category" NotMatch='Services'/>#><#<Content Select="./Product_Name" Optional="true />#>`** | **`<#<Content Select="./Product_Code" Optional="true />#>`** | **`<#<Content Select="./Line_Net_Total" Optional="true />#><#<Conditional Select="./Billing_Unit" Match='Monthly" />#>/mo<#<EndConditional/>#>`** |
   9. Save the changes in Word and return to the Gen Upload
      Documents page.
3. [Re-upload the document to your Gen
   template](https://support.docusign.com/s/document-item?bundleId=gab1637077302276&topicId=sqp1586134339354.html&_LANG=enus "Learn to update your Gen template Word documents.").
4. Select Save and Close and select Save on
   the Do you want to save changes? dialog box.

   You have added the table to your Gen template. When users generate an agreement,
   Gen only imports the Services content. Gen also adds Product Name,
   Product Code, and Line Net Total content
