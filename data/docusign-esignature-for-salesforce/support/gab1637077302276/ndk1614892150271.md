# Exception for Gen Template SOQL Sorting Rules

Learn about the scenario where sorting works differently than the standard SOL sorting
rules.

Salesforce Object Query Language (SOQL) searches your organization’s Salesforce data
for specific information. You can insert Salesforce SOQL code into the TableRow
fields of Gen templates.

Using SOQL code in tables creates custom rules for filtering and sorting data. You
can only use sorting and filtering with child relationship fields.

There is one scenario where sorting works differently than [standard SOQL sorting](https://support.docusign.com/s/document-item?bundleId=gab1637077302276&topicId=mrr1588510303522.html&_LANG=enus "Learn about inserting Salesforce Object Query Language (SOQL) code into Gen templates.").

An administrator wants to add SOQL code for a Gen field with an Account >  Contacts >  Cases >  Status relationship:

The Cases data field has an ORDER BY
value of Status DESC NULLS LAST:

## Expected result

Docusign sorts the output by Status, not
ID.

## Actual result

Docusign sorts the output by ID and not
Status.

![A screenshot displays SOQL results sorted by ID instead of Status.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gab1637077302276/page/images/genwordsortoutput.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg5OTQsInNoZWFmIjoiZ2FiMTYzNzA3NzMwMjI3NiJ9.d-OlW4Q54JbV7BNmwctimjAsNnvGTnFl1ahVByu2Uqg&_LANG=enus "A screenshot displays SOQL results sorted by ID instead of Status.")

There is no workaround for this scenario in the Gen user interface. You can rearrange
the generated content in Microsoft Word tables.
