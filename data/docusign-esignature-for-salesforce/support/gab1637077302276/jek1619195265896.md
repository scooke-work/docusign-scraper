# Add Filters to a Word-Based Gen Template

Learn to use XML code to build custom tables in Word that filter Salesforce
data.

You can use XML code to build custom tables in Word that filter Salesforce data for
Gen agreements. You can use filtering to remove unwanted data from a record and
import only the results you want.

Use this procedure to build a table that filters product types to import only certain
results:

1. Select the Copy button next to the Type anchor
   text.![A screenshot displays the copy button highlighted next to a Gen template anchor text field.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gab1637077302276/page/images/A-screenshot-displays-the-copy-button-highlighted-next-to-a-Gen-template-anchor-text-field.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg5NDMsInNoZWFmIjoiZ2FiMTYzNzA3NzMwMjI3NiJ9.-3-TBPvE9qw8iaBYSPufMFxePE1gL1WImRkrSTBWwOc&_LANG=enus "A screenshot displays the copy button highlighted next to a Gen template anchor text field.")
2. Create an empty table in your Word template and paste the
   Type anchor text into the first cell of the table:

   Note: This field merges every Type
   value from a Salesforce record into the table. Because you only want to see
   the Software type, edit the tag.

   |  |
   | --- |
   | Type: |
   | **`<# <Content Select="./Type__c"/> #>`** |
3. Edit the existing anchor text by adding `[text()='Software']`
   into the text after the tag name `Type__c`.

   Note: Replace the word
   Software with your Salesforce product
   type.

   The updated field reads: `<# <Content
   Select="./Type__c[text()='Software']"/>
   #>`
4. Edit the text to fit Microsoft Word table formatting:
   1. Remove `<# <Content Select="./` at the beginning
      of the tag.
   2. Replace that text with `<# <TableRow
      Select="/Opportunity//OpportunityLineItems/`

   The updated field reads: `<# <TableRow
   Select="/Opportunity//OpportunityLineItems/Type__c[text()='Software']" />
   #>`
5. Copy and paste the field into the first row of your table:

   |  |
   | --- |
   | Type: |
   | **`<# <TableRow Select="/Opportunity//OpportunityLineItems/Type__c[text()='Software']" /> #>`** |
6. Add the Content Select text to the end of your field
   text. The table uses this text to import Salesforce data.

   Important: Paste
   the Content Select text directly at the end of the
   TableRow text, with no spaces between.

   |  |
   | --- |
   | Type: |
   | `<# <TableRow Select="/Opportunity//OpportunityLineItems/Type__c[text()='Software']" /> #><# <Content Select="./Type__c"/> #>` |

   When a user generates an agreement from a Salesforce record, the data in
   the table looks like this:

   |  |
   | --- |
   | Type: |
   | Software |

   Note:
   - This example shows a 1 : Many path: `<# <TableRow
     Select="//Opportunity//OpportunityLineItems/Type[text() =
     ‘Software’ or text() = 'Services']/.." /> #>`
   - To access the Product level from the Opportunity, use the
     `//Opportunity//OpportunityLineItems/Product`
     path. 1 : Many : 1 paths can be difficult to
     use because certain parts of the syntax expression must change.

You have completed the process. You have built a Gen table in
Microsoft Word to filter the results of a Salesforce record.
