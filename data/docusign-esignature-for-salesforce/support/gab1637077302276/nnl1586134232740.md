# Learn About Adding Conditional Logic to a Gen Template Word Document

Learn about adding conditional logic blocks to Gen template Word documents.

You can add conditional logic blocks to Gen template Anchor Text Tags. Conditional logic
creates "if-then" statements that add or exclude text from a generated agreement.

You can use conditional logic to create one Word document that can generate different content
for different customers. For example, you can add conditional logic that hides new customer
language when generating existing customer agreements.

In the Gen template interface, you use document to add conditional logic in a Gen template.
You can only add Match or Does Not Match conditions
in the Gen template user interface.

To use more advanced logic, you can add conditional logic to Microsoft Word documents used in
Gen templates. You can enter text in the document, paste data fields from Salesforce, or a
combination of both.

## Conditional logic example

Your customers from the state of New York have special legal restrictions on the product
you make. You create a Gen template for all of your customer agreements. You add conditional
logic to your Gen template for your New York customers. This logic displays New York legal
clauses when a record's Billing State/Province field equals New York.

## Sample conditional logic blocks

- Match: Conditional logic that only displays text for new
  customers:

  `<# <Conditional Select=""/Opportunity/Type""
  Match=""New Customer"" /> #> Welcome, new
  customer! <# <Content Select=""/Opportunity/Payment_Terms__c""
  /> #> your payment terms <# <EndConditional /> #>`
- Does Not Match: Conditional logic that only displays text for
  existing customers:

  `<# <Conditional Select="/Opportunity/Type"
  NotMatch="New Customer" /> #> Hello, valued
  customer! <# <Content Select=""/Opportunity/Payment_Terms__c""
  /> #> your payment terms <# <EndConditional /> #>`

## Using conditional logic in Gen templates

Use the following guidelines when using conditional logic in Word documents:

- You always start conditional logic blocks with a
  <Conditional> tag and end with an
  <EndConditional> tag.
- You can place <Conditional> and
  <EndConditional> tags on the same line, or split them over
  multiple lines of text.
- You can create a simple string comparison by using the
  <Conditional> tag with Select &
  Match/NotMatch logic.
- You can create complex boolean (true/false) expressions in conditional blocks by using
  <Conditional> with Select & Test
  attributes.
- You can add conditional logic to a Gen template by downloading the template's Word
  document. You then add the text to the document and [re-upload the document to the Gen template](https://support.docusign.com/s/document-item?bundleId=gab1637077302276&topicId=sqp1586134339354.html&_LANG=enus "Learn to update your Gen template Word documents.").
