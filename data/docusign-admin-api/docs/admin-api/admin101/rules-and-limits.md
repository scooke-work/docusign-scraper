# Rules and resource limits

To maintain reliability and stability within our demo and production environments, Docusign operates with certain API call efficiency guidelines. To ensure effective load balancing, we continually monitor the API calls to our backend systems, and we will contact developers that are putting unnecessary burden on the system.

By default, the total number of calls per hour from all apps associated with your account may not exceed 3,000.

Hourly API usage is tracked from the start of one hour to the start of the next hour (for example, 1:00 PM–2:00 PM).

**Note:** A shorter, 30-second burst limit is also implemented for all Admin API calls. This limit can prevent you from making large numbers of calls in very short periods of time. If you exceed the API rate limit, you will receive an error in the API response: "The maximum number of hourly API invocations has been exceeded."
