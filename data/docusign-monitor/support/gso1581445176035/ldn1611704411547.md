# Configure the Blocklist Lookup Tables

Configure the blocklist lookup tables to track Monitor blocklist activity.

Use the block list lookup tables to set and block
locations or IP ranges that are considered suspicious. Docusign recommends using the
Splunkbase Lookup File Editor, available on the [Splunk App for Web
Analytics](https://splunkbase.splunk.com/app/2699/) page, to configure lookup tables.

Note the following:

- All locations' values are set to FALSE by default. This
  indicates they're not suspicious. Set suspicious locations' value to
  TRUE.
- The docusign\_ip\_lookup does not include any predefined
  ranges. Enter this information to use the Block List Overview dashboard.

1. From the App, select Setting > Lookups.
2. From the Lookups page, select Lookup table
   files.
3. Using the provided path, edit
   docusign\_location\_lookup.csv and set suspicious
   locations to TRUE.

   Note:

   The changes in this example can also be made with the Lookup Editor. Find and
   edit files in `\Splunk\etc\apps\docusign_ta\lookups`.

   ![A screenshot of sample blocklist lookup table.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gso1581445176035/page/images/monitorsplunkcountrylookup.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTc1MTcsInNoZWFmIjoiZ3NvMTU4MTQ0NTE3NjAzNSJ9.E_O8H5LBv9soDiFL3R3y23zmZUz69n6hHSQFDtoRhAc&_LANG=enus)
4. Using the Lookup Editor, enter your organization’s subnets and label them as
   FALSE to easily identity traffic from inside the
   network.

   An automatic lookup uses information in the lookup table and assigns fields to all
   docusign:monitor data if there is match. Use these
   fields to identify suspicious events.

   | Blocklist lookup | Field | Field created |
   | --- | --- | --- |
   | docusign\_ip\_lookup | Ip | ip\_blocklisted |
   | docusign\_location\_lookup | location | location\_blocklisted |
