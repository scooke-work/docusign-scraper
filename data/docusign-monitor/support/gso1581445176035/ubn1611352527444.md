# Configure and Enable Alerts

As a Splunk App administrator, you can configure custom alerts.

You can configure custom alerts using data from
Splunk as well as using the Docusign Monitor-generated alerts,

These
Docusign Monitor-generated alerts display in the Splunk Alert
Overview and Investigations dashboard.

- Autoplace scope changed
- Bulk accounts export
- Bulk users export
- CLM folder security modified
- Delegated administrator assigned
- Distance Threshold
- Document retention policy changed
- Envelope bulk send initiated
- Impossible travel
- Legal disclosure settings changed
- Multiple login failures
- Non-domain member activity
- Password changed from new location
- Password rules changed
- Recipient authentication disabled
- Same signer and sender
- SSO bypass enabled for user
- SSO login policy changed
- Unauthorized IP address activity
- Unusual CLM document deletion volume
- Unusual CLM document download volume
- Unusual envelope creation volume
- Unusual envelope deletion volume
- Unusual envelope download volume
- Unusual envelope send volume
- Unusual envelope sign volume
- Watermark configuration changed

Additionally, you can create your own alerts using Splunk data. For example:

- Docusign Alert - Activity Detected from Closed Account
- Docusign Alert - Block List Events

To configure alerts:

1. Enable alerts that you want to configure.
2. Configure the alert schedule.
3. Revise the time range of the alert depending on the schedule. That is, the time
   range for an alert should be the last 60 minutes if it is scheduled
   hourly.
4. Configure additional responses, such as email alerting, or a Slack
   webhook.

   This example shows the  Edit Alert dialog box.

   ![Edit an alert dialog box.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gso1581445176035/page/images/monitorsplunkeditalert.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTc1MjQsInNoZWFmIjoiZ3NvMTU4MTQ0NTE3NjAzNSJ9.Rph7wnkhztdJYY7T5KNqIYLV3nYsoqRc3BMqvprj_6c&_LANG=enus)
5. Validate that these automatic lookups are enabled.

   - docusign:monitor : LOOKUP-docusign\_alert\_lookup
   - docusign:monitor : LOOKUP-docusign\_cim\_lookup
   - docusign:monitor : LOOKUP-docusign\_location\_lookup
   - docusign:monitor : LOOKUP-docusign\_ip\_lookup
   - docusign:monitor : LOOKUP-docusign\_user\_autolookup

   Note: There is no CIM compliance
   without automatic lookups, events correlation with user ids, or alerts correlation
   to the events that generated them.

   Here are the automatic lookups included
   in the Docusign Add-on. The alert, time, location, and user
   automatic lookups always produce new fields. The IP lookup requires user
   inputs.

   | Automatic lookup | Fields input | Field output |
   | --- | --- | --- |
   | docusign:monitor : LOOKUP-docusign\_account\_lookup | accountId | account\_name site\_id |
   | docusign:monitor : LOOKUP-docusign\_alert\_lookup | ds\_action object | alert description  severity  severity\_id |
   | docusign:monitor : LOOKUP-docusign\_cim\_lookup | ds\_action object | action data\_model  object\_category  result |
   | docusign:monitor : LOOKUP-docusign\_location\_lookup | ipAddressLocation.location | location\_blocked |
   | docusign:monitor : LOOKUP-docusign\_ip\_lookup | ipAddress | ip\_blocked |
   | docusign:monitor : LOOKUP-docusign\_user\_lookup | userId | AddedDate company  email  firstName  isAdmin  jobTitle  lastName  permissionProfileName |
