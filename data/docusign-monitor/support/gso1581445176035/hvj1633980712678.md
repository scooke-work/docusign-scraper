# Configure Monitor Data Inputs

Configure Monitor account, monitor, and user data inputs to display in your Splunk
dashboard.

With Splunk data inputs, you can insert data sources (rest API) into Splunk. Use the Docusign Add-on menu to configure these Docusign
Monitor data inputs:

There must be a separate data input for each account in the organization.

Note: To estimate data size, each streamed record is approximately
500 Bytes in size. The total number of events is approximately 10 events for every
envelope sent. However, volume varies based on product activity.

- Docusign Accounts
- Docusign Monitor
- Docusign Users

1. Select Settings >  Data Inputs.

   ![A screenshot of Configure Monitor data inputs types, number, and actions.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gso1581445176035/page/images/monitor_splunk_config_dataInputs.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTc0OTcsInNoZWFmIjoiZ3NvMTU4MTQ0NTE3NjAzNSJ9.leC072f8UJ6sN4RQF4Yel8zOvKMDMy27gi-dcyt_jZk&_LANG=enus)

   Note: You can add the data into
   any index but make sure that all data is collected to the same index.
2. Select +Add new for Docusign Accounts.
3. Enter this information:

   | Information | Provided by: |
   | --- | --- |
   | Name | User |
   | User ID | Docusign |
   | Integrator key | User generated - See your API and key configuration. |
   | RSA private key | User generated - See your API and key configuration. |
   | Auth Server | Docusign Monitor system |
   | Lens Server URL | Docusign Monitor system |
   | fetchLimit | User |

   This example shows a completed Accounts data input entry:

   ![A screenshot of Sample data input field entries for Accounts data.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gso1581445176035/page/images/monsplunkdatainput.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTc0OTcsInNoZWFmIjoiZ3NvMTU4MTQ0NTE3NjAzNSJ9.leC072f8UJ6sN4RQF4Yel8zOvKMDMy27gi-dcyt_jZk&_LANG=enus)

   Note:
   - For every account in your organization, add a new account and enter the
     account information as described in the previous steps.
   - For integrating with your developer (Demo) Docusign
     account to test the integration, you must enter the demo environment
     information for Auth Server and API Server
     URL.
   - For your production environment, you must promote your Demo account
     integration key to a production Docusign account as
     described in the [Go-Live](https://developers.docusign.com/docs/esign-rest-api/go-live/) topic. Once your integration key is
     promoted, use production URLs (removing the -d  in
     the field entries).

   Here is the configuration information for Demo and Production environments for Docusign Accounts, Monitor, and Users source types.

   - Demo environment URLs
     - Auth Server: `https://account-d.docusign.com/oauth/token`
     - API Server/Get Organization Accounts:
       `https://api-d.docusign.net/management/v2/organizations`.
     - Get Account Users:
       `https://demo.docusign.net/restapi/v2.1/accounts/{{Account_Id}}/users`.

       Replace
       the {Account\_Id}  with your account ID
       information.
     - Lens Server:
       `https://lens.docusign.net/api/v2.0/datasets/monitor/stream`
   - Production environment URLs
     - Auth Server: `https://account.docusign.com`
     - API Server/Get Organization Accounts:
       `https://api.docusign.net/management/v2/organizations`.
     - Get Account Users:
       `https://{server}.docusign.net/restapi/v2.1/accounts/{Account_Id}/users`

       Replace the {server} with the
       domain that the account resides on, and
       {Account\_Id} with your account ID
       information.
     - Lens Server:
       `https://lens.docusign.net/api/v2.0/datasets/monitor/stream`
4. Select +Add new for Docusign Monitor and enter the data
   input information.
5. Select +Add new for Docusign Users and enter the data
   input information.

   Note: When entering the
   input data for Users, make sure you replace the
   {{accountid}} with your account ID
   information.
6. Select More Settings to display and set interval,
   sourcetype, host, and index.

   Note: Define
   Sourcetype as described in this table. Failure to set
   the correct sourcetype names can result in problems.

   | Data source | Sourcetype | Description |
   | --- | --- | --- |
   | Docusign Accounts | docusign:accounts | Your organization's accounts |
   | Docusign Monitor | docusign:monitor | Your organization's events |
   | Docusign Users | docusign:users | Your organization's users |
7. Select Input Health from the menu and validate incoming
   data from the Docusign Monitor API.

   If you correctly configured the data inputs, the index contains data from
   these sourcetypes:

   - docusign:accounts
   - docusign:monitor
   - docusign:users
