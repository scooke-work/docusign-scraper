# Use Docusign Add-on Workflow Actions to Search

Use the Docusign Add-on app's workflow actions to create investigation dashboards to
track Monitor events through the Splunk dashboard.

The Docusign Add-on app provides workflow actions that open investigation dashboards directly
from Splunk searches. The workflow actions are:

- docusign\_alert\_workflow
- docusign\_envelope\_workflow
- docusign\_user\_workflow

The following example shows the workflow actions.

## Use Add-on workflow actions

Once you expand an event in Splunk, you can use the drop down to use the workflow
action. These workflow actions are only visible for specific event types and in
certain fields as shown in the following example.

![Screen shot of Docusign Add-on workflow action data.](https://docusign-be-prod.zoominsoftware.io/api/bundle/gso1581445176035/page/images/monitorsplunkworkflowdata.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTc1MjcsInNoZWFmIjoiZ3NvMTU4MTQ0NTE3NjAzNSJ9.OyUs2E1ExObD7uY-8lVyuIRu6fa5LQnCXP86H--GnX4&_LANG=enus)

Here are the different workflow actions and the event types and fields with which they are
associated.

| This workflow action: | Applies to this event type: | Applies to this field: |
| --- | --- | --- |
| docusign\_alert\_workflow | docusign\_alert\_event\_type | alert user |
| docusign\_envelope\_workflow | docusign\_envelope\_event\_type | data.envelopeid user |
| docusign\_user\_workflow | docusign\_monitor\_event\_type | user |

## Field and events in dashboards

The values in the docusign\_cim\_lookup correlate actions and objects with events, and
is based on the Docusign Monitor Schema documentation

There are some fields that are only present in certain events. For example, results of
successes and failures only apply to user logon data.

The following example shows the Docusign User Investigator dashboard. Notice that
although there are Events Over Time there are no
corresponding Event Results Over Time. This occurs because
the types of events listed (Envelope Created / Sent) have no associated result
value. When this occurs, the display includes the note:

`Only certain events have an associated <fieldname>`
