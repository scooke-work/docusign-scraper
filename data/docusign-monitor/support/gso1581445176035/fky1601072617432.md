# Receive Docusign Monitor Alert Email Notifications

Learn about how Monitor sends email alerts for security events.

By default, Docusign organization administrators receive email alerts when
the alert events occur, enabling organization administrators to quickly review and
respond to potential threats.

Note: User events require the Monitor-enabled organization to claim the email
domain that is associated with that user as described in the [Add and Verify a Subdomain to Use
as a Custom Email Domain](https://support.docusign.com/s/document-item?bundleId=rrf1583359212854&topicId=kpy1624481491369.html&_LANG=enus) article. This ensures that the organization is authorized to view that
user's information. If you reserve no email domains, you do not receive information
about these events.

Unless you select specific users as described in the [Select Alert Email Recipients](https://support.docusign.com/s/document-item?bundleId=gso1581445176035&topicId=loo1631913386767.html&_LANG=enus "For each alert, specify which users receive Docusign Monitor alert email notifications.") article, these Docusign organization
administrators receive alert email notifications:

- Docusign organization administrator.
- Delegated administrator with the Security Reporting Delegated Administrator role. See the
  [Add a Docusign
  Administrator](https://support.docusign.com/s/document-item?bundleId=rrf1583359212854&topicId=cvy1583359150599.html&_LANG=enus) article for information about setting up this role.

Note:

- The number of alert emails is limited to ten per hour per user.
- You can stop email notifications by Disabling specific alerts as described in the [Set Up Notifications in Docusign Monitor Settings](https://support.docusign.com/s/document-item?bundleId=gso1581445176035&topicId=ovg1582324205227.html&_LANG=enus "View, enable, or disable Docusign Monitor security alerts for your deployment .") article.
