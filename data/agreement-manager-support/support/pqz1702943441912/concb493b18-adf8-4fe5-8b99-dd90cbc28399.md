# Bulk Metadata Editing

Read about the Bulk Metadata Editing feature, including use cases, typical roles and permissions, and a process overview.

The *Bulk Metadata Editing* feature is used to update the agreement data for a group of documents all at once. This is accomplished by using a pre-formatted, Docusign-generated spreadsheet file. Use this spreadsheet as a template for editing your agreement data in bulk.

Important: The values provided by this flow are considered "user-confirmed". The existing values are overwritten and the fields are excluded from future updates. For example, when the document is reprocessed for additional data points.

Download the spreadsheet for a group of selected documents and edit the agreement data in the file:

The spreadsheet contains rows for each of the selected agreements and columns for all the custom and standard fields in the account. The spreadsheet does not contain any data values for the selected agreements.

Populate the spreadsheet with the relevant metadata and import it back into Agreement Manager:

![A screenshot of the Import data from a spreadsheet dialog box that shows a selected spreadsheet file.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Import-data-from-a-spreadsheet_dialog.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3NzIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.g5Wf2ZbGtYydwJqWu1mn4Gw1DhIDwLDpScDHunxLIfY&_LANG=enus "A screenshot of the Import data from a spreadsheet dialog box that shows a selected spreadsheet file.")

Note: You can use the *Bulk Metadata Editing* feature to edit the metadata for up to 1,000 agreements at a time. For larger jobs, you have the option to edit your documents in batches.

The documents are updated with the user-provided agreement data and existing field values are overwritten with the new metadata. Keep in mind, this only applies to the selected group of documents. Once the upload job is successfully completed, you can search for, filter by, preview, and edit the updated data in Agreement Manager.

## Primary use case

Document migration is the primary use case for the *Bulk Metadata Editing* feature. When organizations migrate documents from external repositories into Agreement Manager, not all agreement-level data can be brought over. For example, legacy attributes from Docusign CLM and Salesforce fields.

This disconnect leads to missing agreement data in Agreement Manager and incomplete search and reporting. Downstream workflows are also impacted. Use the *Bulk Metadata Editing* feature to update the metadata for the migrated documents.

Before using the feature, an administrator or contract manager must create custom extractions for the missing fields. When you download the pre-formatted spreadsheet, the file includes all the custom and standard fields in the account.

Important: Be sure that the custom fields are linked to all the relevant agreement types. If you provide values for fields that are not associated with the corresponding agreement type, the data is skipped. See the [Rules for Updating the Spreadsheet File](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=con3951e9fb-5861-43ad-ba30-253db38295a0.html&_LANG=enus "Read about important rules for updating the pre-formatted spreadsheet file.") topic to learn more about formatting rules and errors.

Download the pre-formatted spreadsheet for the migrated documents and provide the relevant metadata. Import the data back into Agreement Manager to update the values in the selected documents.

**Secondary use cases**

There are a number of secondary use cases for bulk metadata editing, such as:

- **Data cleanup and party name normalization:** Correct party names, normalize internal IDs, and fix inconsistent values across thousands of agreements.
- **Governance and permissions:** Tag your agreements with specific metadata to align with internal governance models. For example, region, business unit, sensitivity level, or owner. Once added to your agreements, these values can drive downstream permissions.
- **Reporting and analytics alignment:** Add classification fields to your agreements to refine the data in your dashboards and reports. For example, industry, segment, product family, or category.

## Permissions and roles

**Permission requirements**

The *Bulk Metadata Editing* feature has *feature*-level and *agreement*-level permission requirements:

- **Feature-level permission:** This feature is available to users with bulk upload access to Agreement Manager. The Upload up to 1,000 files at a time toggle is managed from the User profile page. See the [Agreement Manager Permissions, Profiles, and Roles](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=eut1712157553977.html&_LANG=enus "Read about the available Agreement Manager permissions, including how users and groups are granted access to Agreement Manager agreements and example roles for each permission.") topic to learn more about the bulk upload permission.
- **Agreement-level permission:** Users can only use this feature on agreements to which they have "edit" permission. For example, users with Can Manage or Can Edit agreement-level access.

  Note: Users with Organize and manage all agreements access to Agreement Manager can use this feature on all the agreements in their account.

**Typical roles**

While this feature is available to users with *bulk upload* permissions, it is mostly used by administrators and contract managers. These users have Organize and manage all agreements access to Agreement Manager agreements and can bulk upload documents.

The contract manager role can include teams from across the organization, such as Legal and Contract Operations teams.

## High-level process flow

Follow a three-phase process to use the *Bulk Metadata Editing* feature:

![An infographic that shows the phases of the Bulk Metadata Editing process.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_process-fllow_infographic.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3NzIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.g5Wf2ZbGtYydwJqWu1mn4Gw1DhIDwLDpScDHunxLIfY&_LANG=enus "An infographic that shows the phases of the Bulk Metadata Editing process.")

- **Phase 1: Prepare and Scope**

  Plan and prepare for your agreement updates:

  - Plan for the new data points. Set up the new custom fields in Agreement Manager before downloading the pre-formatted spreadsheet. The spreadsheet includes all the custom and standard fields in the account. Be sure to add the new custom fields to all the relevant agreement types.
  - Determine the scope of the updates. Filter the agreement list table to locate the agreements that you want to update. For example, filter the table by upload job, agreement set, party, or agreement type.
- **Phase 2: Download and Edit**

  Download and edit the pre-formatted spreadsheet for the group of selected documents:

  - Download the spreadsheet file. Select the documents that you want to update from the agreement list table on the Agreement Manager homepage. You can edit up to 1,000 agreements at a time in a single upload job. Larger data sets are processed in batches.
  - Edit the spreadsheet to include the new metadata values. When you edit the spreadsheet, follow the formatting rules and do not change the template structure. For example, do not insert columns and rows. And do not edit the first two rows or the first three columns.
- **Phase 3: Upload and Sync**

  Import the updated spreadsheet file and resolve any upload errors.

  Important: When you import the updated agreement data, the current data is overwritten. This action cannot be undone once saved. The user-provided data is final. The cells must contain a valid value to overwrite the current value. If the cell is empty in the spreadsheet, the current field value is retained.

  - Import the updated metadata file. Use the Import Data option to import data from a spreadsheet. This option is located in the Add (+) menu on the Agreement Manager homepage.
  - Agreement data is synced. When you import the new data, the existing values are overwritten, including the AI-extracted values. Agreement Manager applies all valid cell values and flags formatting errors in the job monitoring hub.

    Resolve the flagged formatting errors and re-import the spreadsheet file. Repeat this step until all the cell values are uploaded successfully.

Once the job succeeds, the updated metadata is reflected in Agreement Manager for your selected documents. You can search for, filter by, preview, and report on the new agreement data.

## Spreadsheet considerations

When you edit the pre-formatted spreadsheet, it is critical that you follow the formatting requirements and spreadsheet rules. During the data import, only the valid cell values are uploaded to Agreement Manager.

Resolve the cell values that failed due to formatting errors before you import the data again. For example:

![A snippet of the Job status for a data import job that shows the error details for a failed file.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Job-status_example-error_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3NzIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.g5Wf2ZbGtYydwJqWu1mn4Gw1DhIDwLDpScDHunxLIfY&_LANG=enus "A snippet of the Job status for a data import job that shows the error details for a failed file.")

Consider these rules when editing the spreadsheet file:

- Maintain the file structure of the pre-formatted spreadsheet file:

  - Always download a fresh spreadsheet from Agreement Manager for the agreements that you want to update.
  - Do not edit the header row, including updating the header names and inserting additional columns.
  - Do not edit the data in the second row of the spreadsheet. This row contains the mapping data for the fields.
  - Do not add or rename the columns.
  - Do not add extra rows. One row per agreement.
- Adhere to strict field-level formatting rules when editing your agreement data. Specific formatting rules exist for text, date, number, currency, duration, multi-value, party, and dropdown fields.

## Entry points for the spreadsheet download

There are two main locations from which you can download the pre-formatted spreadsheet file:

1. **Agreement list table (Agreement Manager homepage):** Apply filters to locate and select the documents that you want to update. Expand the Edit (pencil) dropdown and select the Agreement Data option:

   ![A snippet of the Agreement Manager homepage that highlights the Agreement Data option for a group of selected documents.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Spreadsheet-download_entry-option-1.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3NzIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.g5Wf2ZbGtYydwJqWu1mn4Gw1DhIDwLDpScDHunxLIfY&_LANG=enus "A snippet of the Agreement Manager homepage that highlights the Agreement Data option for a group of selected documents.")
2. Uploaded Agreements **table (Job monitoring hub):** Download the pre-formatted spreadsheet file for the agreements in an upload job. For example, you migrated agreements from an external repository and some of the metadata is missing.

   ![A snippet of the Uploaded Agreements table that shows the Export Agreement Data option in the Actions menu for an upload job.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Spreadsheet-download_entry-option-2.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3NzIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.g5Wf2ZbGtYydwJqWu1mn4Gw1DhIDwLDpScDHunxLIfY&_LANG=enus "A snippet of the Uploaded Agreements table that shows the Export Agreement Data option in the Actions menu for an upload job.")

   1. Go to the job monitoring hub. Expand the Settings (gear) menu on the Agreement Manager homepage and select the Uploads and Processing Status option.

      The All Jobs page opens and the Uploaded Agreements view is displayed.
   2. Locate the relevant upload job. Review the Job Name, Source, Files, and Status columns to locate the relevant upload job.

      Tip: Use the Created date, Status, and Source filters to help you locate the relevant upload job.
   3. Expand the Actions menu for the relevant upload job and select the Export Agreement Data option.

      Tip: The Export Agreement Data option is not available for spreadsheet uploads. Use the Source filter to filter out Spreadsheet uploads.

The View or edit agreement data in a spreadsheet dialog box opens. Edit the File name as needed and select the Download and Edit option. The spreadsheet file is downloaded to your local Downloads folder.
