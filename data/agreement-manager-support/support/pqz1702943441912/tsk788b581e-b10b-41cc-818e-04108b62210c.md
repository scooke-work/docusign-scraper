# Download and Fix Your Metadata Validation Errors

Learn how to download and fix your metadata validation errors from the upload process.

## Before you begin

This task reviews the steps to resolve the errors from the upload process. It relies on you completing the previous phases, including:

- [Download the Pre-Formatted Spreadsheet File](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk1b94afc6-712f-4d71-868e-0787867354d8.html&_LANG=enus "Learn how to download the pre-formatted spreadsheet file for a group of documents.")
- [Edit the Metadata Spreadsheet](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk8710c3f2-c881-4219-96b0-2b287e8b4e03.html&_LANG=enus "Learn how to edit the metadata spreadsheet file to bulk update the agreement data for the selected documents.")
- [Import Your Edited Metadata Spreadsheet](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk5ab1cd5b-00f7-42a9-bd77-728baadff903.html&_LANG=enus "Learn how to import your edited metadata spreadsheet using the Add (+) menu located on the Agreement Manager homepage.")
- [Review Job Results from the Notification Email](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tske7ac3afe-c04e-4acc-95b1-9f83b1b66ed5.html&_LANG=enus "Learn how to view the results of an upload job from the email notification. A system-generated email is sent when the upload process is completed.")

When you import the new data, the existing values are overwritten, including the AI-extracted values. Agreement Manager applies all valid cell values and flags the formatting errors in the job monitoring hub.

Resolve these formatting errors and re-import the spreadsheet file. Repeat this step until all the field values are updated successfully.

In the previous topic, you reviewed how to access the error details from the notification email. This topic reviews the steps to access the job monitoring hub from the Agreement Manager homepage.

1. Log in to your Docusign account.

   The Docusign Home page opens.
2. Select Agreements from the global header.

   The Docusign Agreements page opens.
3. Select Completed from the AGREEMENTS section in the navigation pane.

   The Agreement Manager homepage opens and the Completed Documents view is displayed.

**Access the job status**

4. Expand the Settings (gear) menu and select the Uploads and Processing Status option:

   ![A screenshot of the Completed Documents view that shows the Uploads and Processing Status option highlighted in the Settings (gear) menu.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Completed-documents-view_Actions_Monitor-All-Jobs-option.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A screenshot of the Completed Documents view that shows the Uploads and Processing Status option highlighted in the Settings (gear) menu.")

   The job monitoring hub opens and the All jobs page is displayed. Use this page to access the status of any job, including uploads and processing.

**Review the error details**

5. Go to the Job status page for your upload job:

   ![A snippet of the All jobs page that highlights the View Details option for an upload job.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Uploaded-Agreements_Actions_View-Details_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A snippet of the All jobs page that highlights the View Details option for an upload job.")

   1. Locate the upload job in the Uploaded Agreements table. Apply filters and use the sorting arrows as needed.
   2. Expand the corresponding Actions dropdown and select the View Details option.

   The Job status page opens for the upload job. The files that failed to update are listed in the Error details table, including the reason the data failed to update.
6. Review the errors listed in the Error details table:

   ![A snippet of the Job status page that shows the Error details table, including the reason the files failed to update.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Job-status_Error-details_Fontara-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A snippet of the Job status page that shows the Error details table, including the reason the files failed to update.")

**Fix the errors**

The Errors column describes the reasons that the files were not updated. Use the Download and Fix option to update the field values that failed validation.

Tip: Leave your Agreement Manager window open so you can reference the Error details table. Use the View option to access the associated document. The file opens in the Agreement preview page.

See the [Troubleshoot Spreadsheet Upload Errors](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=ref4dcc5ebd-8791-48fd-98e6-5c5d6bae7bc5.html&_LANG=enus "Look up troubleshooting steps for resolving errors during the metadata spreadsheet upload.") article for steps to resolve common validation errors.

7. Select the Download and Fix option.

   A pre-formatted spreadsheet file is downloaded to your local Downloads folder. This spreadsheet is pre-populated with the agreements that failed validation and were not updated.

   Important: The same formatting requirements and spreadsheet rules apply to this spreadsheet file.
8. Open the pre-formatted spreadsheet.

   Tip: Use the Freeze Panes option under the View ribbon to freeze the first two rows of the spreadsheet. Freezing the Agreement ID and File Name rows helps you find the correct fields to update.

   The spreadsheet opens. It is pre-formatted with the agreements from the Error details table. Update the fields that were noted in the Errors column.
9. Resolve the first error in the list. In this example, the error is, 'Usage fees must be a valid number." This error is displayed when the entered number did not follow the formatting requirement.

   Locate the agreement row and field column for the first error. Enter a value using the correct format:

   ![A snippet of the pre-formatted spreadsheet that highlights the updated agreement and value for one of the errors in the list.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Download-and-fix_Update-usage-fee_example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A snippet of the pre-formatted spreadsheet that highlights the updated agreement and value for one of the errors in the list.")

   Remember: For number fields, enter digits only with no commas. Decimals are approved where the field supports them.

   For example, enter *10000* in the Usage fees (NUMBER) column for the appropriate agreement. The header row includes the field type to help you populate the spreadsheet.
10. Resolve the next error in the list. In this example, the error is, "Parties must follow the format: [Name] | [Role]." This error message is displayed when the Parties value was entered incorrectly or is missing information.

    Locate the agreement row and field column for the error. Enter a value using the correct format:

    ![A snippet of the pre-formatted spreadsheet that highlights the updated agreement and value for one of the errors in the list.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Download-and-fix_Update-parties-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A snippet of the pre-formatted spreadsheet that highlights the updated agreement and value for one of the errors in the list.")

    Remember: When populating the Parties column, enter a party name and role. If you are unsure of the role or do not want to include one, us the format: [Name] | [None]. Use semicolon-separated entries when adding multiple party values in the same cell.

    For example, enter *Fontara | Buyer; Liberty Innovate | Seller* in the Parties column for the appropriate agreement.
11. Resolve the next error in the list. In this example, the error is, "Effective Date must be a valid date and follow the format: MM/DD/YYYY." This error message is displayed when a date value was entered with using the incorrect format.

    Locate the agreement row and field column for the error. Enter a value using the correct format:

    ![A snippet of the pre-formatted spreadsheet that highlights the updated agreement and value for two of the errors in the list.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Download-and-fix_Update-date-field-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A snippet of the pre-formatted spreadsheet that highlights the updated agreement and value for two of the errors in the list.")

    Remember: When populating date fields, enter dates in the standard, U.S. format: *MM-DD-YYYY*.

    For example, enter *01/01/2026* in the Effective Date (DATE) column for the appropriate agreements. There are two agreements in the Error details table with this issue.
12. Continue updating the pre-formatted spreadsheet to fix the errors noted in the Error details table. When all the errors are resolved, save your changes and close the spreadsheet.

**Import the updated spreadsheet**

After populating the pre-formatted spreadsheet with the relevant data values, import the metadata into your account.

13. Return to the Agreement Manager homepage:
    1. From the Job status page, select the Back arrow in the top left. You are returned to the All Jobs page.
    2. From the All Jobs page, select the Back arrow in the top left. You are returned to the Agreement Manager homepage.
14. Expand the Add (+) dropdown and select the Import Data option:

    ![A snippet of the Completed Documents page that highlights the Import Data option in the Add (+) dropdown menu.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Add-menu_Import-Data_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A snippet of the Completed Documents page that highlights the Import Data option in the Add (+) dropdown menu.")

    The Import data from a spreadsheet dialog box opens. You are prompted to drop or select your edited file.
15. Upload the pre-formatted spreadsheet file that you populated with the correct metadata values. This is the spreadsheet that you downloaded from the Job status page. Drop the file in the dialog box of use the Select File option to select the relevant file.
16. Select the Import Data option.

    The dialog box closes and you are returned to the Completed Documents page. A confirmation message indicates that the data was imported.
17. Select the View Details option from the confirmation message.

    The Job status page opens for the upload job.

    ![A screenshot of the Job status page for the upload job that shows all the files were successfully updated.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Job-status_All-files-successfully-updated.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MjAsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.uIbpCRE9sdkaTET0hP0DZGAuSd7Px3DhfG30HwcZwas&_LANG=enus "A screenshot of the Job status page for the upload job that shows all the files were successfully updated.")
18. If errors persist, repeat this process until all the errors are resolved.

You have successfully resolved the validation errors, imported the metadata, and updated the files that previously failed. This completes the process for using the *Bulk Metadata Editing* feature.
