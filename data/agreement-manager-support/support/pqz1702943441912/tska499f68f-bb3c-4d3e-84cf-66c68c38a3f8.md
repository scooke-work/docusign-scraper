# Combine Filters with an Advanced Query String

Learn how to combine filters with an advanced search query to locate relevant documents.

## Before you begin

Before completing this task, review the support articles related to filters and advanced search queries, such as:

- [Perform a Search and Apply Filters to Locate Agreements](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=kcf1713200340695.html&_LANG=enus "Learn how to combine search methods and filters to locate relevant agreements. This topic answers the question: How do I perform a keyword search and apply filters to find my completed agreements?")
- [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.")
- [Build an Advanced Search Query](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tskf302fc1f-74d3-40be-8ca2-e9b038981aa9.html&_LANG=enus "Learn how to build an advanced search query by combining multiple operators and features in a single query string.")

Your advanced query strings work best with limited complexity. Long queries, with many query operators and rules, are more likely to fail or revert to a basic keyword search. As a rule, apply filters for field values instead of encoding the field logic into the query string.

The filters that you apply decide *which* agreements are eligible for inclusion. Your advanced query decides:

- *How* the agreement text matches the criteria.
- *Which* matches should rank higher in the results.

This article shows you how to combine filters with an advanced query string. This task is completed from the  homepage.

1. Log in to your Docusign account.

   The Docusign Home page opens.
2. Select Agreements from the global header.

   The Docusign Agreements page opens.
3. Select Completed from the AGREEMENTS section in the navigation pane.

   The  homepage opens and the Completed Documents page is displayed.

**Apply filters for field values**

Instead of encoding field logic into the query string, filter your agreements for specific field values. For example, filter your agreements by Parties, Agreement Type, or Status.

4. Apply filters to the agreement list table:
   1. Expand (click) the Filters option and select the relevant filter. For example, apply the Parties filter to view agreements for a specific party.

      The filter expands to show the field values.
   2. Choose the appropriate field values from the filter. For example, search for and select all the relevant party names, including variants.
   3. Select the Apply option to apply the selected filter values to your agreement list.
   4. Repeat this step until all the relevant filters are applied.

   The agreement list table updates and the agreements that matched the applied filters are listed in the table.

**Build the advanced query string**

Use a query string to capture search requirements beyond the basic keywords and filters. For example, add requirements to locate specific clause language, capture word variations, find related-terms, and boost relevant terms.

5. Locate agreements that contain specific clause language:
   1. Enter the quoted clause terms in the search field. For example, enter *"liability obligations"* as examples of terms found in force majeure clauses.

      Tip: Choose terms that are typical to the clause, which are
      not next to each other. The presence of these terms indicates the
      existence of the specific clause language.
   2. Add a tilde (~) and proximity number after the quoted terms. For example, update the query string to: *"liability obligations"~5*.
   3. Press the Enter key.

   The query string is applied to your agreement list table. The agreements that contained the clause terms are displayed in the table.
6. Continue building the query string until the relevant results are reflected in the table. See the [Build an Advanced Search Query](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tskf302fc1f-74d3-40be-8ca2-e9b038981aa9.html&_LANG=enus "Learn how to build an advanced search query by combining multiple operators and features in a single query string.") article to learn more about building query strings.

**Save your query string and filters**

Use the Save option to save your query string, including applied filters, for future use. Once saved, run your search at any time to retrieve an updated list of documents that match the query string.

See the [Use Saved Searches to Find Agreement Data](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tskca2d74f2-823d-403d-84a6-d990537cd187.html&_LANG=enus "Learn how to access and run a saved search, including steps to edit and delete searches.") article to learn more about running your saved searches.

7. Save your query string for future use:
   1. Select the Save option located to the right of the filters.

      The New saved search dialog box opens. Complete the required fields to save your search terms.
   2. Enter a unique name for the search in the Name field. A system-generated name defaults automatically. Overwrite this value as needed.
   3. Enter a Description of the search in the text-entry box, including its purpose and use.
   4. Select the Save Search option.

   The dialog box closes. Your search is available in the Saved Searches list. The page title and the breadcrumb navigation are updated to reflect the name of the saved search.
8. Return to the Agreement Manager homepage. Select the Completed Documents option from the breadcrumb navigation in the top left.

   You are returned to the Agreement Manager homepage and the Completed Documents page is displayed.

You have successfully applied filters and entered an advanced query string to locate the relevant agreements and metadata.
