# General Limitations and Considerations for Agreement Manager

This topic answers the question: Where can I see a list of the Agreement Manager limitations and considerations associated with this release?

The table displays the known limitations and considerations for this Agreement Manager release, including high-level descriptions.

Note: Agreement Manager will improve and evolve over time.

| **Limitations and Considerations**  (including regional expansion limitations) | **Description** |
| --- | --- |
| **Data may be processed in a separate region** | During agreement processing, your data may be processed outside of the geographic region where your account is provisioned. See Docusign’s [Subprocessor List](https://www.docusign.com/trust/privacy/subprocessors-list) for details on processing locations. |
| **Agreement Manager AI models support agreements in select languages** | The AI models that power Agreement Manager currently support agreements in Brazilian-Portuguese, English, French, German, Japanese, and Spanish. See the [Language and Regional Support](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=xvp1729795957126.html&_LANG=enus "Read about language support and regional availability for Agreement Manager AI models and features.") topic for additional information. |
| **Agreement Manager features and AI-assisted insights vary by region and language** | Agreement Manager offers different sets of features and AI-assisted insights for customer accounts. These offerings are dependent on the region to which the account is provisioned and the agreement language.  Support for additional features and AI-assisted insights will be rolled out to different regions and languages on an ongoing basis.  See the support matrix on the [Language and Regional Support](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=xvp1729795957126.html&_LANG=enus "Read about language support and regional availability for Agreement Manager AI models and features.") article for additional information. |
| **Agreement Manager AI-assisted insights accuracy and relevance vary by region and language** | The accuracy and relevant of AI-assisted insights may vary depending on the agreement language and customer use case.  Customers should always review AI-assisted insights for accuracy and to determine whether the extracted data point is appropriate for their particular circumstances. |
| **Agreement Manager has ingestion thresholds for Docusign historical completed agreements** | When customers purchase an Intelligent Agreement Management (IAM) plan with Agreement Manager, their historical completed documents are auto-ingested by default.  The ingestion thresholds for Docusign historical completed agreements are:  - 500,000 (North American customers) - 100,000 (International customers)  Important: If your document count exceeds the ingestion threshold, contact your Docusign account representative to submit a special request. |
| **Ingestion and upload limitations** | When uploading or ingesting documents to Agreement Manager:  - Files must be less than 100 MB each - File names must contain fewer than 255 characters - Only supported file types can be uploaded   See the [Agreement Uploads and Ingestion](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=rrs1712255668350.html&_LANG=enus "This topic answers the question: What are my upload options for Agreement Manager, including limitations and required permissions?") article to learn more about these limitations. |
| **Agreement Manager is not fully compatible with Docusign CLM** | Connectivity between CLM and Agreement Manager is expected in early 2026. |
| **Agreement Manager is issued at the account level** | Customers with multiple eSignature accounts will receive one instance of Agreement Manager for each account. |
| **Agreement Manager FedRAMP and GovRAMP limitations** | For customers in regulated industries, Agreement Manager is FedRAMP and GovRAMP authorized:  - FedRAMP (Federal Risk and Authorization Management Program) authorizes Cloud Service Providers (CSPs) for federal agencies - GovRAMP (Government Risk and Authorization Management Program) authorizes CSPs for state, local, and tribal governments  Both frameworks standardize cloud security assessments and compliance.  Restriction: The Agreement Manager email ingestion feature is not supported for FedRAMP and GovRAMP accounts. |
| **Agreement Manager users cannot choose which documents are automatically ingested from eSignature** | All Docusign historical completed agreements are ingested automatically upon provisioning. Users cannot select specific documents or subsets of documents to be auto-ingested into Agreement Manager. |
| **Agreement Manager only displays completed agreements** | Agreement Manager does not display *in-flight* agreements. Note: Customers can manually upload any agreement to Agreement Manager. These agreements may or may not be *completed*. |
| **Granting group permissions for the “Everyone” group has no effect** | Generally, all groups can be granted permissions to view or manage all agreements in Agreement Manager for the account. However, this does not apply to the "Everyone" group. Regardless of whether the user interface (UI) allows administrators to grant permissions for this group, there is no effect. |
| **Sharing agreements with the “Everyone” group does not have any effect** | When sharing agreements, users can select users and groups in the account to grant them view, edit, or manage access. This works for any user or group in the account except for the “Everyone” group, even though that group is selectable in the UI. |
| **Agreement Manager does not support duplicate agreement detection** | Agreement Manager does not recognize when duplicate agreements have been ingested or uploaded to the repository. If the same agreement was uploaded twice, users with the Can Manage permission can delete agreements manually, as needed. |

## Limitation: "Dropdown" attributes (fields) with pre-populated value sets

For attributes (fields) that use dropdowns with pre-populated value sets, only those values are surfaced in your agreements. Agreement processing extracts an attribute value only if it matches one of the values in the dropdown.

When processing cannot locate the value within the agreement, the extracted value is *Other*. For example, if an agreement's payment term is 50 days, the surfaced Payment Term value is *Other*:

This limitation applies to these attributes (fields) and values:

| Attribute (Field) | Value Set |
| --- | --- |
| Assignment (Change of Control) | No, or consent required; Yes, notice required; Yes, restrictions apply; Yes, with conditions |
| Assignment (General) | No, or consent required; Yes; Yes, with conditions |

| Employment Type | Fixed Term; Full Time; Intern; Part Time |

| Renewal Type | Auto-Renew; Evergreen; Fixed Term; Renewal Option; Unspecified |

## Report limitations

When downloading reports to CSV files from the Insights reporting hub, there is an agreement limit of 25,000 agreements for all IAM reports. This count includes your custom reports.

**Suggested workaround:** Adjust the report filters to run the report in batches until all the relevant agreements are downloaded. For example, run the report in batches according to varying date ranges. Once downloaded, combine the data into one CSV file.

Note: The eSignature reports have a different set of limitations. See the [Using Reports](https://support.docusign.com/s/document-item?bundleId=cnu1642812012214&topicId=wou1578456440763.html) article to learn more about eSignature reports for senders and signers.
