# Custom Agreement Type Extractions

Read about custom agreement type extractions, including their use, purpose, and entry points in Agreement Manager.

 

Custom agreement types capture specific, unique information that isn't covered by the standard, Docusign-provided types. For example, when the standard agreement types don't match the organization's taxonomy or vertical.

Other reasons for creating custom agreement type extractions:

- To classify agreements consistently across the Intelligent Agreement Management (IAM) platform
  using relevant type names.
- To include custom data points in search results, filters, dashboards, and reports to reflect
  expanded business-specific types.
- To track the critical, business-specific data in your agreements.
- To provide more robust and tailored data in your agreement insights and reporting.

## Agreement Manager entry point

Custom agreement types are created and managed from the *Agreement Types Manager*. These tasks are completed by users with Organize and manage all agreements access to Agreement Manager.

This feature is accessed from the Agreement Manager homepage under the Settings (gear) menu:

The Manage Agreement Types page opens. A list of standard and custom agreement types for the account is displayed in the Agreement Types table. From this page, administrators and contract managers can add new agreement types and enable them for AI detection.

Use the + Agreement Type option to create new custom agreement types:

The New agreement type dialog box opens. The first part of the configuration process is to provide agreement type details:

![A screenshot of the New agreement type dialog box that is used to provide agreement type details, such as Name and Category.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Custom-agreement-type-extractions_New-agreement-type_dialog-box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc1ODQsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.n9rs-NRMst_5GS3JtC0SDVxVOmmEFGBw4WYYcLfiXmY&_LANG=enus "A screenshot of the New agreement type dialog box that is used to provide agreement type details, such as Name and Category.")

The custom agreement type is pre-populated with standard, Docusign-provided fields to provide structure for the type. These fields are added automatically depending on the selected category from the previous step.

Select additional fields for the custom agreement type to ensure that it meets your business-specific needs:

![A screenshot of the Select fields for type dialog box that prompts you to add custom and standard fields to the new type.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Custom-agreement-type-extractions_Select-fields-for-type_dialog-box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc1ODQsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.n9rs-NRMst_5GS3JtC0SDVxVOmmEFGBw4WYYcLfiXmY&_LANG=enus "A screenshot of the Select fields for type dialog box that prompts you to add custom and standard fields to the new type.")

After saving the new type, the agreement type details page opens. The custom and standard fields that are associated with the type are displayed in the Fields table.

The custom agreement type was created, however, the data points are not surfaced yet in your
agreements. Complete the AI detection setup process to surface the new custom
extraction in your new and historical agreements. See the [Configure New Custom Agreement Types](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=gke1727282238086.html&_LANG=enus "Learn how to create custom agreement types to support organizational use cases.") topic to learn how to complete this task.

## AI detection set up wizard

After configuring the custom agreement type, use the Set Up option to enable the AI detection for the custom extraction. This option has two access points: the Agreement Types table and the agreement type details page.

![A snippet of the agreement type details page that highlights the Set Up option in the top right.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Custom-agreement-type-extractions_agreement-type-details_Set-Up-option_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc1ODQsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.n9rs-NRMst_5GS3JtC0SDVxVOmmEFGBw4WYYcLfiXmY&_LANG=enus "A snippet of the agreement type details page that highlights the Set Up option in the top right.")

The AI detection setup wizard opens. Complete three steps to activate your custom agreement type extractions:

![A screenshot of the setup wizard that introduces the three easy steps to enable the custom agreement type extraction.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Custom-agreement-type-extractions_Setup-wizard_Get-Started.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc1ODQsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.n9rs-NRMst_5GS3JtC0SDVxVOmmEFGBw4WYYcLfiXmY&_LANG=enus "A screenshot of the setup wizard that introduces the three easy steps to enable the custom agreement type extraction.")

2. **Choose good examples of this type of agreement.** This step provides examples to show the AI model how the data displays in your agreements. Once activated, the custom extraction is surfaced in your new agreements by default. The data points are included in your agreement insights, such as filters, searches, dashboards, and reports.

You can select specific historical agreements to process for the new agreement type. See the [Find Custom Agreement Types in Specific Documents](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tskb991328c-dc5f-4d45-9617-dada91a5672a.html&_LANG=enus "Learn how to use the Find On Specific Documents function to surface custom agreement types in selected agreements.") topic to learn about processing your historical agreements for custom types.

## Auto-suggestion for custom agreement types

When uploading or ingesting agreements to Agreement Manager, if an agreement does not match an existing standard or custom type, it is identified as Miscellaneous. If 10 or more similar agreements are identified as Miscellaneous, a custom agreement type is suggested.

For example, 10 charter agreements were uploaded and identified as *Miscellaneous*. The agreements are recognized as being similar and a custom agreement type, *Charter Agreement*, is suggested:

![A snippet of the agreement list table that shows a suggested custom agreement type for a Miscellaneous document.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Autosuggest_New-suggestion-available_dialog-box_charter-agreement-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc1ODQsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.n9rs-NRMst_5GS3JtC0SDVxVOmmEFGBw4WYYcLfiXmY&_LANG=enus "A snippet of the agreement list table that shows a suggested custom agreement type for a Miscellaneous document.")

Use the Create option and follow the steps to create a new custom agreement type. See the [Use Auto-Suggestion to Create Custom Agreement Types](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk2093dc24-1da8-4393-b4cd-526990bc7118.html&_LANG=enus "Learn how to create a custom agreement type from the auto-suggestion feature.") article to learn more about using this feature.

Once created, the custom type is applied to all the relevant documents that were previously identified as Miscellaneous. Depending on the processing queue, it may take an hour or longer for the agreements to reflect the new type.

You can filter the agreement list table using the new custom type:

![A snippet of the agreement list table that shows the new custom type in the Agreement Type filter.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Autosuggest_Agreement-Type-filter_Charter-Agreement-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc1ODQsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.n9rs-NRMst_5GS3JtC0SDVxVOmmEFGBw4WYYcLfiXmY&_LANG=enus "A snippet of the agreement list table that shows the new custom type in the Agreement Type filter.")
