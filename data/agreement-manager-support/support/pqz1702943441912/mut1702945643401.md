# The Agreement Preview Page

Read about the purpose, usage, and key components of the Agreement preview page. This topic answers the question: Where can I see my agreement details and obligations?

The Agreement preview page has two primary components:

![A screenshot of the Agreement preview page that shows the Details pane, including data points that were surfaced within the document.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Navigator_Agreement-Preview-page.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus)

1. The pane on the left displays Details, Obligations, Agreement sets, and Related agreements for the selected agreement:
   - Details **pane:** Displays attributes and attribute values that were either manually created or surfaced within the agreement.
   - Obligations **pane:** Displays a list of obligations that were either manually created or surfaced within the agreement.
   - Agreement sets **pane:** Displays a list of sets to which the agreement has been added.
   - Related agreements **pane:** Displays a list of documents that belong to the same hierarchy as the selected agreement.

   Note: The CLM Attributes pane displays a list of CLM-specific attributes that are associated with the selected document.

   ![A snippet of the Agreement preview page that highlights the CLM Attributes icon in the pane on the left.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-Preview-page_CLM-Attributes-tab_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus)

   This pane is read only and is available for documents that were brought into Agreement Manager from Docusign CLM. Once surfaced in Agreement Manager, you can use these attributes to search and filter the agreement list table.

   **Availability:** This feature is only available to CLM customers who purchased an IAM Enterprise plan. Also, the integration between CLM and IAM must be enabled. See the [Docusign CLM Integration with the IAM Platform](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=cona491d69c-2859-4515-882c-db04db58d439.html&_LANG=enus "Read about Docusign CLM integration with the Intelligent Agreement Management (IAM) platform. CLM customers can extract more value from agreements and use Docusign Iris for important agreement insights.") article to learn more about this integration.
2. The Preview pane on the right displays an image of the full agreement.

Data review is integral to accurate reporting, forecasting, and agreement management.

- Select Download from the top-right corner to download a PDF version of the selected document.

- Select Activity Log to view agreement-level events for the agreement.

For contract managers and analysts, the Agreement preview page and the Details pane provide a look into the actual agreement.

Note: The Agreement type value determines which attributes are surfaced within the agreement. Only attributes that are relevant to the identified agreement type are surfaced and displayed in the Details pane. ![A snippet of the Agreement preview page that shows an example search term that was found within the document.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-preview-page_Search-dialog-box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus "A snippet of the Agreement preview page that shows an example search term that was found within the document.")

- Select the Search (magnifying glass) icon from the top-right corner of the preview pane. A dialog box opens.
- Enter a keyword or search string in the Find in Document field. If the value exists, the keyword or search string is highlighted in the agreement.
- Use the Next and Previous arrows to scroll through the results.

Data reviews are completed to validate the attribute values that were surfaced within the agreement. This is an important aspect of Agreement Manager. Business decisions, negotiations, obligations, and renewals rely on the accuracy of these insights.

## Document-level notifications

Document-level email notifications help you stay informed about upcoming milestones for individual documents. These notifications can be set from the Agreement preview page. Select the Set a notification (bell) icon in the top-right corner to access this function:![A snippet of the Agreement preview page that shows the Set a notification icon.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-preview_Manage-Set-a-notification_snippet.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus)When you select the bell icon, the Renewal Notice Date and Expiration Date notifications are enabled by default. Other notification types can be configured by using the + Add option.

Note: When you enable the notification, you are designated as the recipient and all frequency options are selected. Use the Edit link to update the frequency and recipients for the notifications as needed.

![A screenshot of the Notifications function that shows options for tracking agreement milestones.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Document-level-notifications_Notifications-window.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus)

## The Details pane

The attributes and field values that were manually created or surfaced with AI are displayed in the Details pane. When the Details pane is in *edit* mode, the attribute values can be updated, deleted, and verified.

![A screenshot of the Agreement preview page that highlights the Details pane that is in the edit mode.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-Preview-page_Review-and-edit-details-pane.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus "A screenshot of the Agreement preview page that highlights the Details pane that is in the edit mode.")

When you select either the Edit (pencil) icon or the Review All option, the attribute values become editable. Attributes are grouped based on their usage.

|  |  |
| --- | --- |
| A snippet of the Details pane that shows there are four suggested attribute values in the General section that require review. | Review the suggested details for each group.   - A progress bar is displayed at the top of each group, including the number of suggestions that require review. - An image of the spark icon used to denote the use of AI. A spark icon is displayed next to the values that require validation. - An image of the checkmark indicator that signifies the suggestion has been reviewed. A Review completed (checkmark) icon displays once the value is verified to indicate that the attribute was reviewed. A purple checkmark is also displayed. - Groups with no suggested attribute values are collapsed by default. |

Note: The Party Role is surfaced during agreement processing. This extraction identifies the role that a specific party plays within the context of an agreement. Verify the surfaced Party Role value and if necessary, select a new role from the dropdown:

![A snippet of the Details pane that shows the Party Role dropdown and available roles.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-preview_edit-Details_Party-Roles-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus)Once identified and validated, you can filter agreements to view the agreements where a selected party plays a specific role.

Important: Any changes made to the attribute values in the Details pane do *not* change the content of the actual document.

During a data review, you can:

- Accept the suggested attribute value.
- Delete the suggested value and enter a new attribute value.
- Enter a custom value if the suggested value is not an exact match (applies to attributes with dropdowns).
- View the suggested value within the agreement to validate it against the agreement. Select (click) an attribute to view the value in the context of the agreement. The agreement jumps to the location of the value and highlights it.

When you accept a value, that value becomes saved as the attribute value and the spark no longer displays.

Important: When you place the Details pane in *edit* mode, the ENVELOPE ID field no longer displays in the pane. You cannot update the ENVELOPE ID field for an agreement.

## "AI update available" messaging

When organizations are first onboarded to Agreement Manager, their historical documents are ingested and processed. During processing, attributes and field values are surfaced based on the current AI model. By default, when a new AI model is available, the previously ingested and uploaded agreements are *not* reprocessed for the new insights.

When users interact with insights that were released since their initial onboarding, they are prompted to update their AI model. This messaging also displayed in the Details pane on the Agreement preview page:

![A snippet of the Agreement preview page that highlights the message about the latest AI model.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Reprocessing_Agreement-preview_AI-update-available_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus "A snippet of the Agreement preview page that highlights the message about the latest AI model.")Use the Update Details option to update your AI model, including processing outdated agreements for new attributes and insights. See the [Agreement Manager Reprocessing Experience](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=con44590246-0f0e-43b4-9692-caadb57335ac.html&_LANG=enus "Learn about the Agreement Manager reprocessing experience, including its purpose, trigger points, and job size considerations.") topic to learn more about the reprocessing feature.

## Saving changes to the Details pane

Whether you enter a value or accept the suggested value, be sure to save the Details pane before leaving the page. The Save option becomes active after you add a value or accept a suggested value.

The spark icon displays until a user validates the corresponding value:

- Select the Accept AI Suggestion checkmark and Save your changes to validate the attribute value.
- Update the attribute value manually and Save your changes.

  The entered value is now the verified attribute value - as indicated with a ![An image of the checkmark indicator that signifies the suggestion has been reviewed.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-preview-page_Details-pane_suggestion-reviewed-checkmark.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus) Review completed (checkmark) icon and a purple checkmark.

The document remains in the "review queue" until all the suggested field values have been accepted or updated. Once you Save your changes, the Details pane is reverted back to read-only mode.

## Agreement details from eSignature documents

When preparing agreements for an envelope or template, the preparer can add these agreement detail fields:

- Automatic Renewal (checkbox)
- Contract Value
- Effective Date
- Expiration Date
- Renewal Notice Period
- Renewal Term

By adding these fields and entering their values, the preparer simplifies the review process for the completed agreement and ensures greater accuracy. These values display in the Details pane in Agreement Manager for the completed document. Once in Agreement Manager, these attribute values are used for contract analytics, forecasting, renewals management, and reporting purposes.

## The Obligations pane

Obligations, such as payments, renewal notices, and termination notices are surfaced in agreements with AI. These AI-assisted insights are displayed in the Obligations pane on the Agreement preview page.

![A screenshot of the Agreement preview page that highlights the Obligations pane.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Navigator_Agreement-Preview_Obligations-pane-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1OTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.gbvNuKu9YJ3xgo939Q_amfJ12P78Lej3E64bsF2iJ80&_LANG=enus "A screenshot of the Agreement preview page that highlights the Obligations pane.")

Note: Users can manually capture and define obligations from the Agreement preview page.

Obligations are surfaced in English (U.S.) agreements only. The AI model that processes obligations is run from a U.S. server and can only process English agreements in this region.

If your language and region are not supported, manually create the obligations to include this information in dashboards, reports, and other insights.

Review the [Obligation Management](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=csm1727128259486.html&_LANG=enus "Read about obligation management, including how to use this functionality to capture, track, and fulfill your agreement obligations.
