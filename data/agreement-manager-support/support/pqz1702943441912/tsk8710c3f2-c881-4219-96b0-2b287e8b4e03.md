# Edit the Metadata Spreadsheet

Learn how to edit the metadata spreadsheet file to bulk update the agreement data for the selected documents.

## Before you begin

Edit the spreadsheet to include the new metadata values. When you edit the spreadsheet, follow the formatting rules and do not change the template structure. For example, do not insert columns and rows. And do not edit the first two rows or the first three columns.

Make sure you are editing the Docusign-generated, pre-formatted spreadsheet. See the [Download the Pre-Formatted Spreadsheet File](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk1b94afc6-712f-4d71-868e-0787867354d8.html&_LANG=enus "Learn how to download the pre-formatted spreadsheet file for a group of documents.") article for steps to download the file.

This topic provides suggestions for editing the metadata spreadsheet. The pre-formatted spreadsheet contains no data values. Populate the spreadsheet with the relevant agreement data for the selected documents.

Important: The spreadsheet is pre-formatted to contain system-generated agreements IDs and field-mapping data. Do not edit or delete these values. This data is located in the first two rows and in the first three columns.

1. Open the pre-formatted spreadsheet file that you downloaded previously.

   The spreadsheet file opens. It is pre-formatted to include the selected agreements, mapping data, and all the custom and standard agreements in the account.

**Hide columns in the spreadsheet**

2. Hide the columns that you are not editing. The spreadsheet contains all the custom and standard fields in your account in a scrolling worksheet. Hide the columns to make it easier to update the relevant columns:

   ![A snippet of the pre-formatted spreadsheet that highlights the Hide option for a selected column.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Hide-columns_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ffGJ7rxBvf-E9rBKOpx6jZ4yVK-qrfcAaxLkfD0ccs0&_LANG=enus "A snippet of the pre-formatted spreadsheet that highlights the Hide option for a selected column.")

   1. Select the columns that you want to hide. Hold the Shift key and select the columns that you want to hide.
   2. Right-click at the top of a selected column and select the Hide option.
   3. Repeat this process until the spreadsheet reflects the fields that you want to populate.

   The selected columns are hidden in the spreadsheet. While you can remove columns from the spreadsheet, it is not recommended.

   Warning: Do not remove or re-order the first three columns in the spreadsheet. The Agreement ID (GUID), File Name (TEXT), and Agreement Type (TEXT) columns are integral to successfully updating the selected documents with the new metadata.

**Remove unnecessary agreements (rows)**

3. Remove any agreements that you do not want to update. While you cannot add rows to the spreadsheet, when necessary, you can remove them:

   ![A snippet of the pre-formatted spreadsheet that highlights the Delete option for a selected row.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Delete-row_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ffGJ7rxBvf-E9rBKOpx6jZ4yVK-qrfcAaxLkfD0ccs0&_LANG=enus "A snippet of the pre-formatted spreadsheet that highlights the Delete option for a selected row.")

   1. Select the rows that you want to delete. Hold the Shift key and click on the row numbers for the rows that you want to delete.
   2. Right-click on a selected row select the Delete option.
   3. Repeat this process until the spreadsheet reflects the agreements that you want to update.

   The rows are removed from the table. While you can remove rows from the table, it is not recommended.

   Warning: Do not remove or edit the first two rows in the spreadsheet, including the field names in the header row. The field names in the header row are pre-formatted from Agreement Manager and the second row contains field-mapping data. Both rows ensure that the new metadata is mapped to the correct agreements and fields. Do not alter these rows.

**Populate the data cells**

4. Enter values for the relevant metadata. Be sure to follow the [field-specific formatting requirements](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=refa47e5582-efd8-403b-b39c-45a0c05beb1f.html&_LANG=enus "Look up spreadsheet rules and formatting requirements for using the Bulk Metadata Edit.") when populating the spreadsheet:
   1. Text fields have a 2,000-character limit for custom fields and a 400-character limit for standard fields.

      Important: Depending on how the standard fields are used across the platform, the maximum character limit can vary.
   2. Duration field values are entered using the format, [Number][Space]{Time Unit].
   3. Date field values are entered using the standard, U.S. format, MM-DD-YYYY.
   4. Currency field values are entered using the format, [Number][Space][Currency code].
   5. Multi-values are entered as comma-separated values.

   Tip: If you are entering the same value for all the agreements, populate the cell in row 3 and copy it down to the other rows:![An animated screenshot of the metadata spreadsheet that shows an example of entering the same value for all the agreements.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_populate-data-cells_fill-down_example.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ffGJ7rxBvf-E9rBKOpx6jZ4yVK-qrfcAaxLkfD0ccs0&_LANG=enus "An animated screenshot of the metadata spreadsheet that shows an example of entering the same value for all the agreements.")

**Edit the Parties column**

Use the Parties column to normalize your party data for the selected agreements.

5. Enter the party data in the Parties column. If you copy and paste the values, be sure to follow the correct format, [Party Name] | [Role]. When adding two or more parties, separate the values with a semicolon:

   ![A snippet of the pre-formatted spreadsheet that highlights a value in the Parties column.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Enter-parties-data_format.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ffGJ7rxBvf-E9rBKOpx6jZ4yVK-qrfcAaxLkfD0ccs0&_LANG=enus "A snippet of the pre-formatted spreadsheet that highlights a value in the Parties column.")

   Important: The party role is required when populating cells in the Parties column. If you are unsure of the role or do not want to include one, enter "None" as the role. Use the format, [Party Name] | [None].

   For example, Fontara | None.

**Editing dropdown fields**

When populating dropdown fields, the values must match at least one of the options in the dropdown. For example, the NDA Type field accepts either Mutual or One Way.:![A screenshot of the NDA Type field that has two options, Mutual and One Way.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_example-drop-dwon-field_dropdown-options.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ffGJ7rxBvf-E9rBKOpx6jZ4yVK-qrfcAaxLkfD0ccs0&_LANG=enus "A screenshot of the NDA Type field that has two options, Mutual and One Way.")

6. Enter values for the dropdown field in the relevant cells:

   ![An animated screenshot of the pre-formatted spreadsheet that shows an example of populating a dropdown field (column).](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Bulk-metadata-editing_Enter-dropdown-value.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY4MTMsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ffGJ7rxBvf-E9rBKOpx6jZ4yVK-qrfcAaxLkfD0ccs0&_LANG=enus "An animated screenshot of the pre-formatted spreadsheet that shows an example of populating a dropdown field (column).")

   Tip: Verify your values against the dropdown options in Agreement Manager before adding the data to the spreadsheet.

   If the value that you enter does not match any of the options, the field fails to update for that agreement. An error is logged on the Job status page.

**Review and save your changes**

7. Review the spreadsheet for formatting errors or file structure issues. For example, additional rows and columns or missing agreement IDs and mapping values. Update your data as needed.
8. Save and close the spreadsheet.

You have successfully edited the pre-formatted spreadsheet file with new agreement data for the selected documents. Import the spreadsheet back into your account to update the selected documents. Continue to the next step of the process, [Import Your Edited Metadata Spreadsheet](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk5ab1cd5b-00f7-42a9-bd77-728baadff903.html&_LANG=enus "Learn how to import your edited metadata spreadsheet using the Add (+) menu located on the Agreement Manager homepage.").
