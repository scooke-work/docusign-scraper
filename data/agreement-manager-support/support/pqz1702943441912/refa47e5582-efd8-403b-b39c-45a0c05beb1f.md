# Formatting Requirements for Metadata Editing

Look up spreadsheet rules and formatting requirements for using the Bulk Metadata Edit.

## Prerequisites

When editing the agreement metadata, always use the Docusign-generated, pre-formatted spreadsheet that you downloaded from Agreement Manager. Use the spreadsheet as a template for updating the metadata for the selected documents.

Attention: Before downloading the spreadsheet, be sure that your custom fields are enabled in Agreement Manager. The pre-formatted spreadsheet contains all the custom and standard fields in the account. If you plan on adding custom data, the spreadsheet must contain a column for the custom field. You cannot add columns and rows to the pre-formatted spreadsheet.

Use this reference when you are editing the metadata spreadsheet file and must look up field formatting requirements.

## Rules for using the spreadsheet

Explains the structural requirements for the Docusign-generated, pre-formatted spreadsheet so your upload is accepted.

Do not change the structure of the pre-formatted spreadsheet. If the structure of your spreadsheet does not match the original template, the upload is rejected.

| Component | Rules |
| --- | --- |
| Headers | - Do not change the header row. - Do not change the field names in the header row. |
| Columns | - Do not add columns. - Do not rename the column headers. - Do not edit, reorder, or remove the first three columns, Agreement ID, File Name, and Agreement Type. |
| Rows | - Do not add rows. - One row per agreement. |

While you can remove and reorder the columns, it is not recommended. Retain the original file structure as much as possible to avoid errors.

Tip: Instead of removing or reordering the columns in the spreadsheet, use the Hide function to hide the unnecessary columns.

## No-edit cells, rows, and columns

Identifies the "no-edit" areas of the spreadsheet. These areas must stay exactly as downloaded so your upload is accepted.

| Components | Rules |
| --- | --- |
| Header row (Field names) | Do not edit the field names in the header row. |
| Field-mapping row (second row) | Do not edit or delete the field-mapping values. Agreement Manager relies on this value when updated the field values with new data. |
| Agreement ID column | - Do not delete the Agreement ID column. - Do not edit the IDs in the Agreement ID column.   Agreement Manager uses the agreement ID to match the data rows to the appropriate agreements. |
| Agreement Type column | - Do not delete the Agreement Type column. - Do not edit the Agreement Type value.  Important: Associate your custom fields with the relevant agreement types in Agreement Manager before downloading the spreadsheet. If the field is not associated with the current agreement type, any values that you enter will fail validation. |
| Column layout | - Do not edit, reorder, or remove the first three columns. These values are required for mapping the metadata to the appropriate agreements. - Do not add columns to the spreadsheet. The columns require the correct field-mapping value. This value is generated when you download the pre-formatted spreadsheet.  Keep the original column layout to ensure a successful upload. Removing and reordering the columns is not recommended. |
| Row structure | - Do not add extra rows. Every row represents one of the selected agreements. - Do not delete the agreement ID value from any row. |

## Safe-to-edit cells

You can safely edit the data cells in the existing agreement rows. However, you cannot edit the header row, the field-mapping values, or the Agreement ID and Agreement Type columns.

| Question | Rule |
| --- | --- |
| Which cells can I edit? | You can edit any of the data cells under the existing field columns (header row and field-mapping row) for each agreement.  **Exceptions:**  - Do not update the header row. - Do not update the field-mapping row (second row). - Do not update the Agreement ID values. These values are important to the upload process.  Tip: Follow the required field formatting rules so your updates pass validation. |
| Can I change a field name in the header row? | No. The names default from Agreement Manager. Do not edit or delete the field names in the header row. |
| Can I replace update the Agreement ID or Agreement Type for an agreement? | No. Agreement Manager uses these fields to match rows to agreements and to verify that field data is appropriate for the type. |
| Can I overwrite existing values? | No. Do not overwrite any of the pre-populated data in the spreadsheet. This includes the Agreement ID and Agreement Type values, and the field-mapping data in the second row. |
| Can I leave the fields blank? | Yes, you can leave fields blank if the field is not relevant to the agreement or the agreement type. |

## Field format requirements by type

Lists the required formats for each field type so your updates pass validation.

| Field type | Rule or requirement | Example |
| --- | --- | --- |
| Text (length) | Enter up to 2,000 characters for custom text fields and up to 400 characters for standard fields.  Important: Depending on how the standard fields are used across the platform, the maximum character limit can vary.  Longer values fail validation and an error is logged in the job monitoring hub | Northern District of California |
| Date | Enter dates in the standard, U.S. format, MM-DD-YYYY. | 01-01-2026 (not "2026-01-01") |
| Number | Enter digits only, no commas. Note: Decimals are approved where the field supports them. For example, 2500.75. | 10000 (not "10,000") |
| Currency (and Amount) | Enter an amount followed by the [ISO 4217 currency code](https://www.iso.org/iso-4217-currency-codes.html) using the format, [Number][Space][Currency code]. | 25 USD (not "25.00 U.S. dollars") |
| Duration | Enter a value using the format, [Number][Space][Time Unit]. Common time units are: Year, Month, Week, and Day.  Important: Capitalize the time unit value. This value is case-sensitive. | - 11 Months (not "11 M" or "11 months") - 4 Years (not "4 Y" or "4 years") |
| Multiple-value | Enter comma-separated values for multiple-value cells. | English, German, French (not "English, German, and French") |
| Party and Role fields | Enter party values using the format, [Name] | [Role]. Important:  - The party name and role values are separated by the vertical line (pipe) divider. - Use semicolon-separated entries when adding more than one party value. - The party role is required. Use the format, [Party Name] | [None], if you are unsure of the role. | For example, Party A is the buyer and Party B is the seller. This value is entered as: Party A | Buyer; Party B | Seller |
| Dropdown fields | Values in dropdown fields (cells) must match at least one of the options in the dropdown. If the value is not a valid option for the dropdown, the file is skipped and an error is logged. | For example, if your custom dropdown field has three options, the cell must contain one of those options. |
