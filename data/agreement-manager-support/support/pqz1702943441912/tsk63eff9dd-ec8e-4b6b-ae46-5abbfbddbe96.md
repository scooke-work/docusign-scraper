# View Hierarchies for Selected Parties and Groups

Learn how to locate the agreement hierarchies for selected parties and groups.

Hierarchies are created and maintained at the party-level from the Hierarchies tab. While hierarchies are associated with a specific party, documents from other parties can be added to the hierarchy. This feature supports the more complex contractual multi-party arrangements.

Important:

Access to various hierarchy-related functionalities is based on a
user's document-level access. For example:

- **Viewing documents in a hierarchy.** Users can view the documents in a hierarchy if they have Can View, Can Edit, or Can Manage access to the document.

  If you do not have the appropriate document-level access, the documents are not visible in the
  hierarchy.
- **Adding documents.** When creating and updating hierarchies, users can only add documents to which they have Can Edit or Can Manage access.

  If you do not have the appropriate document-level access, you cannot add the agreement to the hierarchy.
- **Editing hierarchies.** Users can only edit a hierarchy if they have Can Edit or Can Manage access to all the documents in the hierarchy.

  If you do not have the appropriate document-level access, you cannot edit the hierarchy. For
  example, you cannot update details, add documents,
  or remove documents.
- **Rearranging document relationships in a hierarchy.** Users must have Can Edit or Can Manage access to all the documents in a hierarchy to rearrange or change the relationship between documents.

  If you do not have the appropriate document-level access, you cannot rearrange or update the document relationships for a hierarchy.

This topic reviews the steps to access hierarchy details from the party hub. Begin by locating the relevant party or group from the Parties page.

1. Log in to your Docusign account.

   The Docusign Home page opens.
2. Select Agreements from the global header.

   The Docusign
   Agreements page opens. The Inbox view
   is displayed by default.
3. Expand the PARTIES section in the navigation pane on the left and select the All option.

   The Parties page opens and a list of individual and merged parties
   is displayed in the table.
4. Select the relevant party or group from the parties list.
   1. Enter the name of the party or group in the Search parties field and press [Enter].

      Parties and groups matching the keyword are displayed in the table.
   2. Select the party or group from the table.

   The party hub opens for the selected party or group. The
   Overview tab is displayed by default.
5. Select the Hierarchies tab.

   Hierarchies are created and managed from the IAM party hub under the Hierarchies tab. From this page, you are prompted to create new hierarchies or update existing ones.
6. Create a new hierarchy for the selected party or group as needed.
   1. Select the Create Hierarchy option.

      The Create HIerarchy wizard opens. The Define step is displayed.
   2. Complete the Define step. Enter a Title and Description for the hierarchy and select the Next option.

      The Create Hierarchy wizard advances to the Add step.
   3. Complete the Add step. Locate the relevant agreements and select the + Add option for the agreements that you want to add to the hierarchy. Select the Next option.

      The Create Hierarchy wizard advances to the Arrange step.
   4. Complete the Arrange step. Designate the Primary agreement and reorder the other agreements to reflect the appropriate relationship structure. Select the Finish option.

   The wizard closes and you are returned to the Hierarchies tab. The new hierarchy is displayed in the table. See the [Create a Hierarchy of Related Agreements](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk1f4c26ae-7477-4293-8b06-b2354f54271c.html&_LANG=enus "Learn how to create a hierarchy of related agreements for a selected party or group.") article to learn more about creating hierarchies.
7. Update an existing hierarchy as needed.

   Important: You must have Can Edit or Can Manage access to all the documents in the hierarchy to edit the hierarchy.

   1. Select (click) the hierarchy that you want to update.

      The hierarchy details page opens.
   2. Update the hierarchy details. Select the Edit (pencil) icon located to the right of the hierarchy title. Update the Title and Description fields and select the Save option.

      You are returned to the hierarchy details page for the selected hierarchy.
   3. Use the Add Documents option to add additional agreements to the hierarchy.
   4. Update the relationship structure of the hierarchy. Drag the agreements to the appropriate location within the hierarchy. You can also expand the Arrange actions (vertical ellipsis) menu and use the Move options to reorder the agreements.
   5. Remove agreements from the hierarchy. Expand the Arrange actions menu for the relevant agreement and select the Remove from Hierarchy option.
   6. Select the Update option after making the necessary changes.

   The hierarchy is updated. See the [Manage Your Existing Hierarchies](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tskf2e938fc-cb7f-4c65-a7bf-1cf1433722a2.html&_LANG=enus "Learn how to maintain your existing hierarchies, including updating the relationship structure between the related agreements.") article to learn more about updating hierarchies.

You have successfully viewed the hierarchies that are associated with a selected party or group.
