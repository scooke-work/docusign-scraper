# The Agreement List Table

This topic answers the question: What can I do from the agreement list table?

The agreement list table is displayed on the Completed Documents view. The table contains a list of agreements to which the logged-in user has access. This includes their own sent, shared, and received agreements.

![A screenshot of the Completed Documents view that shows a list of documents to which the logged-in user has access.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Completed-view_updated-with-document-count.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0NTIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ACZHxzt7_lfXi8KeUbSzXOgjz0spT5QFz54bA6YYN_c&_LANG=enus)

The total document count is displayed in the bottom-right corner of the view. This count is
updated when you perform a search or apply filters.

Tip: The
Envelope ID column displays the envelope ID that is
associated with the corresponding document. Select the Envelope
ID link to view the envelope details page. See the [View Envelope Details from the Agreement List Table](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk15aecb4b-98ec-4b40-a739-61a3715cba02.html&_LANG=enus "Learn how to view the envelope details for an agreement that was imported from a completed eSignature envelope.") topic to learn more about
viewing envelope details.

Common cards are:

| Card topic | Description |

| We found HR agreements | This card is displayed when your repository has HR agreements and non-HR agreements. Important: The visibility of these cards is dependent on your Agreement Manager permissions and your Intelligent Agreement Manager (IAM) plan. For example, the HR-related cards are visible for accounts that are enabled with IAM for Human Resources (HR).

## Filter options

Use the Filters option to filter the agreement list table by various attributes (fields):

![A snippet of the Completed Documents view that highlights the Filters option, including suggested filters.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Completed-documents-view_Available-filter-pills.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0NTIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ACZHxzt7_lfXi8KeUbSzXOgjz0spT5QFz54bA6YYN_c&_LANG=enus)

Select the filter that you want to apply, choose the relevant filter value, and select the Apply option. The filter is applied to the agreement list table.

**Suggested filters**

You can also select a filter from the list of Suggested filters. For example:

- Use the Sets filter to view agreements in a specific agreement set. For
  example, a project or business unit portfolio.

  Note: Only
  sets that were created by or shared with the logged-in user are available
  for selection.
- Use the Status: filter to focus on agreements at a specific lifecycle stage. For example, Active, Inactive,
  Expiring Soon, Renewing Soon,
  and Effective Soon.
- Use the
  Parties: filter to find agreements involving a specific customer, partner, or vendor.
- Agreement Type: use this filter to view only a particular type of agreement. For example, non-disclosure agreements (NDAs), master service agreements (MSAs), or statements of work (SOWs).
- AI Suggestions: use this filter to surface the agreements
  with AI-recommended insights requiring review. You can also find agreements with
  no suggestions.
- Use the Envelope Completed Date filter to find agreements by when the related envelopes were completed. For example, last month's completions.
- Use the Envelope Sent Date filter to focus on agreements by when their envelopes were sent. For example, a specific reporting period.
- Use the Envelope Type filter to narrow the list by the Docusign eSignature envelope type or template.

**The "View All Filters" option**

Select the View All Filters option to access all the filter options. The Filters dialog box lists all the Applied and Available filters:

![A screenshot of the Filters dialog box that shows additional filters that you can apply to the agreement list.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Completed-documents-table_All-Filters-dialog-box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0NTIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ACZHxzt7_lfXi8KeUbSzXOgjz0spT5QFz54bA6YYN_c&_LANG=enus)

Scroll through the list or perform a keyword search to locate the filter you want to apply. Expand the filter, choose the relevant filter value, and select the Apply option. Repeat these steps to apply additional filters as needed.

After applying the filters, select the Save option at the bottom of the dialog box. The modal closes and the table is updated to reflect that filters that you applied.

Note: Multiple filters and search criteria can be used together.

- Multiple values within the same filter imply an "or" relationship. For example, select two parties to view agreements for either contracting party.
- Multiple values in separate filters with entered search criteria imply an "and" relationship.

## Filter by 'Party Role'

The Party Role is surfaced during agreement processing. This extraction identifies the role that a specific party plays within the context of an agreement.

Once surfaced, you can filter the agreement list table to view the agreements where a selected party played a specific role. For example, use the Party Role filter to view your buy-side and sell-side agreements with a specific party.

The Party Role filter is located under the Parties filter:![A screenshot of the Parties filter that highlights the Party Role dropdown and available roles.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Filter-by-party-role_Parties-filter_Party-Role_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0NTIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ACZHxzt7_lfXi8KeUbSzXOgjz0spT5QFz54bA6YYN_c&_LANG=enus)

Important: When using the Parties filter, you must also select a Party Role value. If you do not know which role to select, accept the default value, All.

Search for and select at least one party from the list and choose the relevant role. The
Party Role filter requires you to select a party before
it can be applied.

See the [Filter for Agreements by Specific Party Role](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk6f2903c9-7ae7-4ad5-8acd-1c4948c98d10.html&_LANG=enus "Learn how to update the agreement list table to view agreements based on the role played by a specific party.") topic to learn more about using the Party Role filter.

## 'Status' column and filter

Agreement statuses provide a quick look at where the agreement is within its life cycle. These statuses are calculated based on an agreement's extracted Effective Date and Expiration Date:

![A snippet of the Completed Documents view that shows the Status filter and filter options.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-statuses_Completed-view_Status-filter_expanded.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0NTIsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.ACZHxzt7_lfXi8KeUbSzXOgjz0spT5QFz54bA6YYN_c&_LANG=enus)

See the [Agreement Statuses](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=con6a3bd080-da17-4d1c-a92a-b9b9340c44fd.html&_LANG=enus "Read about agreement statuses, including how they are calculated, visibility within Agreement Manager, and which types of statuses are being tracked.") topic to learn more about this feature.

## Reprocessing historical agreements

As new or updated AI models are released, the previously ingested agreements can be processed again to surface these data points. Depending on the type of new insights that are available, users can choose to:

- Opt in for reprocessing from the Track obligations badge (obligation-related attributes and insights)
- Reprocess all the outdated agreements for the account (new or updated AI model)
- Reprocess selected agreements (custom agreement types and fields)

Note: See the [Agreement Manager Reprocessing Experience](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=con44590246-0f0e-43b4-9692-caadb57335ac.html&_LANG=enus "Learn about the Agreement Manager reprocessing experience, including its purpose, trigger points, and job size considerations.") article to learn more about these reprocessing features.

Processing happens in the background. While it does not impact your current experience, processing may take some time depending on the number and size of agreements. Once started, processing cannot be canceled.

For example, if the user has obligation management as part of their plan, opting in for additional processing includes obligation-related insights.
