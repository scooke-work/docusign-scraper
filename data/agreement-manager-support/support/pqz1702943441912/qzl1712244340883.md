# Agreement Manager Search Capabilities

Read about Agreement Manager search capabilities, including enhanced search options.

With Agreement Manager, you have a single place to go to search across all of your completed agreements within your account. The search experience shows contextual help to craft search queries using synonyms, concepts, and everyday language.

## Search capabilities and features

The Agreement Manager search capabilities and features provide a robust search experience, including advanced search queries and filters.

| **Capability** | **Description** |
| --- | --- |
| **Advanced search queries** | Enter Boolean operators, proximity searches, term boosting, and wildcards directly in the search field to combine multiple conditions in a single query. Advanced search queries give you more control over how results are matched and ranked.  See the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn more about these queries. |
| **Boolean operators** | Use Boolean operators when performing advanced search queries to combine search terms. The supported operators, *AND*, *OR*, and *NOT*, enable you to specify must-have, optional, and excluded terms in your search query. You can also require or exclude terms with the + (plus) and - (minus) signs.  See the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn more about Boolean operators. |
| **Boosting terms** (Boosting) | Boost terms to give extra weight to specific words or phrases in your advanced search queries. Agreements that contain those terms rank higher in the search results list, without excluding other relevant matches.  This search feature uses a caret (^) symbol followed by a boost factor to prioritize the preceding term. For example, place greater importance on "patient" when searching for "patient registration form". Enter patient^2 registration form in the search field.  See the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn more about boosting terms. |
| **Exact phrase matching** | Use quotes to search for an exact word or phrase. Exact searches match only the specific characters that you type. They do not include variations such as plurals, verb tenses, or partial words.  Note: Quoted searches look for the exact sequence of characters and ignore other matching logic. As a result, these searches often return fewer results than regular keyword searches. See the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn more about exact phrase matching. |
| **Keyword search** | When users search for a keyword or phrase, the system searches the full agreement content to find exact keywords.  Tip: When using quotation marks in a query, the search returns documents that contain the exact word or phrase inside the quotation marks. |
| **Natural language (NL) search queries** | Using conversational language and tone, enter a question or phrase into the search field on the Completed Documents page. The query returns a direct answer to the question and a list of relevant agreements.  Important: Depending on the nature of the question posed, not all queries yield both direct answers and a list of agreements. Some queries may only provide a list of relevant agreements.  Note: The NL search function is currently available in beta. See the *Opt-in for NL search (Join Beta)* section of the [Natural Language Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=fas1739987568042.html&_LANG=enus "Read about how natural language search queries, including components of the Search Results view.") article to join the beta.  See the [Natural Language Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=fas1739987568042.html&_LANG=enus "Read about how natural language search queries, including components of the Search Results view.") article to learn more about NL queries. |
| **Natural language search suggestions** | Users can enter search queries using conversational, everyday language. As you enter keywords or phrases, a list of personalized suggestions is displayed. These suggestions are designed to enhance the discoverability and help users find relevant information more easily. |
| **Party role filter** | The Party Role is surfaced during agreement processing. Once identified, users can filter their agreements, agreement data, and insights by Party Role: A screenshot of the Parties filter that highlights the Party Role dropdown, including filter options.  For example, filter for agreements where a selected party played a specific role, such as Buyer or Seller.  Tip: This filter is available from:  - Agreement list table (on the **Completed Documents** view) - Agreements Dashboard (under the **Insights** tab) - Party list table (on the Parties page) |
| **Proximity search** | Find agreements where two or more words appear near each other, even if they are not side by side.  This search uses a quoted phrase followed by a tilde (~) symbol and a proximity number (N). The search returns results where the terms occur within *N* words of each other.  See the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn more about proximity searches. |
| **Query string search** | Find agreements by combining multiple operators and features in a single query string. You can enter any combination of Boolean logic, phrases, proximity, boosts, and wildcards into a single expression.  See the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn more about query string searches. |
| **Recent searches** | The search remembers recent searches and displays them for easy future reference. Users can also clear recent searches. |
| **Related terms** | The keyword search automatically includes synonyms and related concepts in the search results. A snippet of the Completed Documents page that shows search results and the Show related terms dialog box.  Users can capture all relevant documents even if the exact search term is not used. |
| **Saved searches** | The Save option gives you the option of saving your search criteria for future use. Saved searches include entered keywords, applied filters, and customized columns. |
| **Search feedback** | The Rate as helpful (thumbs up) and Rate as unhelpful (thumbs down) icons give users a chance to rate the search results:  A snippet of the Completed Documents page that shows the feedback icons used to rate search results.  After you select an initial rating, a dialog box opens and you are encouraged to provide additional details.  Restriction: The feedback rating icons only display when you perform a keyword search, including when filters are detected based on the search terms. This function is not available when you use only applied filters. |
| **Semantic search** | Semantic search improves the accuracy of search results by understanding:  - the intent of the search query - the contextual meaning of search terms |
| **Spelling assist** | The search automatically corrects spelling errors in queries. Users can find relevant information even if mistakes exist. |
| **Text normalization** | The search finds the most relevant agreements regardless of the specific word form used in the query. |
| **Type-ahead suggestions** | The search bar provides type-ahead suggestions as the user enters a search query. |
| **Wildcard search** | Perform a wildcard search to match variations of a word without typing every form and related terms. You can use wildcards to capture spelling or naming variations. For example,  - Use an asterisk (\*) to match one or more characters. - Use a question mark (?) to match exactly one character.  See the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn more about wildcard searches. |

These search capabilities lead to a more accurate, flexible, and efficient retrieval of information.

## Exact match (quoted) versus regular (keyword) searches

Quoted searches look for the exact sequence of characters and they ignore other matching logic. These searches return fewer results than regular keyword searches.

| Exact match (quoted) searches | Regular (keyword) searches |
| --- | --- |
| **Purpose:** Used to search for an exact work or phrase. | **Purpose:** Used to search for a general keyword, including variations and related word forms. |
| **Results list:** Small, narrow, focused. This search produces fewer results than the regular keyword search. | **Results list:** Broad, can contain related word forms and variations. This search produces more results than exact searches. |
| **Search behavior:** Exact searches match only the specific characters that you type. | **Search behavior:** The keyword search includes variations, such as plurals, verb tenses, or partial words. |

## Saved search feature

Saved searches are stored sets of search criteria that include keywords, filters, and custom columns. The search defines how to find documents and it displays the data that is relevant to you.

When you run a saved search, the search criteria are applied to all the documents in the repository. Since agreements are regularly added to Agreement Manager, the search results can change over time. You can run a saved search to retrieve an updated list of documents that match the saved criteria.

Important: The Saved Search feature is available at the user level. All account users have permission to save their frequently used search queries.

When saving search criteria, all search parameters are saved, including:

- Keywords or phrases
- Filters
- Date ranges
- Natural language (NL) questions
- Custom column layouts

Important: When saving your search criteria, the exact search term (keyword, phrase, or question) is saved.
