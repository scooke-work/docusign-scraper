# Use Case: Give Groups Permission to View All Agreements in an Account

This topic answers the question: How do I enable access to Agreement Manager agreements for a group of users?

By default, users can only view their own shared, received, sent, and uploaded agreements in Agreement Manager. An administrator can grant access for groups to view all agreements in the account.

Note: User permissions are managed at the account-level. Make sure that you are updating records from Accounts administration instead of Organization administration.

1. Log in to your Docusign account using administrator credentials.

   The Docusign Home page opens.
2. Select Admin from the global header.

   The Admin page opens.

   Important: User and group permissions are managed at the *account*-level. Make sure that you are updating records from Accounts administration instead of Organization administration.If your organization has multiple accounts, verify that you are updating the correct account. If necessary, select Accounts from the pane on the left and choose the appropriate account.
3. Select Groups from the USERS AND GROUPS section in the navigation pane.

   The Groups view opens.
4. Locate the group whose Agreement Manager permissions you want to update:

   1. Enter the group name in the Enter Keyword field.
   2. Select Search.

   The table displays groups matching the search criteria that you entered.
5. Expand the Actions dropdown associated with the group that you want to update and select Manage Agreement Manager Access.

   The group profile view opens for the selected group. The Agreement Manager Access tab is displayed by default.
6. Turn on the View all agreements in this account toggle (blue position) from the Agreement Manager access section.

   By turning on this toggle, you are giving the group, and all group members, access to view all agreements in the account.

   Note: Turn this toggle off (gray position) to remove the group's access to view all agreements in the account.
7. Select Save Changes.

   Next, go to the audit log to verify the changes.
8. Select Audit Logs from the AUDITING section in the navigation pane.

   The Audit Logs view opens. The audit log table displays records of changes to your account.
9. Locate the change in the table to verify the group's settings were successfully updated.
