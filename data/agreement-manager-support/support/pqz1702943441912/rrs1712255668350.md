# Agreement Uploads and Ingestion

This topic answers the question: What are my upload options for Agreement Manager, including limitations and required permissions?

Uploading third-party agreements is an important part of Agreement Manager. There are two upload or ingestion options based on the number of documents:

- **Bulk upload** (more than five documents)
- **Single upload** (up to five documents)

The Add (+) function is available to all users. You can perform single uploads without being assigned Agreement Manager permissions, however, bulk uploads require an additional permission.

An administrator turns on the Upload up to 1,000 files at a time toggle
(blue position) from the user profile page. Agreement Manager permissions.

During processing, attribute (field) values are surfaced from all the agreements uploaded, ingested, or synced to Agreement Manager.

number of documents being processed. When processing large data sets, the total
processing time can exceed a week. ## Historical document ingestion

All historical, completed documents in an existing eSignature account are auto-ingested into Agreement Manager by default upon provisioning. These documents are displayed in the agreement list table on the Completed Documents page.

The number of Docusign historical completed documents that you can ingest is based on the Intelligent Agreement Management (IAM) plan that was purchased. The document ingestion thresholds are:

- 500,000 (North American customers)
- 100,000 (International customers)

Important: If your document count exceeds the ingestion threshold, contact your Docusign account representative to submit a special request.

## Document ingestion and uploads

During the ingestion and upload processes, your documents are moved to Agreement Manager and processed to surface key data points. The primary sources of document uploads and ingestion are:

- Docusign eSignature
- Docusign CLM
- Cloud storage providers
- Email attachments

After the initial ingestion of historical documents, new agreements are stored in Agreement Manager
automatically when they are signed. This includes agreements that use other
Intelligent Agreement Management (IAM) features, such as:

- New signed agreements from eSignature and CLM
- Workflow Builder workflow sends
- API sends
- Integration sends, such as from Microsoft Dynamics and Salesforce

These ongoing agreements are stored in Agreement Manager as they are completed.

CAUTION:

Optical character recognition (OCR) converts images of text into machine-readable text. OCR is limited to the first 100 pages of a document. For documents with more than 100 pages, embed the text directly in a document before ingesting or uploading the file to Agreement Manager. For example, a text-based PDF.

Data extraction and AI processing considers the first 500 pages of "text". This limitation ensures performance and accuracy as large documents can impact processing times.

## Single document uploads

All users can upload up to five documents to Agreement Manager from their local drives and third-party cloud storage apps. Once uploaded, these documents are visible to users and groups with permission to view the user's sent agreements. These users must have at least View all agreements in this account access to Agreement Manager.

Limitations of single document upload include:

- File names must be fewer than 255 characters
- Files must be less than 100 MB
- Only supported file types can be uploaded

When successfully uploaded, the file displays in a list view. If processing exceptions exist, the
user receives a notification.

Important: Users with *bulk upload*
permission are prompted to assign access to the documents they are uploading -
even if fewer than five documents are being uploaded. See the *Access controls
for bulk document uploads* section in this topic to learn more about
these controls.

## Bulk document uploads

Bulk upload is available when you want to upload more than five files, such as uploading a folder of files. An administrator must grant the Bulk upload permission for users to use this function. This is done by turning on the Upload up to 1,000 files at a time toggle for a user.

Limitations of bulk upload:

- File names must be fewer than 255 characters
- Maximum of 1,000 files per bulk upload job
- Folder structure is not maintained when uploading folders and sub-folders
- Files must be less than 100 MB
- Only supported file types can be uploaded

## Access controls for bulk document uploads

During the bulk upload process, the Allow access step gives you the option to:

- Add the uploaded documents to an agreement set
- Share files with other users and groups
- Skip this step

Important: The Allow access step always displays for users with bulk upload permissions, regardless of the number of documents being uploaded.

**Add to agreement sets:**

Agreement sets are named groups of agreements that can be shared with users and groups. Sets are used to organize agreements for productivity and data privacy, including managing access at scale.

Tip: See the [Agreement Set Management](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=zed1730843285476.html) article to learn more about agreements sets.

**Share files with others:**

When sharing files with other users and groups, an access level is assigned. The choices are: Can View, Can Edit, and Can Manage. When users upload documents to Agreement Manager, they are set as the *owner* for the files in the upload job.

Note: By default, users are given Can Manage access to their uploaded and completed agreements. This includes the documents that are ingested from eSignature.

Tip: See the Direct Sharing and Agreement Downloads article to learn more about agreement access levels.

If permissions are not set during the upload, the files are visible by the owner and anyone with the View all agreements in this account permission.

**Skip this step:**

If you choose to skip the Allow access step, the files are added to Agreement Manager but they won't be organized or shared with others.

## Email attachment ingestion

Import your executed agreements from email attachments directly into Agreement Manager. This streamlined approach to ingestion reduces manual effort, improves data accuracy, and drives better collaboration.

Restriction: Email ingestion is currently available to U.S.-based customers only. Users outside of the U.S. will experience the existing user interface until this feature is generally available in early 2026.

Include the repository's unique email address in your ongoing email threads to import the
attached documents automatically:![A screenshot of the Import agreement straight from your inbox dialog box that highlights the Copy Address option.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Email-ingestion_Copy-Addresss-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3MDksInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.qAQx_cuf-u0qP4YPfOU11-tlmOBQno6BCBss3Y2Euyg&_LANG=enus "A screenshot of the Import agreement straight from your inbox dialog box that highlights the Copy Address option.")

Tip: **Agreement Manager email addresses**

- Add the repository email address to your Contacts list for ongoing use.

  The next time you import agreements to Agreement Manager, the email address is already in your Contacts
  list. You do not have to copy the email address every time
  you want to import email attachments.
- Add the Agreement ManagerUploader@docusign.net email alias to your Safe Senders list.

  If your ingestion fails, this email alias will send you a notification. Depending on your security policies, the email may be blocked or marked as spam. Add the email address to your Safe Senders list to ensure that you receive failure notifications.

Important: **License and permission requirements**

By default, all users can upload or import up to five agreements at a time. However, to ingest email attachments, the email must come from a registered account user. Only registered account users with the appropriate upload permissions can use this feature.

The Upload up to 1,000 files at a time permission is required to ingest more than five email attachments at a time. This toggle is turned on from the user profile page by an administrator.

Limitations of email attachment ingestion:

- File names must be fewer than 255 characters
- Only supported file types can be ingested

Attention: The maximum file size for email
attachment ingestion is limited by your email provider. The common file size
limit for popular providers, like Gmail, Outlook, and Yahoo is 25 MB. Some
providers, like Outlook, might have lower limits for certain account types, such
as accounts with internet service providers (ISPs). For these accounts, the file
size limit is 20 MB.

See the [Import Documents from Email Attachments](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=ask1749568023897.html&_LANG=enus "This topic answers the question: How do I add documents from email attachments to Agreement Manager? Learn how to import documents from email attachments to Agreement Manager for processing.") article to learn more about using this function.

## Auto-suggestion for custom agreement types

When uploading or ingesting agreements to Agreement Manager, if an agreement does not match an existing standard or custom type, it is identified as Miscellaneous. If 10 or more similar agreements are identified as Miscellaneous, a custom agreement type is suggested.

For example, 10 charter agreements were uploaded and identified as *Miscellaneous*. The agreements are recognized as being similar and a custom agreement type, *Charter Agreement*, is suggested:

![A snippet of the agreement list table that shows a suggested custom agreement type for a Miscellaneous document.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Autosuggest_New-suggestion-available_dialog-box_charter-agreement-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3MDksInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.qAQx_cuf-u0qP4YPfOU11-tlmOBQno6BCBss3Y2Euyg&_LANG=enus "A snippet of the agreement list table that shows a suggested custom agreement type for a Miscellaneous document.")

Use the Create option and follow the steps to create a new custom agreement type. See the [Use Auto-Suggestion to Create Custom Agreement Types](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk2093dc24-1da8-4393-b4cd-526990bc7118.html&_LANG=enus "Learn how to create a custom agreement type from the auto-suggestion feature.") article to learn more about using this feature.

Once created, the custom type is applied to all the relevant documents that were previously identified as Miscellaneous. Depending on the processing queue, it may take an hour or longer for the agreements to reflect the new type.

You can filter the agreement list table using the new custom type:

![A snippet of the agreement list table that shows the new custom type in the Agreement Type filter.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Autosuggest_Agreement-Type-filter_Charter-Agreement-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3MDksInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.qAQx_cuf-u0qP4YPfOU11-tlmOBQno6BCBss3Y2Euyg&_LANG=enus "A snippet of the agreement list table that shows the new custom type in the Agreement Type filter.")

Important: The agreements contain attributes and field values that were surfaced with AI. Complete a data review to validate these values, including the Agreement Type. ## Monitor uploads

The Monitor All Jobs function gives you access to job details for agreement uploads and processing jobs. This job monitoring hub provides a centralized location to view and track the status of these jobs.

Access this function from the Settings (gear) menu on the Agreement Manager homepage:

![A snippet of the Completed Documents view that shows the Uploads and Processing Status option in the Settings dropdown.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Completed-documents-view_Actions_Monitor-All-Jobs-option.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3MDksInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.qAQx_cuf-u0qP4YPfOU11-tlmOBQno6BCBss3Y2Euyg&_LANG=enus "A snippet of the Completed Documents view that shows the Uploads and Processing Status option in the Settings dropdown.")

The All Jobs page opens. The Uploaded Agreements view is displayed, including the job details and status information the upload jobs.

![A screenshot of the All jobs page that is used to monitor all upload jobs, including the job name, number of documents, and status.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Completed-documents-view_Monitor-All-Jobs_All-JobsUploaded-tab.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY3MDksInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.qAQx_cuf-u0qP4YPfOU11-tlmOBQno6BCBss3Y2Euyg&_LANG=enus "A screenshot of the All jobs page that is used to monitor all upload jobs, including the job name, number of documents, and status.")

The created date, job name, number of documents, and status for all uploads are displayed in the table. Use this view to:

- Monitor job progress
- Cancel an upload job
- View upload failures and processing exceptions
- Download a list of failed, skipped, or canceled files
- View the status of initiated upload jobs, such as in-progress, completed, failed, and canceled
- View the documents that were uploaded for a selected job

Tip: Use the Actions dropdown to Browse
