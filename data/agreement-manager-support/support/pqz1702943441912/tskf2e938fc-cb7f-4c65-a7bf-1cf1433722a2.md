# Manage Your Existing Hierarchies

Learn how to maintain your existing hierarchies, including updating the relationship structure between the related agreements.

## Before you begin

The selected party or group must have existing hierarchies to complete various maintenance tasks.

Important:

Access to various hierarchy-related functionalities is based on a
user's document-level access. For example:

- **Viewing documents in a hierarchy.** Users can view the documents in a hierarchy if they have Can View, Can Edit, or Can Manage access to the document.

  If you do not have the appropriate document-level access, the documents are not visible in the
  hierarchy.
- **Adding documents.** When creating and updating hierarchies, users can only add documents to which they have Can Edit or Can Manage access.

  If you do not have the appropriate document-level access, you cannot add the agreement to the hierarchy.
- **Editing hierarchies.** Users can only edit a hierarchy if they have Can Edit or Can Manage access to all the documents in the hierarchy.

  If you do not have the appropriate document-level access, you cannot edit the hierarchy. For
  example, you cannot update details, add documents,
  or remove documents.
- **Rearranging document relationships in a hierarchy.** Users must have Can Edit or Can Manage access to all the documents in a hierarchy to rearrange or change the relationship between documents.

  If you do not have the appropriate document-level access, you cannot rearrange or update the document relationships for a hierarchy.

Perform these tasks to maintain your existing hierarchies:

- Update the hierarchy details
- Add documents to the hierarchy
- Change the primary agreement
- Reorder the agreements in the hierarchy
- Remove agreements from the hierarchy

This topic reviews the steps to update an existing hierarchy from the party hub. Begin by locating the relevant party or group from the Parties page.

1. Log in to your Docusign account.

   The Docusign Home page opens.
2. Select Agreements from the global header.

   The Docusign
   Agreements page opens. The Inbox view
   is displayed by default.
3. Expand the PARTIES section in the navigation pane on the left and select the All option.

   The Parties page opens and a list of individual and merged parties
   is displayed in the table.
4. Select the relevant party or group from the parties list.
   1. Enter the name of the party or group in the Search parties field and press [Enter].

      Parties and groups matching the keyword are displayed in the table.
   2. Select the party or group from the table.

   The party hub opens for the selected party or group. The
   Overview tab is displayed by default.
5. Select the Hierarchies tab.

   Hierarchies are created and managed from the IAM party hub under the Hierarchies tab. From this page, you are prompted to create new hierarchies or update existing ones.

   ![A screenshot of the Hierarchies tab for a selected party or group that highlights an existing hierarchy.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Contract-Hierarchy_Hierarchies-tab_Select-hierarchy_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc4NTYsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.9jEFiUrDPtdJq67TbNqC-bFBUVYqhiA9-YZMI9HA3GE&_LANG=enus "A screenshot of the Hierarchies tab for a selected party or group that highlights an existing hierarchy.")
6. Select (click) the hierarchy that you want to update.

   The hierarchy details page opens for the selected hierarchy. Perform the various maintenance tasks as needed.

   ![A screenshot of the hierarchy details page for a selected hierarchy that lists the related agreements.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Contract-Hierarchy_Hierarchy-details-page.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc4NTYsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.9jEFiUrDPtdJq67TbNqC-bFBUVYqhiA9-YZMI9HA3GE&_LANG=enus "A screenshot of the hierarchy details page for a selected hierarchy that lists the related agreements.")
7. Update the hierarchy details.
   1. Select the Edit (pencil) icon located next to the title at the top of the page.

      The hierarchy details page opens.
   2. Update the Title and Description fields.
   3. Select the Save option.

   The page closes and you are returned to the hierarchy details page. The title and description are updated to reflect the changes.
8. Add documents to the hierarchy.
   1. Select the Add Documents option.

      The Add related agreements page opens.
   2. Scroll to the table at the bottom of the page. This is the table of available documents.
   3. Enter search criteria and apply filters to locate the relevant documents.
   4. Select the + Add option associated with the documents that you want to add to the hierarchy.
   5. Select the Update option.

   The page closes and you are returned to the hierarchy details page. The additional documents are listed in the Related Agreements section.
9. Designate a new primary document.

   ![A snippet of the hierarchy details page that highlights the Arrange actions menu options.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Contract-Hierarchy_Create-Hierarchy_Arrange-actions-menu.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTc4NTYsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.9jEFiUrDPtdJq67TbNqC-bFBUVYqhiA9-YZMI9HA3GE&_LANG=enus "A snippet of the hierarchy details page that highlights the Arrange actions menu options.")

   1. Expand the Arrange actions (vertical ellipsis) menu for the relevant agreement.
   2. Select the Make Primary option.

   The Related Agreements section updates and the new Primary document is displayed at the top of the table.
10. Reorder the agreements to create structured relationships between them. Consider these options:

    - Hover the cursor over the dots in the File Name column for the agreement that you want to move. When the "grabber" cursor displays, drag the agreement to the appropriate location.
    - Expand the Arrange actions menu and choose to move the agreement up, down, or to a different level.Repeat this step until the hierarchy reflects the appropriate relationship structure between the agreements. Select the Update option.

    The hierarchy details page reflects the new agreement relationships.
11. Remove unnecessary agreements.
    1. Expand the Arrange actions (vertical ellipsis) menu for the relevant agreement.
    2. Select the Remove from Hierarchy option.
    3. Select the Update option. Your changes are saved.

    The hierarchy details page updates and the agreement no longer displays in the Related Agreements list.
12. Return to the Parties page. Select Parties from the breadcrumb navigation at the top of the page.

    You are returned to the Parties page. The search criteria and filters are no longer applied to the table.

You have successfully updated an existing hierarchy for a selected party or group.
