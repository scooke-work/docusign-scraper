# Agreement Set Management

Read about agreement set management including how to use sets to control access to agreements. This topic answers the question: What are agreement sets and how can I use them?

 

## Definition and overview

Agreement sets are named groups of agreements that can be shared with users and groups. These sets are task- and action-oriented. Contract managers use agreement sets to organize agreements for productivity and data privacy, including managing access at scale.

Note: Contract managers, or any users that are assigned the Organize and manage all agreements permission, have access to agreement set management. See the [Agreement Manager Permissions, Profiles, and Roles](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=eut1712157553977.html&_LANG=enus "Read about the available Agreement Manager permissions, including how users and groups are granted access to Agreement Manager agreements and example roles for each permission.") article to learn more about Agreement Manager permissions and using agreement sets for access control.

Consider this use case:

- Contract managers oversee thousands of sales agreements and need an efficient way to grant access to subsets of these agreements. For example:
  - The Renewals team needs access to all master service agreements (MSAs) expiring in a quarter without auto-renewal.
  - The U.S. Sales Team needs access to all MSAs under U.S. jurisdiction.
- With agreement sets, contract managers have a way to grant access to both teams to the appropriate agreements.

Important: There is a 1,000 set limit per account. You can create up to 1,000 agreement sets per account. If you have multiple accounts, each account can have up to 1,000 sets.

After applying the smart filters and saving the set, a recurring rule dynamically adds and removes agreements based on the fields specified in your filters. Agreements are selected if they match conditions across filters. Consider these examples:

- You applied the Agreement Type and Parties filters to the set. The agreements must match the selected agreement type *AND* parties to be added to the set.
- You applied the Agreement Type filter to the set and selected multiple values. The agreements that match *ANY* of the selected values are added to the set.

  Tip: Some filters, such as Agreement Type and Renewal Type, support multiple values. When multiple values are selected, the agreements must match one of the values to be added to the set.

Important: The smart filters feature does *not* support these types of filters:

- **Rolling date filters:** the filter value is based on a relative time period that updates as time passes, such as "last 7 days".
- **Duration-based filters:** the filter value is based on a length of time, such as: "within 30 days".

You can update the smart filters for an agreement set at any time. Expand the More actions (three dots) menu for the set that you want to update and select Edit smart filters:

## Agreement set usage by role

Agreement set management is primarily used by contract managers, not administrators. Consider this example:

- *Administrators* manage users and groups in their organization, including granting access to software and features in the Admin portal.
- *Contract managers* create and manage sets to make sure the right people have access to relevant agreements. They also add agreements into sets and share sets with the appropriate users and groups.
- *Users and groups* view agreements in a set to focus on agreements that need their immediate attention.

Note: The separation between 'user and group management' and 'agreement set management' is intentional to address access control concerns. For example, IT administrators should not be exposed to sensitive agreement data. Contract managers use agreement sets to ensure that only the appropriate users have access to sensitive agreement data.

## Agreement sharing and access permissions

Before using the Share function, or sharing agreements in general, it's important to recognize the different access levels. They are: Can Manage, Can Edit, and Can View.

![A screenshot of the agreement access permissions for the direct sharing function.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Share-an-agreement_available-access-permissions.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTczMDcsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.4fBhIdQwi60mtUu7kLw09mDcz15TqJNIMsKkOGsT9Cw&_LANG=enus "A screenshot of the agreement access permissions for the direct sharing function.")

These permissions provide varying access to the agreements being shared:

- Can Manage:: Users can edit attribute values, and delete, share, and remove access to the agreement. Users with this access can also add the agreement to agreement sets.
- Can Edit:: Users can edit attribute values and share and download the agreement.
- Can View:: Users can read and download the agreement.

Important: These document permissions are used when setting access controls for
document uploads and shared agreement sets. See these topics to read more about access controls:

- [Agreement Uploads and Ingestion](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=rrs1712255668350.html&_LANG=enus "This topic answers the question: What are my upload options for Agreement Manager, including limitations and required permissions?")
- Agreement Set Management
- [Share an Agreement Set](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=pvb1736186431453.html&_LANG=enus "Learn how to share an agreement set with individual users and groups. This topic answers the question: How do I share groups of agreements with the appropriate people?")

By default, all users have Can Manage access to their uploaded and completed agreements. This includes the documents that are ingested from eSignature. Both senders and recipients are given Can Manage access to their completed documents that are synced from eSignature.

Note: Users can only share an agreement with individuals and groups within their account. Users must have Can Edit or Can Manage access to an agreement to share the agreement with others:

- Users with *manage* access can share an agreement and assign *edit*, *manage*, or *view* access.
- Users with *edit* access can share an agreement and assign *edit* or *view* access.
- Users with *manage* access can revoke user and group access to an agreement.
