# Rules for Updating the Spreadsheet File

Read about important rules for updating the pre-formatted spreadsheet file.

The *Bulk Metadata Editing* feature uses a pre-formatted spreadsheet to update the agreement data for a group of documents. This process relies on strict adherence to file structure and field-level formatting rules.

Tip: Bookmark the [Formatting Requirements for Metadata Editing](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=refa47e5582-efd8-403b-b39c-45a0c05beb1f.html&_LANG=enus "Look up spreadsheet rules and formatting requirements for using the Bulk Metadata Edit.") article for future reference. For example, to verify the spreadsheet rules and field format requirements.

**File structure implications**

File structure rules relate to the column and row structure of the pre-formatted spreadsheet. Retaining the structure of the Docusign-generated, pre-formatted spreadsheet is key to successfully using the *Bulk Metadata Editing* feature.

If the file structure does not match the pre-formatted spreadsheet (template), the upload is rejected. For example, if you upload the wrong spreadsheet file, such as a CSV file generated from another Agreement Manager feature.

**FIeld-level rule implications:**

The field-level rules apply to the cell values that you enter in the spreadsheet. The columns in the spreadsheet represent the custom and standard fields in your account. Formatting rules are in place to ensure consistency and data normalization.

When you import the updated spreadsheet, if a field value (cell) fails validation, Agreement Manager:

- Applies all the valid field values in the spreadsheet.
- Skips the invalid field values (cells). The files that fail to update and are logged in the Error details table for the upload job. The error log includes field-specific error messages for each of the failed files.

**View your error details**

After importing the data file, you will receive an email notification when the upload job is completed. The email shows the number of files that failed to update, including an option to view details.

## File structure rules

When you edit the pre-formatted spreadsheet file, follow the file structure rules to ensure a successful data import. If the file structure of your spreadsheet does not match the original template, the upload is rejected.

| Rule | Description |
| --- | --- |
| Start from the Docusign-generated, pre-formatted spreadsheet. | Always download a fresh spreadsheet for the agreements that you want to update. Edit only the data cells. Do not edit the header row or mapping values. |
| Do not edit the header row. | Keep the column names exactly as they are displayed in the downloaded file. |
| Do not edit the field-mapping values. | The second row of the spreadsheet contains mapping information for the fields. Do not edit these values. |
| Do not edit, reorder, or remove the first three columns, Agreement ID, File Name, and Agreement Type. | The first three columns in the spreadsheet are used to map the agreement metadata to the correct agreements. Do not change these columns to avoid errors during the upload process. |
| Do not add columns. | Keep the original column layout to ensure a successful upload. Tip: Instead of removing or reordering the columns in the spreadsheet, use the Hide function to hide the unnecessary columns. While you can remove and reorder the columns, it is not recommended. Retain the original file structure as much as possible to avoid errors. |
| One row per agreement. | The pre-formatted spreadsheet is populated with identifying and mapping data for the selected documents. Each row represents an agreement.  - Do not add extra rows. - Do not delete the agreement ID for any row. - Do not update the agreement type for any row. |
| Do not change the Agreement ID column. | Agreement Manager relies on the agreement ID to apply the metadata changes to the correct agreement.  - Do not edit the agreement ID values. - Do not add agreement ID values. - Do not remove agreement ID values. |

## Field-level rules

The field-level rules dictate the appropriate formats for editing your agreement data. When fields fail validation due to formatting errors, the details are listed on the Job status page.

| Field type | Formatting rule |
| --- | --- |
| Text fields | There is a 2,000 character limit for custom text fields and a 400-character limit for standard fields.  Important: Depending on how the standard fields are used across the platform, the maximum character limit can vary.  When the value exceeds the maximum, it fails validation and an error is logged in the job monitoring hub. |
| Date fields | Date fields use the standard, U.S. format, MM-DD-YYYY. |
| Number fields | Numbers are entered without commas. For example, 10000 (not 10,000). Note: Decimals are approved where the field supports them. For example, 2500.75. |
| Currency fields | Enter an amount followed by the appropriate [ISO 4217 currency code](https://www.iso.org/iso-4217-currency-codes.html). Use the format, [Number][Space][ISO currency code]. For example, 25 USD. |
| Duration fields | Duration fields use the format, [Number][Space][Time unit]. For example, 1 Year, 6 Months, 3 Weeks, 10 Days. |
| Multiple-value fields | When fields contain multiple values, complete the field as a comma-separated list of values. Note: This option is available for custom fields that were configured for multiple-values. If you enter multiple values for a field that was not defined for this type of input, the field fails to update. |
| Party and Role fields | The Parties value is entered with both the party name and role for each party:  - Separate the party name and role with a vertical line (pipe) divider. - Separate party values with a semi-colon. - Use the format: [Name] | [Role]; [Name] | [Role];   For example, PartyA (Buyer) and PartyB (Seller) is entered as: PartyA | Buyer; PartyB | Seller;  Note: When updating the Parties column, the Role is required. If you are unsure of the party role or do not want to include it, use the format: [Name] | [None]. |
| Dropdown fields | Dropdown fields must contain data that matches at least one of the expected dropdown values. For example, if you enter a value that is not a valid option for that field, the field fails validation. |

## Overwriting and clearing values

When you import the updated spreadsheet file, your edited values replace the current agreement data. This action includes the extracted values. Additionally, your user-provided metadata is considered "confirmed" and the fields are excluded from future updates. For example, when the document is reprocessed for additional data points.

Important: Review your spreadsheet carefully before uploading it to Agreement Manager. Once the job succeeds, the imported values cannot be undone. You cannot revert to the previous agreement data.

If you cleared values from the pre-formatted spreadsheet, the upload process removes the existing value. When you clear a value, any user-provided data is deleted and the field is empty. A new value is generated the next time the agreement is processed.
