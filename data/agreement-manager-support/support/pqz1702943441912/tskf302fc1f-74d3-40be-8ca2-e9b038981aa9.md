# Build an Advanced Search Query

Learn how to build an advanced search query by combining multiple operators and features in a single query string.

## Before you begin

Before completing this task, review the [Advanced Search Queries](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=conce9d9428-f27e-49ad-9001-3c7a0820ae78.html&_LANG=enus "Read about the advanced search query feature that gives you control over how your results are matched and ranked.") article to learn about the supported advanced query operators.

Advanced search queries give you more control over how results are matched and
ranked. You can find agreements by combining multiple operators and features in a
single query string.

Enter any combination of Boolean logic, phrases, proximity, boosts, and wildcards
into a single expression. The search returns agreements that match the query
string.

This article walks you through the steps to build an advanced search query.

1. Log in to your Docusign account.

   The Docusign Home page opens.
2. Select Agreements from the global header.

   The Docusign Agreements page opens.
3. Select Completed from the AGREEMENTS section in the navigation pane.

   The Agreement Manager homepage opens and the Completed Documents page is displayed.

**Start simple and broad**

When you enter a valid query string, the search returns a focused, predictable result set. Test each piece independently and start simple to ensure that your query string is valid and effective.

The goal of this query is to locate vendor agreements that contain non-standard limitation of liability language for legal review.

4. Perform a broad, initial search using a keyword or term. Enter the term in the search field and press the Enter key. For example, search for *liability* to view any agreements that mention "liability" in the text:

   ![A snippet of the Completed Documents page that shows a search term and the number of results.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_start-simple_enter-search-term.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A snippet of the Completed Documents page that shows a search term and the number of results.")

   The search returns any agreement that mentions "liability" anywhere in the text.

   Tip: Apply the Agreement Type filter to limit the scope to specific types of vendor contracts. Expand the Filters option, select the Agreement Type filter, choose the relevant types, and select Apply.

**Target a specific clause**

In the previous step, you located documents based on a broad keyword search to locate the term anywhere in a document. Next, narrow the search results to show agreements that contain an exact phrase.

5. Overwrite the existing query string and enter an exact phrase in the search field. Enter the quoted clause phrase and press the Enter key.

   For example, update the query string to:

   ```
   |  |  |
   | --- | --- |
   |  | "limitation of liability" |
   ```

   ![A snippet of the Completed Documents page that shows an exact phrase query string and the resultant document count.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_add-party-requirement.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A snippet of the Completed Documents page that shows an exact phrase query string and the resultant document count.")

   The search results are narrowed to show agreements that contain the exact phrase.

**Capture common variations**

An exact phrase search can narrow your search results list too much by excluding common clause variants. Update your query string to capture common clause variations.

6. Use Boolean logic to capture common clause variations in your search query. Update the existing query string so the exact phrase is grouped with the clause variants. Separate each quoted variant with a Boolean operator.

   For example, update the query string to:

   ```
   |  |  |
   | --- | --- |
   |  | ("limitation of liability" OR "limitation of damages" OR "liability cap") |
   ```

   ![A snippet of the Completed Documents page that shows the grouped clause variations and the resultant document count.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_add-agreement-type-requirement.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A snippet of the Completed Documents page that shows the grouped clause variations and the resultant document count.")

   The search results are expanded to include the common clause variations.

**Add proximity to find clause thresholds**

Your results list contains documents that contain the relevant clause, including clause variations. Next, update your query string to find specific clause language. The proximity search returns documents where specific words are near each other, as is typical in the clause.

For example, find agreements where liability cap language appears near references to contract values or fees. The documents that match this query requirement are likely to contain a defined cap amount.

7. Use the proximity search to locate specific clause details. Add a quoted phrase followed by ~ (tilde) N, where N is the maximum distance (number of words) allowed between terms. Use parentheses to capture variations in the clause details.

   For example, update the query string to:

   ```
   |  |  |
   | --- | --- |
   |  | ("limitation of liability" OR "limitation of damages" OR "liability cap") AND ("annual contract value"~5 OR "fees paid"~5) |
   ```

   ![A snippet of the Completed Documents page that shows the updated query string and the resultant document count.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_add-clause-requirement_proximity.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A snippet of the Completed Documents page that shows the updated query string and the resultant document count.")

   The query returns agreements where liability cap language appears *near* references to contract values or fees. For example, when referencing a specific cap amount.

   Tip: Apply the Parties filter to focus on a specific vendor. Expand the Filters option, select the Parties filter, choose the relevant vendor, and select Apply.

**Exclude standard, boilerplate, and template agreements**

After building your query string, consider an additional rule to clean up the search results. For example, use Boolean operators to exclude standard, boilerplate, and template agreements. These agreements add irrelevant noise to the query results.

8. Use Boolean operators to exclude standard, boilerplate, and template agreements from the results. Add a NOT statement to the query string that excludes these types of agreements.

   ```
   |  |  |
   | --- | --- |
   |  | ("limitation of liability" OR "limitation of damages" OR "liability cap") AND ("annual contract value"~5 OR "fees paid"~5) NOT (template OR standard OR boilerplate) |
   ```

   ![A snippet of the Completed Documents page that shows the updated query string and resultant document count that excludes standard, boilerplate, and template agreements.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_exclude-examples-tests-templates_Boolean.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A snippet of the Completed Documents page that shows the updated query string and resultant document count that excludes standard, boilerplate, and template agreements.")

   The search results are updated to exclude the standard, boilerplate, and template agreements.

   Tip: Apply the Status filter to limit the query results to active, live contracts. Expand the Filters option, select the Status filter, choose the Active option, and select Apply.

**Boost preferred terms and phrases**

Boosting is used to favor some terms, variants, or phrases over others when
ranking results. This is done without excluding any agreements outright. Review
the components of your query string and boost any relevant terms or phrases.

9. Boost the term or phrase that you want to prioritize in the search results: Add a caret (^) followed by a boost number. This weighted number indicates the relative importance of the associated term or phrase.

   For example, update the search string to boost the clause name:

   ```
   |  |  |
   | --- | --- |
   |  | ("limitation of liability"^3 OR "limitation of damages" OR "liability cap") AND ("annual contract value"~5 OR "fees paid"~5) NOT (template OR standard OR boilerplate) |
   ```

   A caret (^) and the boost number are added to the "limitation of liability" phrase:

   ![A snippet of the Completed Documents page that highlights the boosted phrase.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_prioritize-term-phrase_Boosting.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A snippet of the Completed Documents page that highlights the boosted phrase.")

   1. Locate the term or phrase that you want to prioritize when ranking results.
   2. Add a caret (^) to the end of the quoted term or phrase. The caret (^)
      goes on the outside of the quote.
   3. Add the weighting number to indicate the relative importance of the term or phrase.

      Important: Large boosts can produce unintuitive relevance. Use small integers to start.
      The suggested range is 2–5. Boosts work within the advanced query, however,
      filters still control which agreements are included in the results. If a boosted
      term never appears in the results, it has no effect.
   4. Press the Enter key.

   The query results are re-ordered to account for the boosted term or phrase.

**Save your query string**

Use the Save option to save your query string for future use. This function saved your query string, applied filters, and column customizations. Once saved, run your search at any time to retrieve an updated list of documents that match the query string.

See the [Use Saved Searches to Find Agreement Data](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tskca2d74f2-823d-403d-84a6-d990537cd187.html&_LANG=enus "Learn how to access and run a saved search, including steps to edit and delete searches.") article to learn more about running your saved searches.

10. Save your query string for future use:

    ![A screenshot of the New saved search dialog box that is used to save your search criteria, filters, and column customizations for future use.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_New-saved-serach_LoL-example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A screenshot of the New saved search dialog box that is used to save your search criteria, filters, and column customizations for future use.")

    1. Select the Save option located to the right of the filters.

       The New saved search dialog box opens. Complete the required fields to save your search terms.
    2. Enter a unique name for the search in the Name field. A system-generated name defaults automatically. Overwrite this value as needed.
    3. Enter a Description of the search in the text-entry box, including its purpose and use.
    4. Select the Save Search option.

    The dialog box closes. Your search is available in the Saved Searches list. The page title and the breadcrumb navigation are updated to reflect the name of the saved search:

    ![A snippet of the saved search, including the query string and resultant document count.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Advanced-search-queries_saved-search-page.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY0MzgsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.w3kUHdl2eqIubJgQpFO_-AjSbv5LN2PPwbkYFcvNib8&_LANG=enus "A snippet of the saved search, including the query string and resultant document count.")
11. Return to the Agreement Manager homepage. Select the Completed Documents option from the breadcrumb navigation in the top left.

    You are returned to the Agreement Manager homepage.

You have successfully built an advanced search query by combining operators, terms, and modifiers into a single query string.
