# Agreement Statuses

Read about agreement statuses, including how they are calculated, visibility within Agreement Manager, and which types of statuses are being tracked.

Agreement statuses provide a quick look at where the agreement is within its life cycle. Users gain clarity on which agreements are active or inactive without manual tracking:

![A screenshot of the Completed Documents view that highlights the Status column in the agreement list table.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-statuses_Completed-view_Status-column_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1NTUsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.-7zu4GnWbqwoYo98Shf7cWEeh7sAibIQKBEH51Ardc0&_LANG=enus)

These statuses are calculated based on an agreement's extracted Effective Date, Expiration Date, and Renewal Type. The statuses are re-calculated and updated daily, based on the current date.

Note: A Renewal Type value is not required for calculating an agreement's status. However, if an extracted value exists, it is taken into consideration when calculating the status. For example, if an agreement has an Expiration Date and the Renewal Type is Evergreen, the agreement is always Active.

There are two types of agreement statuses:

- **Primary status:** indicates whether an agreement is *active* or *inactive*.
- **Secondary status:** defines the agreement's current state, such as *expiring soon*, *renewing soon*, and *effective soon*.

## Primary statuses

The primary statuses are calculated based on an agreement's extracted Effective Date and Expiration Date values. If the agreement has an extracted Renewal Type, this value is also taken into consideration when calculating the status.

These statuses indicate whether the agreement is Active or Inactive:

![A snippet of the agreement list table that highlights examples of the primary status in the Status column.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-statuses_Primary-statuses_highlighed.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1NTUsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.-7zu4GnWbqwoYo98Shf7cWEeh7sAibIQKBEH51Ardc0&_LANG=enus)

| Status | Calculation Explanation |
| --- | --- |
| Active | The agreement is in-effect, or active. Active agreements are calculated based on the effective and expiration dates:  - Effective date is equal to or before the current date (the agreement is "active") - Expiration date is equal to or after the current date (the agreement has not "expired")  For agreements that are missing either the effective date or the expiration date:  - No  Expiration date, but the Effective date is equal to or before the current date (the agreement is Active) - No Effective date, but the  Expiration date is equal to or after the current date (the agreement is Active) |
| Inactive | The agreement is not in-effect, or inactive. Inactive agreements are calculated based on the effective and expiration dates:  - Effective date is after the current date (the agreement is "not yet effective") - Expiration date is before the current date (the agreement has "expired")  Important: If both the Effective date and the Expiration date are missing or null, the agreement is Inactive. |

Note: If the agreement has an extracted Renewal Type, this value is also taken into consideration when calculating the status.

When available, AI-generated sub-text is displayed under the primary status:

![A snippet of the agreement list table that highlights examples of the Status sub-text.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-statuses_Status-column_subtext_highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1NTUsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.-7zu4GnWbqwoYo98Shf7cWEeh7sAibIQKBEH51Ardc0&_LANG=enus)

This sub-text is generated based on the agreement's extracted Effective
Date, Expiration Date, and Renewal Type. For example, the
sub-text indicates when an agreement is expiring or renewing and displays the
relevant date.

## Secondary statuses

While the primary statuses define whether an agreement is Active or
Inactive, the secondary statuses define the agreement's
current state. For example: Renewing Soon,
Expiring Soon, and becoming Effective
Soon:

![A snippet of the agreement list table that highlights examples of secondary statuses.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-statuses_Secondary-statuses_highlighed.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1NTUsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.-7zu4GnWbqwoYo98Shf7cWEeh7sAibIQKBEH51Ardc0&_LANG=enus)

As with primary statuses, if an agreement has an extracted Renewal Type, this value is taken into consideration when calculating the secondary status.

| Primary Status | Secondary Status | Calculation Explanation |
| --- | --- | --- |
| Active | Expiring Soon | These agreements are Active but will expire soon:  - Expiration date is within 30 days of the current date - Renewal type is not Auto-Renew or Evergreen |
| Active | Renewing Soon | These agreements are Active and will renew soon:  - Expiration date is within 30 days of the current date - Renewal type is Auto-Renew or Evergreen |
| Inactive | Effective Soon | These agreements are Inactive and are not yet effective:  - Effective date is within 30 days of the current date |

The associated renewal, expiration, or effective date is displayed in the sub-text for the secondary status.

Note: The secondary statuses are only displayed in the agreement list table in Agreement Manager. These statuses do *not* display for the agreement lists found under the Insights menu.

## Filter your agreements by "Status"

Once identified, use the Status filter to filter the agreement list table by status:

![A screenshot of the Status filter that shows selected filter values.](https://docusign-be-prod.zoominsoftware.io/api/bundle/pqz1702943441912/page/images/Agreement-statuses_Status-filter.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE3OTY1NTUsInNoZWFmIjoicHF6MTcwMjk0MzQ0MTkxMiJ9.-7zu4GnWbqwoYo98Shf7cWEeh7sAibIQKBEH51Ardc0&_LANG=enus)Select the Filters option and locate the Status filter. Expand the filter, choose the relevant agreement statuses, and select Apply. The agreement list updates and the documents that match the applied filters are displayed in the table.

## Status column and filter in Agreement Manager

The Status column and filter are associated with the agreement list table and can be found on the:

- Completed page
- Agreements page in the party hub
- Detailed report view for a selected dashboard or report

Important: If you previously customized the columns in the agreement list table, you must manually display the Status column in the table. See the [Display the Status Column in the Agreement List Table](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk57b92226-d6e4-4725-99c4-79ebf42e9f8b.html&_LANG=enus "Learn how to customize the agreement list table to display agreement status information.") topic to learn about this task.
