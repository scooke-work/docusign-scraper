# 2025 Agreement Manager Release Notes

Look up past release notes from 2025 for Docusign Agreement Manager, including features, bug fixes, and other changes.

Note: The features and enhancements referenced in this topic are available in Demo accounts, Production accounts, or both. Availability is dependent on the ramp schedule and can vary across regions and languages.

The following fixes, new features, and product enhancements have been delivered since the last release.

## Release Notes | October 31, 2025

The following fixes, new features, and product enhancements have been delivered since the last release.

| **Type** | **Category** | **Description** |
| --- | --- | --- |
| Enhancement | Analyze | **Agreement statuses:** Users can gain clarity on which agreements are active or inactive and quickly understand the state of an agreement:  - The primary statuses, Active and Inactive, are calculated based on the current date versus an agreement's effective and expiration dates. - Secondary statuses are used to further define the current state of an agreement, including: Expiring Soon, Renewing Soon, and Effective Soon.  See the [Agreement Statuses](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=con6a3bd080-da17-4d1c-a92a-b9b9340c44fd.html&_LANG=enus "Read about agreement statuses, including how they are calculated, visibility within Agreement Manager, and which types of statuses are being tracked.") article to learn more about this enhancement. |
| Enhancement | Analyze | **Auto-suggestion for custom agreement types:** When uploading agreements to Agreement Manager, if an agreement does not match an existing standard or custom type, it is identified as *Miscellaneous*. If 10 or more similar agreements are identified as *Miscellaneous*, a custom agreement type is suggested.  For example, 10 charter agreements were uploaded and identified as *Miscellaneous*. The agreements are recognized as being similar and a custom agreement type, *Charter Agreement*, is suggested:  A snippet of the agreement list table that shows a suggested custom agreement type for a Miscellaneous document.  Use the Create option to create the suggested custom agreement type. See the [Use Auto-Suggestion to Create Custom Agreement Types](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=tsk2093dc24-1da8-4393-b4cd-526990bc7118.html&_LANG=enus "Learn how to create a custom agreement type from the auto-suggestion feature.") article for steps to complete this task. |
| Enhancement | Analyze | **Expiration date extension for auto-renewing agreements:** When an auto-renewing agreement reaches its expiration date, the date is extended based on the agreement's renewal term. This action happens automatically and requires no user input.  For example, if the expiration date is January 1, 2025 with a 1-year renewal term, the expiration date updates to January 1, 2026 when the current date equals January 1, 2025.  Important: This feature relies on the accuracy of the agreement's Expiration Date, Renewal Type, and Renewal Term values. Although these values are extracted from your agreements, you should review and validate them for accuracy. See the [Validate Renewal-Related Attribute Values for an Agreement](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=sah1712251075016.html&_LANG=enus "Learn how to validate renewal-related attribute values for a selected agreement.") article to learn more about completing a data review.   The expiration date extension feature applies to agreements with a Renewal Type of Auto Renew or Evergreen.  See the [Renewals Management](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=hjg1712251035061.html&_LANG=enus "Read about how Agreement Manager supports renewal management, including available tools, use cases, and key features. This topic answers the question: What are my renewal management options for Agreement Manager agreements, including tools and key features?") article to learn more about this enhancement. |
