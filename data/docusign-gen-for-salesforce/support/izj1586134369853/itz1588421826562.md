# Learn About Adding Document Rules to a Word-Based Gen Template

Learn about using document rules to set conditional logic in Gen templates.

You can use document rules in a Gen template to enable conditional document
generation. Your Gen templates can generate different agreement types based on
conditions present in Salesforce records. Conditional generation is useful when a
Gen template has multiple versions of a contract.

## Document Rule example

Fontara has a Gen template named Global Sales Contract. The administrator
adds three different contract versions to the Gen template: English, French, and
Brazilian Portuguese.

The administrator builds a document rule in the Global Sales Contract template using
this logic:

- IF: Billing Country >  Equals >  Brazil
- THEN: Generate Brazilian Portuguese Contract

A user selects the Global Sales Contract button on a Brazilian customer's record. Docusign generates a Brazilian Portuguese-language agreement from
the record. Without the document rule, the user must manually select the Portuguese
template before generating.

## Document rule fields

Administrators can use the following Salesforce fields to create a document rule:

- BOOLEAN (using true or false)
- CURRENCY
- DATE
- DOUBLE
- EMAIL
- PERCENT
- PHONE
- PICKLIST
- STRING
- URL

## Notes on document rules

- You cannot use commas in number field rule values (CURRENCY, DOUBLE, PERCENT,
  and so on.) Example: Use 9000 instead of
  9,000.
- Do not use quotation marks in string field rule values (STRING, PICKLIST, URL,
  and EMAIL.)
- Do not use quotation marks for Boolean field rule values
  (true and false.)
- Do not use quotation marks for date field rule values that follow the YYYY-MM-DD
  format.
- You can use Date Literals in Salesforce Object Query Language (SOQL) syntax.
  Example: LAST\_WEEK, LAST\_n\_DAYS:n, and so on. Follow this link for more
  information about [entering conditional date values](https://developer.salesforce.com/docs/atlas.en-us.soql_sosl.meta/soql_sosl/sforce_api_calls_soql_select_dateformats.htm).
- Use double field rule values when using decimals for large numbers. Example: Use
  9000000000.00 to search for
  9000000000.)
- Checkbox field rule values are Boolean.
  - Selected check boxes equal true (or True or TRUE.)
  - Unselected check boxes equal false (or False or FALSE.)
- Use the currency prefix before multi-currency field rule values. Example: Use
  USD9000.00 for $9000.00.
- You can duplicate any condition with a field value, whether or not it has a
  conditional value.
- You cannot save a document rule until you add a rule name and conditions for all
  fields.
