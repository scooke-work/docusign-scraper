# I Created New Docusign Users, But They Cannot Use Gen

Learn to fix issues that prevent new users from generating agreements.

I [added new Docusign users](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=dzl1586134206658.html&_LANG=enus "Learn to add Docusign Gen, CLM, Negotiate, and eSignature users and manage permissions on the user management page.") to
our Salesforce organization.

In Docusign User Management, I see a green check mark with a status of
Successful next to both users.

One user has not received a Docusign activation email.

Another has activated their user, but cannot generate agreements.

## Resolution 1: Reset the Existing User's Password

Cause: If a user has an invalid Docusign password,
Gen actions will fail.

Use this procedure to fix the issue:

1. [Log in to the connected Docusign account](https://account.docusign.com/#/username) as an
   administrator.
2. Go to Admin >  Users.
3. Select Actions next to the user's name and choose
   Reset Password.

   ![A screenshot displays the reset password option selected in the actions menu.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/genresetpassword.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MjgsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.RYelE47L4z9TSin1YUYQUXuM0R9DILA_cpYf0HNrFZg&_LANG=enus "A screenshot displays the reset password option selected in the actions menu.")

   A message displays: A reset password email has been sent to (user
   name).

   You have completed the process. Docusign sends the user
   [a new activation email](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=ymi1586134213877.html&_LANG=enus "Learn how to activate your Docusign CLM, Gen, Negotiate, or eSignature for Salesforce user.").

## Resolution 2: Add Generator Permissions to the User

Docusign does not grant Generator permissions to new users. Add
Generator permissions for new users who cannot generate agreements.

Use this procedure to fix the issue:

1. [Open Docusign Apps Launcher as
   an administrator.](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=etv1586134206174.html&_LANG=enus "Learn to open the Docusign Apps Launcher for Salesforce package in your organization.")
2. Select User Management in the left column.
3. Select the arrow ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/Arrowhead.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MjgsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.RYelE47L4z9TSin1YUYQUXuM0R9DILA_cpYf0HNrFZg&_LANG=enus) next
   to the user and choose Edit Permissions from the dropdown menu.
4. Select Generator from the Docusign Gen
   dropdown menu.
5. Select Apply.![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/A-screenshot-displays-the-edit-permissions-page-with-generator-selected.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MjgsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.RYelE47L4z9TSin1YUYQUXuM0R9DILA_cpYf0HNrFZg&_LANG=enus)

   A message displays: Successfully edited permissions for (user
   name).

   You have completed the process. Docusign sends the user
   [a new activation email](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=ymi1586134213877.html&_LANG=enus "Learn how to activate your Docusign CLM, Gen, Negotiate, or eSignature for Salesforce user.").
