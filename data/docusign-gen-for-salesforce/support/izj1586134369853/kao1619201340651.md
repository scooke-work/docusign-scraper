# Use Group Tags in a Word-Based Gen Template

Learn how to use group tags to gather a cluster of related data. Group tags can filter a
large amount of data to get targeted results.

Use this procedure to build Group tags and add them to your Word-based Gen template:

Example: You have a catalog in your Salesforce organization with multiple
product types. Your catalog is grouped by software,
service, and hardware product
categories. You want to generate contracts that show products grouped by type. You
add Group tags to your Gen template.

When your users generate agreements, Docusign groups the available
products by type.

Use this procedure to add Group tags to your Gen template:

1. Select Copy next to the Opportunity Product
   Name anchor text.![A screenshot displays the Copy button highlighted next to the OpportunityProductName tag.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/Copy-OpportunityProductName-tag.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTgzMjYsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.IK1MYnZGen_Rl6w515A1OSZ-m6AiORbP0J4S38mwFlU&_LANG=enus)
2. Add a table with XML tags for product name and category to your Word document:
   1. Open the Gen template Add Documents page and
      select your Word document.

      The document preview page opens.
   2. Select Download and open the document in
      Microsoft Word.
   3. Create an empty, two-column table in your Gen Word template.
   4. Enter Name in the column 1 header and
      Category in the column 2 header.
   5. Paste the Name anchor text that you copied from
      Gen into the cell below the Name column:

      |  |  |
      | --- | --- |
      | Name | Category |
      | `<# <Content Select="./Name"/> #>` |  |
   6. Enter `<# <TableRow Select="./GroupItems/Item" />
      #>` at the start, to fit the formatting recognized by a Word
      table. The updated field reads: `<# <TableRow
      Select="./GroupItems/Item" /> #><# <Content Select="./Name"
      /> #>`

      Note: In the following example, replace the word
      `Item` with the list you are using. For example,
      `<# <TableRow Select="./GroupItems/OpportunityLineItems" />
      #>`

      |  |  |
      | --- | --- |
      | Name | Category |
      | `<# <TableRow Select="./GroupItems/Item" /> #><# <Content Select="./Name" /> #>` |  |
   7. Return to the Gen template and select Copy next
      to the Category tag.
   8. Paste the Category anchor text into the cell below
      the Category column:

      | **Name** | **Category** |
      | --- | --- |
      | `<# <TableRow Select="./GroupItems/Item" /> #><# <Content Select="./Name" /> #>` | `<# <Content Select="./Category"/> #>` |
3. Enter the Group Select tags just above your table in the
   Word document:
   1. `<# <Group Select ="/Opportunity//OpportunityLineItems"
      By="./Category" /> #>`
   2. Here are all `<# <Content Select="./GroupName" />
      #>` items:
4. Add the **`<# <EndGroup/> #>`** tag just below the
   table in your Word document.

   Important: The
   finished table with header and footer tags displays as follows:

   `<# <Group Select ="/Opportunity//OpportunityLineItems"
   By="./Category" /> #>`

   Here are all `<# <Content Select="./GroupName" />
   #>` items:

   | Name | Category |
   | --- | --- |
   | `<# <TableRow Select="./GroupItems/Item" /> #><# <Content Select="./Name” /> #>` | `<# <Content Select="./Category"/> #>` |

   `<# <EndGroup/> #>`
5. [Re-upload the document to your Gen
   template](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=sqp1586134339354.html&_LANG=enus "Learn to update your Gen template Word documents.").
6. Select Save and Close and select
   Save on the Do you want to save
   changes? dialog box.

   You have added your table with Group tags. The tags you added perform
   the following functions:

   | Tag | Purpose |
   | --- | --- |
   | `/Opportunity//OpportunityLineItems` | Finds all of your Opportunity Line Items on a record. |
   | `By="./Category"` | Filters Opportunity Line Items by category. |
   | `<# <Content Select="./GroupName" /> #>` | Required for any Group by tag type. **Do not customize this tag or it will not work correctly**. |
   | `<# <TableRow Select="./GroupItems/Item" /> #>` | Required for any Group by tag type. |
   | `<# <EndGroup/> #>` | Ends the group tag. Must be placed outside the table. |

   When a user generates an agreement from a Salesforce record, the tables
   display as follows: ![A screenshot displays the generated output of the Group tables that you built in this process.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/Group-tag-generated-output.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTgzMjYsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.IK1MYnZGen_Rl6w515A1OSZ-m6AiORbP0J4S38mwFlU&_LANG=enus)

You have completed the process. You have added a table in your
Word document using Group XML tags. When users generate agreements with this Gen
template, Docusign generates separate tables for each product
type.
