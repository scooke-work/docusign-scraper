# I Want to Clone My Sandbox Gen Template and Use It in Production

Learn to move Gen templates from one Salesforce organization to another.

I built a Gen template for my Salesforce sandbox organization. I want to use the template
in my production organization without completely rebuilding it.

Can I copy my sandbox Gen template and import it to production?

## Resolution 1: Create Sandbox Data Tags in Your Production Organization

Cause: Gen templates created in a sandbox do not work in Production.
The sandbox template's Word document contains anchor text tags from the Sandbox
organization.

If sandbox anchor text tags don't exist in production, the Word template finds nothing to
merge and fails.

You can fix the issue by creating the template's sandbox fields in your production
organization.

Use this procedure to fix the issue:

1. [Edit the Gen template Upload Documents page as a Salesforce
   administrator](https://support.docusign.com/s/document-item?topicId=nyj1621662912687.html&_LANG=enus).
2. Select a document name.

   The document preview page opens in a new tab or window.
3. Select Download and open the document in the Microsoft Word
   program. Note the Salesforce data field tags that are present.
4. Log out of your sandbox session and log on to your Salesforce production
   organization as an administrator.
5. [Edit the Gen template Add Fields page as a
   Salesforce administrator](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=was1621662565775.html&_LANG=enus):
6. Add data fields that you are missing from your sandbox Gen template.
7. [Replace the Word document from your production
   Gen template with the sandbox document](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=sqp1586134339354.html&_LANG=enus "Learn to update your Gen template Word documents.").

   You have completed the process. You have added sandbox data fields to your
   production organization and uploaded the sandbox Word document. You can use the
   sandbox Gen template in your production organization.

## Resolution 2: Create an Edit Fields Button in Salesforce Production and Sandbox Organizations and Copy/Paste JSON

Cause: You have not enabled Edit Fields buttons in your Salesforce
organizations. You can turn on an option to show Edit Fields buttons on your Gen
templates. You can use these buttons to copy the raw JSON data from sandbox Gen templates
and paste them into production templates.

You can also create a custom button that exposes the JSON code. You can paste that JSON
code from sandbox Gen templates into production templates.

## Step 2: Add the Edit Fields Button to Your Sandbox Docusign Gen Template Layout

1. Select Page Layouts on the **Docusign Gen
   Template** page.
2. Select **Docusign Gen Template layout**.
3. Select the Quick Actions menu.
4. Quick Actions in the Salesforce Classic Publisher: Select the
   wrench button that displays.

   The bar turns blue and exposes a button menu.
5. Drag the Edit Fields button to the Quick Actions
   in the Salesforce Classic Publisher bar.
6. Select Save.

   ![A screenshot displays the Edit Fields button being dragged to the quick actions in the Salesforce classic publisher menu.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/gencreateeditfieldspagelayout.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0NDgsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.gBW5K8SLS3fLZzg2E0txTOlah6_BiyfkbyY1lSGjjk0&_LANG=enus "A screenshot displays the Edit Fields button being dragged to the quick actions in the Salesforce classic publisher menu.")

   **Docusign Gen Template** layout. Users can see the button
   when they open the Docusign Gen Templates tab. Move to Step 3 to
   create an Edit Fields button in your Salesforce Production organization.

## Step 3: Add the Edit Fields Button in Your Production Organization

1. Open another tab in your browser and sign into your Salesforce Production
   organization.
2. Follow the instructions from Step 1 and Step 2 to add the Edit Fields button to
   your production organization. You can paste sandbox Gen template data into these
   production Gen template fields.

   You have completed step 3. Move to Step 4 to expose the JSON in your Sandbox
   organization.

## Step 4: Expose Sandbox Gen Template Data

1. [Open Docusign Apps Launcher as
   an administrator.](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=etv1586134206174.html&_LANG=enus "Learn to open the Docusign Apps Launcher for Salesforce package in your organization.")
2. Select the **Docusign Gen Templates** tab and select the Gen
   template you want to copy for production.
3. Select Edit Fields. The Edit Fields dialog box opens:

   ![A screenshot displays the edit fields page with object mappings and signer mappings highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/geneditfieldsjson.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0NDgsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.gBW5K8SLS3fLZzg2E0txTOlah6_BiyfkbyY1lSGjjk0&_LANG=enus "A screenshot displays the edit fields page with object mappings and signer mappings highlighted.")

   You have completed Step 4. This dialog box contains the raw JSON data from
   your sandbox Gen template. These are the Gen template anchor text tags that you can
   paste into your production Gen template. Keep this browser tab open and open a new tab
   for Step 5.

## Step 5: Create a New Blank Gen Template in Production

1. Open a new browser tab and log on to your production Salesforce organization as an
   administrator.
2. Select the **Docusign Gen Templates** tab and select
   New.
3. Create a basic Gen template:
   1. Add Template Name: Enter a name for your template.
   2. Select Data Source: Select a Salesforce object to import
      data from.
   3. Select Template Editor: Select Microsoft
      Word.
   4. Select Create.
   5. Select Save and Close >  Save.
4. Select the **Docusign Gen Templates** tab and select your new
   Gen template.
5. Select Edit Fields.

   You have completed Step 5. A dialog box opens, displaying the fields from
   your new production Gen template. Keep this browser tab open. Move to Step 6 to paste
   the JSON data from your sandbox template to your production template.

## Step 6: Paste the Sandbox JSON Data Into Your Production Gen Template

1. Copy the sandbox Object Mappings JSON data to your production
   Gen template:
   1. Select the sandbox Gen browser tab.
   2. Copy the data from the first Object Mappings
      field.
   3. Delete the data from the first production Object
      Mappings field.
   4. Paste the sandbox data into the empty production field.
   5. Repeat these steps for all Object Mappings fields.
2. Copy the sandbox Signer Mappings JSON data to your production
   Gen template:
   1. Select the sandbox Gen browser tab.
   2. Copy the data from the first Signer Mappings
      field.
   3. Delete the data from the first production Signer
      Mappings field.
   4. Paste the sandbox data into the empty production field.
   5. Repeat these steps for all Signer Mappings fields.

   Note: You can also copy and paste
   the sandbox Name, Source Object, and
   Steps Completed data.
3. Select Save.

   ![A screenshot displays the Edit Fields dialog box with sandbox Gen template JSON data highlighted in the production object and signer mappings fields.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/geneditfieldsjson.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0NDgsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.gBW5K8SLS3fLZzg2E0txTOlah6_BiyfkbyY1lSGjjk0&_LANG=enus "A screenshot displays the Edit Fields dialog box with sandbox Gen template JSON data highlighted in the production object and signer mappings fields.")

   You have completed the process. You have created a production Gen template
   with the data from a sandbox template. Now that you have exposed the Edit Fields
   button, you can copy any Gen template between Salesforce organizations.
