# Learn About Using Gen Template Filtering to Import Only the Data You Need

Learn about using XML code to build custom tables in Word that filter Salesforce data.

You can use XML code to build custom tables in Word that filter Salesforce data for
Gen agreements. You can use filtering to remove unwanted data from a record and
import only the results you want.

## Filter Tag example

You have a catalog with multiple product types. You want to generate agreements from
Salesforce records that only display a certain product type. You have a
Type picklist with values for Software, Services, and
Hardware.

You build a Gen template with Microsoft Word. You add a Word table with custom XML
filtering to import only Software results.

## Sample Filter tag expressions

The following are sample expressions for filter tags:

- Only display results when one field matches certain values:

  `<#
  <TableRow Select="//Opportunity//OpportunityLineItems/Type[text() =
  ‘Software’ or text() = 'Services']/.." /> #>`
- Only display results when two fields match certain values:

  `<#
  <TableRow Select="/Opportunity//OpportunityLineItems[contains
  (./Product_Family__c,'Services') and ./Product_Rating__c/text() = '5']"
  /> #>`
- Only display results when a currency field isn’t negative:

  `<#
  <TableRow Select="//Opportunity//OpportunityLineItems/
  UnitPrice[not(contains(.,'-'))]/.." /> #>`
