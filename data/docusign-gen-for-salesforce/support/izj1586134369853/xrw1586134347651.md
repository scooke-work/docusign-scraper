# Learn About Formatting Gen Percentage, Currency, Date, Time, and Number Fields

Learn about how you can enhance Gen field formatting to customize your
templates.

You can enhance Gen field formatting to customize your templates.

Gen templates automatically match the Salesforce settings for percentage, date, time,
number, and currency field formatting.

Use the following guidance when adding these fields in a Gen template:

## Percentage Field

Gen template percentage fields have enhanced formatting options and requirements. These
include:

- Ability to include or exclude the percentage sign (%)
- Accepting as many decimal places as specified on the field, including
  .00
- Inheriting Salesforce locale format for decimal places separators: comma as
  opposed to period (123,00 as opposed to 123.00)
- Inheriting Salesforce locale format for thousand separators: comma as opposed to period
  (123,456,789.00 as opposed to 123.456.789,00)

## Date Field

Date fields inherit the same format as specified in Salesforce Locale settings. Short Date
and Custom Date options can also be used.

- Short Date options: (using January18, 2020 as an example)
  - 18/1/2020
  - 18-01-20
  - 18-Jan-20
  - 18-01-2020
  - 18.01.2020
  - 18-Jan-2020
  - 18 January 2020
  - 1/18/2020
  - 01-18-2020
  - 01/18/2020
  - 01/18/20
  - Jan-18-2020
  - Jan 18, 2020
  - January 18, 2020
  - 2020-01-18
  - 2020-Jan-18
  - 2020/01/18
  - 2020 January 18
- Custom Date: Enable a text field where users can customize
  dates.
  - Example 1: (English)
    - **Date to display**: Wednesday, March 14, 2020
    - **User input**: dddd, MMMM D, YYYY
  - Example 2: (Spanish)
    - **Date to display**: Miercoles, 14 de Marzo de 2020
    - **User input**: dddd, D de MMMM de YYYY
- No validation is needed on custom input: Salesforce default
  preferences for Language and Date Format dictate the Date Format on a Gen template.
  Example: A user's Salesforce language is set to French. The Date
  Format is set to D MMMM YYYY. The Gen output format results in 6 janvier 2020, not 6
  January 2020.

## Date & Time Field

Date & Time field formatting in Gen templates matches the formatting set in the
connected Salesforce organization. Users can customize dates when you enable a Date &
Time text field.

This field includes the following options:

- Locale Default: display the same format as specified in
  Salesforce Locale
- Short Date Options: (using January 18, 2019 as an example)
  - 18/1/2019
  - 18-01-19
  - 18-Jan-19
  - 18-01-2019
  - 18.01.2019
  - 18-Jan-2019
  - 18 January 2019
  - 1/18/2019
  - 01-18-2019
  - 01/18/2019
  - 01/18/19
  - Jan-18-2019
  - Jan 18, 2019
  - January 18, 2019
  - 2019-01-18
  - 2019-Jan-18
  - 2019/01/18
  - 2019 January 18
- Custom Date: Enable a Text Field where users can customize
  dates:
  - Example 1: (English)
    - Date to display: Wednesday, March 14, 2019
    - User input: dddd, MMMM D, YYYY
  - Example 2: (Spanish)
    - Date to display: Miercoles, 14 de Marzo de 2019
    - User input: dddd, D de MMMM de YYYY
    - - No validation is needed on custom input

## Number Field

Number field formatting in Gen templates matches the formatting set in the connected
Salesforce organization. This includes: Gen, Negotiate, eSignature, and CLM for Salesforce
(Docusign Apps Launcher).

This includes:

- Accepting as many decimal places as specified on the field, including .00
- Inheriting Salesforce locale format for decimal places separators: comma as opposed to
  period (123,00 as opposed to 123.00)
- Inheriting Salesforce locale format for thousand separators: comma as opposed to period
  (123,456,789.00 as opposed to 123.456.789,00)

## Currency Field

Gen uses the ASP.Net core-based defaultCurrencyIsoCode to determine currency formatting.

Currency field formatting in Gen templates matches the formatting set in the connected
Salesforce organization. This includes:

- Accepting as many decimal places as specified on the field, including .00
- Inheriting Salesforce locale format for decimal places separators: comma as opposed to
  period (123,00 as opposed to 123.00)
- Inheriting Salesforce locale format for thousand separators: comma as opposed to period
  (123,456,789.00 as opposed to 123.456.789,00)
- Inheriting Salesforce currency locale for currency symbol.
- An additional option is now provided for users to print a Currency value without
  Symbol/ISO code

## Other CurrencyIsocodes

In addition, there are a number of CurrencyIsocodes which are only
specific to Salesforce and are not identified by .Net. Docusign has added
a dictionary to ensure the correct formatting.

The additional codes are as follows:

ANG, AOA, AWG, BBD, BIF, BMD, BSD, CSD, CVE, FJD, FKP, GHS, GIP, GMD, GNF, GYD, KMF, KPW,
KYD, LRD, MGA, MRU, MUR, MWK, MZN, NAD, PGK, SBD, SCR, SDG, SHP, SLL, SRD, STN, SZL, TOP,
TZS, UGX, VUV, WST, XPF
