# Use Does Not Match Conditional Logic in a Word-Based Gen Template

Learn to create "does not match" conditional logic to add or remove text when a
Salesforce field is not present.

Use this procedure to create "does not match" logic. Does not
match logic adds or removes text when a Salesforce field is not present.

1. Select the Add Fields arrow. The arrow has no text until you
   select it.
2. Select Copy![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/Arrowhead.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY1OTIsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.QsyYqvRBOoSFLMUtiIc4ywIJ7Ir_ayO0LXMuGnmDWsg&_LANG=enus)
    next to a field and choose
   Options. ![A screenshot displays the description field with the copy menu open to options and remove.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/genconditionaloptionsdropdown.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY1OTIsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.QsyYqvRBOoSFLMUtiIc4ywIJ7Ir_ayO0LXMuGnmDWsg&_LANG=enus "A screenshot displays the description field with the copy menu open to options and remove.")

   The Table Options dialog box opens.
3. Create does not match logic:
   1. Select Include other content if the conditions set for this field are
      met.
   2. Select does not match.
   3. Enter keywords to avoid matching. For example, use does not match >  Closed Lost to avoid generating agreements for opportunities you did not win.![A screenshot displays the display options field with the values set to does not match and closed lost.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/A-screenshot-displays-the-display-options-field-with-the-values-set-to-does-not-match-and-closed-lost.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTY1OTIsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.QsyYqvRBOoSFLMUtiIc4ywIJ7Ir_ayO0LXMuGnmDWsg&_LANG=enus "A screenshot displays the display options field with the values set to does not match and closed lost.")
   4. Select Save.

   The Add Fields page opens.
4. Select Copy next to the anchor text tag you updated.
5. Add the conditional logic to your Gen template:
   1. Open the Word document that you are using in your Gen template.
   2. Paste the copied anchor text into the document.
   3. Add any custom text to the tag. If you use Welcome back, valued
      customer! as custom text, the tag that results is:`<#
      <Conditional Select="/Opportunity/Description" NotMatch="New Customer" />
      #> Welcome back, valued customer! <# <EndConditional/>
      #>`
6. Select Finish when you are done editing.

   Note: You can create more advanced
   conditional logic than match and does not match. See the Conditionals topics in [Gen Advanced Features](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=pui1586134216490.html&_LANG=enus "Learn to use advanced actions when configuring your Gen templates.") for more information.

You have completed the process. You added
does not match conditional logic to your Gen template.
