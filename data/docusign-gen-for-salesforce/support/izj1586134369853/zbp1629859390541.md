# Learn How to Create and Manage Docusign Users in Salesforce

Learn how to grant Docusign app access to your Salesforce
users.

There are several ways that you can grant Docusign app access to your
Salesforce users:

[Docusign user management](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=uwf1586134275195.html&_LANG=enus)
:   Create Docusign users with the Docusign Apps Launcher
    user management page.

[Docusign Login permission set](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=xxm1623778416524.html&_LANG=enus)
:   Create Docusign users by adding Salesforce users to the Docusign
     Login permission set.

Single sign-on (SSO)
:   Create Docusign users with the Docusign Login
    permission set and your SSO identity provider.

When you grant Docusign access to a Salesforce user, Docusign searches for a matching Docusign name
and email address. If matching credentials exist, Docusign links
the Salesforce and Docusign users. If matching credentials do not
exist, Docusign creates a new user.

Important: Never change the name or email address of a
Salesforce or Docusign user after activation. If Docusign and Salesforce credentials don't match, the user may be
unable to send envelopes and access Docusign objects.

You can use the following methods to grant Docusign access to Salesforce
users:

## A new Docusign user activates from an email link

A Docusign administrator assigns CLM, Gen, Negotiate, or eSignature
for Salesforce app permissions to a Salesforce user.

The user receives an email from Docusign, selects the activation
link, and creates a password.

The user selects a Docusign
Authorize button on a record and enters their Docusign and Salesforce credentials.

## A new Docusign user activates with SSO credentials

A Docusign administrator assigns CLM, Gen, Negotiate, or eSignature
for Salesforce app permissions to a Salesforce user.

The user selects a Docusign
Authorize button on a record and enters their SSO
credentials.

## An existing Docusign user activates with standard or SSO credentials

You can grant app access to an existing Docusign user on the user management
page. Existing Docusign users can activate their Docusign app access from an email link or with SSO credentials.

Important: Make sure the Salesforce user's name and email
address match those of the existing Docusign user. If the
Salesforce user's credentials differ from the existing Docusign
user's, Docusign creates a new user.

A Docusign administrator assigns CLM, Gen, Negotiate, or eSignature
for Salesforce app permissions to a Salesforce user.

The user selects a Docusign
Authorize button on a record and enters their credentials:

- Standard authentication: The user enters Salesforce and Docusign credentials
- SSO authentication: The user enters SSO credentials

## A user with Docusign Login permissions self-activates

You can let existing Docusign users activate their Docusign app access by adding them to the Docusign Login permission
set. Existing users can self-activate using Docusign and
Salesforce credentials or with SSO credentials. This permission set lets you manage
user activation in Salesforce, not on the Docusign user management
page.

A Salesforce administrator adds a Salesforce user to the Docusign
Login permission set.

The user selects a Docusign
Authorize button on a record and enters their credentials:

- Standard authentication: The user enters Salesforce and Docusign credentials
- SSO authentication: The user enters SSO credentials

## A new Docusign user activates with SSO just-in-time provisioning

You can create new Docusign users by enabling Docusign SSO
and adding Salesforce users to the Docusign Login permission set.
Users can self-activate Docusign app access using SSO credentials.

Important:

- You must enable just-in-time user provisioning for Docusign in your SSO
  identity provider.
- You can only use just-in-time provisioning to add new eSignature for Salesforce users. You
  can add Gen, Negotiate, or CLM permissions after you create eSignature
  users.

A Salesforce administrator adds a Salesforce user to the Docusign
Login permission set.

The user selects a Docusign
Authorize button on a record and enters their SSO
credentials.
