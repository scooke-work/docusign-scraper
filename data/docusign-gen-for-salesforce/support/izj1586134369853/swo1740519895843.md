# Use Hyperforce with Docusign Apps Launcher

Learn to add domains to your allowlist so that Docusign apps work with
your Hyperforce organization.

[Hyperforce](https://help.salesforce.com/s/articleView?id=000388902&type=1) is a Salesforce infrastructure architecture
that is built for the public cloud. Hyperforce infrastructure is composed of code
rather than hardware. Hyperforce uses enhanced standards for compliance, security,
agility, scalability, and privacy.

You must update your Salesforce security settings to use Docusign apps in
your Hyperforce organization. Otherwise, Salesforce denies connections to your Docusign account and causes the integration to fail.

You can update your security settings by adding either IP ranges or
domains to your allowlist.

If you want to allow IP addresses, use the following ranges:

- [Salesforce](https://ip-ranges.salesforce.com/ip-ranges.json)
- [Docusign eSignature](https://www.docusign.com/trust/security/esignature)
- [Docusign CLM](https://www.docusign.com/trust/security/clm)

However, due to the dynamic nature of cloud environments, Salesforce recommends allowing
domains, rather than IP ranges.

Use this procedure to add domains to your allowlist:

1. Access the Trusted URLs page as a Salesforce
   administrator:

      Setup.

   The Trusted URLs page opens.
2. Add a trusted URL to the list:
   1. Select New Trusted URL.
   2. API Name: Enter a simple display name, like
      salesforce for salesforce.com.
   3. URL: Enter
      https://\*.salesforce.com.
   4. Select Save.![A screenshot displays the API Name and URL fields of the Trusted URLs page filled with Salesforce domain information.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/A-screenshot-displays-the-API-Name-and-URL-fields-of-the-Trusted-URLs-page-filled-with-salesforce-domain-information.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTYzMTQsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.uGHQ4F0O7iwFq6mF88el_XImakXijwy2LfOW5sNmYX8&_LANG=enus "A screenshot displays the API Name and URL fields of the Trusted URLs page filled with Salesforce domain information.")
3. Repeat the process to add the following domains:

   - https://\*.force.com
   - https://\*.lightning.com
   - https://\*.visualforce.com
   - https://\*.docusign.com
   - https://\*.docusign.net
4. (Optional) [Add domains from this list](https://help.salesforce.com/s/articleView?id=xcloud.setup_domains.htm&type=5) if you
   control user or server access to the internet through allowlists.
5. (Optional) For IP-restricted user profiles. To access Docusign
   Apps Launcher, Salesforce users must have valid authorization to the Docusign connected app.
   1. Copy all restricted Hyperforce IPs from your login history.
   2. Add the restricted Hyperforce IPs to your allowlist.

   You have added Hyperforce IPs to your allowlist. For the complete list of
   Hyperforce IPs, visit the [Salesforce Help site](https://help.salesforce.com/s/articleView?id=release-notes.rn_hyperforce_inboud_ips.htm&release=260&type=5).

You have completed the process. You have added domains to
your allowlist. You can use Docusign Apps Launcher apps in your
Hyperforce organization.
