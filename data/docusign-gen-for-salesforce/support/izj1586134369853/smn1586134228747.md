# "Could Not Generate Document [100] Error Occurred Reference ID" Displays When I Try to Generate or Preview a Document

Learn to fix the "Could Not Generate Document [100] Error Occurred Reference ID"
error.

Salesforce users are having problems with Gen. The "Could not generate document [100] error
occurred reference ID" error displays in the following scenarios:

- Administrators upload new documents to a Gen template
- Users try to generate agreements
- Users try to preview agreements

## WHEN DOCUMENT GENERATION FAILS FOR ALL USERS:

## Resolution 1: Audit Field Changes in Your Salesforce Organization

Cause: If you change or delete Salesforce fields that are present in
your Gen template, the template fails.

Use this procedure to fix the issue:

1. Use [Field History Tracking](https://help.salesforce.com/s/articleView?id=xcloud.tracking_field_history.htm&type=5) in your Salesforce organization to
   see if fields have been changed or deleted.
2. If fields have been changed or deleted, update your Gen template:
   1. [Remove all fields that no longer
      exist](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=vxs1586134339522.html&_LANG=enus) in your Salesforce organization.
   2. [Update all fields that changed](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=irc1586134297835.html&_LANG=enus) in your Salesforce organization.
   3. [Remove and update the fields](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=ffk1586134297621.html&_LANG=enus) on your Gen template Word document.
   4. [Upload the new version of your Word
      document.](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=gxl1586134297397.html&_LANG=enus).

   You have completed the process. You have updated your Gen template to
   reflect changes or deletions to your Salesforce fields.

## Resolution 2: Paste Your Template Content Into a New Word Document

Cause: Gen templates sometimes have hidden formatting in their Word
documents, like macros or hidden XML data. These hidden elements can prevent the template
from generating agreements. Pasting all text from your Word document into a new,
unformatted Word document fixes issues with hidden formatting.

Use this procedure to fix the issue:

1. [Edit the Gen template Upload Documents page as a Salesforce
   administrator](https://support.docusign.com/s/document-item?topicId=nyj1621662912687.html&_LANG=enus).
2. Select a document name.

   The document preview page opens in a new tab or window.
3. Select Download and open the document in the Microsoft Word
   program.
4. Paste the template's text into a blank Word document:
   1. Copy all document content from your Gen template's Word document.
   2. Create a new blank Word document and paste the copied content.
   3. Save the new Word document to your device. Use the same file name as the
      original Gen template document.
5. Return to the Gen Upload Documents page and select
   X next to the old document version. ![A screenshot of the Upload Your Word Templates section displays the Generated File Name, File Suffix, and File Format fields.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/deletewordtemplate.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MzAsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.eUaKjJTpdfQIksVRtig-pf3P7omLler9pAGwSS-1uJY&_LANG=enus "A screenshot of the Upload Your Word Templates section displays the Generated File Name, File Suffix, and File Format fields.")

   A message displays: File was successfully
   deleted.
6. Select Upload Files and follow the prompts to upload the new
   version from your device.

   A message displays: File was successfully uploaded.
7. Select Save and Close.
8. Select Finish and choose Save in the
   Do you want to save changes? dialog box.

   You have completed the process. Your old Gen template has been replaced with
   the new template, which has all the same content, but no hidden formatting.

## Resolution 3: Restore System Administrator Status to Your Integration User

Cause: The Salesforce administrator credentials that you use to
connect Docusign and Salesforce is called the integration user. Follow
this link for more information: [Apps Launcher Integration User Best Practices](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=rhi1657226496755.html&_LANG=enus "Learn how best to manage the integration user that connects your Salesforce and Docusign accounts.")

If you revoke the administrator status of your integration user, the integration fails
and Gen does not work.

Use this procedure to fix the issue:

1. As an administrator, select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/salesforce-gear_.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MzAsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.eUaKjJTpdfQIksVRtig-pf3P7omLler9pAGwSS-1uJY&_LANG=enus) and choose
   Setup.
2. Enter and select Users in the Quick Find field.
3. Restore System Administrator status to the installing Salesforce user. Or use
   different Salesforce administrator credentials to [reconnect the accounts](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=zfp1586134208988.html&_LANG=enus "Learn to connect your Salesforce organization to your Docusign account.").

   You have completed the process. You have restored the access of the
   integration user.

## Resolution 4: Check Your Salesforce Storage

Cause: Gen fails if you don't have enough storage available in your
Salesforce organization. When users generate agreements, document creation fails at the
Preview step. One of the following error messages display:

- There were some problems with your document generation. Unable to generate
  the following files: <file name>
- Unable to create a new ContentVersion for the file name : <file name>
  and SfContentDocumentId.

Use this procedure to check the issue:

1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MzAsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.eUaKjJTpdfQIksVRtig-pf3P7omLler9pAGwSS-1uJY&_LANG=enus) and choose Setup.
2. Enter and select Storage Usage in the Quick Find field.
3. Check the Storage Type section.

   Note: If the Percent Used column adds up to
   100%, you are out of storage.

   You have completed the process. You have checked the storage capacity of
   your Salesforce organization. If there is not enough storage, delete data or contact
   your Salesforce representative to purchase additional storage.

## Resolution 5: Check Your Salesforce API Limits

Cause: Gen fails if you have reached your organization's API limit.
When users generate agreements, document creation fails at the Preview step.

Use this procedure to check the issue:

1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MzAsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.eUaKjJTpdfQIksVRtig-pf3P7omLler9pAGwSS-1uJY&_LANG=enus) and choose Setup.
2. Enter and select System Overview in the Quick Find field.
3. API Usage: Check to see if you have reached your limit.

   Note: If the column adds up to 100%, you have reached your
   limit.

   You have completed the process. You have checked the API limits of your
   Salesforce organization. If you have reached the limit, wait until your levels reset
   or contact your Salesforce representative to increase limits.

## Resolution 6: Save Your Word Document in the Correct Format

Cause: You can only use .PDF or .DOCX file types in Gen templates.
If a Gen template document is saved as .DOC or another unsupported format, the template
fails.

Use this procedure to fix the issue:

1. [Edit the Gen template Upload Documents page as a Salesforce
   administrator](https://support.docusign.com/s/document-item?topicId=nyj1621662912687.html&_LANG=enus).
2. Select a document name.

   The document preview page opens in a new tab or window.
3. Select Download and open the document in the Microsoft Word
   program.
4. Open your device's Downloads folder and verify that the file extension is
   .DOCX.
5. If the file is in a different format, open it and save a copy in .DOCX
   format.
6. Return to the Gen Upload Documents page and select
   X next to the old document version.![A screenshot of the Upload Your Word Templates section displays the Generated File Name, File Suffix, and File Format fields.](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/deletewordtemplate.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MzAsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.eUaKjJTpdfQIksVRtig-pf3P7omLler9pAGwSS-1uJY&_LANG=enus "A screenshot of the Upload Your Word Templates section displays the Generated File Name, File Suffix, and File Format fields.")

   A message displays: File was successfully
   deleted.
7. Select Upload Files and follow the prompts to upload the new
   version from your device.

   A message displays: File was successfully uploaded.
8. Select Save and Close.
9. Select Finish and choose Save in the
   Do you want to save changes? dialog box.

   You have completed the process. You have replaced your old Gen template with
   a .DOCX version.

## WHEN DOCUMENT GENERATION FAILS FOR ONE USER:

## Resolution 1: Check If the User Is Active

Cause: If a Gen user has not activated their account, the document
generation process fails. To have Active status, users must create a Docusign password, and authorize the Gen app upon first use.

Use this procedure to fix the issue:

1. [Open Docusign Apps Launcher as
   an administrator.](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=etv1586134206174.html&_LANG=enus "Learn to open the Docusign Apps Launcher for Salesforce package in your organization.")
2. Select User Management in the left column.
3. Check the Status column.
4. Inform Pending users that they must [activate their user accounts](https://support.docusign.com/s/document-item?bundleId=izj1586134369853&topicId=ymi1586134213877.html&_LANG=enus "Learn how to activate your Docusign CLM, Gen, Negotiate, or eSignature for Salesforce user.").

   You have completed the process. When the user completes activation, they can
   generate agreements from Salesforce records.

## Resolution 3: Grant "Enabled Custom Setting Definitions" Permissions to Generating Users

Cause: Docusign for Salesforce users require
Enabled Custom Setting Definitions access to use Gen. If a user
cannot use Gen, add their profile to the Enabled Custom Setting
Definitions list.

Use this procedure to fix the issue:

1. Open the Permission Sets page as an administrator:
   1. Select the gear ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MzAsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.eUaKjJTpdfQIksVRtig-pf3P7omLler9pAGwSS-1uJY&_LANG=enus) and choose
      Setup.
   2. Enter and select Permission Sets in the Quick Find
      field.

   The Permission Sets page displays.
2. Open **Docusign Document Generator** Permission Sets page:
   1. Select **Docusign Document Generator**.
   2. Select Custom Setting Definitions.
   3. Select Manage Assignments.

   The DocuSign Document Generator page
   opens.
3. Add the user to the permission set: Salesforce Profile to the **Enabled Custom
   Setting Definitions** list.
   1. Select Manage Assignments.
   2. Select Add Assignment.
   3. Select the check box next to the user.
   4. Select Next.
   5. Select Assign.
   6. Select Done.

   You have completed the process. You have granted Docusign
   Gen for Salesforce access to the user.

## Resolution 4: Have the User Change Their Locale

Cause: Generation fails when Gen does not support your Salesforce
locale. Have the user switch to a known-supported locale to see if this resolves their
issue.

Have your affected user follow this procedure to fix their issue:

1. Access your Salesforce Language & Time Zone
   settings:
   1. Select the Salesforce user icon ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/izj1586134369853/page/images/salesforceusericon.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTg0MzAsInNoZWFmIjoiaXpqMTU4NjEzNDM2OTg1MyJ9.eUaKjJTpdfQIksVRtig-pf3P7omLler9pAGwSS-1uJY&_LANG=enus) and choose
      Settings.
   2. Select Language & Time Zone.
2. Locale: Select English (United
   States) and select Save.

   A message displays: Your settings have been successfully
   saved.
3. Empty your browser cache and restart the browser.
4. If switching to English (United States) resolves the issue, switch to a
   commonly-used locale with the same language. For example, if your locale is French
   (Guinea) try French (France).
5. Select Save.

   The affected user has completed the process. They have updated their locale
   to one that Gen supports.
