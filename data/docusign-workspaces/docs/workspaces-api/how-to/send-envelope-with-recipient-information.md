# How to send an envelope with recipient information

This topic demonstrates how to create an envelope in a workspace with the [Workspaces API](https://developers.docusign.com/docs/workspaces-api/) and send it with recipient information using the [eSignature REST API](https://developers.docusign.com/docs/esign-rest-api/).

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PowerShell](https://github.com/docusign/code-examples-powershell).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {DOCUMENT\_ID} | A GUID value that identifies a document within a workspace. To return a list of your workspace document IDs, use the [WorkspaceDocuments: getWorkspaceDocuments](https://developers.docusign.com/docs/workspaces-api/reference/accounts/workspacedocuments/getworkspacedocuments/) method. |
| {ESIGN\_BASE\_PATH} | A string value that forms the root of the URL required to make calls to the eSignature REST API. The base path value in the example code is set to target the developer environment at   https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the `GetUserInfo` call in production. |
| {SIGNER\_EMAIL} | A string value for the email address where the signer will receive a notification of the signing request. |
| {SIGNER\_NAME} | A string value for the full name of the signer. |
| {WORKSPACES\_BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at   `https://<domain>`  Where `<domain>` is `https://api-d.docusign.com/v1` for the developer environment and `https://api.docusign.com/v1` for production. |
| {WORKSPACE\_ID} | A GUID value that identifies your workspace. To return a list of your workspace IDs, use the  [Workspaces: getWorkspaces](https://developers.docusign.com/docs/workspaces-api/reference/workspaces/workspaces/getworkspaces/) method. |

## Step 1. Obtain your OAuth token

To make the API calls shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

Wondering which grant type suits your integration? See [Authenticate](https://developers.docusign.com/platform/auth/) for guidance on your options to get an OAuth token.

Workspaces OAuth access tokens are the same ones used for our eSignature REST API, although you must request the `signature` scope and one or more Workspaces scopes to call the Workspaces API.

See  [Workspaces authentication](https://developers.docusign.com/docs/workspaces-api/auth/) for details and the full list of Workspaces scopes.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers. These same headers will be used for both API calls shown below. Construct your headers like this, but make sure to substitute your valid OAuth access token for {ACCESS\_TOKEN}.

Although this how-to uses two different APIs, both accept the same access token and headers because the `signature` scope required by the eSignature REST API also applies to the Workspaces API.

1

2

3

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}" \

'--header' "Accept: application/json" \

'--header' "Content-Type: application/json")

## Step 3. Construct the request body for a workspace envelope draft

Next, construct a request body for a workspace envelope draft. Set an `envelope_name` and insert the appropriate workspace document ID for {DOCUMENT\_ID}.

1

2

3

4

5

printf \

'{

"envelope\_name": "Example Workspace Envelope",

"document\_ids": ["'"${document\_id}"'"]

}' >> $request\_data

## Step 4. Call the Workspaces API

Now that you have constructed your API headers and request body, make a `POST` request to the [Workspaces: createWorkspaceEnvelope](https://developers.docusign.com/docs/workspaces-api/reference/workspaces/workspaces/createworkspaceenvelope/) endpoint and include the request body to create a workspace envelope draft. Here’s how you make the call, but make sure to substitute your API account ID for `{API_ACCOUNT_ID}`, your workspaces base path for `{WORKSPACES_BASE_PATH}`, and the desired workspace ID for `{WORKSPACE_ID}`. Save the `envelope_id` from the response; you’ll need it in Step 6 to update and send the envelope.

1

2

3

4

5

Status=$(curl -s -w "%{http\_code}\n" -i \

--request POST ${workspaces\_base\_path}/accounts/

${account\_id}/workspaces/${workspace\_id}/

envelopes \

"${Headers[@]}" \

--data-binary @${request\_data} \

--output ${response})

## Step 5. Construct the request body to update and send the envelope

Now that you have created an envelope that is associated with a workspace, use the eSignature REST API to update the envelope with tabs and recipient information. Set the [status](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/envelopes/liststatus/) to `sent` to send the envelope.

## Step 6. Call the eSignature REST API

Make a `PUT` request to the [Envelopes:update](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/envelopes/update/) endpoint and include the request body to send the workspace envelope with recipient information. Here’s how you make the call, but make sure to substitute your API account ID for `{API_ACCOUNT_ID}`, your eSignature base path for `{ESIGN_BASE_PATH}`, and the envelope ID saved from step 4 for the `{envelope_id}`.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/Workspaces/eg003SendEnvelopeWithRecipientInfo.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/Workspaces/Examples/SendEnvelopeWithRecipientInfo.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/workspaces/examples/Work003SendEnvelopeWithRecipientController.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/workspaces/examples/sendEnvelopeWithRecipientInfo.js), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/Workspaces/eg003SendEnvelopeWithRecipientInfo.ps1)

Status=$(curl -s -o "${response}" -w "%{http\_code}" \

--request PUT "${esign\_base\_path}/v2.1/accounts/$

{account\_id}/envelopes/${envelope\_id}" \

"${Headers[@]}" \

--data-binary @"${request\_data}")
