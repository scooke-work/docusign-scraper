# How to create a workspace

This topic demonstrates how to create a workspace using the Workspaces API.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PowerShell](https://github.com/docusign/code-examples-powershell).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {WORKSPACES\_BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at   `https://<domain>`  Where `<domain>` is `https://api-d.docusign.com/v1` for the developer environment and `https://api.docusign.com/v1` for production. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

Wondering which grant type suits your integration? See [Authenticate](https://developers.docusign.com/platform/auth/) for guidance on your options to get an OAuth token.

Workspaces OAuth access tokens are the same ones used for our eSignature REST API, although you must request the `signature` scope and one or more Workspaces scopes to call the Workspaces API.

See  [Workspaces authentication](https://developers.docusign.com/docs/workspaces-api/auth/) for details and the full list of Workspaces scopes.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid OAuth access token for {ACCESS\_TOKEN}.

1

2

3

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}" \

'--header' "Accept: application/json" \

'--header' "Content-Type: application/json")

## Step 3. Construct the request body

Next, create the request body. The only body field required to create a workspace is name.

1

2

3

4

printf \

'{

"name" : "'"${workspace\_name}"'"

}' >> $request\_data

## Step 4. Call the Workspaces API

Now that you have constructed your API headers and request body, make a POST request to the [Workspaces: createWorkspace](https://developers.docusign.com/docs/workspaces-api/reference/workspaces/workspaces/createworkspace/) endpoint and include the request body to create the workspace. Here’s how you make the call, but make sure to substitute your API account ID for `<{account_id}>`.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/Workspaces/eg001CreateWorkspace.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/Workspaces/Examples/CreateWorkspace.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/workspaces/examples/Work001CreateWorkspaceController.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/workspaces/examples/createWorkspace.js), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/Workspaces/eg001CreateWorkspace.ps1).

1

2

3

4

5

Status=$(curl -s -w "%{http\_code}\n" -i \

--request POST ${base\_path}/accounts/$

{account\_id}/workspaces \

"${Headers[@]}" \

--data-binary @${request\_data} \

--output ${response})
