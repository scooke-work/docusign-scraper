# How to bulk upload agreements

This topic demonstrates how to bulk upload agreements using Agreement Manager API.

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at   https://<domain>/v1  Where <domain> is api-d.docusign.com for the developer environment and api.docusign.com for production. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different OAuth grant types: Authorization Code Grant, Implicit Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

Agreement Manager OAuth access tokens are the same ones used for our [eSignature REST API](https://developers.docusign.com/docs/esign-rest-api/esign101/auth/), although you must request one or more Agreement Manager scopes to call the Agreement Manager API. The endpoint calls made in this how-to require that you request the Agreement Manager-specific `adm_store_unified_repo_read` scope when obtaining your access token.

See [Agreement Manager Authentication](https://developers.docusign.com/docs/agreement-manager-api/auth/) for details and the full list of Agreement Manager scopes.

## Step 2. Create bulk upload job and extract upload URLs

After you have a valid OAuth access token, you are ready to create a new bulk upload job. Make sure to substitute your valid OAuth access token for `{ACCESS_TOKEN}`, and to provide your base path and account ID for `{base_path}` and `{account_id}`.  This example uses 5 included documents; after the bulk upload job is created, one upload URL will be returned for each. Record these upload URLs and the bulk upload job ID, which will be needed for the next step.

## Step 4. Mark the bulk upload job complete

After uploading each document to the bulk upload job, call the POST [BulkJob: uploadCompleteBulkJob](https://developers.docusign.com/docs/agreement-manager-api/reference/agreementmanager/bulkjob/uploadcompletebulkjob/) endpoint to trigger ingestion of the documents and AI extraction.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/Navigator/eg003BulkUpload.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/Navigator/Controllers/Nav003BulkUploadController.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/navigator/examples/Nav003BulkUploadDocumentsController.java), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/Navigator/eg003BulkUpload.ps1).
