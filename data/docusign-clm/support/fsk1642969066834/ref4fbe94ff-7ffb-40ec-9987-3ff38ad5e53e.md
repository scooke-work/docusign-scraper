# Search Examples for Modernized Search in Docusign CLM

Look up search examples for modernized search in Docusign CLM. Administrators and users can review these examples to understand how the product interprets search keywords and produces search results.

This topic shows examples of searches in the modernized search interface in Docusign CLM. Each example describes how the product interprets the search keywords and describes the search results.

Note: A keyword search must contain at least four characters to return results.

| Example Search Keywords | Explanation | Search Results |
| --- | --- | --- |
| conf | The search can find all words that start with these letters. | The search finds all words starting with the letters c-o-n-f. Example results are confident, Confidential, and confidentiality, as well as conflict or confirm. |
| "force majeure event" | Enclosing the keywords in quotation marks finds the *exact* phrase. You do not want contracts that just mention "force" and "majeure" separately. Use this method for any of the following purposes:   - Locating a precisely defined specific concept - Finding section headers, clause titles, and so on - Finding a specific combination of words or terms in a specific sequence | The search finds contracts containing the exact phrase force majeure event. |
| third-party indemnification | The search can find any word from this group of words. | The search finds third, party, and indemnification (even if they do not display together). The search first looks for results with all words and then results with some or at least one of the words. Documents with more occurrences display higher in the list of results. |
| affect | The search can find the word and all stemmed versions of the word. | The search finds affect, affected, and affected parties. |
