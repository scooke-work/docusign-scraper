# Reconcile Data Using Writeback in Docusign CLM

Learn how to reconcile data using Writeback in Docusign CLM.

When the Data Reconciliation workflow step runs after a workflow is triggered in CLM or
Salesforce, the step will generate a task for the user if mismatches are found between
the tracked content value in the document generation field of your Word document and the
corresponding Salesforce value.![CLM writeback reconcile task1](https://docusign-be-prod.zoominsoftware.io/api/bundle/pxt1643324456371/page/images/CLM-writeback-reconcile-task1.jpg?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTg5NzUsInNoZWFmIjoicHh0MTY0MzMyNDQ1NjM3MSJ9.jZxvBJQKeLwNBleoK156tBAT8Mn96RXZ-e8ImCngrCs&_LANG=enus "CLM writeback reconcile task1")

For each field assigned in the data reconciliation task, the user can open the field to
review both the value in Salesforce as well as the value in the document. From this
view, the user can choose to update the Salesforce value from the CLM UI, or ignore the
mismatch and make no changes to the Salesforce value.![CLM writeback reconcile task2](https://docusign-be-prod.zoominsoftware.io/api/bundle/pxt1643324456371/page/images/CLM-writeback-reconcile-task2.jpg?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTg5NzUsInNoZWFmIjoicHh0MTY0MzMyNDQ1NjM3MSJ9.jZxvBJQKeLwNBleoK156tBAT8Mn96RXZ-e8ImCngrCs&_LANG=enus "CLM writeback reconcile task2")

In order to complete the Data Reconciliation step, the user must review and select an
outcome for each field in the reconciliation task.
