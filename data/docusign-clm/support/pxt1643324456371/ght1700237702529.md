# User Profile Properties in CLM.CM

Look up the user profile properties for CLM.CM. Administrators can set these properties
when creating or managing users.

The following user profile properties are available in CLM.CM. Administrators can set these
properties when creating or managing users.

| Profile Property | Description |
| --- | --- |
| Login Name | Specifies a login name. This property is required for users that must log on to Docusign CLM. A login name must be between 2 and 100 characters and must be unique to CLM.CM. After a user's login name gets set, it cannot be changed. To change a login name, you must delete the user and re-create the user. You can search for users in the CLM address book by login name. |
| Permission Profile | Assigns a profile that has a defined set of capabilities. The assigned profile determines what actions a user can perform. Note: For more information, see the documentation about [permission profiles](https://support.docusign.com/s/document-item?bundleId=pxt1643324456371&topicId=wji1582562825539.html&_LANG=enus "Read about permission profiles in Docusign CLM and CLM.CM. Administrators can assign a profile with defined capabilities to determine what actions a user can perform."). |
| Delegated Permission | Assigns delegated permissions to a user in the Full Subscriber role. Users who get assigned a permission will have access to any areas you indicate; however, the users still respect the folder and document security that is set for full subscribers. Note: For more information about delegated permissions, see the [Limited Administrative Permissions with Delegated Permissions in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=pxt1643324456371&topicId=bra1576610051531.html&_LANG=enus "Read about delegated permissions in Docusign CLM. Administrators can assign permissions to users in Docusign CLM (or full subscribers in CLM.CM). Users with delegated permissions have access to assigned administrative functions.") topic. |
| User Persona | Assigns a persona that changes the look and feel of the CLM.CM interface based on a user's job function. |
| Managed by | Specifies another CLM.CM user who can manage workflow tasks that are assigned to this user. The user that is listed in the Managed by field can see workflow tasks that are assigned to this user in the workflow inbox. The assignee can also respond on behalf of the user when the user is unavailable to take action. |
| Group Memberships | Optionally adds the user to a security group. You can create the user without adding the user to a group. You can edit the user later to add the group. You can also edit the group and add the user to the group. |
| Enable User Date/Disable User Date | Defines a start and end date for a user to be active in your account. For example, you want to provide access to a contractor for a temporary project. You can specify the start and end dates of the project. The contractor can log on to your CLM.CM account only for the duration of the project. |
