# Bulk Import Users CSV Format

Look up header row values to use in the CSV for bulk importing users in
CLM.CM.

The CSV for bulk import consists of a header row and a rows of user data to add. The CSV
can contain both new and existing users, but existing users are not updated when you bulk
add users. You must run a separate bulk update to update existing users.

**Header row values**

The first row of the CSV is the header row and defines each of the columns. Header values
must match the header row values listed below. Header values are not required to be in the
order listed and are not case-sensitive.

**Required columns**

The CSV must contain the Email header row. The rest of the header row values shown below
are optional.

Table 1. Acceptable header row values

| Header Row Value | Description |
| --- | --- |
| Email | The user's complete email address—required column. |
| FirstName | The user's first name. |
| LastName | The user's last name. |
| Role | The user’s role. If a role is not listed, the user receives the role of "Full Subscriber.” When updating users, they maintain their current role. |
| Groups | This is a pipe (‘|’) delimited list of the group names that the user on the row is a member. The user is removed from groups they are currently a member of that are not listed and added to groups listed. If the group does not exist in the account, the group is created, and the user added as a member. |
| RemoveGroupsNotListed | Valid values for this field are "true" or "false.” If set to "true," the user is removed from all groups not listed in the Groups column. If set to "false," the user remains in existing groups and is added to groups listed in the Groups column. The default value is "true" when the Groups column has no data, and a RemoveGroupsNotListed column is not specified. |
| Active | Valid values for this field are "true" or "false.” If set to "true," the user is added to the Address Book. If set to "false," the user is removed from the Address Book. |
| Title | The user's title that displays in the address book. |
| Department | The user's department that displays in the address book. |
| Company | The user's company that displays in the address book. |
| Address1 | The user's address- first line. |
| Address2 | The user's address- second line. |
| Address3 | The user's address- third line. |
| City | The user's city name. |
| State | The user's state/province. |
| PostalCode | The user's postal code. |
| PhoneNumber | The user's phone number. |
| FaxNumber | The user's fax number that displays in the address book. |
| Persona | The user's persona should a valid name of a persona already created in the account. |
| ManagedBy | The user's manager should be a valid email address of a user in the account. |
| SendActivationEmail | Valid values for this field are "true" or "false.” Entering "true" sends an activation email to the user. |
| EnabledStartDate | The start date when this user can access the account. |
| EnabledEndDate | The end date when this user can access the account. |
| PortalUser | Valid values for this field are "true" or "false.” Indicates if they have only portal access to Docusign CLM, otherwise full access to the UI. |
| ExemptFromUserSync | Valid values for this field are "true" or "false.” Indicates if the Salesforce user sync process should not manage the user. |
