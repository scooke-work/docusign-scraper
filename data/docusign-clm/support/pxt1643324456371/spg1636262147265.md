# Build a Document Rule to Set Your CLM Trash Retention Policy

Learn how to build a document rule to move files to the trash folder in Docusign CLM.
Administrators can build the rule to efficiently eliminate unwanted documents from the
account.

Administrators can use smart rules to build logic and automate actions in
Docusign CLM. One type of smart rule is a document rule. Document rules automate
actions on documents when events happen.

You can build a document rule to move files to the trash folder. This rule can help you
eliminate unwanted documents from your account.

Use this procedure to build a rule that moves files to the trash when a document attribute
value changes to Canceled.

1. Log in to Docusign CLM as an administrator.
2. Select Admin in the navigation header.

   An administrative sidebar opens.
3. Select Smart Rules under Automation Tools.

   The Smart Rules page opens.
4. Select Rule > New Document Rule.

   The Edit Document Rule page opens.
5. Complete the following fields to define a name, event, and rule executor for
   the rule:

   Name
   :   Specify a descriptive name for the rule.

   Event
   :   Specify Document Indexed.

   Execute as
   :   Select one of the following options for executing the rule:
       - Initiator executes the rule by using
         the security rights of the user who takes action.
       - your\_user\_name
          executes
         the rule by using your security rights.
       - Specific User executes the rule by using the security rights of
         the administrator whose name or email address you
         specify.

         Note: We recommend
         creating a generic CLM administrator to use for
         automation. This way, your automation processes are not
         tied to users who might leave or change
         roles.

   After completing the fields, your rule has a name, an assigned event,
   and an assigned executor.
6. Define filters for the rule:
   1. Select the Filter drop-down menu, and select
      In Folder.
   2. Select the Select Folder link, select the folder
      where the rule will run, and select the Select
      button.

      The product adds the filter to the bottom of the Filters
      section.
   3. Select the Filter drop-down menu, select
      Has Attribute Value, select an attribute
      group and attribute field, and type Canceled in
      the Attribute Value field.
   4. Select Add Filter.

      The product adds the filter to the bottom of the Filters
      section. The following example shows filters added to the rule
      definition.

   ![Add Filters](https://docusign-be-prod.zoominsoftware.io/api/bundle/pxt1643324456371/page/images/Add-Filters.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTg1OTYsInNoZWFmIjoicHh0MTY0MzMyNDQ1NjM3MSJ9.yx2fiSFjDPygQDKsTgGM3CYvSDj8c696IneV7ATDc4g&_LANG=enus "Add Filters")
7. Create an action to move the document to the trash:
   1. Select the Action drop-down menu, and select
      Move Document.
   2. Enter /your\_clm\_account\_name/Trash/
      in the Or Enter Path field.

      /DocuSign/Trash/
   3. Select the Add Action link.

      The product adds the action to the bottom of the Actions section.
   4. Select Save.

      The product saves the rule and displays the defined rule on the
      Smart Rules page. The rule shows a
      Disabled status.
8. Enable the new document rule:

   1. Select the checkbox next to the new rule.
   2. Select Rule > Enable.

   The rule's status changes to Enabled.

You have completed the process. When document attribute values
change to Canceled, the documents automatically move to the trash
folder. When the trash folder gets emptied, these files get permanently removed from all
Docusign servers. The removed files no longer add to your account storage
limits.
