# Configure Writeback in Docusign CLM Workflows using the Data Reconciliation Workflow Step

Learn how to configure Writeback in Docusign CLM Workflows.

The data reconciliation workflow step allows you to use Writeback at any point in
your CLM workflow to reconcile the data fields in your document with the values in
the Salesforce record. Docusign recommends using Writeback in CLM workflows at the
end of the negotiation process once a document has been signed. For information
about setting up workflows in CLM, see [Workflow Management in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=yks1643320936212&topicId=kuq1584137317950.html&_LANG=enus).

Note: The data reconciliation workflow step requires a Microsoft
Word document in parameters and an assignee.

In Tracked Names, specify the tracked content TrackNames that are provided
to you in the Document Generation form configuration. To locate the TrackNames:

1. Navigate to Admin > Automation Tools > Document Generation > Forms.
2. Open your Document Generation form.
3. Select a form field. If the form field is linked to Salesforce, open the
   Merge Tags tab. To see if a form field is linked to
   Salesforce, review the Link to Salesforce Field checkbox
   located under Field Value.
4. In the Merge Tags tab, copy the field name that displays
   within quotation marks after TrackName=.
5. Paste the field name into the Tracked Name field in the
   data reconciliation workflow step. Use a pipe delimiter to separate multiple
   field names. Do not add spaces. The name must be an exact match without the
   quotation marks.

   The data reconciliation workflow step will output writeback fields
   and results as an .xml file. This output includes a summary of which fields
   had mismatches, which fields in the document are linked to Salesforce, and
   whether the user wrote new values to Salesforce or ignored the mismatched
   fields.
