# Permission Profiles in Docusign CLM and CLM.CM

Read about permission profiles in Docusign CLM and CLM.CM. Administrators can assign a
profile with defined capabilities to determine what actions a user can perform.

Each Docusign CLM user needs an assigned permission profile. Each profile has a defined set of
capabilities that determines what actions a user can perform. Administrators assign
permission profiles when creating users in Docusign CLM.

When setting security on documents and folders, access can also be determined by a user’s
permission profile. For more information about defining folder access based on user
profiles, see [Limited Administrative Permissions with Delegated Permissions in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=pxt1643324456371&topicId=bra1576610051531.html&_LANG=enus "Read about delegated permissions in Docusign CLM. Administrators can assign permissions to users in Docusign CLM (or full subscribers in CLM.CM). Users with delegated permissions have access to assigned administrative functions.").

Your CLM account type determines which permission profiles are available and how you
assign them.

Note: To see what CLM account type you have, see [Identify Your Docusign CLM Account
Type](https://support.docusign.com/s/document-item?bundleId=fsk1642969066834&topicId=xzt1631832579827.html&_LANG=enus).

## Permission Profiles in Docusign CLM

In Docusign CLM, profiles are assigned in Docusign Admin at the account level. The
following profiles are available for assignment:

CLM Admin
:   Assigns administrative rights for Docusign CLM. This profile has access to
    everything in your Docusign CLM repository.
:   An account must have at least one CLM Admin profile. This profile is not the
    same profile as an organization administrator but can be the same user.

CLM User
:   Assigns basic user rights for Docusign CLM. This profile is the most common
    permission profile. CLM users can perform the following activities:

    - Access folders, documents, and tasks
    - View, edit and upload documents
    - Run reports

      Note: Users cannot create or edit
      reports by default.
    - Change their own passwords

CLM Viewer
:   Assigns guest rights for Docusign CLM. This profile provides a minimal user
    role with limited access to Docusign CLM.
:   CLM viewers can perform the following activities:

    - Upload or view documents in Docusign CLM.
    - Use search
    - complete eForms
    - Change their own passwords in user preferences
:   CLM Viewers cannot perform the following activities:

    - Move, edit or share documents
    - Interact with tasks
    - View reports, create reports, or access contacts

## Permission Profiles in CLM.CM

In CLM.CM, profiles are assigned in the CLM application. The following profiles are
available for assignment:

Guest
:   Is the basic user role. Guest users can execute searches and complete eForms. Guests can
    also create (upload), view, and download documents. Guests can change their
    passwords in the Preferences tab. Guests *cannot*
    perform the following activities:

    - Participate in workflows
    - View or create reports
    - Access the address book
    - Create or edit eForms
    - Move, edit, or share documents

Limited Subscriber
:   Is a user role similar to Guest. Limited subscribers can execute searches and complete
    eForms. Limited subscribers can change their passwords in the
    Preferences tab. Limited subscribers *cannot*
    perform the following activities:

    - View or create reports
    - Create contacts
    - Edit or create eForms

Full Subscriber
:   Is the most common user role. Full subscribers have full access to view, edit, and upload
    documents. Full subscribers can execute reports and participate in
    workflows. Full subscribers cannot edit reports and have some expanded
    account settings in Preferences.

User Administrator
:   Is a delegate role for some administrative tasks. User administrators can manage users,
    create and edit mailing lists, and administer some account-level
    preferences. User administrators cannot create or edit security groups. An
    account can have multiple user administrators.

Super Administrator
:   Is the highest-level administrator in your account. Super administrators can access
    everything in your Docusign CLM repository, including all users' home
    folders. Use this role for designating a repository administrator. An
    Account can have multiple super administrators.
