# Language Support and Settings in Docusign CLM

Read about language support and settings in Docusign CLM. Administrators can configure
language settings to support a localized interface that uses a preferred
language.

In Docusign CLM, administrators can configure language settings. These settings provide support
for creating a localized interface that uses a preferred language. The Docusign CLM user
interface supports the following languages:

- Dutch
- English
- French
- German
- Japanese
- Portuguese (Brazil)
- Spanish (Latin America and Spain)

Important: In Docusign CLM, you can store documents and
attribute values that are written in non-supported languages. However, you do so at your
own risk, and you must use UTF-8 encoding for the text. For document generation,
customer-generated content that uses a non-supported language might cause problems.
Examples of customer-generated content are form instructions or content in a template.

Docusign CLM supports the most widely used languages. If you have an urgent need for language
support, you can contact Docusign Support to submit a request. However, we cannot
guarantee approval of the request.
