# XPath in Docusign CLM

Read about how XPath is used in Docusign CLM for Workflows, Smart Rules, and other features.

XPath syntax is used in several places throughout Docusign CLM to access data formatted as XML. Docusign CLM can use XPath syntax in Document Generation templates, for document attributes, Smart Rules, and Workflows.

Note:

- For more information on using XPath syntax in Document Generation templates, see [Document Generation Word Template Merge Tags in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=uqj1643324072491&topicId=uno1576795541214.html&_LANG=enus).
- For more information on using XPath syntax for Smart Rules, see [Use Cases and Sample Smart Rules in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=qaz1642962729191&topicId=ttl1576610045092.html&_LANG=enus).
- For more information on using XPath syntax for document attributes, see [Configuration for XML Formatted Manifest Files in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=lun1683559393884&topicId=gov1576795527087.html&_LANG=enus).
- For more information on using XPath syntax in Workflows, see [Workflow Expression Builder in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=yks1643320936212&topicId=kxb1576610102148.html&_LANG=enus) and “Evaluate an XPath” in [Automated Document Management Workflow Steps in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=yks1643320936212&topicId=hpj1576610096401.html&_LANG=enus).
