# Assign Delegated Permissions to a User in Docusign CLM

Learn how to use delegated permissions to assign administrative permissions. Administrators can use these steps to create delegated permissions and enable them for Docusign CLM users.

## Before you begin

[Create Delegated Permissions in Docusign CLM](https://support.docusign.com/s/document-item?bundleId=pxt1643324456371&topicId=hyn1576610087430.html&_LANG=enus "Learn how to use delegated permissions to assign administrative permissions. Administrators can use these steps to create delegated permissions and enable them for Docusign CLM users.")

Administrators can assign delegated permissions to users. These permissions give users limited administrative abilities in Docusign CLM. Delegated permissions give your organization flexibility when managing work in CLM.

Note:

- In Docusign CLM, administrators can assign delegated permissions only to users with the CLM User permission profile.
- In CLM.CM, super administrators can assign delegated permissions only to CLM users with the Full Subscriber permission profile.
- To see what CLM account type you have, see the topic [Identify Your CLM Account Type](https://support.docusign.com/s/document-item?topicId=xzt1631832579827.html&bundleId=dsa1643667928464&_LANG=enus).

1. Select Admin from the navigation header.

   The Admin sidebar opens.
2. Select Users under User Management.

   The Users page opens.
3. Select a user to assign delegated permissions to.
4. Select a permission from the Delegated Permission drop-down list.
5. Select Save.

The delegated permission is now assigned to the user.
