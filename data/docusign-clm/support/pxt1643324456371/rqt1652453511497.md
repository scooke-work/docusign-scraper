# Docusign CLM Trash Folder Management

Read about the trash folder. To manage this folder, administrators can remove all trash,
set retention policies, and create schedules and smart rules.

When CLM users delete files and folders, these items get sent to the trash folder. The trash
folder contains all CLM content that has been deleted but not permanently removed.

The following list describes important information about the trash folder:

- Users can delete only CLM files and folders that they have View, Edit & Delete access
  to.
- Items in the trash do not display in search results.
- Deleted content moves to a trash folder that is named for the deletion date (in
  YYYYMMDD format).
- Deleted folders maintain their original structure. Parent folders and subfolders keep the
  same grouping.
- Users with View, Edit & Delete access can view and restore files and folders that are in
  the trash.

  Note: Items moved to the Trash folder will retain
  the security settings from the original folder they were in.
- The trash folder can be purged to permanently remove all deleted content.

  Important: Data that
  gets purged from the trash folder is removed from all Docusign datacenters. This
  purged data can never be restored.
