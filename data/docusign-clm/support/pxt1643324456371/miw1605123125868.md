# Start Workflow Agreement Action in Docusign CLM

Learn about the Start Workflow Agreement Action in Docusign CLM. Administrators can use
this agreement action to pass envelope metadata to CLM as custom XML data from
eSignature.

When you select Start Workflow as part of your Agreement Rule configuration, your
Docusign eSignature admin account passes envelope metadata to CLM as custom XML data. In
Add Actions, select **Start Workflow**, your Docusign CLM account type, and the
desired workflow.

When setting up your workflow, you can use the Evaluate an XPath workflow step in CLM
Workflow Designer to grab specific metadata from the XML to update document attributes
and complete other post-signature tasks.

In the following workflow example, the custom XML data contains two text fields, Employee
ID and Department, that are pulled into CLM from the envelope data delivered by
Agreement Actions.![Agreement Actions in CLM 4](https://docusign-be-prod.zoominsoftware.io/api/bundle/pxt1643324456371/page/images/Agreement-Actions-in-CLM-4.jpg?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTkwMTUsInNoZWFmIjoicHh0MTY0MzMyNDQ1NjM3MSJ9.YZu81YBBBMHVgWZhc7XbD26xwKsEGXIyC8RxIA_U4SU&_LANG=enus "Agreement Actions in CLM 4")

To grab the specific meta-data in this example, the workflow uses a series of
*Evaluate an XPath* steps to look for the relevant data in the XML. Listed
below are the XPaths to some of the common fields in eSignature Documents:

- **Recipient Name:**

  */Params/envelope/recipients/signers/item/name/text()*
- **Envelope Custom
  Field**

  */Params/DocuSignEnvelopeInformation/EnvelopeStatus/CustomFields*
- **Document Custom
  Field:**

  */Params/envelope/recipients/signers/item/tabs*
- **Template Name:**

  */Params/envelope/templates*
- **Document
  Name:**

  */Params/envelope/envelopeDocuments/item[order/text()=1]/name/text()*

  For
  information on Workflow Designer, see [Docusign CLM Workflows Overview](https://support.docusign.com/s/document-item?bundleId=yks1643320936212&topicId=uxf1651607363522.html&_LANG=enus).
