# Docusign CLM Service Limits

Read about service limits in Docusign CLM. These limits enable CLM to operate safely and efficiently.

Service limits enable Docusign CLM to operate safely and efficiently. Administrators should be aware of these limits for account management, planning, and other purposes.

The following table describes the service limits that the product enforces:

Table 1.  Docusign CLM Service Limits

| Product Area | Restricted Item | Limit |
| --- | --- | --- |
| AI-Assisted Review add-in for Microsoft Word | Number of rules in a playbook | Exceeding 50 rules might slow down processing time when running the playbook |
| AI-Assisted Review add-in for Microsoft Word | Number of characters in a playbook rule | 1000 characters |
| Document administration | Number of Secure File Transfer Protocol (SFTP) concurrent connections | 20 connections |
| Document comparison | File size | 128 MB |
| Document preview | Page limit for document preview (all preview formats) | 250 pages Note: Documents over 250 pages are downloadable and available for actions in CLM. However, these documents are not viewable in the preview window. |
| Document attributes | Attribute values | 400 characters |
| Document attributes | Attribute name in an attribute group | 50 characters |
| Document attributes | Attribute group name | 50 characters |
| Document attributes | Rows in a CSV that you use to populate values for attributes | Exceeding 10,000 rows can degrade product performance. |
| Document and folder management | Special characters in folder and document names | Prohibited special characters are as follows:  - / - \ - : - \* - ? - “ - < - > - |   Note: The characters # and & are not prohibited but are automatically replaced in External Object Storage (EOS) folders |
| Document and folder management | Folder name | 100 characters Note: Folder names cannot begin or end with a space. These names also cannot end with a period. |
| Document and folder management | Folder description | 255 characters |
| Document and folder management | Document name | 255 characters Note: Document names cannot begin or end with a space. These names also cannot end with a period. |
| Document and folder management | Number of document versions | 1000 versions |
| Document and folder management | Number of direct child objects (documents and subfolders) that can exist within a parent folder | 10,000 objects |
| Document and folder management | File size in document preview | 1 MB for TXT files  256 MB for all other files |
| Document and folder management | Inbound emails for an account | 500 emails per hour |
| Document and folder management | Combined file size when you send one or more documents for external review manually (non-workflow) | 25 MB (all files combined) |
| Document generation | Size for a document that gets created or uploaded through a document generation configuration | 200 MB  Exceeding this size can result in errors. Tip: We recommend limiting the size to 50 MB or less. Generations take longer for documents larger than 50 MB. Generation time can also depend on the number of merges. |
| Document generation | Format of a “Link to CSV Document” dropdown field | Specifying a CSV file that exceeds 10 KB changes the field from a picklist to a type-ahead list. |
| Document generation | File size of the CSV file for a “Link to CSV Document” dropdown field | 1 MB or less (recommended size)  When a CSV gets loaded into a document generation form, the CSV gets loaded into the user's browser.  Larger files can cause delays in loading the form and searching in the dropdown field.  Docusign cannot guarantee support for files that exceed 25 MB. |
| Document generation | Size of the document generation form (based on the number of fields added) | 256 MB  Exceeding this size prevents the form from being saved. Tip: The form size is not checked until you attempt to save. To avoid losing your work, we strongly recommend creating smaller, focused forms as a standard practice. |
| Document uploads | Uploaded document size | 1.5 GB |
| Document uploads | Number of uploaded documents submitted at one time (through the File > Upload menu action or through drag and drop) | 20 documents |
| Electronic signature requests from CLM to eSignature | Size of each document or envelope that CLM sends to eSignature | - 25 MB - 100 MB when "chunked upload" is enabled for the account   Note: Chunked upload permits uploading a PDF file to eSignature in separate pieces. To request enablement of chunked upload, contact Docusign Support. |
| Electronic signature requests from CLM to eSignature | Number of pages in each document or envelope that CLM sends to eSignature | See the [file size limitations for envelopes](https://support.docusign.com/s/document-item?bundleId=gbo1643332197980&topicId=akk1578456465691.html&_LANG=enus). |
| Electronic signature requests from CLM to eSignature | eSignature API | See the [eSignature API rules and resource limits](https://developers.docusign.com/docs/esign-rest-api/esign101/rules-and-limits/). |
| Party management | Total number of parties | Reaching or exceeding 20,000 parties causes performance issues when you select parties or search for parties. |
| Reporting | Report size when exporting to CSV | 5,000 record entries Note: A report that exceeds 5,000 records must be saved to a folder location in Docusign CLM. The report can then be exported as a CSV or XLSX file and saved locally. |
| Search | Amount of content that CLM can extract and index from each document | 1 MB of content |
| Search | Number of search results | 100 results by default for legacy search Note: Customers can request an increase (up to 1000 results).   1000 results by default for modernized search |
| Search | Size limit for a file to be indexed for search | 268 MB  Any file larger than 268 MB cannot be indexed and will not show up in searches. |
| Workflows | Data size for the output from a workflow expression evaluation | 1 MB |
| Workflows | Expression execution limit for a workflow instance Important:  - In UAT environments, this limit is effective for new workflow revisions that are created on or after August 12, 2025. - In Production environments, this limit is effective for new workflow revisions that are created on or after September 2, 2025. | 300 expressions during a 15-minute time period  When the limit is reached, the product pauses the workflow instance. The product also sends an email to users who are defined in the “Advanced Workflow Error Notification User” account setting. If no action is taken on the paused workflow before the scheduled purge day, the product aborts and purges the workflow. |
| Workflows | Size for each individual variable for the following accounts:  - Accounts with workflows that execute in a user acceptance testing (UAT) environment - Newly created accounts with workflows that execute in a Production environment | 5 MB |
| Workflows | Total size of all variables in a single workflow for the following accounts:  - Accounts with workflows that execute in a UAT environment - Newly created accounts with workflows that execute in a Production environment | 25 MB |
| Workflows | CSV file size for the Create Variable from CSV workflow step | 1 MB |
| Workflows | Number of workflows that can be started within one hour per account | 100 workflows (UAT environments)  1,000 workflows (Production environments) |
| Workflows | Workflow executions Note: A workflow execution is counted whenever a new workflow is initiated or a workflow resumes from an asynchronous step (such as sending for signature). | 500 executions every 15 minutes per account (UAT environments)  12,000 executions every 15 minutes per account (Production environments) |
| Workflows | Number of executed steps that you can view for a workflow in the user interface | 1,000 steps |
| Workflows | Activities per workflow instance Note: A workflow activity is counted whenever a workflow step is executed. | 100,000 activities per instance for existing workflow revisions (versions) that were published before April 28, 2022 (UAT) and May 13, 2022 (PROD)  10,000 activities per instance for new workflow revisions (versions) that were published after April 28, 2022 (UAT) and May 13, 2022 (PROD) |
| Workflows | Number of activities per workflow that can be executed consecutively  Number of activities per workflow that can be executed synchronously without a human activity step or asynchronous activity step | 5,000 activities |
| Workflows | Number of days that a workflow can remain under “Executing” or “Paused” status Note: When the limit is reached, the product aborts the workflow revision. After the abort, Docusign CLM schedules the workflow for purging, according to the [purge schedule](https://support.docusign.com/s/document-item?bundleId=yks1643320936212&topicId=gqw1655998280555.html&_LANG=enus). | 365 days (Production environments)  90 days (UAT environments) Note: If a workflow gets paused by the system due to reaching any service limit, purging begins 30 days after the pause. |
| Workflows | Purge schedule for workflows under “Complete” or “Failed” status | The value for the Days after Workflow Completes and Days after Workflow Fails account settings must be between 7 and 30 days. |
| Workflows | Workflow names | 100 alphanumeric characters |
| Workflows | Workflow variable names | 250 alphanumeric characters |
| Workflows | Number of recipients in the Email Document workflow step | 50 recipients |
| Workflows | Attachment size in the Email Document workflow step | 20.48 MB total for all attachments |
| Workflows | Size of the combined document that the Combine Documents workflow step creates | 200 MB |
| Workflows | Number of files that the Combine Documents workflow step can combine | 250 files |
