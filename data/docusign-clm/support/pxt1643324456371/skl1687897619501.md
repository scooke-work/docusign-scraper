# User Personas in Docusign CLM and CLM.CM

Read about user personas in Docusign CLM and CLM.CM. Administrators can use personas to
help simplify a user's experience and actions while working in the application.

Personas are role-based user settings. CLM administrators use personas to create a
unique display in the interface for menu, toolbar, and navigation options. Personas are
created from groups and are based on the group name. In a user's profile, you can assign
a single persona.

Personas can help simplify a user's experience and actions while working in the application.
Different buttons and menu items get displayed, based on the user's persona or job
function.

For example, a training manager sees different buttons in the user interface than an HR
manager. This can help make end-user training happen in minutes or hours rather than
days or weeks.
