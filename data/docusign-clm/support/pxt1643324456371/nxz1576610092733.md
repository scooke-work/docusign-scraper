# Unique Usernames

Read about usernames in Docusign CLM. Administrators can create a user and set the
username, which can be associated only with a single Docusign CLM account.

On August 25, 2017, Docusign CLM introduced a change that affects customers who are in
multiple Docusign CLM accounts. The change to how usernames are managed provides better
security and helps ensure that the product can provide more enterprise capabilities.

Each username can now only be associated with a single Docusign CLM account. Usernames must
be unique across an entire Docusign CLM environment. Users in multiple accounts need to log on
to each account separately.

## What is a username?

- Each user will have a unique user name.
- Each user is tied to one and only one Docusign CLM Account.
- The username is a string but does not have to look like an email address.
- Usernames can only begin with an alpha or numeric character.
- The email address must currently be unique in each account. This means only one user can be tied to an email address in the same account.
- Usernames are unique to the Docusign CLM environment.
- Each username must be between 3 and 100 characters.
- Only Super Administrators or User Administrators can create users and set the usernames. End users can not set or choose the username.
- Once the username is set, it cannot be changed.

## How to create a user?

When you create a new user in Docusign CLM, you will need to also include the username in addition to the email address. The username can be the email address provided that the username/email address is unique. The username must be unique in the entire Docusign CLM environment. If the username is not, an error will be returned and you will need to enter a new username.

## How to create a user via the REST API?

When you create a new user via the REST API, then you can pass both the username and the email address. If the username is not provided then the email address will be used as the user name.

If the username is not unique, then an error is returned.

## How to create a user via SOAP?

When you create a new user via the SOAP API, then the email address will always be set as the user name. Their is no provision in the SOAP API to pass both the username and the email address.

If the email address is not unique, then an error is returned.

If the user is created, then the username and the email address are the same.

## How to create a user via Bulk Import(CSV) ?

When you create a new user in Docusign CLM via Bulk Import, a new column (called Username) has been added.

If you do not add the column to the csv and do not provide a username, then the import uses the email address as the username.

If the username has been added, then the import process uses the username as the username and the email address as the email address.

If the email address or the username is not unique, then an error is returned.

## What happened to my username if I'm part of multiple accounts?

If a user is only in one Docusign CLM account, then the email address will be automatically converted to the username.

If a user is in multiple Docusign CLM accounts, then in the default account, the username will be the current email address of the user. All other accounts will include the Account ID for each user. Additionally, an email will be sent to each user that is in multiple accounts with their new username.

Lets take John Smith as an example. John Smith is in 3 different accounts in the Docusign CLM production environment. Here is the account list:

- Acme Production, Account ID 12345 (default)
- Acme Sandbox, Account ID 9899
- Acme Development, Account ID 7674

After the conversion has completed, John will have three unique user names, one for each Docusign CLM Account.
