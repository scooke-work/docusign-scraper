# CLM API 101

Developing with the CLM API is only available for CLM customers with a production account. To purchase a plan to gain access to the CLM API, [contact Docusign Sales](https://www.docusign.com/contact-sales).

The Docusign CLM API enables developers to interact with Docusign CLM documents, data, and services. Most objects and services available in the Docusign CLM web application are supported in the API. Users and groups are managed through [Docusign Admin](https://www.docusign.com/products/admin). See [Object API](https://developers.docusign.com/docs/clm-api/clm101/object-api/) for more information.

For additional issues or requests, visit the [Docusign Support Center](https://support.docusign.com).

The Docusign CLM API contains the following logical sub-APIs for working programmatically with Docusign CLM:

- [Object API](https://developers.docusign.com/docs/clm-api/clm101/object-api/): API methods that create, read, update, and delete (CRUD) Docusign CLM objects, such as folders.
- [Task API](https://developers.docusign.com/docs/clm-api/clm101/task-api/): API methods that expose Docusign CLM services that are invoked as asynchronous tasks, such as zipping files or running searches.
- [Content API](https://developers.docusign.com/docs/clm-api/clm101/content-api/): API methods for uploading, downloading, and versioning documents in Docusign CLM.

## Base URLs

All API URLs use the following format:

```
https://[base url]/[version]/[Docusign Account Id]/[resource identifier]
```

Docusign CLM is distributed across several data centers, and the base URLs for the Object, Task, and Content APIs are data center–specific. You can determine the data center associated with an account by looking at the subdomain of the URL after you log in to the Docusign CLM web application (for example, na11 or s1.us). Content API URLs are properties of the document and folder objects and are discoverable at run time.

You can also get your account-related URLs, including the base CLM API URLs, by making a GET request to these endpoints:

**UAT**: `https://authuat.springcm.com/api/v2/{Docusign API Account ID}/account`

**Production**: `https://auth.springcm.com/api/v2/{Docusign API Account ID}/account`

To find your Docusign API Account ID, log in to your eSignature account and select **Admin** from the top menu, then select **Apps and Keys** from the left navigation. Copy your **API Account ID** value.

**Note**: Similar to endpoints for resources, these endpoints require:

- CLM to be enabled for your account.
- A valid access token in the authorization header. You must request the `springcm_read` scope when you request the access token. See [Authentication](https://developers.docusign.com/docs/clm-api/clm101/auth/) for details.

## Request and response formats

The Docusign CLM Object and Task APIs support JSON for all requests and responses. The Content API by default will download documents in the native format in which they were uploaded. For certain document types, Docusign CLM will also maintain a copy of the document in additional formats such as *.PDF*, *.PNG*, or *.TXT*. See the [Content API reference](https://developers.docusign.com/docs/clm-api/reference/content/) for more information.
