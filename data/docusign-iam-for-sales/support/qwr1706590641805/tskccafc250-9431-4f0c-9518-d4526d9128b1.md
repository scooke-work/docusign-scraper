# Build a Workflow to Be Triggered By a Start with Docusign Action

Learn to build a workflow that can be triggered by a Start with Docusign action.

## Before you begin

Important: This feature is only available to IAM for
Sales customers. Contact [Docusign support](https://support.docusign.com/s/?language=en_US&_gl=1*74elk4*_gcl_au*MTY5ODQyMjA2MC4xNzY5NjM1OTA1LjE5ODM3MTQzNjguMTc2OTYzNjIwNC4xNzY5NjM2MjA0) or [Docusign
sales](https://www.docusign.com/contact-sales) for more information.

Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account
administrator.

You can use Workflow Builder to automate agreement actions from
your Docusign account.

Docusign Apps Launcher is a package of Docusign
apps that you can use in your Salesforce organization. These apps include Docusign eSignature for Salesforce, Docusign Gen
for Salesforce, and Docusign CLM for Salesforce.

You cannot build Workflow Builder workflows in the Apps Launcher user interface. However, you
can start Workflow Builder workflows from Salesforce records to create agreements with
Salesforce and Docusign data.

Before you can start workflows from Salesforce, you must create a compatible
workflow.

Use this procedure to build a Workflow Builder workflow that can be started from
Salesforce:

1. As a Docusign administrator, create your workflow:
   1. Log on to your Docusign account and go to Agreements >  Workflows.
   2. Select Create Workflow.

      The Creating your workflow dialog box
      displays.
   3. Select New Canvas and select
      Continue.

      The Creating your workflow dialog box
      closes. The workflow builder page displays.
   4. Select Add workflow start.

      The Add workflow start dialog box displays.
2. Select a method: Select either From a Link or From an API
   Call from the dropdown menu and select
   Next.

   Note: You cannot select other methods when building workflows that start in Salesforce.

   ![A screenshot displays the What starts this workflow step with From a Link and From an API Call highlighted in green.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/What-starts-this-workflow-with-From-a-link-and-From-an-API-Call-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU5MDUsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.IwmK-3Bpm6pxWyuq_tY4KZYvzSuPru91pKtVQMaKKnk&_LANG=enus "A screenshot displays the What starts this workflow step with From a Link and From an API Call highlighted in green.")
3. Set up variables for dynamic information: Add the following variables:
   1. Select New Variable and choose
      Text from the dropdown menu.
   2. Enter `recordId` in the field.
   3. Create another text variable and enter `objectName` in the field.
   4. Create another text variable and enter `applicationName` in the field.
   5. Create another text variable and enter `orgId` in the field.

      ![A screenshot displays the Set Up Variables step with recordId, objectName, applicationName, and orgId variables highlighted in green.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Start_Workflow_Builder_workflows_from_Salesforce_Set_up_variables_for_dynamic_information.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU5MDUsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.IwmK-3Bpm6pxWyuq_tY4KZYvzSuPru91pKtVQMaKKnk&_LANG=enus "A screenshot displays the Set Up Variables step with recordId, objectName, applicationName, and orgId variables highlighted in green.")
4. Will this API call be started by a human or an automated
   process?:
   1. Select Human.
   2. Select Apply.

   The page refreshes. Your From an API call start
   step displays.
5. Complete your workflow:
   1. Add more steps to your workflow.
   2. Select Review & Publish when you have
      finished building your workflow.

You have completed the process. You have built a workflow that can
be triggered by a Send with Docusign action. When a user selects a
workflow's Start button, the workflow starts.
