# Link Multiple Writeback to Salesforce Steps

Learn to link two or more Writeback to Salesforce workflow steps.

Before you begin
:   - Required: Installation of the Salesforce app
      from the Docusign App Center. Review the [Installing Extension Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more information.
    - Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account administrator.
    - Required: Access to a suitable template to use for your workflow
      step. Review the [Template Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=xnw1709852160602.html&_LANG=enus) topic for more information.

You can use a Writeback to Salesforce step to update Salesforce records
with information from your Docusign workflow. The Writeback to
Salesforce step updates Salesforce fields, creates new records, and more.
Administrators link workflow fields to Salesforce fields in the Writeback to
Salesforce step. When task recipients enter data in workflow fields, Docusign updates the information in the linked Salesforce
fields.

You can configure more than one Writeback to Salesforce step in the same
workflow.

Example: You want data from your workflow to write back to
different fields from Account and Opportunity records. You build a writeback step
linked to the Account object and add one group of fields. You create a second step
linked to the Opportunity object and add fields there. A workflow participant enters
information into a Docusign web form. The participant enters an
Account Number and an Opportunity Amount. Docusign updates the
linked Account and Opportunity fields.

Use this procedure to add multiple Writeback to Salesforce steps to your workflow:

1. [Configure a Writeback to Salesforce
   step](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=pen1706132774147.html&_LANG=enus "Learn to add and configure a Writeback to Salesforce step to update Salesforce records.").
2. Add a Writeback to Salesforce step as a Docusign administrator:
   1. Go to Agreements > Workflows and select Edit next to a workflow
      name.

      The workflow editor page displays.
   2. Select the Add a step
      ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_add_step_plus_sign.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NjcsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.YwNIwpOLDG8izw7TqZlZeL0Y_MwAHGVALfTIJ3ZK1pg&_LANG=enus) icon.

      The Add New Step dialog box displays.
   3. Search for steps: Enter `Writeback to
      Salesforce` in the search field and select the
      Writeback to Salesforce step that displays.

      The workflow editor page displays with the Writeback to
      Salesforce step added.
3. Name your Writeback to Salesforce step:
   1. Select Configure on the Writeback to Salesforce
      step.

      The configuration dialog box displays.
   2. Step Name: Keep or edit the name. It is helpful
      to add the name of the Salesforce object, like `Account -
      Writeback to Salesforce`.
4. Where will you write to?
   1. Select connection: Select the field and choose a
      Salesforce organization that you connected to Docusign. Multiple options display if you connected more than one Salesforce
      organization.
   2. (Optional) Add New Connection: Select this
      button if you want to [connect more than one Salesforce
      organization](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=ogk1719256147008.html&_LANG=enus).
   3. (Optional) Manage Connections: Select this
      button if you want to [add, edit, refresh, or delete a
      Salesforce connection](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=uaw1719259450391.html&_LANG=enus).
   4. Salesforce object: Search for an object or
      choose one from the dropdown menu.
   5. Write settings: Select an option from the list
      that opens:

      - Update or Create: Update a record or
        create a new record if none exists.
      - Update: Update data on an existing
        record.
      - Create: Create a new record.
   6. Select Next.

      ![A screenshot displays a Writeback to Salesforce workflow step with the Connection, Salesforce object, and Write settings highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_Writeback_Where_will_you_write_to_Salesforce_object.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NjcsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.YwNIwpOLDG8izw7TqZlZeL0Y_MwAHGVALfTIJ3ZK1pg&_LANG=enus "A screenshot displays a Writeback to Salesforce workflow step with the Connection, Salesforce object, and Write settings highlighted.")
5. Which fields are you writing to?: Select workflow fields to
   map to Salesforce fields. When workflow recipients enter data, Docusign updates Salesforce records.
   1. Select Add or Remove Fields.

      The Add or Remove Fields page displays.
   2. Select the check box next to each Salesforce field that you'd like to
      use as a workflow variable. You'll be able to use these fields in later
      steps.

      ![A screenshot displays the Add or Remove Fields page with two Salesforce fields selected and highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_Writeback_to_Salesforce_Add_or_Remove_Fields.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NjcsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.YwNIwpOLDG8izw7TqZlZeL0Y_MwAHGVALfTIJ3ZK1pg&_LANG=enus "A screenshot displays the Add or Remove Fields page with two Salesforce fields selected and highlighted.")
   3. Select Save and Exit.

      The Add or Remove Fields dialog box closes. The fields that you
      selected display.
   4. Fields: Add a variable or step for each
      field:

      - Manual entry: Add text for your new
        variable and select Save.

        Note: The name that you enter must exactly
        match an existing field. For example, the rule fails if you
        enter CompanyName for the
        Company\_Name
        variable.
      - Select a field from the start step or another step in this
        workflow:

        Note: Steps like Read
        from HubSpot and Read from
        Salesforce display fields from up to five
        records. If more than five records are present, Salesforce
        logic decides which five to display. Selecting fields from
        multiple records helps you automate processes like mapping
        multiple Salesforce contacts to multiple Docusign participants.![A screenshot displays Read from Salesforce fields highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_Read_Workflow_field_selections.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NjcsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.YwNIwpOLDG8izw7TqZlZeL0Y_MwAHGVALfTIJ3ZK1pg&_LANG=enus "A screenshot displays Read from Salesforce fields highlighted.")

        You can use the fields that you select here in
        later steps like Collect Data with
        Webforms or Send Documents for
        Signature.
   5. (Optional) Overwrite if field value exists:
      Select this check box if you want the data entered by a workflow
      participant to overwrite existing Salesforce data.
   6. Repeat this process for each variable displayed.
   7. Select Next.

      ![A screenshot displays the Account ID Salesforce field and the Overwrite if field value exists checkbox highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_Writeback_to_Salesforce_Which_fields_are_you_writing_to.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NjcsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.YwNIwpOLDG8izw7TqZlZeL0Y_MwAHGVALfTIJ3ZK1pg&_LANG=enus "A screenshot displays the Account ID Salesforce field and the Overwrite if field value exists checkbox highlighted.")
6. Which record are you writing to?: Create a rule that
   matches a Salesforce field with an equivalent workflow field. This allows
   Workflow Builder to interact with Salesforce data.
   1. Salesforce field: Select a Salesforce field to
      match with a Docusign field.

      Note: When you select a
      (reference) type field, a new field opens
      below it. Choose a child object from the new field.
   2. Operator: Choose an option like Equal
      to, Less than, and so on.
   3. Workflow field: Select a Docusign field to match with your Salesforce field. The following
      options display:

      - Manual entry: Add text for your new
        variable and select Save.
      - Select a field from the start step or another workflow
        step.
   4. Select Done to save your rule.

      ![A screenshot displays a Salesforce field, an operator value of Greater than or equal to, and a workflow field all highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_Writeback_step_Which_record_are_you_writing_to_Rule.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NjcsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.YwNIwpOLDG8izw7TqZlZeL0Y_MwAHGVALfTIJ3ZK1pg&_LANG=enus "A screenshot displays a Salesforce field, an operator value of Greater than or equal to, and a workflow field all highlighted.")
7. (Optional) Add more rules:
   - +AND: Workflow Builder only imports Salesforce data that
     matches all of your rules.
   - +OR: Workflow Builder imports Salesforce data that matches
     any of your rules.
8. Select Apply.

   The Writeback to Salesforce dialog box closes. The page displays your new step.

You have completed the process. You created and linked two
Writeback to Salesforce steps for related objects. You can repeat this process to map
other objects like Contact and Case. When this step is triggered in a workflow, it
updates records for all objects you mapped. Workflow participants are not required to
enter data for these fields.
