# Prepare Your Salesforce Users to Start Docusign Workflows

Learn to prepare your Salesforce users so that they can start Docusign workflows from Salesforce.

Important: Do not follow these steps if you already have
Docusign Apps Launcher v7.4 or higher installed. Only follow
these steps to add a Start
Docusign Workflow  action to an organization with v7.3 or
lower installed.

Before you begin
:   - Purchase a Docusign Intelligent Agreement
      Management (IAM) plan.
    - Install Docusign Apps Launcher version 7.4 or
      higher.
    - [Build a workflow to be
      started by your Salesforce flow](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=ovh1741282680990.html&_LANG=enus).

Before standard Salesforce users can start Docusign workflows, you
must assign the necessary permissions. Use this procedure to assign the necessary
Docusign and Salesforce user permissions:

1. Log on to your Docusign account as an administrator and go to Admin >  Permission Profiles.
2. Copy the DS Sender profile:
   1. DS Sender: Select Actions and
      choose Copy from the dropdown menu.

      The Copy Permission Profile dialog box opens.
   2. Name: Enter Workflow Builder DS
      Sender.
   3. Select the User Permissions tab.
   4. Scroll to Workflows and choose
      Use from the dropdown menu.
   5. Select Add.

   The Copy Permission Profile dialog box closes.
3. Add users to the new Workflow Builder DS Sender profile. Only add
   users that must start Workflow Builder workflows from Salesforce:
   1. Go to Admin >  Users.

      The Users page opens.
   2. Select Actions next to the user that you want to
      add and choose Edit from the dropdown menu.
   3. Permission Profile: Choose Workflow Builder DS
      Sender from the dropdown menu.
   4. Select Save.

   The user's page closes.
4. Assign Flow User permissions to Salesforce users who must
   start Docusign workflows:
   1. Log on to your Salesforce organization as an administrator, select the
      gear icon ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/salesforcegear.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4ODQsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.G8TMgHTkdEvg1Ftv3TVX3NntW6ImSPUgclUmQTY6PEM&_LANG=enus), and
      choose Setup from the dropdown menu.
   2. Enter and select Users in the Quick Find
      field.
   3. Select Edit next to a user.

      The User Edit page opens.
   4. Select the Flow User check box and select
      Save.

      ![A screenshot displays the Flow User check box selected and the Save button highlighted on the User Edit page.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/A-screenshot-displays-the-Flow-User-check-box-selected-and-the-Save-button-highlighted-on-the-User-Edit-page.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4ODQsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.G8TMgHTkdEvg1Ftv3TVX3NntW6ImSPUgclUmQTY6PEM&_LANG=enus "A screenshot displays the Flow User check box selected and the Save button highlighted on the User Edit page.")

   The User Edit page closes. Users must [re-activate their Docusign access in Salesforce](https://support.docusign.com/s/document-item?bundleId=vbw1648698462634&topicId=ymi1586134213877.html&_LANG=enus). After re-activating, users
   can start Docusign workflows.

You have completed the process. You have assigned the necessary Docusign and Salesforce permissions. After re-authorizing their Docusign users, these users can start Docusign
workflows from Salesforce.
