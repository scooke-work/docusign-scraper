# Create a Connect Configuration to Add Salesforce Merge Fields to a Workflow Step

Learn how to create a Connect configuration to add Salesforce merge fields to your workflow's Send Documents for Signature step.

## Before you begin

- Set up a webhook listener. Review URL to Publish in [Connect Configuration Settings](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=zjq1665170940873.html&_LANG=enus "Docusign Connect sends updates to other applications. Look up the settings that you use to set up Docusign Connect.") for more information.
- Enable Connect for your Docusign account.
- Enable Workflow Builder for your Docusign account.

Note: If you want to follow only selected groups of users, create the groups first. See [Add Groups](https://support.docusign.com/s/document-item?bundleId=pik1583277475390&topicId=mbg1583277362677.html&_LANG=enus) for details.

Docusign Connect sends updates (event messages or
webhooks) about your Docusign activity to other
applications. Create a Connect configuration to add a Salesforce merge
field in your workflow's Send Documents for Signature step.

The Send Documents for Signature step uses your Connect configuration to access
Salesforce fields from a Docusign envelope template. If you do not
create a Connect configuration, you cannot add a Salesforce field to a Send
Documents for Signature step.

After creating a Connect configuration, you can add a Salesforce merge field to your
workflow's Send Documents for Signature step.

Use this procedure to create a Salesforce Connect configuration in your Docusign account:

1. Log in to your Docusign account as an administrator and go to Admin >  INTEGRATIONS >  Connect.
2. Select Add Configuration and choose Salesforce from the dropdown menu. ![A screenshot displays the Add Configuration dropdown menu with Salesforce highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/A-screenshot-displays-the-Add-Configuration-dropdown-menu-with-Salesforce-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MjIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.GMJX3Ly7HqSv8Ujzzr3y1TB8zmduSGwakg89mO9NSo4&_LANG=enus "A screenshot displays the Add Configuration dropdown menu with Salesforce highlighted.")

   The Connect with Salesforce dialog box opens.
3. Salesforce Environment: Choose an environment to connect to Docusign:

   - Production: Connect to a Salesforce production environment.
   - Test: Connect to a Salesforce sandbox environment.
   - URL: Enter your Salesforce environment's URL.
     - Sample URL for OAuth: https://[your\_salesforce\_instance\_url\_here]/services/oauth2/authorize
     - Sample URL for other:
       https://[your\_salesforce\_instance\_url\_here]/services/Soap/u/39.0
4. Salesforce SOAP Web Services API Version: Select the appropriate API version from the dropdown menu.
5. Salesforce Login Method: Choose a method to connect Docusign and Salesforce:

   Important: When creating your Connect configuration, enter the same Salesforce administrator credentials you used to [install the Salesforce app](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=jxo1706295810553.html&_LANG=enus).

   - OAuth (Open Authorization) - Recommended:
     1. Select Connect with OAuth. The Salesforce login page displays.
     2. Enter Salesforce administrator credentials and select Log In.
     3. Allow Access?: Select Allow.
   - Salesforce username and password - Less secure:
     1. Salesforce Username: Enter your Salesforce administrator username.
     2. Salesforce Password and Security Token: Enter your Salesforce administrator password and security token. Append your security token to your password. For example, if your password is MyPassword and your security token is XXXXXX, enter MyPasswordXXXXXX.
     3. Select Connect.![A screenshot displays the Connect with Salesforce dialog box with Production, API 54, and OAuth login method selected.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/A-screenshot-displays-the-Connect-with-Salesforce-dialog-box-with-Production-API-54-and-OAuth-login-method-selected.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MjIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.GMJX3Ly7HqSv8Ujzzr3y1TB8zmduSGwakg89mO9NSo4&_LANG=enus "A screenshot displays the Connect with Salesforce dialog box with Production, API 54, and OAuth login method selected.")

   The Connect for Salesforce Configuration page displays.
6. Complete the setup of your connect configuration:
   1. System Settings: Choose Active from the Connection dropdown menu.
   2. Sender Selectable Items: Select the checkbox for each Salesforce object that you want senders to access.
   3. [Choose other Connect options like Associated Users, Envelope Events, Recipient Events, and so on](https://support.docusign.com/s/document-item?bundleId=vob1727899215236&topicId=ctv1583277395112.html&_LANG=enus)

      ![A screenshot displays the Connection field set to Active and the Sender Selectable Items section highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/A-screenshot-displays-the-Connection-field-set-to-Active-and-the-Sender-Selectable-Items-section-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MjIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.GMJX3Ly7HqSv8Ujzzr3y1TB8zmduSGwakg89mO9NSo4&_LANG=enus "A screenshot displays the Connection field set to Active and the Sender Selectable Items section highlighted.")
7. Select Save.

You have completed the process. You have connected your Salesforce
organization to your Docusign account. You can add Salesforce merge
fields to your workflow's Send Documents for Signature step.
