# Supported Salesforce Field Types

Learn about the field types that you can use in Salesforce workflow
steps.

Note: Docusign continues to add support for new field
types. If you can't access new field types or configurations, open your Salesforce step and save it again.

When you build
Salesforce workflow steps, you match
Salesforce fields with other workflow fields. Workflow Builder formats data from Salesforce fields in specific ways.

You can add the following field types to Salesforce steps:

| Field type | Description |
| --- | --- |
| Boolean | Add true/false options. |
| Checkbox | Add a checkbox to select true/false or yes/no options. |
| Currency | Add a unified currency field to capture, map, and manage monetary amounts and currency codes throughout workflows. |
| Date | Add a date with no time. Unlike adding dates into string values, workflows can use date fields to schedule actions. |
| Datetime | Add a date and time. Unlike adding dates and times into string values, workflows can use datetime fields to schedule actions or timestamp events. |
| Double | Add a number with decimal points. |
| Dropdown | Add list of allowed values in a dropdown menu. |
| Email | Add an email address. You must use valid formatting, like user@company.com. |
| Encryptedstring | Add a field for encrypted text that can be used for secure data handling, such as storing sensitive information like passwords or API keys. Workflows can use this data securely, without showing it in plain text. |
| ID | Add the unique identifier of a Salesforce record, like a company or contact. |
| Integer | Add a whole number that can be positive, negative, or zero, and does not include fractions or decimals. |
| List | Add a list of items. |
| Phone | Add a phone number. You must use a valid country code, like `+61 xxx xxx xxxx` |
| Picklist | Add a list from which users can make selections. |
| String | Add human-readable text, like words and sentences. |
| Text | Add fields into which users can enter text. |
| Textarea | Add text entry fields with attributes like minimum and maximum character limits, placeholder text, and validation rules. |

## Date field formatting

Due to Salesforce date restrictions, you can only use the following date formats in a
Writeback to Salesforce workflow step:

- YYYY-MM-DD
- YYYY-MM-DDThh:ssZ
- YYYY-MM-DDThh:ss.sssZ

- YYYY/MM/DD
- YYYY/MM/DDThh:ssZ
- YYYY/MM/DDThh:ss.sssZ

## Phone number field formatting

Phone number format restrictions

Due to Salesforce restrictions, you can only use the following formats in
Writeback to Salesforce or Read from Salesforce steps:

- PhoneNumberFormat.E164
- PhoneNumberFormat.INTERNATIONAL
- PhoneNumberFormat.NATIONAL

Country code formatting

Add a country code if you
are using a phonenumber field in the following Workflow Builder
for Salesforce options:

- Read from Salesforce
- Writeback to Salesforce:
  - Update or Create
  - Update

Add the country code in front of the contact number. For example,
for a phone number with a code of +61, use the following format: `+61
xxx xxx xxxx`.

If your Writeback to Salesforce step has the
Create option selected, you are not required to use a
country code. However, using a country code is still recommended.

Correcting phone numbers

- Docusign adds the United States country code of
  `+1` to Salesforce records lacking a country
  code.
- Workflow task recipients can change incorrect country codes.
- Workflow task recipients can also change incorrect phone numbers.

Example 1:: A workflow sends a task from a Salesforce
record with no country code. Rather than refusing to advance the workflow task,
Docusign automatically adds the United States country code
of `+1` to the phone number. The organization is in the
United States, so the workflow recipient keeps the `+1`
country code and completes the task. The Writeback to Salesforce step adds the
`+1` country code to the
record.

Example 2: A workflow sends a task from a
Salesforce record with no country code. Docusign adds the
United States country code of `+1` to the task. The
recipient knows that the organization is from Australia and changes the country
code to `+61`. The recipient completes the task. The
Writeback to Salesforce step changes the record's country code to
`+61`.

Example 3: A workflow
sends a task from a Salesforce record with an incorrect phone number of
`1234`. Rather than refusing to advance the workflow
task, Docusign sends the task with an editable phone number.
The recipient enters the correct phone number of `+1
555-666-7777` and completes the task. The Writeback to Salesforce
step changes the record's phone number from `1234` to
`+1 555-666-7777`.

## Checkbox field formatting

You can connect Salesforce checkbox fields to workflow checkbox fields. This is
useful when you want your workflow to read from and writeback to Salesforce yes/no
fields.

Example: Your Salesforce organization has an Express
Shipping checkbox field. The checkbox has values of
Checked = Yes (true) and Unchecked = No
(false).

You want workflow task recipients to add express shipping to their orders by
selecting the checkbox. You configure the following steps in your workflow:

- Read from Salesforce: You add the Salesforce
  Express Shipping (checkbox) field.
- Collect Data with Web Forms: You add an
  Express Shipping checkbox field and map it to the
  Salesforce checkbox.
- Writeback to Salesforce: You map the Salesforce Express
  Shipping field to the web form field.

A workflow task recipient selects the Express Shipping
checkbox on the web form and completes the task. The Writeback to Salesforce step
adds a check to the record's Express Shipping checkbox.

## Dropdown field formatting

You can connect Salesforce dropdown fields to workflow dropdown fields. This is
useful when you want your workflow to read from and writeback to Salesforce dropdown
fields.

Example: Your Salesforce organization has a Product
Size dropdown field. The dropdown field has values of
Large, Medium, and
Small.

You want workflow task recipients to select a size when ordering shirts. You
configure the following steps in your workflow:

- Read from Salesforce: You add the Salesforce
  Product Size (picklist) field.
- Collect Data with Web Forms: You add a
  Product Size (picklist) dropdown field and map it
  to the Salesforce checkbox.
- Writeback to Salesforce: You map the Salesforce
  Product Size field to the web form field.

A workflow task recipient selects the Medium option from the
web form's Product Size dropdown menu and completes the task.
The Writeback to Salesforce step changes the record's Product
Size value to Medium.
