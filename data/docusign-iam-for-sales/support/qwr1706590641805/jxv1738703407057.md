# Add Salesforce Merge Fields to a Send Documents for Signature Step

Learn to add a Salesforce merge field to a Send Documents for Signature step.

Before you begin
:   - Required: Installation of the Salesforce app
      from Docusign App Center. Review the [Installing Extension Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more information.
    - Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account administrator.
    - Required: [Create a Connect configuration to use Workflow Builder with
      Salesforce](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=xza1742958339120.html&_LANG=enus).
    - Required: [Build a Docusign envelope template](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=gou1591745811909.html&_LANG=enus) containing [Salesforce merge fields from
      a single object](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=wnq1738608337037.html&_LANG=enus "Learn to fix the \"This template is not supported\" error in a Workflow Builder Send Documents for Signature step.").

You can access a Docusign envelope template in a workflow by adding a Send Documents for
Signature step. When your envelope template contains Salesforce merge
fields, you can add the Source Record ID to a workflow. Docusign uses these merge fields to update Salesforce records when
recipients sign agreements.

Example: You create a workflow that contains a Read from
Salesforce step. You connect an envelope template containing Salesforce fields to
your Send Documents for Signature step. Your envelope template contains fields for
the Contact object.

When you configure the Send Documents for Signature step, you add Contact ID
to the Source Record ID field. The Contact ID variable
contains the Title field.

A workflow recipient signs an agreement. The recipient updates the Title field from
HR Assistant to HR Manager when
signing. Docusign updates the recipient's title to HR
Manager on the contact record.

Use this procedure to add a Source Record ID to your Send
Documents for Signature step:

1. Build a [Docusign envelope
   template](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=gou1591745811909.html&_LANG=enus) containing Salesforce merge fields [from a single object](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=wnq1738608337037.html&_LANG=enus "Learn to fix the \"This template is not supported\" error in a Workflow Builder Send Documents for Signature step.").

   Note: If your envelope template contains merge fields from more than one
   object (Account, Opportunity, and so on) [an error displays](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=wnq1738608337037.html&_LANG=enus).
2. [Create a basic
   workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus) containing [Read from
   Salesforce](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=idr1713985144297.html&_LANG=enus) and [Send Documents for Signature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) steps.
3. Add a Salesforce signer to your Send Documents for Signature step:
   1. Who is the signer?: Add a Salesforce Contact as a
      signer.
   2. Select Next.
4. Add a Salesforce merge field to your Send Documents for Signature step:
   1. Connect document fields (Merge Fields).
   2. Source Record ID: Select a field from your Read
      from Salesforce step. ![A screenshot displays the Connect document fields (Merge Fields) section with Contact ID highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/A-screenshot-displays-the-connect-document-fields-section-of-a-Workflow-Builder-for-Salesforce-Get-Signatures-step.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NDYsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.qI2Dgp2rUkFGJ93d_S2LgK17fZbu5fa0CBy3OzCeZv0&_LANG=enus "A screenshot displays the Connect document fields (Merge Fields) section with Contact ID highlighted.")
   3. Select Next.
   4. Select Apply.

   The Send Documents for Signature dialog box closes.

You have completed the process. You have added a Source Record ID to
your Send Documents for Signature step. When task recipients change field data, Docusign updates the corresponding Salesforce record fields.
