# Configure a Store Files in Salesforce Step

Learn to add and configure a Store files in Salesforce step to add files to Salesforce
records.

## Before you begin

- Required: Installation of the Salesforce app from Docusign App Center. Review the [Installing Extension Applications for
  Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus)
  topic for more information.
- Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account
  administrator.
- Required: Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add steps like
  [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus), [web forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and more to gather recipients,
  documents, and merge fields for your workflow.

The Store files in Salesforce step adds signed agreements and
certificates of completion to Salesforce records. Administrators choose files to
store, a location to store them, and a naming convention for the files. The Store
files in Salesforce step decides the name and storage location of agreements sent
from workflows.

Example: You build a workflow with a Store files in Salesforce
step. Your workflow sends an agreement to three recipients. When all three
recipients sign the agreement, Docusign uses Store files in
Salesforce variables to name the agreement and add it to the correct record.

Note: Currently, the Store files in Salesforce step limits file sizes
to 10 MB.

Use the following procedure to configure a Store files in Salesforce step for your
workflow:

1. Add a Store files in Salesforce step as a Docusign administrator:
   1. Go to Agreements > Workflows and select Edit next to a workflow
      name.

      The workflow editor page displays.
   2. Select the Add a step
      ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_add_step_plus_sign.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NzIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.sPe-eWhrmfT4MuyYBkdMyUBhJ58IDd-IntyF96V3PzY&_LANG=enus) icon.

      The Add New Step dialog box displays.
   3. Search for steps: Enter `Store files
      in Salesforce` in the search field and select the
      Store files in Salesforce step that displays.

      The workflow editor page displays with the Store files in
      Salesforce step added.
2. Name your Store files in Salesforce step:
   1. Select Configure on the Store files in
      Salesforce step.

      The configuration dialog box displays.
   2. Step Name: Keep or edit the name. It is helpful
      to add the name of the Salesforce object, like `Account -
      Store files in Salesforce`.
3. What are you storing?:
   1. Select a file from the dropdown menu.
   2. Select Next

      ![A screenshot displays the What are you storing section of a Store files in Salesforce step with the file selection highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_Store_Files_What_are_you_storing.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NzIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.sPe-eWhrmfT4MuyYBkdMyUBhJ58IDd-IntyF96V3PzY&_LANG=enus "A screenshot displays the What are you storing section of a Store files in Salesforce step with the file selection highlighted.")
4. Where are you storing it?
   1. Connection: Select the field and choose a Salesforce organization that
      you connected to Docusign. Multiple options display if
      you connect more than one Salesforce organization.
   2. (Optional) Add New Connection: Select this button if you want to [connect more than one Salesforce
      organization](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=ogk1719256147008.html&_LANG=enus).
   3. (Optional) Manage Connections: Select this
      button if you want to [add, edit, refresh, or delete a
      Salesforce connection](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=uaw1719259450391.html&_LANG=enus).
   4. Select Record: Select a field from the start
      step or another step in this workflow:

      - Manual entry: Add text for your new
        variable and select Save.

        Note: The name that you enter must exactly
        match an existing field. For example, the rule fails if you
        enter CompanyName for the
        Company\_Name
        variable.
      - Select a field from the start step or another step in this
        workflow:

        Note:
        - You can only select records from other workflow
          steps that are built on the same object as this step
          (Account, Contact, and so on.)
        - Steps like Read from HubSpot
          and Read from Salesforce
          display fields from up to five records. If more than
          five records are present, Salesforce logic decides
          which five to display. Selecting fields from
          multiple records helps you automate processes like
          mapping multiple Salesforce contacts to multiple Docusign participants.
        - You can use the fields that you select here in later
          steps like Collect Data with
          Webforms or Send Documents
          for Signature.![A screenshot displays Read from Salesforce fields highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_Read_Workflow_field_selections.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NzIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.sPe-eWhrmfT4MuyYBkdMyUBhJ58IDd-IntyF96V3PzY&_LANG=enus "A screenshot displays Read from Salesforce fields highlighted.")

      The field that you selected displays.
   5. Select Next.

      ![A screenshot displays the Select Connection and Select Record fields with the Next button highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_Store_files_in_Salesforce_Where_are_you_storing_it.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NzIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.sPe-eWhrmfT4MuyYBkdMyUBhJ58IDd-IntyF96V3PzY&_LANG=enus "A screenshot displays the Select Connection and Select Record fields with the Next button highlighted.")
5. What will you call it?: Use variables and text to create a
   custom file name for your agreements:
   1. Add Variable: Use the search field or dropdown
      menu to select variables to add to the agreement name.
   2. Add Text: Enter text to add to the
      agreement name and select Add.

      Note: The File name field
      displays the naming convention. As you add variables and text, the
      name updates. Example: File name:
      [Account ID][Envelope ID]MSA.

      ![A screen capture displays the Store files in Salesforce step settings with the Add Variable and Add Text features used to create a file name.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_Store_files_What_will_you_call_it.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NzIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.sPe-eWhrmfT4MuyYBkdMyUBhJ58IDd-IntyF96V3PzY&_LANG=enus "A screen capture displays the Store files in Salesforce step settings with the Add Variable and Add Text features used to create a file name.")
   3. Select Apply.

   The Store file in Salesforce dialog box closes. The page displays your new step.

You have completed the process. You have added a Store files in
Salesforce step to your workflow. When signers complete an agreement, Docusign adds the agreement and certificate of completion to the
specified record.
