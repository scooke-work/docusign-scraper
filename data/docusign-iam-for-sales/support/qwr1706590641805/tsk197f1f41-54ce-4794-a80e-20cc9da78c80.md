# Use the Record Count Variable to See How Many Records a Workflow Finds

Learn to use the Record Count field to find how many records are returned in a Read
from Salesforce step.

## Before you begin

- Required: Installation of the Salesforce app from Docusign App Center. Review the [Installing Extension Applications for
  Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus)
  topic for more information.
- Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account
  administrator.
- Required: Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add steps like
  [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus), [web forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and more to gather recipients,
  documents, and merge fields for your workflow.

You can use a Read from Salesforce step to import information
from your Salesforce records to use in workflow tasks.

The Read from Salesforce step displays fields from up to five
records. Selecting fields from multiple records is useful for processes involving
multiple people, like mapping several Salesforce contacts to several workflow participants.

Record Count is an Integer (number) variable that tracks how
many records your Read from Salesforce step finds. You can use Record Count in a
Branching Rule step to send your workflow down the
following paths:

- True Path: Proceed with the workflow if the record
  count is greater than zero.
- False Path: Take alternative action if the record count
  is zero.

The Record Count variable ensures that your workflows don't fail when no
records exist.

Example: You build a workflow to verify employee data. The
workflow contains a Read from Salesforce step with a Branching Rule below it. The
Branching Rule states that if the Read from Salesforce step returns zero records,
the workflow moves to the False path. A user starts the
workflow that finds zero records and triggers the False path. The workflow sends a
Docusign Web Form to the recipient to collect their details.
The recipient submits the form and the workflow uses that data to create a new
record in your Salesforce organization.

Note: If you don't see
`Record Count`, open your Read from Salesforce step in the
designer and save it.

Use this procedure to create a Branching Rule that works with your Read from
Salesforce step:

1. Add a Branching Rule step as a Docusign administrator:
   1. Go to Agreements > Workflows and select Edit next to a workflow
      name.

      The workflow editor page displays.
   2. Select the Add a step
      ![A screenshot displays a white plus sign in a purple square.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_add_step_plus_sign.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NDIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.nirTdVX2HfKJkRZCF2sVjZ_tn2iwdUjAXHv_hJEKPsw&_LANG=enus "A screenshot displays a white plus sign in a purple square.") icon.

      The Add New Step dialog box displays.
   3. Search for steps: Enter and select
      `Add a Branching Rule`.

      - When adding the step between existing steps, select which path
        to move the steps that come after the new rule. Then select
        Configure.
      - If you are adding the step to the end of your workflow, select
        Configure.

   The Add conditional rule dialog box displays.
2. Configure your Branching Rule:
   1. What will the rule check? Select the field and
      choose Record Count from the dropdown menu.
   2. What operator will the rule use? Choose an option like Equal
      to, Less than, and so on.
   3. What result will the rule look for? Enter a
      value between zero and five. Read from Salesforce steps cannot return
      more than five records.
   4. Select Done.

   ![Workflow Builder Branching Rule configuration showing Record Count selected as the field, an operator such as Equal to, and a numeric value between 0 and 5.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Read_from_Salesforce_or_HubSpot_Record_Count_used_in_rule_.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4NDIsInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.nirTdVX2HfKJkRZCF2sVjZ_tn2iwdUjAXHv_hJEKPsw&_LANG=enus "Workflow Builder Branching Rule configuration showing Record Count selected as the field, an operator such as Equal to, and a numeric value between 0 and 5.")
3. (Optional) Select + AND or + OR
   to add conditions to your rule. You can configure up to five conditions.
4. (Optional) Select the option Merge true and false
   branches? if you want to combine the branches to continue the
   workflow. This optional setting is useful if you want to combine the paths to
   use a single-step outcome.

   Important: Variables and files generated from
   steps inside of a merged Branching Rule will not be available to use in
   steps occurring after the merge. For more details, see [Merging branches](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=jaf1716918359288.html&anchor=jaf1716918359288__section-426&_LANG=enus).
5. Select Apply.

The workflow editor page displays with the Branching Rule step added.
