# Format String Values in Salesforce Steps

Learn to create concatenated values in Salesforce step string fields.

You can create Salesforce workflow steps that read and update Salesforce data. You
can add string fields to Salesforce steps.

String fields are human-readable text like words, IDs, or simple labels. Workflow Builder uses
string fields as text, not as number or date values used in calculations or
scheduling.

Because Workflow Builder formats string value fields as rich text inputs, you can enter text and add variables in the same field. This is useful when you want Salesforce to store a single, human-readable value built from multiple pieces of data.

Important: The following guidelines apply to formatting Workflow Builder for Salesforce string fields:

- Add text and variables in Writeback to Salesforce fields and rules.
- Add text and variables in Read from Salesforce rules.
- Add multiple variables by selecting the string field's + icon.
- Add up to three string variables in one string field.
- Enter a space after text if you want to add variables after it.
- Add only string variables. You cannot add Boolean, array, or other field types.

## Writeback to Salesforce field formatting

You can add text and variables when formatting Writeback to Salesforce fields. Select a Salesforce field and link it to variables from other workflow steps. You can enter text and up to three string values to a single string field.

Example 1: You create a workflow that includes a
Writeback to Salesforce step. You build the step to create a
new Account record if one doesn't exist. You want each new record's Account Name field
to contain the following content:

- The words `New Customer`
- The `First Name` and `Last Name` variables of a
  Salesforce contact

In the Which fields are you writing to? section, you add
the Account Name string field. You enter the words
`New Customer:`  followed by a space. You select the
field's + icon and add the contact's First
Name and Last Name variables.

A
workflow recipient completes a task for a new customer named Alex Smith. The
Writeback to Salesforce step in your workflow creates a new record. The Account Name
field of the new record displays this text: New Customer: Alex
Smith.![A screen capture displays the process of entering text and variables into the Account Name field of a Salesforce step.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_and_HubSpot_concatenated_string_fields.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MzksInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.Rs3nVUPK0vwMGcL_Nb8d1LnW7KA0SN_7dUvYvgB-5U8&_LANG=enus "A screen capture displays the process of entering text and variables into the Account Name field of a Salesforce step.")

Example 2: You create a workflow that includes a Writeback
to Salesforce step. Your organization has a custom string field named
Products Purchasing. When customers purchase multiple products,
you want all products combined in a single field, rather than in separate fields.

In
the Which fields are you writing to? section, you add the
Products Purchasing string field. You add Opportunity string variables for
`Product 1`
`Product 2`, and `Product 3`.

A workflow
recipient completes a task for a customer that is purchasing cups, straws, and lids.
The Writeback to Salesforce step adds the three products to the customer's
Salesforce record. The record's Products Purchasing field displays this text:
Cups, Straws, Lids.

## Read from Salesforce and Writeback to Salesforce rule formatting

You can add text and variables when formatting Read from and Writeback to Salesforce rules. When you create a rule, you can select a Salesforce string field and link it to workflow fields from other steps. You can format the matching workflow field with text and up to three string values.

Writeback to Salesforce example: You create a workflow that includes a Writeback
to Salesforce step.

In the Which record are you writing
to? section, you add the Owner ID string
field. You select the Equal to operator. For
Workflow field, you select the field's
+ icon and add the Owner ID,
First Name, and Last Name
variables.

A workflow recipient completes a task for a customer with an owner
named Terry Jones. The Writeback to Salesforce step only updates Salesforce records
that match the Owner ID, First Name, and Last Name of Terry Jones.![A screen capture displays the process of adding three variables into the workflow field of a Salesforce step rule.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_for_Salesforce_Rule_concatenated_string_fields.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MzksInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.Rs3nVUPK0vwMGcL_Nb8d1LnW7KA0SN_7dUvYvgB-5U8&_LANG=enus "A screen capture displays the process of adding three variables into the workflow field of a Salesforce step rule.")

Read from Salesforce example: You create a workflow that includes a Read from
Salesforce step.

In the Which records are you reading
from? section, you add the Owner string field.
You select the Equal to operator. For Workflow
field, you enter the words `Owner name and email:` followed by a space. You select the field's +
icon and add the Signer: Name and Signer:
Email variables from a web form.

A workflow recipient
completes a task for a customer named Fontara with an owner named Terry Jones. The
Read from Salesforce step adds records with an owner name of Terry
Jones and an email of terry.jones@fontara.com.
Your workflow can use fields from these records as variables in later
steps.
