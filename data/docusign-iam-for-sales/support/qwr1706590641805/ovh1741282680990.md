# Build a Docusign Workflow to Be Started By a Salesforce Flow

Learn to build a Docusign workflow that can be started by a Salesforce
flow.

Before you begin
:   - Purchase a Docusign Intelligent Agreement
      Management (IAM) plan.
    - Required: Installation of the Salesforce app
      from Docusign App Center. Review the [Installing Extension Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more information.
    - Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account administrator.
    - Required: Access to a suitable template to use for your workflow
      step. Review the [Template Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=xnw1709852160602.html&_LANG=enus) topic for more information.

Workflow Builder automates agreement actions from your Docusign account.

Salesforce Flow is a workflow builder that you can use to automate agreements from your
Salesforce organization.

You can link Salesforce flows to Docusign workflows by adding a **Start Docusign
Workflow Builder** workflow step to your flow.

When a flow starts, Salesforce starts your linked Docusign workflow. Linking flows with
workflows builds an automation chain that creates agreements from Salesforce and Docusign data.

To use a Start Docusign workflow builder workflow step, create a
Docusign workflow that can be started by a
Salesforce flow.

Use this procedure to create a workflow that can be started by a flow:

1. As a Docusign administrator, create your workflow:
   1. Log on to your Docusign account and go to Agreements > Workflows.
   2. Select Create Workflow.

      The Create a workflow page displays.

   4. Select + Add workflow start.

      The Add workflow start page displays.
2. What starts this workflow?: Choose one of the two
   supported options from the dropdown menu:

   ![A screenshot displays the What starts this workflow step with From a Link and From an API Call highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/What-starts-this-workflow-with-From-a-link-and-From-an-API-Call-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4ODksInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.ZPO46lA-wsj_GNQSEFg15eUDk8PNASb_qlGDYv2U4bI&_LANG=enus "A screenshot displays the What starts this workflow step with From a Link and From an API Call highlighted.")

   - (Option 1: Recommended) From an API Call: This
     workflow starts when an API call starts the workflow. Use the following
     procedure:
     1. Select From an API Call from the dropdown
        menu.
     2. Select Next.
     3. Set up variables for dynamic information: Add
        up to ten variables, including the Record ID
        variable:

        Important: Make each
        variable a Text type. Name one variable
        Record ID (case sensitive) and the
        others Variable 1 through
        Variable 9. This is the only variable
        format that you can use in a Salesforce flow.

        1. Select New Variable and choose
           Text from the dropdown menu.
        2. Enter Variable 1 in the field.
     4. Repeat this process for any additional variables and select
        Next.![A screenshot displays the Set Up Variables step with Variable 1 highlighted in green.](https://docusign-be-prod.zoominsoftware.io/api/bundle/qwr1706590641805/page/images/Workflow_Builder_start_step_for_Salesforce_Flow.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4ODksInNoZWFmIjoicXdyMTcwNjU5MDY0MTgwNSJ9.ZPO46lA-wsj_GNQSEFg15eUDk8PNASb_qlGDYv2U4bI&_LANG=enus "A screenshot displays the Set Up Variables step with Variable 1 highlighted in green.")
     5. Will this API call be started by a human or an automated
        process?:
        1. Select Automated process.
        2. Select Apply.
   - (Option 2) From a Link: This workflow starts when a participant
     opens a shared link or a link embedded on an external website. Only use this
     option for non-agreement request flows, or when you do not need to see
     status updates.
     1. Select From a Link from the dropdown
        menu.
     2. Select Apply.

   The page refreshes. Your start step displays.
3. Complete your workflow:
   1. Add more steps to your workflow. These are the steps that are started
      from the [Configure a Start Docusign Workflow](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=azl1741206795047.html&_LANG=enus) step in your
      Salesforce flow.
   2. Select Review & Publish when you have
      finished building your workflow.

You have completed the process. You have built a workflow that can
be started by a Salesforce flow. When your flow starts, Salesforce starts your
workflow.
