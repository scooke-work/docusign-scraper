# Agreement Desk for Salesforce Report Filters

Learn about the filters that you can use to build Agreement Desk for Salesforce reports.

Important: This feature is only available to IAM for
Sales customers. Contact [Docusign support](https://support.docusign.com/s/?language=en_US&_gl=1*74elk4*_gcl_au*MTY5ODQyMjA2MC4xNzY5NjM1OTA1LjE5ODM3MTQzNjguMTc2OTYzNjIwNC4xNzY5NjM2MjA0) or [Docusign
sales](https://www.docusign.com/contact-sales) for more information.

You can build custom Salesforce reports to filter and display your [Docusign
Agreement Desk for Salesforce requests](https://support.docusign.com/s/document-item?bundleId=map8c30aa94-7e9f-4950-92fc-dd22c6d9f835&topicId=con007504df-80b7-41f7-aa5f-1daa72bcc257.html&_LANG=enus "Learn about using Docusign Agreement Desk in Salesforce to streamline your agreement process."). To build your report, you add
columns, rows, and filters. These data points determine the Agreement Desk requests
that Salesforce delivers in your report results.

You can sort and filter your Docusign for Salesforce data based on
the following data points:

Table 1. Agreement Desk for Salesforce report filters

| Filter type | Purpose |
| --- | --- |
| Docusign Agreement Desk Requests: ID | The unique system ID for the Agreement Desk entry itself. Use this option to find one specific record or prevent duplicates in a list. |
| CreatedDate | The date the agreement request was first started. Use this option for measuring time to signature or how many new requests were created in a time span. |
| LastActivityDate | The last time someone interacted with the request, like editing or sending an email. Use this option to identify information like stalled agreements that haven't been updated recently. |
| Owner | The person currently responsible for the request, like a legal or sales representative. Use this option to view who is managing your agreements. |
| RecordID | The ID of the Salesforce record. Use this option to view the record that this agreement is linked to. |
| RecordName | The name of the linked Salesforce record, like Burlington Textiles. Use this option to display business context for the agreement. |
| RecordType | The type of record displayed. Use this option to filter different types of Salesforce records, like sales opportunities or partner accounts. |
| RequestID | The unique identifier specific to a Docusign request. Use this option to track a request across the Docusign IAM platform. |
| RequestType | The agreement's categorization, like MSA, NDA, or SOW. Use this option to view the types of agreements that your team is working on. |
| Status | The stage of an agreement. Use this option to view the progress of your agreements, like Draft, In Review, Signed, or Canceled. |
| Submitter | The person who originally completed the intake form or sent the email to start the process. Use this option to view the people responsible for initiating agreements. |
| Title | The subject line or name given to the specific agreement request. Use this option to identify a deal without opening a request. |
