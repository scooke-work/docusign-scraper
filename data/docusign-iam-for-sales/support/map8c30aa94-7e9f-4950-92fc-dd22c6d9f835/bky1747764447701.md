# Learn About Using Agreement Manager in Salesforce

Learn about using Docusign Agreement Manager to manage your Docusign agreements in Salesforce.

Important: This feature is only available to IAM for Sales
customers. Contact [Docusign support](https://support.docusign.com/s/?language=en_US&_gl=1*74elk4*_gcl_au*MTY5ODQyMjA2MC4xNzY5NjM1OTA1LjE5ODM3MTQzNjguMTc2OTYzNjIwNC4xNzY5NjM2MjA0) or [Docusign sales](https://www.docusign.com/contact-sales)
for more information.

Agreement Manager is a smart repository for your Docusign agreements. You can use Agreement Manager to
store, manage, and analyze your agreements.

More than just document storage, Agreement Manager is powered by [Docusign Iris](https://support.docusign.com/s/document-item?bundleId=fzd1707173174972&topicId=usy1712258503642.html&_LANG=enus), an AI tool that identifies and stores key attributes of your agreements. Agreement Manager makes it easy for users to find agreements, access vital information quickly, and gain valuable insights. Administrators can configure Agreement Manager to limit user access to agreements, ensuring that users can only access the agreements they need.

## What Agreement Manager displays

Note: Because Docusign uses AI to extract key information from your agreements, completed envelopes take some time to display in Agreement Manager.

- The Docusign Completed Agreements component displays agreements sent from the
  following applications:
  - eSignature for Salesforce
  - CLM for Salesforce
  - Agreement Desk for Salesforce
  - Workflow builder for Salesforce
- The component displays agreements sent both before and after you add the component to your layouts.
- The component displays agreements sent from the source record and its child records. It does not display agreements from parent objects.
- Account records display agreements from the account and all of its child objects.

## Agreement Manager file security

Docusign security settings control who can view agreements in the
Docusign Completed Agreements component.

Example: A Salesforce user has no access to an agreement named
Fontara NDA in the connected Docusign
account. The user opens a Fontara record. The Docusign Completed
Agreements component does not display the Fontara NDA
agreement.

## Examples of what Agreement Manager displays

Example 1: A user sends agreements from an account named
Fontara. The user also sends agreements from the Fontara account's opportunity
record. On the Fontara account record, the Docusign Completed
Agreements component displays agreements from the account and opportunity.

Example 2: A user sends agreements from a Fontara contact. They
also send agreements from the related Fontara account. On the Fontara opportunity
record, the Docusign Completed Agreements component displays
agreements from the opportunity, but not the account.
