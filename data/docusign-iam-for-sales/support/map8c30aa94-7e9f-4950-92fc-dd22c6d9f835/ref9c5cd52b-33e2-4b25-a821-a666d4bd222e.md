# Agreement Desk for Salesforce User Access

Learn about granting Salesforce user access to the Docusign Agreement
Requests component.

Important: This feature is only available to IAM for
Sales customers. Contact [Docusign support](https://support.docusign.com/s/?language=en_US&_gl=1*74elk4*_gcl_au*MTY5ODQyMjA2MC4xNzY5NjM1OTA1LjE5ODM3MTQzNjguMTc2OTYzNjIwNC4xNzY5NjM2MjA0) or [Docusign
sales](https://www.docusign.com/contact-sales) for more information.

Salesforce users require the following access to use Agreement Desk:

- [A Docusign Apps
  Launcher user](https://support.docusign.com/s/document-item?bundleId=srl1586134368658&topicId=uwf1586134275195.html&_LANG=enus).
- [A Docusign Intelligent Agreement Management (IAM) user with Agreement Desk access](https://support.docusign.com/s/document-item?bundleId=msb1736279079453&topicId=yao1749836146880.html&_LANG=enus).

Docusign administrators grant access to these user types. Approved users activate their access by selecting a Docusign button or component and entering their Docusign and Salesforce credentials.

## How does the​ component display for different users?

User access dictates how the Docusign Agreement Requests
component displays on a layout. Depending on the access granted, Docusign displays or hides the component on Salesforce
layouts.

| Salesforce user access | How Docusign Agreement Desk displays |
| --- | --- |
| Docusign Apps Launcher and Docusign IAM access | Users can see the Docusign Agreement Desk​ component and act on the agreements displayed. |
| Docusign Apps Launcher access only | Users see a blank component with a spinning wheel icon instead of the Docusign Agreement Desk​ component. |
| No Docusign user access | The Docusign Agreement Requests component does not display on records. |
