# Update Salesforce Settings to Add Docusign IAM Workflow Actions

Learn to update your Salesforce settings to add Docusign IAM
workflow actions on your layouts.

## Before you begin

If you want to start Docusign workflows from Salesforce records,
you can add Start with Docusign actions to your layouts. The
Start with Docusign action displays workflows that you can start
from your records. Users can start Workflow Builder workflows from a record without logging
on to Docusign.

You can use the IAM Launcher Setup interface to add a ​Start with Docusign​ action to your layouts. Access the interface by going to Docusign Setup >  Manage Connections and the IAM Launcher >  Manage​​.

Before you can add workflow actions to your layout, you must update your Salesforce
settings.

Use this procedure to update your Salesforce settings to add Docusign IAM workflow actions on your layouts:

1. Access a page layout as a Salesforce administrator:
   1.    2. Enter and select Object Manager in the Quick Find
      field.

      The Object Manager page displays.
   3. Select a layout from the LABEL column.

      The object's Details page displays.
   4. Select Page Layouts.

      The Page Layouts page displays.
   5. PAGE LAYOUT NAME: Select the page layout that you
      want to update.

      The selected page layout displays.
2. Override the default global publisher layout for this object:
   1. Select Fields in the object layout menu.
   2. Salesforce Mobile and Lightning Experience Actions:
      Select the override the predefined actions link.

      Note: If you do not see the link, your layout does not need updating. You can close the page without taking action.

      The bar displays a list of actions.
   3. Select Save on the object layout menu.

      ![A screen capture displays selecting the Quick Actions in the Salesforce Classic Publisher wrench icon and saving the Account layout changes.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/IAM_Launcher_override_the_global_publisher_layout.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0MDYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.MVVnutmSiW917VGaP79TacXwlB3BiSCxSAquMeI8V7Q&_LANG=enus "A screen capture displays selecting the Quick Actions in the Salesforce Classic Publisher wrench icon and saving the Account layout changes.")

      A Saving layout message displays. The Page Layouts page opens.
3. Repeat the process for each layout to which you want to add Docusign workflow actions.
4. Access the Trusted URLs page:
   1. Enter and select Trusted URLs in the Quick Find
      field.

      The Trusted URLs page displays.
   2. Select New Trusted URL.

      The Trusted URLs page displays.
   3. API Name: Enter
      `Contentful_CDN`.
   4. URL: Enter
      `https://cdn.contentful.com`.
   5. Select Save.

      ![A screenshot displays the Trusted URLs page with Contentful CDN entered in the API Name and contentful dot-com in the URL field.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/IAM_for_Salesforce_Trusted_URLs_for_Contentful.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0MDYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.MVVnutmSiW917VGaP79TacXwlB3BiSCxSAquMeI8V7Q&_LANG=enus "A screenshot displays the Trusted URLs page with Contentful CDN entered in the API Name and contentful dot-com in the URL field.")

      The page refreshes, displaying your new trusted
      URL.
5. Access the Session Settings page:
   1. Enter and select Session Settings in the Quick
      Find field.
   1. Uncheck Lock sessions to the IP address from which they
      originated.
   2. Select Save.

You have completed the process. You have updated the necessary Salesforce settings. You can
add Docusign workflow actions to this layout.
