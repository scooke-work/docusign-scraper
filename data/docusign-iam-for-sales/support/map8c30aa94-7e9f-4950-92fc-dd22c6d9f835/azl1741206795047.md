# Configure a Start Docusign Workflow Step in Salesforce Flow

Learn to configure a Salesforce Flow to start a Docusign workflow.

Before you begin
:   - Purchase a Docusign Intelligent Agreement
      Management (IAM) plan.
    - Install Docusign Apps Launcher version 7.4 or
      higher.
    - [Prepare Your Salesforce Users to Start Docusign Workflows](https://support.docusign.com/s/document-item?bundleId=map8c30aa94-7e9f-4950-92fc-dd22c6d9f835&topicId=bmk1744912007661.html&_LANG=enus "Learn to prepare your Salesforce users so that they can start Docusign workflows from Salesforce.").
    - [Build a workflow to be
      started by your Salesforce flow](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=ovh1741282680990.html&_LANG=enus).

Important: Assign [Workflow Builder permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) to all Docusign
users (not administrators or senders) who must start workflows.

You can link Salesforce flows to Docusign
workflows by adding a Start Docusign
workflow step to your flow.

When a flow starts, Salesforce starts your linked workflow. Linking flows with
workflows builds an automation chain that creates agreements from Salesforce and Docusign data.

Use this procedure to add a **Start Docusign workflow** step to your Salesforce
flow:

1. Open a Salesforce flow and add a **Start Docusign workflow** step:
   1. Locate the point in your flow where you want to add your action.
   2. Hover over the circle icon and select the plus +
      icon that displays.![A screen capture displays the Add Element plus icon in a Salesforce Flow.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/A-screen-capture-displays-the-Add-Element-plus-icon-in-a-Salesforce-Flow.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU2MDQsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.14V1AnEWpQFKEUzLK9ifIHQq3btIpAEB_aQ826kxE0Q&_LANG=enus "A screen capture displays the Add Element plus icon in a Salesforce Flow.")
   3. Choose Action from the Add
      Element dialog box.
   4. Search: Enter and select `Start Docusign Workflow`.
2. Configure a Start Docusign workflow action:
   1. Label: Enter `Start Docusign
      workflow`.
   2. API Name: Keep or change the default name.
   3. Instance Name: Enter a string value for the name
      of the workflow instance.
3. Workflow ID: Add the workflow ID.
   1. As a Docusign administrator, go to Agreements > Workflows and select your workflow name.
   2. Copy the alpha-numeric workflow ID at the end of the URL. Do not copy
      the slash mark. ![A screenshot displays the end of a Docusign workflow URL highlighted in a green box.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/A-screenshot-displays-the-end-of-a-Workflow-Builder-workflow-URL-highlighted-in-a-green-box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU2MDQsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.14V1AnEWpQFKEUzLK9ifIHQq3btIpAEB_aQ826kxE0Q&_LANG=enus "A screenshot displays the end of a Docusign workflow URL highlighted in a green box.")
   3. Return to your flow action and paste the ID (without slash mark) in the
      Workflow ID field.
4. Add workflow variables to your action:
   1. Select the switch next to a workflow variable that you want to include
      in your action.

      The switch displays as blue and says
      included.
   2. Enter and select a variable name in the field.
   3. Repeat this process for up to ten variables. Each variable name must
      exactly match the variable name in your workflow.
5. Select Save >  Activate.

   A message displays: Your flow was
   activated.

You have completed the process. You added a Start Docusign workflow
action to your Salesforce Flow. When your flow starts, it starts your linked
workflow.

Note: Users can only see whether the workflow
instance has failed in the Docusign logs.

Users will
not be able to [monitor the workflow instance process](https://support.docusign.com/s/document-item?language=en_US&bundleId=yff1696971835267&topicId=vcp1698707326210.html&_LANG=enus). To do so, the Docusign administrator can go to Docusign’s platform to see the progress of the workflow
instance.
