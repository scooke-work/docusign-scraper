# Configure a Read from Salesforce Step

Learn to add and configure a Read from Salesforce step to import data from Salesforce records.

Before you begin
:   - Required: Installation of the Salesforce app from Docusign App Center. Review the [Installing Extension Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more information.
    - Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account administrator.
    - Required: Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add steps like [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus), [web forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and more to gather recipients, documents, and merge fields for your workflow.

You can use a Read from Salesforce step to import data from
Salesforce records to your workflow.
Administrators link workflow fields to Salesforce fields in the Read from Salesforce
step. When task recipients open an envelope, they see Salesforce data imported from
the record. Importing Salesforce data improves efficiency and reduces errors by
eliminating manual data entry.

Example: You build a workflow with a Read from Salesforce step. In the Read from Salesforce step, you link to a Salesforce record's Amount field. When a workflow recipient opens a contract, they see the record's Amount field pre-filled.

Use this procedure to create a Read from Salesforce step in a workflow:

1. Add a Read from Salesforce step as a Docusign administrator:
   1. Go to Agreements > Workflows and select Edit next to a workflow name.

      The workflow editor page displays.
   2.       The Add New Step dialog box displays.
   3. Search for steps: Enter and select `Read from Salesforce`.

      The workflow editor page displays with the Read from Salesforce step added.
2. Name your Read from Salesforce step:
   1. Select Configure on the Read from Salesforce step.

      The configuration dialog box displays.
   2. Step Name: Keep or edit the name. It is helpful to add the name of the Salesforce object, like `Account - Read from Salesforce`.
3. Where will you read from?
   1. Connection: Select the field and choose a
      Salesforce organization that you connected to Docusign. Multiple options display if you connect more than one Salesforce
      organization.
   2. (Optional) Add New Connection: Select this button if you want to [connect more than one Salesforce organization](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=ogk1719256147008.html&_LANG=enus).
   3. (Optional) Manage Connections: Select this
      button if you want to [add, edit, refresh, or delete a
      Salesforce connection](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=uaw1719259450391.html&_LANG=enus).
   4. Salesforce object: Search for an object or choose one from the dropdown menu.
   5. Select Next.

      ![A screenshot displays a Read from Salesforce workflow step with the Connection and Salesforce object fields highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Workflow_Builder_for_Salesforce_Read_Where_will_you_read_from_object.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU1MTQsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.STiVdSBl8Cl2DKJWOmQ1QVv7fkHZgFBgTLgvmxZXT34&_LANG=enus "A screenshot displays a Read from Salesforce workflow step with the Connection and Salesforce object fields highlighted.")
4. Which fields would you like to read?: Select workflow
   fields to map to Salesforce fields. When workflows start, Docusign adds this data to agreements.
   1. Select Add or Remove Fields.

      The Add or Remove Fields page displays.
   2. Select the check box next to each Salesforce field that you'd like to
      use as a workflow variable. You'll be able to use these fields in later
      steps.

      Tip: When you select objects on the
      left, related objects open. The Source column
      displays each field's object path. ![A screen capture displays the Add or Remove Fields page with related objects highlighted and fields selected.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Workflow_Builder_for_Salesforce_Add_or_Remove_Fields.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU1MTQsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.STiVdSBl8Cl2DKJWOmQ1QVv7fkHZgFBgTLgvmxZXT34&_LANG=enus "A screen capture displays the Add or Remove Fields page with related objects highlighted and fields selected.")
   3. Select Save and Exit.

      The Add or Remove Fields dialog box closes. The fields that you
      selected display.
   4. Select Next.
5. Which records are you reading from?: Create a rule that matches Salesforce fields with equivalent workflow fields. Workflows use this rule to read Salesforce data. You can create as many rules as you want:
   1. Salesforce field: Select a Salesforce field to match with a Docusign field.

      Note: When you select a (reference) type field, a new field opens below it. Choose a child object from the new field.
   2. Operator: Choose an option like Equal
      to, Less than, and so on.
   3. Workflow field: Select a Docusign workflow field to match with your Salesforce field:

      - Manual entry: Add text for your new variable and select Save.

        Note: The name that you enter must exactly match an existing field. For example, the rule fails if you enter CompanyName for the Company\_Name variable.
      - Select a field from the start step or another step in this
        workflow:

        Note: Steps like Read
        from HubSpot and Read from
        Salesforce display fields from up to five
        records. If more than five records are present, Salesforce
        logic decides which five to display. Selecting fields from
        multiple records helps you automate processes like mapping
        multiple Salesforce contacts to multiple Docusign participants.![A screenshot displays Read from Salesforce fields highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Workflow_Builder_for_Salesforce_Read_Workflow_field_selections.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU1MTQsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.STiVdSBl8Cl2DKJWOmQ1QVv7fkHZgFBgTLgvmxZXT34&_LANG=enus "A screenshot displays Read from Salesforce fields highlighted.")

        You can use the fields that you select here in
        later steps like Collect Data with
        Webforms or Send Documents for
        Signature.
   4. Select Done to save your rule.

      ![A screenshot displays a Salesforce field, an operator value of Greater than or equal to, and a workflow field all highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Workflow_Builder_for_HubSpot_Read_Which_record_are_you_reading_from_Greater_than.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU1MTQsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.STiVdSBl8Cl2DKJWOmQ1QVv7fkHZgFBgTLgvmxZXT34&_LANG=enus "A screenshot displays a Salesforce field, an operator value of Greater than or equal to, and a workflow field all highlighted.")
6. (Optional) Add more rules:
   - +AND: Workflow Builder only imports Salesforce data that
     matches all of your rules.
   - +OR: Workflow Builder imports Salesforce data that matches
     any of your rules.
7. How many records should this step return?:
   - A single record: Returns the first record that
     matches your rule.
   - Multiple records: Returns up to five records that match your
     rule.

     Note: If more than five records are present, Salesforce logic
     decides which five to display.
8. (Optional) Preview this step: See examples of how this rule will work with up to ten Salesforce records:
   1. Select the Preview this step link.

      The Add Data to Preview Step page opens.
   2. Keep or edit the data provided in the field.
   3. Select Preview Salesforce Data.

      ![A screenshot displays the Read from Salesforce Add Data to Preview Step page of a Read from Salesforce workflow step.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Read_from_Salesforce_Add_Data_to_Preview_Step.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU1MTQsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.STiVdSBl8Cl2DKJWOmQ1QVv7fkHZgFBgTLgvmxZXT34&_LANG=enus "A screenshot displays the Read from Salesforce Add Data to Preview Step page of a Read from Salesforce workflow step.")

      The page refreshes, displaying Salesforce data that your Read from Salesforce rule found.
   4. Select Close.

      The Add Data to Preview Step page closes.
   5. Select Apply.

   The Read from Salesforce dialog box closes. The workflow page displays your new step.

You have completed the process. You have built a Read from Salesforce step in your workflow.
When this step is triggered in a workflow, it imports field data from the Salesforce
records that you mapped.
