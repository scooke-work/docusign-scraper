# Connect Configuration Settings

Docusign Connect sends updates to other applications. Look up the
settings that you use to set up Docusign Connect.

Docusign Connect notifies other applications when a Docusign event you want to know about occurs. You can use Connect to kick off downstream
workflows. This topic describes the settings that are available for custom Connect
configurations. To learn more, see [Create a Connect Configuration](https://support.docusign.com/s/document-item?bundleId=map8c30aa94-7e9f-4950-92fc-dd22c6d9f835&topicId=xwi1583277389681.html&_LANG=enus "Connect notifies other applications when a Docusign event you want to know about occurs. Learn how to create a Connect configuration.").

| Setting | Description |
| --- | --- |
| Name | A name for the Connect configuration. |
| URL to Publish | The web address of your Connect listener. The URL must use the HTTPS protocol and SSL/TLS connections. It may include query parameters. Connect uses port 443 for HTTPS by default. You can specify port 1443, 2443, 3443, 4443, 5443, 6443, 7443, 8443, or 9443 in the URL instead. Example: https://listener.example.com:1443/listener.aspx The SSL/TLS certificate for your web server must chain to a certificate authority on Microsoft's trust list. Self-signed certificates do not work. You may use free certificates from Let’s Encrypt. |
| Enable Log (Account-Level Connect only) | This option enables logging, which can help you troubleshoot any problems. This setting is on by default. It is automatically enabled for Organization-Level Connect configurations. |
| Require Acknowledgment (Account-Level Connect only) | This option is selected by default. Requires Connect to wait for an acknowledgement from your listener after sending a message. Connect records a successful message transfer when your listener returns an HTTP 200 status code. This functionality is automatically enabled for Organization-Level Connect configurations. |
| Data Format | The Data Format you want to receive. This setting is available only for custom Connect. Important: You cannot change this setting after you save the configuration. The options are:  - REST v2.1: JSON (default) - Legacy: XML |
| Event Message Delivery Mode | If the configuration is for a third-party integration or partner, verify the format to use. Important: You can't change this setting after you save the configuration.  The options are:   - Aggregate: Groups event messages for periodic sending. The   information about the event is from when the system sent the   message. - Send Individual Messages (SIM) (default): Sends individual   messages for each event. The information about the event is   from when the event occurred. This option is only available   for JSON configurations. |
| Trigger Events | The events you want to follow. |
| Include Data | The data you want Connect messages to include. The available options depend on the Data Format you chose. |
| Associated Users/Groups | The users or groups of users you want to follow or exclude.  - All Users (includes new users):   Connect sends updates for all account users.  - Select Users/Groups to include: Connect sends updates for the   account users or groups that you select. - Select Users/Groups to exclude:   Connect does not send updates for the account users or   groups you select. |
| Include HMAC Signature | Generates a Hash-based Message Authentication Code (HMAC) and includes it in the message header. |
| Include OAuth | Enables OAuth for Connect messages to your web server. Enable OAuth for the account first. |
| Include Basic Authentication Header | Enables Basic Authentication for Connect messages to your web server. |
| Enable Mutual TLS | Adds Mutual Transport Layer Security (TLS) authentication and authorization to the configuration. Mutual TLS only initiates when your listener sends a valid Certificate Request message during the TLS handshake protocol. Docusign cannot initiate Mutual TLS because it is the client. When you use it with OAuth, the access token request to your authorization server includes the X.509 certificate. |
