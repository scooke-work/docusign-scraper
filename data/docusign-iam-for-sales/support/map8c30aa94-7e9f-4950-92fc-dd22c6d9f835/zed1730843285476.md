# Agreement Set Management

Read about agreement set management including how to use sets to control access to
agreements. This topic answers the question: What are agreement sets and how can I use
them?

 

## Definition and overview

Agreement sets are named groups of agreements that can be shared with users and groups. These
sets are task- and action-oriented. Contract managers use agreement sets to organize
agreements for productivity and data privacy, including managing access at
scale.

Note: Contract managers, or any users that are assigned the
Organize and manage all agreements permission, have
access to agreement set management. See the [User and Group Management](https://support.docusign.com/s/document-item?bundleId=map8c30aa94-7e9f-4950-92fc-dd22c6d9f835&topicId=eut1712157553977.html&_LANG=enus "Read about user and group management tasks, such as managing agreement access and granting bulk upload permissions.")
topic to learn more about Agreement Manager permissions and using agreement sets for access
control.

Consider this use case:

- Contract managers oversee thousands of sales agreements and need an efficient way to grant
  access to a subset of these agreements. For example:
  - The Renewals team needs access to all master service agreements (MSAs)
    expiring in a quarter without auto-renewal.
  - The U.S. Sales Team needs access to all MSAs under U.S.
    jurisdiction.
- With agreement sets, contract managers have a way to grant access to both teams to the
  appropriate agreements.

Important: There is a 100 set limit per account. You
can create up to 100 agreement sets per account. If you have multiple accounts, each
account can have up to 100 sets.

## Smart filters

You can set up smart filters that add agreements to your sets automatically. This feature
streamlines the agreement management process by updating sets based on agreement
attribute values.

You can apply Smart filters to sets during set creation:

![A snippet of the New agreement set view that shows the Smart filters step used to add and remove agreements based on attribute values.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Agreement-set-management_New-agreement-set_step-2-Smart-filters-snippet.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MDIsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.cfDkX0CWXwDZe_EccH9OodkVLGLWC7KH8h0wemHg45c&_LANG=enus "A snippet of the New agreement set view that shows the Smart filters step used to add and remove agreements based on attribute values.")

After applying the smart filters and saving the set, a recurring rule dynamically adds and
removes agreements based on the fields specified in your filters. Agreements are
selected if they match conditions across filters. Consider these examples:

- You applied the Parties and Agreement
  Type filters to the set. The agreements must match for the
  selected party *AND* the selected agreement type to be added to the
  set.
- You applied the Agreement Type filter to the set and
  selected multiple values. The agreements that match *ANY* of the selected
  values are added to the set.

  Tip: Some filters, such as
  Parties and Agreement
  Type, support multiple values. When multiple values are
  selected, the agreements must match one of the values to be added to the
  set.

Important: The smart filters feature does *not* support
these types of filters:

- **Rolling date filters:** the filter value is based on a relative time
  period that updates as time passes, such as "last 7 days".
- **Duration-based filters:** the filter value is based on a length of
  time, such as: "within 30 days".

You can update the smart filters for an agreement set at any time. Expand the
 menu for the set that you want to
update and select Edit smart filters:

![A snippet of the All agreement sets view that shows the Edit smart filters option in the menu.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Agreement-set-management_Actions_Edit-smart-filters-highlighted.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MDIsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.cfDkX0CWXwDZe_EccH9OodkVLGLWC7KH8h0wemHg45c&_LANG=enus "A snippet of the All agreement sets view that shows the Edit smart filters option in the menu.")

## Agreement set usage by role

Agreement set management is primarily used by contract managers, not administrators. Consider
this example:

![A diagram that illustrates how administrators, contract managers, and users or groups use agreement set management.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Agreement-Set-Management_Contract-mgradmin-diagram.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU4MDIsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.cfDkX0CWXwDZe_EccH9OodkVLGLWC7KH8h0wemHg45c&_LANG=enus "A diagram that illustrates how administrators, contract managers, and users or groups use agreement set management.")

- *Administrators* manage users and groups in their organization, including granting
  access to software and features in the Admin portal.
- *Contract managers* create and manage sets to make sure the right people have access to
  relevant agreements. They also add agreements into sets and share sets with the
  appropriate users and groups.
- *Users and groups* view agreements in a set to focus on agreements that need their
  immediate attention.

Note: The separation between 'user and group management' and 'agreement set
management' is intentional to address access control concerns. For example, IT
administrators should not be exposed to sensitive agreement data. Contract managers
use agreement sets to ensure that only the appropriate users have access to
sensitive agreement data.
