# Learn About Searching for Agreements with Agreement Manager for Salesforce

Learn to search for agreements in Agreement Manager for Salesforce.

Important: This feature is only available to IAM for Sales customers. Contact [Docusign support](https://support.docusign.com/s/?language=en_US&_gl=1*74elk4*_gcl_au*MTY5ODQyMjA2MC4xNzY5NjM1OTA1LjE5ODM3MTQzNjguMTc2OTYzNjIwNC4xNzY5NjM2MjA0) or [Docusign sales](https://www.docusign.com/contact-sales) for more information.

The Docusign Completed Agreements component displays Docusign agreements on Salesforce records. You can use Agreement Manager to [search for your agreements in Salesforce](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=qzl1712244340883.html&_LANG=enus).

With Agreement Manager, you have a single place to go to search across all of your completed agreements within your account. The search experience shows contextual help to craft search queries using synonyms, concepts, and everyday language. These search capabilities lead to a more accurate, flexible, and efficient retrieval of information.

## Finding the Agreement Manager search field in Salesforce

Access the Agreement Manager search feature in Salesforce by opening a record that contains the [Docusign Completed Agreements component](https://support.docusign.com/s/document-item?bundleId=pwq1747797355782&topicId=cmv1746652068556.html&_LANG=enus). A Salesforce administrator may have placed the component on a tab labeled Docusign Agreements or something similar. The component features the word Completed at the top.

- The Docusign Completed Agreements component displays agreements sent from the following applications:
  - eSignature for Salesforce
  - CLM for Salesforce
  - Agreement Desk for Salesforce
  - Workflow builder for Salesforce
- The component displays agreements sent both before and after you add the component to your layouts.
- The component displays agreements sent from the source record and its child records. It does not display agreements from parent objects.
- Account records display agreements from the account and all of its child objects.

## Agreement Manager for Salesforce search capabilities

You can use the following capabilities in the Agreement Manager for Salesforce search field:

| **Capability** | **Description** |
| --- | --- |
| **Boolean search** | Users can combine search terms using Boolean operators to refine their search results. Agreement Manager currently supports the *AND* operator. |
| **Fuzzy search** | The search results contain agreements that closely match a user's query, even if it's not an exact match. Users can find relevant agreements based on approximate search terms. |
| **Keyword search** | When users enter a keyword or phrase, the system searches the full agreement content to find exact keywords. |
| **Natural language (NL) search queries** | Users can enter a natural language question into the search field and receive both search results and direct answers. See the [Natural Language Search Queries](https://support.docusign.com/s/document-item?bundleId=map8c30aa94-7e9f-4950-92fc-dd22c6d9f835&topicId=fas1739987568042.html&_LANG=enus "Read about how natural language search queries, including components of the Search Results view.") topic to learn more about NL queries. |
| **Natural language search suggestions** | When users enter a keyword or phrase, personalized suggestions are displayed. These suggestions are designed to enhance discoverability and help users find relevant information more easily. |
| **Recent searches** | The search remembers and displays recent searches for easy future reference. Users can also clear recent searches. |
| **Related concepts search** | The search automatically includes synonyms and related concepts in the search results. Users can capture all relevant documents even if the exact search term is not used. |
| **Semantic search** | Semantic search improves the accuracy of search results by understanding:  - the intent of the search query - the contextual meaning of search terms |
| **Shareable searches** | A shareable link is generated when a search is performed, including applying one or more filters. Receivers of the link have access to agreements in the search results list depending on their permissions. |
| **Spelling assist** | The search automatically corrects spelling errors in queries. Users can find relevant information even if mistakes exist. |
| **Text normalization** | The search finds the most relevant agreements or clauses regardless of the specific word form used in the query. |
| **Type-ahead suggestions** | The search bar provides type-ahead suggestions as the user enters a search query. |
| **Wildcard search** | Users can use wildcard characters in their search to find agreements that contain variations of a word or phrase. Agreement Manager currently supports the use of asterisks and quotation marks. |
