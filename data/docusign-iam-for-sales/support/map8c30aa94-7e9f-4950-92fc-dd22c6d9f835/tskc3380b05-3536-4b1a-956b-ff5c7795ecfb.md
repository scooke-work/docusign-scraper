# Add a List of Participants to a Salesforce Workflow Step

Learn to add a list of participants to a Salesforce workflow step.

## Before you begin

- Required: Installation of the Salesforce app from Docusign App Center. Review the [Installing Extension Applications for
  Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus)
  topic for more information.
- Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account
  administrator.
- Required: Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add steps like
  [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus), [web forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and more to gather recipients,
  documents, and merge fields for your workflow.

You can dynamically map recipients from a Read from Salesforce
step to a Send Documents for Signature step. Docusign can import a list of contacts related to a Salesforce
record and add them as envelope recipients. Senders are not required to enter
recipient names and contact information manually.

You can use the following related list objects to add agreement recipients:

| Parent Object | Related List (Child) | Common use Case |
| --- | --- | --- |
| Account | Contacts | Map multiple contacts as agreement signers |
| Opportunity | Opportunity Contact Role >  Contact | Map contact roles (for example, buyer, approver) as participants |
| Opportunity | Opportunity Line Items | Read multiple product lines |

The following logic applies to creating Salesforce participant lists:

- The Read from Salesforce step adds up to five related records per parent object.
  If more than five records exist, Salesforce logic decides which five records to
  add.
- You can only map participant lists in the Read from Salesforce step. You cannot
  map participant lists in Writeback to Salesforce or other Salesforce steps.
- You can map up to 10 participant fields per related object, like Full Name,
  Email, and so on.
- When you select a related list object, Docusign reads fields across multiple
  objects, like Opportunity >  Opportunity Contact Role >  Contact.

Example: You want to create a workflow that
automatically sends an NDA agreement to multiple Salesforce contacts. You add a Read
from Salesforce step to your workflow. You create a Salesforce Contacts participant
list by mapping the following Salesforce and Docusign fields:

| Salesforce field | Docusign field |
| --- | --- |
| Account / Contacts / Department | participantRoleName |
| Account / Contacts / Full Name | participantName |
| Account / Contacts / Email | participantEmail |
| Account / Contacts / Business Phone | participantPhone |

You add a Send Documents for Signature step to your workflow.
You choose the Add dynamic list of participants option and
add the Salesforce Contacts list. You assign a recipient role of
`Signer` to the list.

A user starts a workflow from a Salesforce account with five contacts.

Result: Docusign creates an agreement for all
five contacts, with signature fields for each contact's department, full name,
email, and business phone. Docusign sends the NDA to all contacts
automatically, with no manual entry needed.

Use this procedure to add a list of participants to a workflow:

1. [Create a basic workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus) containing [Read from Salesforce](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=idr1713985144297.html&_LANG=enus) and [Send Documents for Signature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) steps.
2. Configure your Read from Salesforce step:
   1. Open the step to the Which fields would you like to
      read? section.
   2. Select Add or Remove Fields.

      The Add or Remove Fields page displays.
   3. Select the Related Lists tab on the left.
   4. Select the check box next to each Salesforce related list field that
      you'd like to use.
   5. Select Save and Exit.

      ![A screenshot displays the Add or remove fields section with Salesforce field check boxes highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Read_from_Salesforce_Related_Lists_Contacts_fields_selected.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0ODYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.GcaTz-r4hWbbf5m67zItKZXorfbnz38pcCeRVpbImPw&_LANG=enus)

      The Add or Remove Fields dialog box closes.
      The related list object that you selected displays.
   6. Select the Map button next to the related
      list.

      ![A screenshot displays the Which fields would you like to read section with a Map button highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Read_from_Salesforce_Related_Lists_Map_Contacts_button.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0ODYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.GcaTz-r4hWbbf5m67zItKZXorfbnz38pcCeRVpbImPw&_LANG=enus)

      The Map to Docusign Object dialog box
      displays.
3. Map your Salesforce and Docusign objects and fields:
   1. Docusign object: Select a Docusign object from the dropdown menu.

      The Map Salesforce fields to your Participant
      object section expands.
   2. Salesforce fields: Select a Salesforce field to
      match with each Docusign field.
   3. Repeat this process for each displayed field.
   4. Select Save Mapping.

      ![A screenshot displays the Map to Docusign object dialog box with a Docusign object and fields highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Read_from_Salesforce_Map_to_Docusign_Object_dialog_box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0ODYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.GcaTz-r4hWbbf5m67zItKZXorfbnz38pcCeRVpbImPw&_LANG=enus)

      The Map to Docusign Object dialog box closes.
   5. Select Next.
   6. Complete the rest of your Read from Salesforce step and select
      Apply.
4. Configure your Send Documents for Signature step:
   1. Open the step to the Confirm recipient details
      section.
   2. Match the template's [name] recipient to one of this
      workflow's participants: Select Add
      Participant from the dropdown menu.
   3. Select Add participant.

      ![A screenshot displays the Confirm recipient details section with the Select and Add Participant fields highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Read_from_Salesforce_Related_Lists_Add_Participant.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0ODYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.GcaTz-r4hWbbf5m67zItKZXorfbnz38pcCeRVpbImPw&_LANG=enus)

      The Add Participant dialog box displays.

      Note: A participant is a placeholder for someone who
      will take action at specific steps in your workflow. Docusign uses the participant information to
      assign that role to a person when the workflow
      runs.
   4. Add a dynamic list of participants: Select the
      switch icon to turn it on.

      The switch moves to the right.
   5. Dynamic list: Choose the dynamic recipients list
      from your Read from Salesforce step.

      ![A screenshot displays the Add Participant dialog box with a Salesforce Dynamic Recipients list highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Read_from_Salesforce_Dynamic_Recipients_Add_Participant_dialog_box.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0ODYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.GcaTz-r4hWbbf5m67zItKZXorfbnz38pcCeRVpbImPw&_LANG=enus)
   6. Role: Enter a role name for your recipients
      list, like `Approver`.

      Tip:
      - Use unique role names: Make sure the role names that you’re bringing in
        from Salesforce are unique. If there are multiple roles with
        the same name in your incoming list, Docusign only imports the first one.
      - Use the same role name in your source system and
        eSignature template: Make sure the role names
        from Salesforce match the recipient roles in your eSignature
        template. If they don’t, the workflow will likely fail.
   7. Select Confirm Role.

      ![A screenshot displays the Add Participant dialog box with a Salesforce Dynamic Recipients list and a Role highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/map8c30aa94-7e9f-4950-92fc-dd22c6d9f835/page/images/Send_Documents_for_Signature_step_add_dynamic_Read_from_Salesforce_list.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODIxNTU0ODYsInNoZWFmIjoibWFwOGMzMGFhOTQtN2U5Zi00OTUwLTkyZmMtZGQyMmM2ZDlmODM1In0.GcaTz-r4hWbbf5m67zItKZXorfbnz38pcCeRVpbImPw&_LANG=enus)

      The Additional guidance dialog box displays. Select
      OK to close the dialog box.
   8. Select Add.

      The Add Participant dialog box closes.
   9. Select Next.
5. Complete the rest of your Send Documents for Signature step and select
   Apply.

You have completed the process. You have added a list of participants to your workflow. When
the workflow starts, Docusign creates an agreement and adds all mapped
contacts and fields.
