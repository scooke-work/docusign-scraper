# Check the Status of a Flow That Failed to Start a Docusign Workflow

Learn to check the status of a flow that failed to start a workflow.

Salesforce administrators can build a Start Docusign Workflow Builder
workflow step to link Salesforce flows with workflows. When a flow starts, Salesforce starts the
linked workflow. Linking flows with workflows builds an automation chain that
creates agreements from Salesforce and Docusign data.

If you want to view the status of a workflow, you can [check the workflow process in your Docusign account](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=vcp1698707326210.html&_LANG=enus).

If a Flow fails to start your workflow, it does not display in your Docusign account. You can only view the logs in Salesforce.

Use this procedure to check the status of a flow that failed to start a workflow:

1. Select the Docusign Logs tab.

   The Docusign Logs page opens.
2. Select a log from the Docusign Log column.

   The log details page opens for that item.

You have completed the task. You viewed the details of a flow that
failed to start a workflow.
