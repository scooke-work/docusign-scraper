# Configure Dynamic Recipients

Read about how to configure dynamic recipients in a Web Form, making certain roles optional to streamline the envelope generation process.

When importing a template with a Role field that is not always needed, you can make it optional so it is included only when necessary.

For example, if your template includes "Applicant" and "Co-Applicant" roles, but the co-applicant is not always required to sign, you can configure the template to include that role only when needed.

The primary recipient completes the form and enters their own and the additional recipient's name and email, which are used when the envelope is generated.

In this example, the Co-Applicant is optional. The primary recipient enters their name and email, leaves the Co-Applicant fields blank, and those fields are excluded when the envelope is generated.

To make a recipient optional:

- Select the recipient field (for example, the Co-Applicant), and clear the Required option.
- Activate the form. For more information, see the [Activate a Web Form](https://support.docusign.com/s/document-item?bundleId=gmi1660583110357&topicId=iri1669070601520.html&_LANG=enus) and [Reactivate a Web Form](https://support.docusign.com/s/document-item?bundleId=gmi1660583110357&topicId=crq1669076320407.html&_LANG=enus) guides.
- The recipient field displays on the form without the Required star.
- The primary recipient enters their name and email in the Primary Recipient field and leaves the Additional Recipient field blank. The additional recipient role is excluded when the envelope is generated.

You can hide the co-applicant fields with conditional rules, such as hiding them when a primary recipient is present, so they are excluded when generating the envelope. For more information about adding rules, see the [Add Rules to a Web Form](https://support.docusign.com/s/document-item?bundleId=gmi1660583110357&topicId=kff1668121038223.html&_LANG=enus) guide.
