# Add Web Form Permissions for Users

Read about how administrators can assign and manage Web Forms permissions for non-administrators using permission profiles or support tickets.

Administrators automatically have Web Forms access. Non-administrators need assigned permissions. If the Admin tab is missing in Web Forms, an administrator must assign your permissions profile. For information about template and form permissions, see the [Permission Profile Options](https://support.docusign.com/s/document-item?bundleId=pik1583277475390&topicId=mfo1583277360424.html&_LANG=enus) guide.

Administrators and form creators can share form access with non-administrators, who can view, edit, or activate a shared form based on their assigned access level.

## Granting Template Permissions

Administrators must first grant users template permissions. This establishes the baseline permission level for that user. If your account allows separate form permissions, administrators must also grant a form permission. To manage web-form-only permissions separately from template permissions, you can [submit a support ticket](https://support.docusign.com/s/articles/How-Do-I-Open-a-Case-in-the-DocuSign-Support-Center).

## Granting Web Forms Permissions

If necessary, administrators must grant user-specific form permissions. This level can match or be more restrictive than the template permission, but not more permissive.

## Permission Level Definitions

The specific meanings of the different permission levels (none, use, create, share) are the same for both template permissions and form permissions.

- **None:** Users can't access any forms or templates.
- **Use:** Users can access shared forms and templates, but can't edit them.
- **Create** or **Share:** Users can create and edit forms or templates.

Administrators can assign permissions through two main methods:

1. **Permission profiles:** This is the recommended method. Permission profiles allow you to manage permissions for groups of users efficiently.
2. **Support tickets:** This method is used when you must manage form permissions separately from template permissions. This is less common but can be useful in specific situations.

   Note: Forms permissions can't exceed templates permissions. For example, you can't add a form **Create** permission to a user who has the template **Use** permission. If the form selection you want is not activated, check that the template permission level is sufficient.
