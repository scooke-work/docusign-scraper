# Run a Web Form Data Report

Learn how to use the Web Forms data report in Docusign to track field-level form responses, filter results by date, status, and other fields, and generate visual summaries that highlight key trends.

The Web Form data report is a Document Data report in Docusign that provides field-level

For example, a Real Estate agency uses a form to collect loan applications. Analysts use the Web Forms data report to track completed applications, review application volume over the last year, and identify trends. These insights help the Marketing team focus promotions and ad spending where they are most effective.

1. Go to the Insights page.
2. Select Reports > Web Forms.
3. Use the dropdown to select a form to analyze from your active forms.
4. Select a date range from the Date Range field.

   The available date ranges are:
   - Last 30 days: Previous 30 calendar days.
   - This Month: From the first day of the current month to 24 hours ago (default).
   - Last Month: First to last day of the previous calendar month.
   - Last 90 days: Previous 90 calendar days.
   - Last 3 Months: First to last day of the previous three calendar months.
   - Last 6 Months: First to last day of the previous six calendar months.
   - This Year: January 1 to 24 hours ago in the current year.
   - Last Year: January 1 to December 31 of the previous year.
   - Last 12 Months: First to last day of the previous 12 calendar months.
   - Custom: Select a custom date ranging from April 1, 2023 to 24 hours ago.
5. Select Run report.

   The report displays all form fields as columns, plus these columns:

   - Created On: The date the form response was created. A "response" occurs when users complete the form and create an envelope. They can then complete the envelope, setting the status to Completed, or abandon it, leaving the status as Sent.
   - Envelope Id: The envelope ID.
   - Status: The envelope status: Sent or Completed.
   - Completed On: The date when the form response envelope was signed and completed.
6. By default, all report columns display. To hide or rearrange columns, select Edit Columns, adjust the settings in the Edit Columns window, and select Done.
7. Apply filters to refine report data by setting conditions for specific form fields.
   1. Select Edit Filters > Add a condition, and choose the field, operator, and value for each condition. Available conditions depend on the selected field type.
   2. Select Save >  Run report.

   The report displays a list of the responses for your form that meet all the
   conditions you specified.
8. Analyze your data further by summarizing the data for each field type. You can
   display the data as a visual chart or a data table.
   1. Select the Summarize Results By dropdown and choose a form field to summarize.
   2. Select Run report.

      Your data summary displays as a bar or pie chart with a data table, depending on the field type. For bar charts, you can switch to a table view by selecting More Actions (...).
   3. For bar charts, you can switch to a table view by selecting More Actions (...).
9. Download the report data as a CSV file. After you run the report, select Download a report (down arrow) icon.

   Your data downloads as a CSV file to your local machine.
10. To reuse a report, select Save Report, enter a title and description, and select Save.

    Your custom report is saved and is displayed in the Reports list under Custom.

You have configured and run a Web Forms data report, applied filters, summarized results, downloaded the data, and optionally saved the report for future use.
