# Prepare a Template to Import to a Web Form

Read about how to prepare an eSignature template so it meets all requirements before importing it into the Web Form Builder.

Before you create a form from a template, prepare the template, then copy and import it into the Web Form Builder, where it becomes a form template.

Note: The form template is a copy of the eSignature template that you imported into the Web Form Builder. In your eSignature instance, the Template tab shows only the eSignature template, while the Web Form Builder shows only the form template.

You can access the form template in the Web Form Builder anytime to review or make changes to a form. To view the template, go to the Signature page of the builder and select the Template Edit (pencil) icon.

To edit template settings and fields, select More Actions (...) at the top of the form and select Edit Template Settings and Edit Template Fields.

Each template field maps one-to-one to a form field, and you cannot change a field’s type after import. For example, you cannot import a text field from a template and then change it to a dropdown field in the form.

Before you import a template to the Web Form Builder, make sure that the template meets these requirements:

- The template must include at least one document.
- The primary recipient must have at least one text, checkbox, dropdown, or radio field. Otherwise, nothing is imported from the template to the form.

  Note: Other eSignature field types (such as drawing, formula, and attachments) still work in the eSignature template but are not imported into the form.
- Each recipient must have a unique role name. For example, do not assign the role name "Applicant" to two recipients. Instead, use "Applicant" and "Co-Applicant".
- The first recipient in the signing order must have the signing action Needs to Sign.
  - If you have only one recipient, that recipient must have the Needs to Sign signing action.
  - If you have multiple recipients, the first recipient must have the Needs to Sign signing action.

- Data labels must be non-blank and descriptive. A data label is a field identifier visible only to the sender. If a label is blank or has a non-descriptive identifier, update it to a more descriptive name. For more information about labels, see the [Create a Data Label on a Template](https://support.docusign.com/s/document-item?bundleId=gmi1660583110357&topicId=yis1665092718418.html&_LANG=enus "Read about how to create clear, descriptive data labels on template fields so senders can easily identify each form field.") guide.
- The template cannot include I, Witness, or Notary recipients.
- The template cannot include Salesforce, Healthcare, or other merged fields.
- Recipients must use email as the delivery option.
