# Sending a Web Form

Learn how to send Web Forms directly from the Docusign web app to specific recipients, streamlining processes with sender-initiated forms and customizable sender controls.

You can start a web form directly from the Docusign web app and send it to specific recipients by email. This shift from public self-service URLs to sender-initiated forms streamlines processes that require a sender to prefill information or target specific recipients.

## Sender-initiated web forms

Unlike public URL web forms, where the envelope is created only after the recipient submits the form,
sender-initiated web forms generate a Docusign envelope immediately once the sender
selects Send.

The workflow for a sender-initiated form is:

1. **Initiation:** A sender logged into Docusign selects an active web form and chooses the Send option.
2. **Pre-fill:** The sender enters the recipient's information (name and email) and can optionally
   complete specific fields designated for the sender.
3. **Delivery:** Recipients receive a standard Docusign envelope email.
4. **Form completion:** When recipients select the email link, they go to the web form to review the sender’s information, complete any remaining fields, and then proceed to the standard signing ceremony.

## Sender views and controls

Process builders can customize the sender's experience through specific configurations:

- **Sender view:** A Data Pre-fill Page in the Web Forms Builder collects sender-only data for internal use (such as envelope custom fields) and is hidden from recipients.
- **Sender controls:** Process builders use sender controls to decide whether senders can pre-fill each field. Fields can be set to Hide (default), Display (optional), or Required for the sender. Sender controls are available in the Properties tab for each field in the Web Forms Builder.

## Requirements and limitations

- **Licensing:** Basic sending with the Docusign web app is available for standard Web Forms.
  Advanced capabilities, such as multi-recipient support or sending with
  the API, require the Advanced Web Forms add-on for eSignature plans but
  are included in IAM plans.
- **Unsupported fields:** Web forms with Attachment fields do not currently support sending.
