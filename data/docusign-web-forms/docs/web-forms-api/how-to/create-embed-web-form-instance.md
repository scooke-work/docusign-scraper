# How to create and embed a web form instance

The Web Forms API is available in all developer accounts, but only in certain production account plans. Contact [Docusign Support](https://support.docusign.com/s/?language=en_US) or your account manager to find out whether the Web Forms API is available for your production account plan.

This topic demonstrates how to create a web form instance and embed it in your app’s HTML using the Docusign JS library. After the signer fills out the form, they will be directed to the embedded signing experience, where they will sign the document.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Prerequisites

Web form configurations cannot be created through the API. Therefore, before you create a web form instance, you must [create](https://support.docusign.com/s/document-item?bundleId=gmi1660583110357&topicId=euk1665093155272.html) and [activate](https://support.docusign.com/s/document-item?bundleId=gmi1660583110357&topicId=iri1669070601520.html) a web form configuration in your Docusign account. If you are running this from our Quickstart or launcher projects, you will receive instructions for how to create a web form configuration by uploading a compatible JSON configuration file.

If you choose not to use our launcher projects, you can follow along with this guide by creating a web form configuration with the name “Web Form Example Template” and adding the following fields:

- A text field with the API reference name “PhoneNumber”
- A text field with the API reference name “Company”
- A text field with the API reference name “JobTitle”
- A checkbox with the API reference name “Yes”

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at   https://<domain>/webforms/v1.1  Where <domain> is apps-d.docusign.com/api for the developer environment. |
| {CLIENT\_USER\_ID} | A unique identifier for a user that should originate from your own system. This value can be anything your backend system will use to track and retrieve individual web form instances. Examples include employee IDs, email addresses, and surrogate key values. |

## Step 1. Obtain your OAuth Token

To make the API calls shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

Web Forms OAuth access tokens are the same ones used for our [eSignature REST API](https://developers.docusign.com/docs/esign-rest-api/esign101/auth/), although you must request one or more Web Forms scopes to call the Web Forms API. The API calls made in this how-to require that you request the eSignature REST API `signature` scope and Docusign Web Forms–specific scopes `webforms_instance_read`, `webforms_instance_write`, and `webforms_read` when obtaining your access token.

See [Authentication](https://developers.docusign.com/docs/web-forms-api/plan-integration/authentication/) for guidance on your options to get an OAuth token.

First, get the ID of the web form configuration of which you will be creating an instance. Call the [Configurations:listForms](https://developers.docusign.com/docs/web-forms-api/reference/webforms/configurations/listforms/) endpoint, passing the value `Web Form Example Template` to the `search` query parameter. ## Step 4. Construct the request body

Next, create the request body. Set the `clientUserId` to a unique value that will be used to identify the user who will fill out the form.

Use the `formValues` object to prefill fields in the web form instance. The keys in this object correspond with the API reference names of the fields in the web form configuration. You can find and edit these API reference names in the [Properties tab](https://support.docusign.com/s/document-item?bundleId=gmi1660583110357&topicId=wor1676501633875.html) in the web forms UI. The `expirationOffset` parameter determines the amount of time in hours until the instance will expire. The form instance can be submitted and a corresponding envelope can be generated as long as the instance has not yet expired.

## Step 5. Call the Web Forms API

Now that you have constructed your API headers and request body, make a POST request to the [Instances:createInstance](https://developers.docusign.com/docs/web-forms-api/reference/webforms/instances/createinstance/) method and include the request body to create a new instance of the web form.

Here’s how you make the call, but make sure to substitute your API account ID for {ACCOUNT\_ID} and form ID for {FORM\_ID}.

## Step 6. Embed the Web Form in your HTML

The best practice for embedding a web form instance in your app’s HTML is to use the Docusign JS library. The URL that will load the JS library is <https://js-d.docusign.com/bundle.js> for the developer environment and <https://js.docusign.com/bundle.js> for the production environment.

After the JS library has been loaded, construct the `webFormsOptions` object. The values in this object will define how the web form is rendered on the page. Set the value of the `instanceToken` parameter to the instance token returned in the response of the `CreateInstance` API call. The other optional parameters control various elements of the embedded form’s style. Use the `hideProgressBar` parameter to control whether the progress bar appears in the form. The `minHeight` parameter inside the `iFrameStyles` array determines the height of the iframe where the form is embedded.

Call `docusign.webforms`, passing the `webFormsOptions` object and the `formUrl` returned from the previous API call as parameters. Finally, mount that object in the HTML element of your choice. In this example it is mounted in a `<div>` with the `id``docusign`.
