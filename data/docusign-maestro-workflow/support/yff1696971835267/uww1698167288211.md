# Configure a Send Documents for Signature Step

Learn how to add and configure a workflow step to send a document for signature.

## Before you begin

- Required: Workflows Create permissions or be an account administrator.
- Required: A Prepare eSignature Template or one or more Prepare Document Template steps prior to the Send Documents for Signature step in your workflow.
- Recommended: Open the template in another window to refer to as you set up your step. This will help you correctly configure your Send Documents step.
- Review the information in the topic [eSignature Requirements and Other Information](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=qkf1716338686555.html&_LANG=enus "Read about some important requirements to consider when configuring a Send Documents for Signature step.").

You use a Send Documents for Signature step to present a document to one or more participants to view, sign, or receive a copy. This step uses the output from either a Prepare eSignature Template or one or more Prepare Document Template steps. If your workflow contains multiple Prepare Document Template steps, you can send the generated documents together in one envelope.

For the signing session and all notification steps, Workflow Builder applies your account's default signing brand. This helps ensure brand consistency at key customer touchpoints.

1. Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus "Learn the basics of how to create an agreement workflow. Process creators and account administrators can create workflows to automate the agreement process.").
2. Add the steps that you want to precede your Send Documents step. For example, a Set Up Invite step or a Collect Data with Web Forms step.
3. Add either a Prepare eSignature Template step or one or more Prepare Document Template steps. These steps produce the documents that you will send for signature.
4. Select Add a step >  Send Documents for Signature.
5. Select the documents that you want to send.
   1. Click the dropdown for the primary document and make a selection.
   2. (Optional) Select the Add Document dropdown and make a selection. If your workflow has multiple Prepare Document Template steps, you can add more generated documents.
   3. (Optional) Reorder the documents by using the grab handles to click and drag them into the desired order of presentation.

      Note: For additional information and limitations around adding and reordering documents, see [Managing Multiple Documents in Workflow Builder Envelopes: Order and Limits](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=con30310a57-3e53-40f2-a7f0-d9e099874eaa.html&_LANG=enus "Read about how to add, reorder, and combine multiple documents or templates in a Send Documents for Signature step, and learn about the limits for document types, quantity, and recipient mapping.")
   4. (Optional) Add supporting documents that are available in your workflow. Select Add Document > Add Supporting Document and select the document variable.

      These are ancillary files that do not contain any recipient roles or fields for data input. Examples of supporting documents include Salesforce files and Web Forms attachments.
   5. Select Next.

   Recipients for the templates you selected are added to the step
6. Choose how the send the envelope.

   For start methods that have a known initiator, Manually in Docusign and From an API call, you can choose between **Automatic** and **Manual** sending and on behalf of either the workflow publisher or the user who starts the workflow. See [Understand Manual and Automatic Sending and Sender Options](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=con50a84a0e-9d14-4028-98fa-e1b2d8405b48.html&_LANG=enus "Learn about manual and automatic sending options for envelopes in your workflows, including how to choose who appears as the sender.") for more information about these options.

   If your workflow uses the From an Agreement Desk Request start method, you can choose automatic sending on behalf of the Request submitter. See [Sender Impersonation and Consent for Agreement Desk Workflows](https://support.docusign.com/s/document-item?bundleId=map4317ef56-de36-43e2-9921-c76a445c3276&topicId=con85f610e6-2584-423e-840e-8823eb1ed065.html&_LANG=enus).
7. Confirm recipient details.
   1. Review and assign workflow participants to each template recipient role.
   2. Select the notification method (Email or Text message).
   3. Provide Name, Email, and Phone Number (for SMS/WhatsApp) using workflow variables or manual entry.

      Note: If the participant's information was defined in an earlier step, those details are applied here. You can keep the definitions or change them. Changes will apply to all subsequent steps.
8. Add envelope custom fields (ECFs).

   If your account uses these metadata fields for envelope classification, you must select values for any required ECFs. With text-type ECFs, you can use workflow variables to define values. ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/add-ecfs.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU3NzYsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.6YjiO3uQsXWa0JdQLUHmnOyYfo-UmR5dtUZer1wKaLE&_LANG=enus)
9. Select Apply.

Your Send Documents for Signature step is configured. When this step is triggered in a workflow, it creates an envelope in your account, accessible from your Agreements page. If you share access to your envelopes, these workflow-generated envelopes are included. A process preparer (the user who starts the workflow) who uses your workflow must have shared access to view the envelope. For information on sharing access to envelopes, see [this help topic](https://support.docusign.com/s/document-item?bundleId=oeq1643226594604&topicId=obf1692128351600.html&_LANG=enus).
