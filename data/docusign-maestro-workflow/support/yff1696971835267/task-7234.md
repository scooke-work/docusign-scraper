# Configure a Create Request Step

Learn how to add a Create Request step to a workflow to generate automatically or interactively a new Agreement Desk request.

You use a **Create Request** step to initiate an Agreement Desk request directly from your workflow. This step creates a complete, ready-to-work request package by pulling in data and documents generated in earlier workflow steps or passed through API calls.

**Execution Modes:** You can configure this step to run in one of two modes:

- **Automatic:** The system generates the request automatically and unattended, making it ideal for high-volume, zero-touch workflows. Data for the request is mapped directly from previous workflow steps or starting variables.

  Note: To support external participants (such as outside vendors or customers), you can place a Collect Data with Web Form step prior to the Create Request step and map the captured form fields directly into Automatic mode.
- **Interactive:** A designated internal participant manually completes the request form. This mode is designed for CRM integrations. Internal teams can initiate and route requests directly from their existing workflows.

**Example use case (Automated Contract Renewal):** You can set up an automated workflow triggered by an API call. When a contract is 30 days from expiring, your CRM triggers the workflow, passing data into a **Create Request** step running in Automatic mode. The system silently generates a renewal request for your team without any manual intervention.

**Example use case (High-Value Contract Approval):** You can build an internal web form using Workflow Builder to collect contract data from your team. If a submitted contract's value exceeds $50,000, the workflow automatically creates an Agreement Desk request and routes it to the appropriate VP for review and approval.

Important:

This step is **not available** for workflows that use the From an Agreement Desk request start method.

## Before you begin

- Ensure **Agreement Desk** is enabled and active for your organization's Docusign account.
- Create a workflow with a supported start method (such as From a Link, From an Event, From an API Call, or Manually in Docusign).

1. In the workflow designer, select **Add a step**.
2. Select **Create Request** > **Configure**.
3. Select the **request type** for the new request from the list.

   Note: This list displays the custom request types configured in your organization's Agreement Desk account.
4. Select how the form will be filled out: **Automatic** or **Interactive**.
5. Construct the **Request title** using manual text, up to five workflow variables, or a combination of both.
6. Map workflow variables to the request form fields.

   - *Automatic mode:* Mapped values cannot be changed by a user. If required fields from the request type are left unmapped, the workflow will ignore the requirement, leave the fields blank, and still successfully create the request.
   - *Interactive mode:* The assigned participant can manually change any mapped values when completing the form.
7. Configure **Documents** by choosing how to pull documents into the request.

   - Include all workflow documents.
   - Manually select specific documents to include.
   - Do not include any documents.

   *Note:* As part of the mapping experience, you can also map specific workflow documents directly to attachment fields in the request form.
8. Select **Apply**.

Your **Create Request** step is configured.
