# Limits for Branching Rules

This topic provides a detailed explanation of how branching limits work within the rule configuration for the Add a Branching Rule and Add a Multi Branch Rule steps in Workflow Builder. It covers specific limits and how path merging and counting logic affects the ability to add subsequent branch rules.

This topic provides a detailed explanation of how branching limits work for the two types of branching rules for adding simple True/False or complex multi-branching to your workflow. For general how-to information on setting up branching rules, see the topics [Configure a Branching Rule](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=mnv1698169910928.html&_LANG=enus "You can create conditional rules in your agreement workflow. These rules validate a condition and branch the workflow based on whether the condition is met or not.") and [Configure a Multi Branch Rule](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=xtg1749585007265.html&_LANG=enus "Learn how to use the Add a Multi Branch Rule step in Docusign Workflow Builder to create workflows with multiple paths based on specific conditions. This feature allows for dynamic processes that adapt based on data collected from web forms.").

## Branching limits

When you are building a workflow with branching rules, the system enforces a maximum of 8 branch rules per path. A path is an individual outcome of a branching rule, and includes all of the subsequent steps executed after that outcome is met. This limit can include a mix of both multi branch and simple True/False branch rules.

Of these 8 total branch rules, no more than 2 can be configured as Multi Branch Rules. (Allowed combinations are provided in the details that follow.) These limits apply only to the branching rule steps. Other types of steps added to the paths do not count towards the branching rules limits.

Maximum limit
:   You can add up to 8 branch rules on a single path.

Multi branch rule limit
:   Within the 8 total branch rules, you can configure a maximum of 2 Multi Branch rules.

Counting examples
:   - For an existing True/False branch rule, you can add up to 7 more branches on both the true and false paths; no more than 2 of these 7 branches can be Multi Branch rules.
    - For an existing Multi Branch rule, you can add up to 7 more branches to each path (rule outcome); only 1 of the 7 branches can be a Multi Branch rule.

Allowed combinations
:   - 2 Multi Branch Rules + 6 True/False Branches
    - 1 Multi Branch Rule + 7 True/False Branches
    - 0 Multi Branch Rules + 8 True/False Branches

Enforcement
:   For each single path in your workflow, the count is cumulative from the first branching rule on through to the end of the workflow.

    - If the path contains 8 branches, no additional branch rules can be added.
    - If the path contains 2 Multi Branch Rules, no additional Multi Branch rules can be added.

## Path counting and merges

When you add a step within an existing flow, the system calculates the branch limit based on the longest possible sequence of branches leading to that point.

- Longest Path Measure: The limit is always measured against the longest possible path within the workflow structure.
- Impact of Merges: After two or more branches merge back into a single path, the logic must still account for all possible originating paths. This can restrict what subsequent steps or branches can be added post-merge. This ensures that every path leading to the merge point honors the branching rule limit.
- Pre-Branch Counting: The same counting logic applies to the structure preceding a new branch, ensuring the total path length remains under the 8-branch limit. In other words, the count begins at the first branch in a workflow and continues through to the end.

## Multi branch rule specifics

For a Multi Branch Rule, each of its possible outcomes is treated as an independent path for the purpose of the branch count.

- Independent Paths: Every path within a Multi Branch Rule—for all specific conditions met and the final "no conditions met" path—is considered separately when checking the branch count limit.
- Further Branching: An additional Multi Branch Rule (or other branch type) can be added on each of the specific paths generated by the current Multi Branch Rule, including the "no conditions met" default path. Maximums for total branch rules and multi branch rules apply.

## Examples

Branching rules are complex. This section presents some examples that illustrate where branches can be added in a workflow and how the system’s path counting logic operates. The ability to add a new branch rule is determined by the path's current branch count and the overall structural complexity of the orchestration workflow at that specific point.

**Example: Workflow with the maximum allowed 8 branching rules.**

In this workflow, there is one maximized path with 8 simple True/False rules. The count from 1 to 8 is shown on each rule to show the counting logic, so “True / False count = 1” is the first rule, “True / False count = 2” is the second rule, and so on.

Adding branching rules to this workflow is limited as follows:

- No additional rules of either type can be added along the original True path in this workflow.
- No additional rules of either type can be added to the last branch where the count has reached the maximum allowed. (Labeled “True / False count = 8”)
- At the open paths (the “+” signs), rules of either type can be added up to the maximums allowed. For example, for the original False path, you could add 7 more True/False rules or a combination of up to 2 Multi Branch and up to 5 True/False rules.

**Example: Linear workflow - no merging**

In this workflow, the initial Multi Branch rule does not merge and a variety of nested rules are in use. All configured paths have the maximum number of rules (8), meaning no more can be added at any point on those paths. The count from 1 to 8 is shown on each rule to show the counting logic, so "True / False count = 1" is the first rule, "True / False count = 2" is the second rule, and so on.

Adding branching rules to this workflow is limited as follows:

- No additional rules of either type can be added along the 3 paths that have the maximum number of branches.
- At the open paths (the “+” signs), rules of either type can be added up to the maximums allowed. For example, on the right-most path that originates with a Multi Branch Rule, depending on where you are in the rule chain, you could add up to 7 more True/False rules or a combination of up to 1 Multi Branch and up to 6 True/False rules.
