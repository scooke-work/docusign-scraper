# Web Form Requirements and Data-Mapping Information

Review the configuration rules, standalone formatting options, and precise field-matching requirements needed to map data between web forms and agreement templates.

Using a web form in a workflow carries certain requirements for the type of web form used. There are also details that the process builder should keep in mind when mapping data between a web form and a template.

## Stand-alone web forms

You can use any existing stand-alone web form on your account that you have access to. A stand-alone web form is not tied to a template but exists on its own to use to gather data for an agreement process.

There are two ways to create stand-alone web forms.

Create a blank form
:   Create a new web form that is not associated with a template. On the Templates page, create a new web form. Use the Create Blank Form instead of selecting a template to use.
:   Set up your web form pages and add the questions to show to participants. Consider customizing the API reference name for each question. Creating a meaningful name can help process creators correctly identify and use the web form data fields in workflow steps.

Create a workflow version from an existing web form
:   Copy an existing web form to create a stand-alone version. On the Templates page, locate the web form you want to copy. ## Mapping data between web forms and templates

In a workflow, you can use web forms to gather data from participants. As an option, you can map data from other sources to prepopulate web form fields. This data can be strings for values like text, dates, email addresses, and phone numbers, and also numerical values.

**Currency**

When a web form uses a currency field type, the submitted value becomes a currency variable in Workflow Builder. You can map currency variables to Number (Currency) fields in templates and you can use them in branching rules.

Web form currency fields can also receive currency data from other sources, such as starting variables in a workflow that uses an API start method.

When mapping a currency field, there are three versions of the field:

- **Currency field name**: contains the currency code and numeric amount
- **Currency field name (Double Value)**: contains the numeric amount
- **Currency field name (Currency Code)**: contains the currency code

**Checkboxes, radio buttons, and dropdowns**

You can also use checkboxes, radio buttons, and dropdowns in your web form. For these field types, to map data between a web form and an eSignature template, there are some specific requirements. To ensure values for these field types populate correctly, the following value names must match exactly.

|  |  |  |
| --- | --- | --- |
| **Field type** | **eSignature template** | **Web form** |
| Checkbox | Checkbox Values | API value for each checkbox |
| Radio Button | Radio Button Values | API value |
| Dropdown | Values | API value |

## Web forms attachments

Workflow Builder supports stand-alone web forms with attachment fields. Participants can use attachment fields to upload files through the web form. You can map these attachment fields in a Send Documents for Signature as **supporting documents**. Supporting documents are ancillary files with no recipient roles or fields.

Some important points to note:

- Supporting documents cannot be used as the primary document for signature.
- Web form attachment fields can be mapped only to a Send Documents for Signature step; you cannot map it to an attachment field on an eSignature template.

Read about how to create a web form with an attachment field in [Build a Web Form with Attachment Fields](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=ymm1721679198157.html&_LANG=enus "Read about how to build or update a Web Form with attachment fields or import them from a template that includes attachments.").
