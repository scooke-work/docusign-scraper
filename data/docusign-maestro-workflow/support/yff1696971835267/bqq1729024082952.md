# Update an Existing Workflow to Use a Start Method

Learn how to update an older, published workflow by defining a required start method before saving and publishing new edits.

Older published workflows that do not have a start method defined are not impacted by
the new workflow start method. Any running instances for these workflows will
continue, uninterrupted. However, to update an existing workflow definition (or
when creating a new workflow), process builders will need to pick a workflow start
method. You cannot publish edits to an existing workflow until you add a workflow
start method.

You can read more about the start methods in this help topic [Workflow Start Methods](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=ztb1727892686033.html&_LANG=enus "Review the different automated and manual trigger methods available to initiate processes in Workflow Builder, including links, events, API calls, and CLM integrations.").

Depending on which method you want to use, and whether your workflow has starting
variables, the updates required for your workflow will differ. Some workflow start
methods no longer use starting variables

Review the following table for details on the Link, API Call, and CLM start methods.
The Manually in Docusign requires the more significant updates. Those details are
provided in the procedure following the table.

Tip: You can view your workflow's starting variables
by selecting either the From CLM or From an API
Call start method.

To use the Manually in Docusign  start method for a workflow
that has starting variables designed to collect information, you'll need to make several updates.

1. Create a stand-alone Web Form with all the fields needed to replace the
   starting variables used previously. Activate the web form.
2. Edit your workflow, and if your workflow starts with a Set Up
   Invite step, delete it.
3. Add an Collect Data with Web Forms step as the first
   step in your workflow. Use the web form you created to gather the information
   needed to start the workflow. Assign the step to the person who will start the
   workflow. If this person is not already a participant on the workflow, add them
   (e.g., "Preparer" or "Sender").
4. Add any additional steps you want the person starting the web flow to complete.
5. At the end of the starter's tasks, add a Confirmation
   Screen step to let them know they have completed all their
   assigned steps in the workflow.
6. If any of your existing steps rely on the original starting variables, edit
   those configurations and select variables from the web form added at the
   beginning of your workflow.
7. Edit the Start and select Manually in Docusign.

   You will get a warning that starting variables were created and will be
   deleted. Since you created a web form to capture this information, you can
   continue.
8. Publish the workflow.

The workflow is updated for a start manually from within Docusign. To launch this process, the preparer selects Run Workflow from the
Workflows list. The workflow takes them directly to the web
form to provide the information needed.
