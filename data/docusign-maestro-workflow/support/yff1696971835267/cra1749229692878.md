# Configure a Prepare Document Template Step

Learn how to incorporate a document template into your process to generate customized agreements automatically using workflow variables.

## Before you begin

- Recommended: Access to an existing document template to use for your workflow step. For more information about document templates, see the [Document Templates](https://support.docusign.com/s/document-item?bundleId=map332c3ebc-8fca-4fac-9b9f-c7206adc3985&topicId=conc8f06624-6ef6-40a3-bbc8-c5c7fe18746b.html&_LANG=enus) guide.

Use the Prepare Document Template step to prepare customized agreements automatically using workflow variables and document templates. While this step is often used for dynamic customization, it also supports static documents.

**Key Capabilities:**

- **Dynamic Content:** Uses conditional logic to show or hide content based on mapped values. For more information about how you can use conditional logic with document templates, see [this help topic](https://support.docusign.com/s/document-item?topicId=ziz1707785438108.html&_LANG=enus).
- **Dual Mapping:** Map workflow data to both **Sender** fields (prepopulated as part of the generated document) and **Recipient** fields (assigned to a signer).
- **Static Document Support:** Your document template does not require fields. You can use this step to include standard, non-customized files in your agreement process. Static documents are available for subsequent steps like Send Documents for Signature.
- **Multi-Document Support:** Each step generates **one** document. You can add multiple Prepare Document Template steps to a single workflow to generate multiple files, and send them all together in one Send Documents for Signature step.

**Mapping Your Data**

When configuring this step, you can map workflow variables to these field types:

| **Field type** | **Description** |
| --- | --- |
| **Sender Fields** | Data prepopulated from prior steps (like a web form). Editable by the reviewer participant in a manual review, allowing for a quality control check before the process continues. Note: Include sender fields in your template to use the **manual review** feature. If the template lacks these fields, the workflow cannot support a manual review. |
| **Recipient Fields** | Placeholders for information the signer must provide or verify. Mapping data here gives them a "head start" on their form. |

Note: If using a static document, data mapping is skipped as there are no fields to prepopulate.

A typical workflow starts with a Collect Data with Web Forms step to gather details from your user. Next, the Prepare Document Template step takes that information and uses it to complete your document automatically. Once the document is ready, a Send Documents for Signature step routes it to your signers.

1. Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus "Learn the basics of how to create an agreement workflow. Process creators and account administrators can create workflows to automate the agreement process.").
2. Add the steps that you want to precede your Prepare Document Template step, such as a Collect Data with Web Forms step.
3. Select Add a step >  Prepare Document Template >  Configure.
4. Select the document template to use for the step.
5. If your workflow start method is Manually in Docusign or From an API Call, select whether you want to include a manual review of the generated document. This review as a quality control measure. The workflow is paused after the document is generated to allow the reviewer to inspect it, and potentially edit values.

   Note: You can only use manual review for document templates that contain **sender fields**. If your selected template lacks sender fields, the system disables this option.

   1. Toggle the option to No to turn on manual review.
   2. Select the workflow participant who will complete the review.

      ![Manual review options for a Prepare Document Template step.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/manual_review_select_reviewer.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU3MzksInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.QdJrWEASqbkGeZ_pzfz8TWKkzHeVbJudjnskQIROKRE&_LANG=enus)
   3. By default, the reviewer is permitted to edit sender field data for the document. Clear this option if you want to restrict editing.
6. Map workflow variables fields to the template's fields. You can map variables to two types of template fields:

   - **Sender fields:** Prepopulates data that becomes part of the document. These fields can be reviewed during a manual review.
   - **Recipient fields:** Prepopulates input fields for the signer, which they can see or edit during the signing session.You can select variables from any prior step, such as a web form. Fields with a red asterisk require a mapping. Map variables to optional fields as needed.

   Note: If a document template sender field was used in a prior step, it will be automatically mapped for you.

   ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/map_fields_dt.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU3MzksInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.QdJrWEASqbkGeZ_pzfz8TWKkzHeVbJudjnskQIROKRE&_LANG=enus)
7. Specify a name and file format for your generated document. Adding some type of unique identifier to the document name helps avoid creating multiple documents with the same or similar names.
   1. Use the title of the document template or select workflow variables to customize the name of each generated document. If you use variables, the title of the generated document is dynamically constructed using the specified variables. These variables are concatenated with underscores to form the final filename.
   2. Select the file format. Options: .docx; .pdf

      ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/doc-gen-file-name.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU3MzksInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.QdJrWEASqbkGeZ_pzfz8TWKkzHeVbJudjnskQIROKRE&_LANG=enus)
   3. Select Apply.

The Generate Document step is added to your workflow and configured. You can use the generated document in subsequent workflow steps, such as a Send Documents for Signature step.
