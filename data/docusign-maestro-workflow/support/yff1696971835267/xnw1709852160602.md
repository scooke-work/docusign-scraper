# Template Requirements

Important information for process builders to consider before starting to develop
workflows that include a Prepare eSignature Template step. When you prepare a template to use in a Send Documents for Signature step, there are several requirements and limitations that must be respected to configure the step
successfully.

When you configure a Prepare eSignature Template workflow step, the template you select for the step is
subject to certain requirements and limitations. If your template does not meet these
requirements, or if it uses any of the unsupported features listed, your
step will not process successfully. Docusign will continue to develop
the Workflow Builder service, and potentially address some of these limitations in future
releases.

Note: In addition to these template requirements, you should also
review this topic [General Limitations and Considerations for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=otm1696972807664.html&_LANG=enus "Review important account limits, supported regions, and functional considerations before building automated processes in Workflow Builder.").

## Troubleshooting template issues

When you configure a Prepare eSignature Template step in a workflow, you may encounter an error message:
This template is not supported. The error message lists the
unsupported conditions. Either edit your template to remove these features or select
a different template. You can find details about the unsupported features in the
following section.

## Requirements

To use a Prepare eSignature Template step in your workflow, you employ an eSignature template. Your
template must meet certain requirements. (These requirements are subject to change
in subsequent releases.)

Recipient types
:   Only the Needs to Sign, Receives a Copy, and Needs to View recipient types
    are supported.

Unique recipient roles and recipient names
:   The recipient roles and any named recipients must be unique values. Your
    template cannot have duplicate role or recipient names.

Recipients
:   Your template must have at least one configured recipient.

Signing groups
:   Signing group recipients are not supported.

Conditional routing
:   Routing group recipients are not supported.

ID Verification
:   ID verification specified on your template is supported. You can also add a separate
    workflow step for ID verification. If the template and the Verify
    Someone's Identity step use the same workflow method for the same
    participant name, the result (identity claim) is recorded on the eSignature
    Certificate of Completion. For details, see the topic [Identity Claims](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=vwg1731931235105.html&_LANG=enus "Learn about identity claims and how you can use them to streamline your automated agreement processes.").

Signature type
:   The following signature types are not supported:

    - ICP Smart Card
    - ICP-Brasil com Carimbo do Tempo
    - DS EU Advanced
    - DS EU Qualified with in person ID check
    - IDV - DS EU Advanced
    - IDV - DS EU Qualified

Delivery method
:   Workflow Builder supports all Docusign multi-channel delivery options for notifications: email, SMS,
    and WhatsApp.

Branding
:   When a template has a signing brand, that brand is used for all emails and signing sessions when you use the template in a Send Documents for Signature step.
:   If you choose a signing brand at the workflow level and opt to have it override the template brand, it can replace the template brand. The only exception to this override is when the template branding is locked. A template is brand-locked when an eSignature template owner or administrator turns off the **Allow senders to edit the logo and brand colors** option in the template's Advanced Options. In that case, the template's branding always applies and the workflow cannot change it.
:   See also: [Set Branding for a Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=task-6976.html&_LANG=enus "Choose a signing brand for your workflow to make sure all related emails and signing sessions use your custom logos and colors.")

Multi-recipient templates and direct signing
:   On a template with multiple recipients, only the first
    recipient can be routed directly to signing. Additional recipients follow the
    standard send for signature notification and routing process. See the topic [Configure a Send Documents for Signature Step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus "Learn how to add and configure a workflow step to send a document for signature.") for more information.

Envelope custom fields
:   Required envelope custom fields (ECFs) that are blank on your template must be mapped to a workflow variable or have a hardcoded value entered when configuring the subsequent Send Documents for Signature step. You cannot finish configuring or saving that sending step if a required ECF remains unmapped or empty.

    Your template can have optional ECFs that are blank or ECFs with hard-coded values. Blank
    ECFs will remain blank for envelopes sent through a Workflow Builder
    workflow.

Email subject
:   The template must have a message for recipients with an email subject line
    specified.

Field labels best practice
:   Adding descriptive field labels to your template fields helps process
    builders create accurate data connections.

Field types for data mapping
:   You can map workflow data to the following field types:

    - Text
    - Note
    - Name
    - Email
    - Phone
    - Number
    - Checkbox
    - Radio button
    - Dropdown

    Other field types cannot be used for data mapping but are still valid
    for recipients to complete.

Pre-fill fields
:   Empty pre-fill fields are not supported. If your template contains pre-fill
    fields they must have values entered.

Merge fields
:   Merge fields, including document generation fields, are not supported. Only
    data gathered through the workflow (such as on a web form) can be mapped to
    a document text field.

Elastic templates are not supported
:   You cannot use an elastic template in a Prepare eSignature Template step.
