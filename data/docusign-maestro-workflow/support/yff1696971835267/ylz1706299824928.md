# Connect Events for Workflows

Read about how Docusign administrators can use Docusign Connect events to track certain workflow events.

When certain workflow events occur, Workflow Builder publishes Connect trigger events. You can
also use other Connect events to start a workflow.

## Workflow events

Workflow Builder publishes the following Connect trigger events:

- workflow-started
- workflow-completed
- workflow-failed

These workflow instance events are available to use in a custom Connect
configuration. For example, you can use these Connect events to trigger a
notification or record data in another application.

These workflow events are documented in the [Connect Events](https://support.docusign.com/s/document-item?bundleId=vob1727899215236&topicId=lry1608071169447.html&_LANG=enus) guide.

## Use Connect events to start a workflow

With the Event start method, a workflow starts when a
specified Connect event occurs. The workflow starts automatically without manual
oversight or intervention. Workflows can be started based on events occurring within
the same Docusign account.

Learn more about the Event start method in the help topic
[Workflow Start Methods](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=ztb1727892686033.html&_LANG=enus "Review the different automated and manual trigger methods available to initiate processes in Workflow Builder, including links, events, API calls, and CLM integrations.").
