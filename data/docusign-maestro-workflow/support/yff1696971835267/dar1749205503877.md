# Capture ID Evidence Data

Learn how to configure your workflow to enable ID Evidence to capture data.

## Before you begin

To capture ID Evidence data from a workflow step, you must follow these prerequisites:

- You must use either a Verify Someone’s Identity or a
  Risk Assessment step whose configuration enables data
  storage. For more information about ID Evidence limitations in Workflow Builder, read the
  [ID Evidence in
  Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kmm1749205469137.html&_LANG=enus "Read about ID Evidence integration with Workflow Builder and how it helps deliver evidence of the ID verification process.") page.
- You are a Docusign administrator or a workflow creator as specified in the [Workflow Builder Roles and Permission Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus "Read about the different roles involved with creating and using workflows with Docusign Workflow Builder. Each role requires specific user permissions.") page.
- You have installed and authorized the application that stores ID Evidence data on your account. Read the [Install Apps in the Docusign App Center](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ctg1698170340729.html&_LANG=enus) and [ID Evidence in Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kmm1749205469137.html&_LANG=enus "Read about ID Evidence integration with Workflow Builder and how it helps deliver evidence of the ID verification process.") pages for authorized applications to store ID Evidence files.

As a process builder, you want to capture data from a Verify Someone’s
Identity or a Risk Assessment step. For
example, you want to capture the names of the participants who verified their
identity and the verification process results. You may also want to capture a copy
of the identity document participants used to identify themselves. Once ID Evidence
has captured the data, you want to access it through a PDF file in a specific
repository.

1. Create a new workflow or edit an existing workflow.
2. Ensure that your workflow includes a Verify Someone’s
   Identity or Risk Assessment step.

   If your workflow does not include such steps, add one as described in the
   [Configure a Verify
   Someone's Identity Step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=qht1698126463108.html&_LANG=enus "Learn how to configure a Verify Someone's Identity step in an agreement workflow. You use an identity verification step to verify a participant's identity using Docusign's ID Verification process.") or in the [Configure a Risk Assessment Step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=tsk6ecf58be-0285-4608-92cd-66252f6e07a5.html&_LANG=enus "Learn how to configure the Risk Assessment step in a workflow. This step gathers participant risk signals for dynamic orchestration, data-driven decisions, and simplified compliance.") page.

   Important: The identity verification configuration that you use in
   your Verify Someone’s Identity or Risk
   Assessment step configuration must allow data
   storage. For more information, read the [Add Identity Verification Configurations](https://support.docusign.com/s/document-item?bundleId=pik1583277475390&topicId=dnh1583277456343.html&_LANG=enus) or the [Configure a Risk Assessment Step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=tsk6ecf58be-0285-4608-92cd-66252f6e07a5.html&_LANG=enus "Learn how to configure the Risk Assessment step in a workflow. This step gathers participant risk signals for dynamic orchestration, data-driven decisions, and simplified compliance.") help pages.
3. Add other steps as needed to complete your workflow.
4. Select Add a step > Store files and choose the storage application where you want to archive your
   ID Evidence file.

   For example, Store files in Google Drive.

   Workflow Builder adds your step to the workflow.
5. Select Configure.

   Workflow Builder opens the configuration panel.
6. Configure your Store files step.

   Important: Ensure you select ID Evidence file in the Select a file or envelope from any earlier step in this workflow list.

   For more information about configuring a Store files step, read the following pages:

   - [Configure a Google Store Files Step in an Agreement Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=wqt1698167316926.html&_LANG=enus "Learn how to configure a Google Drive storage step to automatically archive documents generated in earlier workflow steps.")
   - [Configure a Dropbox Store Files Step in an Agreement Workflow](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=rgp1706110555492.html&_LANG=enus)
   - [Configure a SharePoint Store Files Step in an Agreement Workflow](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=haj1706107382131.html&_LANG=enus)
   - [Configure a Microsoft OneDrive File Archive Step in an Agreement Workflow](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=wir1706111649199.html&_LANG=enus)
   - [Configure a Box Store Files Step in an Agreement Workflow](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=dfb1706110409817.html&_LANG=enus)
7. Select Apply.
8. Select Review and Publish.

   The Review before publishing panel opens.
9. Review and publish your workflow as explained in the [Review and Publish a Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=iqm1698272226447.html&_LANG=enus "As a process builder, publishing a workflow validates your steps are properly configured and makes the workflow available to your account members. Once published, preparers can start instances of your workflow.") page.

## Next steps

Your workflow is now ready for use. ID Evidence will capture participants’ data and export it to the appropriate directory. Open this directory to access the PDF export file.
