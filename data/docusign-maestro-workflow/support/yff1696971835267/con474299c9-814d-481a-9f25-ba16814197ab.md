# Agreement Actions and Workflow-Powered Rules

Learn how workflow-powered rules transform Agreement rules into automated processes using Workflow Builder, enabling richer, multi-step processes while maintaining trusted configurations.

For customers who use Agreement rules to archive or export agreements, Workflow Builder can take those existing rules and turn them into automation steps inside a workflow. This lets you keep using the rules you trust today, while expanding them into richer, multi‑step automations that run in Workflow Builder.

When you save an Agreement rule that uses an Archive to action, Docusign creates a Workflow Builder‑powered system workflow behind the scenes. You can then use that rule to generate an editable workflow that uses the rule's event as the trigger and its archive action as one of the workflow steps. Administrator permissions are required both to create rules and to use them as automation steps in a workflow.

## What are workflow-powered rules?

Workflow‑powered rules are Agreement rules that run on Workflow Builder behind the scenes.

When you save an Agreement rule that uses an **Archive to**, DocuSign creates and publishes a **system workflow** in Workflow Builder that represents that rule. This conversion happens automatically:

- The rule appears in the **Agreement Rules** list in the Agreement Actions Admin view, using the same configuration UI you use today.
- In the Agreement Rules list, these rules are labeled to indicate they are **powered by Workflow Builder**.
- The underlying system workflow:

  - **Does not** appear in your  **Workflows** list.
  - **Does not** count against your standard workflow allowance.

From an administrator's perspective, you still create and manage a "rule." Behind the scenes, Workflow Builder is now responsible for running it.

## Turning a rule into an automation step in a workflow

For supported actions, Workflow Builder can use a workflow‑powered rule as the starting point for a richer, multi‑step workflow. You can add additional steps to the new workflow that starts with the completed envelope event specified by the agreement rule.

When you choose to add automation steps from an Agreement rule:

- Docusign creates an **editable user workflow** based on that rule.
- The workflow is configured with:

  - A **From an Event** start method for an envelope completed event. The event conditions match the rule's parameters. For example, the rule is designed to archive all completed envelopes sent by a specific email address. In the workflow start method, the event conditions specify that the sender's email must equal the address used in the rule.
  - A workflow step that performs the same **Archive to** action defined in the rule.

This new workflow appears in your standard  **Workflows** list and, once published, counts against your workflow allowance, as defined by your plan. Your original rule remains in the Agreement Rules list. The system workflow tied to that rule continues to run it, while the user workflow you created lets you extend that same logic into broader automations.

Note: Changes to a rule you use to create a workflow are NOT reflected in your workflow. The workflow uses a system-generated copy of the rule that cannot be modified. To make changes, edit your workflow details directly.

For detailed steps on this process, see the help topic [Create a Workflow from an Agreement Rule](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=tsk9166c067-1d6e-49a9-91c8-641cda5f7f65.html&_LANG=enus "Learn how to generate a workflow from an Agreement Rule, using the rule's event as the automation trigger and its archive action as the initial workflow step.").

## Who can use workflow-powered rules

Workflow‑powered rules are available to IAM customers who have both Workflow Builder and Agreement Actions. Only new rules created for Archive and Export actions will be workflow-powered. Legacy Agreement rules that existed **before** this feature was released are not changed. They continue to run on the existing Agreement Actions infrastructure.

You must have administrator permissions to create and edit Agreement rules and turn a rule into a workflow.

## Limitations

Workflow-powered rules are a new Docusign IAM offering, so there are some significant limitations. The scope and function of these Agreement Action rules will expand over time. For now, please keep these limitations in mind as you explore this new feature:

- **Only Archive to rules are supported.** Other rule actions, such as Create Another Envelope, are not supported at this time.
- If an Agreement Actions Archive to rule is specified with any of the following conditions, the rule will be a legacy-style rule and not workflow-powered. You cannot use this type of rule to create a workflow.

  - **Archive to CLM.** Specifies archiving to CLM.
  - Customized envelope setting that specifies combining all envelope documents into a ZIP file but excludes the Certificate of Completion.

## Activity logs for workflow-powered rules

Workflow-powered rules generate activity logs that capture key details such as the envelope acted upon, action type, execution status, and the date and time of execution.

These logs are accessible from the Agreement Rules list and are stored separately from other logs.

For more detailed instructions, refer to the topic [Access Rule Logs for Agreement Rules](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=tsk1244004b-76be-4f59-b82e-08bbc5700972.html&_LANG=enus "Learn how to access execution logs for agreement rules used to trigger a Workflow Builder process.").
