# How to Start a Workflow

Learn how the different start methods determine how to start a workflow.

Published workflows have a defined start method. Depending on which start method is applied,
the workflow is initiated in different ways. For all methods, the workflow must be
published to start an instance.

Once a workflow instance is started, it has 120 days to complete. If the instance is still in process after 120 days, it will expire.

Note: Paused workflows cannot generate new instances, regardless of the
start method. The workflow status must be "Published" to start an instance.

From a Link
:   Once published, a workflow with a link start has a URL. Use the URL to embed it on your
    website or share directly with the participant who will start a workflow
    instance. Select the link icon on the Workflows list
    page to copy the URL. ![Screenshot of the Workflows list showing the link icon for copying the URL.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/workflowlink.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU5MTAsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.JQx5-_ZNbAKDXjnBYTKgVMJD8VQcE55fOKwzq6ZH6TM&_LANG=enus "Screenshot of the Workflows list showing the link icon for copying the URL.")
:   You can make this link accessible for participants to start an agreement
    process themselves. Typically, you use the URL if your workflow is
    configured to start with an anonymous participant in an unmonitored
    scenario.
:   For example, you could post the link on your website for your customers to
    use to start an application process. Or you might use the link in an email
    marketing campaign to encourage prospects to sign up for an event.

Manually in Docusign
:   Use the Run Workflow option from the Workflows
    list more actions menu to start the workflow. When prompted, you can enter a
    unique name for the workflow instance in the Instance
    name field. If you do not add a name, the workflow's default
    instance name is used.
:   Next, complete the workflow steps assigned to the first participant.
    Typically, the first participant is directed to a web form to provide
    participant information for other workflow participants. This method is
    similar to using a template to create an envelope and completing the
    recipient information for any template roles must be defined.
:   You must have Workflows: Prepare permissions to start a
    workflow with this method. ![Screenshot of the Run Workflow option in the action menu for a published workflow.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/runworkflow.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU5MTAsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.JQx5-_ZNbAKDXjnBYTKgVMJD8VQcE55fOKwzq6ZH6TM&_LANG=enus "Screenshot of the Run Workflow option in the action menu for a published workflow.")

Docusign CLM
:   After you publish the workflow, you create a CLM workflow that
    triggers it.
:   To test this workflow without triggering it from CLM, select Run
    Workflow from the Workflows list more
    actions menu. Enter dummy data for the workflow variables.

From an API Call: Human
:   Select the link icon on the Workflows list page to copy the URL and
    use it to construct the API call. For this start method, the participant
    initiating the workflow will be redirected to the first step of the workflow
    when the API is triggered.
:   To test this workflow without making an API call, select Run
    Workflow from the Workflows list more
    actions menu. Enter dummy data for the workflow variables.

From an API Call: Automated process
:   Select the link icon on the Workflows list page to copy the URL and
    use it to construct the API call. For this start method, an external system
    starts the workflow and not a participant. If there are any participants in
    this workflow, you must invite them with a Set Up
    Invite step.
:   To test this workflow without making an API call, select Run
    Workflow from the Workflows list more
    actions menu. Enter dummy data for the workflow variables.
