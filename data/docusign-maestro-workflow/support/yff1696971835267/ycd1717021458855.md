# Status Rules for a Send Documents Step

Read about how you can use branching rules to check the status of an envelope generated from a
prior workflow step.

If your workflow includes a Send Documents for Signature step, you should include
branching rules to account for certain scenarios that lead to unplanned envelope events.
By using branching rules to check the status of an envelope, you can control what
happens in the workflow.

There are three statuses that you can check for:

- declined
- voided
- completed

## Recipient declines to sign

If any recipient declines to sign, the workflow will still move to the next step. To control
the workflow progress, add a branching rule after the Send Documents step to check
for the decline status.

Example rule details to stop a workflow if a participant declines to sign:

- Configure the rule to check the Send Documents step variable `Envelope
  Status`, with the operator `Equal to`, for the result
  `declined`.
- Add a confirmation step to the True branch. Configure it to show a
  custom message to the Send Documents step participant, such as "Sorry you
  declined to sign this document".
- Add a confirmation step to the False branch. Configure it
  to display a Tasks Complete message.
- Continue your workflow steps on the False branch of the
  rule. These additional steps define the workflow process after the envelope
  successfully completes. ![Screenshot of a workflow example of a branching rule for status = declined.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/exampledeclinedrule.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4NDMsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.54N0ALt11MFJgiKY2NGZ0v0E6dM0wMO6JcLjgC42IVY&_LANG=enus "Screenshot of a workflow example of a branching rule for status = declined.")

## Sender voids an in progress envelope

If the sender voids an envelope, any in progress signer is redirected to a page explaining the
voided status. The workflow will continue to the next step unless you add a
branching rule to check if the envelope `Envelope Status` is
`Equal to` the result `voided`.

Continue your workflow steps on the False branch of the rule.
These additional steps define the workflow process after the envelope successfully
completes.

## Envelope completes successfully

The Send Documents for Signature workflow step will continue to the next step when an
envelope reaches a terminal state (declined, voided, or completed). With a branching
rule, you can ensure that the workflow continues to the next step if and only if the
envelope reaches a terminal state of completed. This rule provides the appropriate
guardrails for subsequent steps, such as writing data from the envelope back to
Salesforce. These steps are executed only when an envelope completes.

To confirm a successful result, add a branching rule after the Send Documents step to check if
the envelope `Envelope Status` is `Equal to` the
result `completed`. Continue your workflow steps on the
True branch of the rule.
