# Mapping Data From Web Forms

Learn the structural requirements and formatting rules for mapping advanced data types, such as checkboxes, radio groups, and dropdowns, from a web form to an agreement template.

For a Send Documents for Signature step, you use a prepared template or generated document to create an envelope. As an option, you can map data fields from other sources to prepopulate the document fields. This mapping is done in the Prepare eSignature Template or Prepare Document Template step. This data can be strings for values like text, dates, email addresses, and phone numbers, and also numerical values. You can also map data to these advanced field types: checkbox, radio group, and dropdown fields.

For the advanced field types, to map data between a web form and an eSignature template or generated document, there are some specific requirements. The web form API value must match the eSignature *value* property exactly. In addition, the web form API value does not allow spaces. You can use underscores or other characters to separate words for these values.

## Example: Radio group mapping

This example illustrates how you would prepare a radio group field to allow data mapping between a web form and an eSignature template.

In this example, there is a radio group field in an eSignature template for marital status. The Radio Button Values property uses underscores to concatenate the option text.

## Admin setting for mapped web form data visibility

When you map data from a web form to fields in an eSignature template, an account-level admin setting controls whether those mapped values are visible to all recipients when the envelope is sent.

When the **When an envelope is sent, write the initial value of the field for all recipients** setting is turned off, mapped values only appear to the recipient who is assigned those fields. Other recipients do not see the values until that recipient completes their fields and finishes signing.

To make mapped values visible to all recipients at send time, your Docusign administrator must turn on this setting in the **Admin** panel under **Sending Settings**. For more information about this setting and its effects, see [Docusign Support Center.](https://support.docusign.com/s/articles/How-do-I-share-the-information-in-all-fields-with-all-recipients-when-sending-a-document?language=en_US)

Note:

This setting does not affect document templates or generated documents.
