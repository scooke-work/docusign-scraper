# Review a Migrated Workflow

Learn how to review and publish a draft workflow after Docusign
automatically migrates it from an older format. When you publish the draft, that will
activate the new features and replace the active older version.

Once Docusign migrates your workflow draft, you must review the
draft for accuracy. As with any draft, when you're ready to use the updated workflow,
you must publish the changes.

1. Locate the workflow that you want to review, then select the more
   actions menu and choose Edit. If there is
   a saved draft of the workflow, this will continue to edit that draft.

   In the workflow designer, you should see the migration information banner.![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/designer_migrated_workflow.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4NzMsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.VpGhrJhY4Ql_ft5s_r3aGKn8mFzEAg9uvNtKhsSyyXU&_LANG=enus)
2. Review the migrated workflow to ensure that all steps, participant information,
   and conditional paths are correct.
   - The Get Signatures step has been separated into two steps: Prepare
     eSignature Template and Send Documents for Signature. Review the
     configuration settings for each step to confirm the template, mapped,
     fields, participant information, and so on, is correct.
   - Participant information that was previously in an Update Participant
     Information step is now automatically distributed to the relevant steps
     where it is used.
3. (Optional) Add any new features that you want to use, such as the Prepare
   Document Template step or merging paths of branching rules.
4. Once you have reviewed and finished updating the draft, select
   Publish Changes.

Your migrated workflow using the latest Workflow Builder features is published and ready to run.
