# Workflow Builder Activity Logs

Read about activity logs for workflows. Account Administrators, workflow administrators, process builders, and preparers can use these logs for a full audit trail of actions on workflow definitions and instances.

Activity Logs provide full visibility into all actions taken across your workflows, enabling better governance, simplified troubleshooting, and clear accountability for all workflow-related tasks. Activity logs are available at the account, workflow, or instance level and can be exported as a CSV. Customers use audit capabilities to check the status of a workflow or confirm that a workflow was executed as expected.

## Key features

The activity logs provide users with details about events for both workflow definitions and individual instances.

- **Comprehensive Tracking**: Logs capture a wide range of activities, including user actions, system events, and workflow configuration changes.
- **Filtering and Sorting**: Quickly locate specific events or actions based on criteria such as user, date, event type, or resource.
- **Audit Trail**: Maintains a secure and immutable record of all activities for compliance and auditing purposes.
- **Exporting**: Logs can be exported for further analysis.

## Use cases

- **Security Auditing**: Track user access and changes to sensitive data or configurations.
- **Troubleshooting**: Identify the root cause of errors or unexpected behavior by reviewing recent activities.
- **Compliance**: Demonstrate adherence to regulatory requirements by providing a detailed activity history.
- **Operational Insights**: Analyze usage patterns and system performance over time.
