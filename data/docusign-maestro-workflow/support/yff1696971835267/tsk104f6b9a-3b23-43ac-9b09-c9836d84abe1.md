# Create a Workflow from a Web Form

Learn how to quickly convert an existing Docusign web form and its linked eSignature template into a fully configured Workflow Builder process in a single step, saving significant setup time.

## Before you begin

- You must have access to a configured web form that is linked to an eSignature template. (The web form doesn't need to be active to use it it to create a workflow.)
- Creating workflows requires specific user permissions. For more information on required permissions to complete this task, see [Workflow Builder Roles and Permission Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus "Read about the different roles involved with creating and using workflows with Docusign Workflow Builder. Each role requires specific user permissions.").

This topic explains how to create a Workflow Builder workflow directly from an existing web form. This feature allows you to quickly convert an existing Docusign Web Form and its linked eSignature template into a fully configured Workflow Builder workflow in a single step, saving significant setup time.

As a process builder, you can start a new workflow from a web form from various eSignature locations: from the main action menus of the Templates, Home, and Agreements pages, or from the Workflow Builder Workflows view.

When you create a workflow from a web form, the system automatically creates new workflow versions of the web form and its linked eSignature template to use in the workflow. The new web form is added to your list of web forms, named "Workflow Copy - [original web form name]", and activated. The web form is stand-alone and is not associated with any template. The workflow you create will have the following structure:

- The workflow's start method will be set to From a Link.
- Recipients from the template are automatically added to the workflow's list of participants.
- The workflow will include three main process steps: a Collect Data with Web Forms, Prepare eSignature Template, and Send Documents for Signature.
- The first signer role on the template is the participant role assigned to the web forms step.
- The workflow ends with a Show Confirmation Screen assigned to the participant assigned to the web forms step.

1. In your Docusign account, start the workflow creation process using one of the following methods:

   - From the Templates page: Select one of the WEB FORMS views from the left navigation. Locate the form you want to use, select the more actions menu, and choose Add Automation with Workflow Builder.
   - From the Home page: Select Create a Workflow > Start from Web Form. Select the form you want to use and click Continue.
   - From the Workflow Builder Workflows view on the Agreements page: Select Create Workflow > Start from Web Form. Select the web form you want to use and click Continue.

   The Preparing your workflow... message displays while your new workflow is created.
2. Review the steps of your new workflow. All steps are configured with participant and field mapping details.

   - Participant information is created from the eSignature template recipient roles.
   - The Collect Data with Web Forms step uses the new workflow version of your web form. The first recipient role from the template is assigned as the workflow participant.
   - The Prepare eSignature Template step uses the new workflow version of your template. Web form data is mapped to fields in the eSignature template.
   - The Send Documents for Signature step uses the prepared eSignature template.
