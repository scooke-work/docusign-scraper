# Configure a Multi Branch Rule

Learn how to use the Add a Multi Branch Rule step in Docusign Workflow Builder to create workflows with multiple paths based on specific conditions. This feature allows for dynamic processes that adapt based on data collected from web forms.

A multi branch rule creates a step where the workflow path diverges into up to 15 different paths based on a condition. It allows you to create workflows where the next step depends on the information collected earlier, such as selections made in a web form.

The Multi Branch Rule allows a workflow path to diverge into multiple different paths. For example, you can use it to send different documents to users based on their state of residence. Your rule would have a path for each possible state entered by the user.

Multi branch rules provide the following support:

- Define up to 15 conditional branches for a single rule evaluating a single workflow variable.
- Currently, it only supports an "equals" comparison. This means the value collected (for example, from a web form) must exactly match the value specified in the rule for a specific branch to be taken. Future enhancements may include additional comparison types. The comparison matching is case insensitive.
- Each rule has a No Conditions Met branch. This fallback branch ensures that your workflow has a valid path to proceed on if none of the branch conditions are met.
- Merge two or more branches to simplify your workflow design and eliminate the need to duplicate follow on steps.
- Workflow monitoring provides visibility into which branch was executed for each workflow instance. [Monitor a Running Workflow Instance](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=vcp1698707326210.html&_LANG=enus "Learn how to check the progress of the agreement instances you start from a workflow.")

Tip: By comparison, the Branching Rule step described in [Configure a Branching Rule](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=mnv1698169910928.html&_LANG=enus "You can create conditional rules in your agreement workflow. These rules validate a condition and branch the workflow based on whether the condition is met or not.") offers a true/false branch with a broader selection of comparison types. This rule is useful for testing conditions using a variety of comparison types (such as 'greater than,' 'less than,' or 'contains').

1. Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus "Learn the basics of how to create an agreement workflow. Process creators and account administrators can create workflows to automate the agreement process.").
2. Add a Collect Data with Web Forms step. The web form will collect the information you will use in the rule for your branching step. For example, a form might collect a user's name, email address, rental date, and the state where their license was issued.
3. Select Add a step >  Add a Multi Branch Rule after your Collect Data with Web Forms step.

   - If you are adding the step between existing steps, select which path to move the steps that come after the new rule. Then select Continue.
   - If you are adding the step to the end of your workflow, continue to [the next step](https://support.docusign.com/s/document-item?language=en_US&bundleId=yff1696971835267&topicId=xtg1749585007265.html&_LANG=enus&anchor=xtg1749585007265__step_w1040ab1b7b3b5_w1041ab1b7b3_w1042ab1b7_w1043ab1).![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/multibranchrule.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4NDgsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.YTf0Up-wGRiqE98cyevZFh4wGiqgQOjBRtJonVfb6DY&_LANG=enus)

   The configuration panel for the branching rule opens. The rule starts with one configurable branch and a No Conditions Met branch.
4. Select the variable for the rule to check against and specify a value to compare to. Then select Done.

   Example: Add a multi branch rule after a Collect Data with Web Forms step to automatically determine which contract to send for signature.
   - Select the variable to evaluate. Select the web form step and the specific variable that you want to evaluate from that form.
   - Define Branch 1. Enter the specific value that you want to compare against the information the workflow participant will provide in the web form variable. For example, if the variable is "ContractType", you might enter "Standard" or "Premium" here. For matching, this value is case-insensitive.
   - (Optional) Name the branch. Enter a descriptive name for the branch to easily identify it on the workflow canvas.
5. (Optional) Select Add Branch +  to add branches to your rule. You can configure up to 15 branches.
6. (Optional) Choose if any branches should merge. Select at least two branches to merge.

   ![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/merge_branches_selection.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4NDgsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.YTf0Up-wGRiqE98cyevZFh4wGiqgQOjBRtJonVfb6DY&_LANG=enus)

   Important: Variables and files generated from steps inside of a merged branching rule will not be available to use in steps occurring after the merge. For more details, see [Merging branches](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=jaf1716918359288.html&anchor=jaf1716918359288__section-426&_LANG=enus).
7. Select Apply.

Your Multi Branch Rule step is configured.

## Next steps

**Add steps to each branch**: After creating the branches, add steps to each branch, such as Send Documents for Signature steps with different documents. Each branch can have its own unique set of workflow steps.
