# Send a Copy of a Completed Agreement in a Workflow

Learn how to configure a workflow to automatically send a copy of a completed agreement to a recipient who isn't a signer.

## Before you begin

- **Permissions:** You must have Administrator or appropriate Process Builder permissions to create workflows.
- **Template Setup:** Template requirements vary by the document creation step type:
  - **Document Template:** Include a "Signer" recipient without assigned fields. Their role and receipt order will be designated during workflow configuration. See [Configure a Prepare Document Template Step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=cra1749229692878.html&_LANG=enus "Learn how to incorporate a document template into your process to generate customized agreements automatically using workflow variables.").
  - **eSignature Template:** Add the copy recipient using the 'Receives a Copy' or 'Needs to View' role. Position this recipient within the signing order to ensure they receive the document in the desired state, either completed or still in process. See [Configure a Prepare eSignature Template Step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=tskd83cc960-fb4b-4534-9573-1041591501e2.html&_LANG=enus "Learn how to incorporate an eSignature template into your process, prefill recipient fields using upstream variables, and route completed documents to a secure repository.").

Once an agreement is fully signed and completed, you'll often need to provide a copy to someone who wasn't part of the signing process. For example, for record-keeping or internal review.

To achieve this routing, first properly configure your template (eSignature or document template), and then use the template in the Send Documents for Signature step. Make sure to review the Template Setup listed for this task. With the right set up, you can ensure that the right person automatically gets their copy of the completed document.

1. In the workflow editor, add and configure one or more Prepare Document Template steps or a Prepare eSignature Template step. These steps produce the documents that you will send for signature.

   Important: The template you select must meet the prerequisites listed for this task.
2. Add the Send Documents for Signature step and select the documents you want to send.
3. Configure the step to confirm which workflow participant will perform each template role. Depending on which type of document you're sending, do one of the following:

   - **Generated document**:
     - Add the participant to the template recipient without assigned fields.
     - Specify what the recipient must do: In the drop-down menu, select either Needs to view or Receives a copy.
   - **Prepared eSignature template**: Assign the participant to the copy recipient role defined on the template.
4. Confirm the signing order:
   1. Ensure the Set signing order option is selected.
   2. Assign the copy recipient to the **last** position in the signing order to guarantee they receive the completed document.
5. Provide the required participant information using workflow variables or manual entry. (Name and Email. Phone Number for Text message or WhatsApp delivery.)
6. Click Apply to save the changes to the step configuration.
7. Complete and publish the workflow.

The workflow is configured to automatically email the completed agreement to the copy recipient.
