# Configure a Branching Rule

You can create conditional rules in your agreement workflow. These rules validate a
condition and branch the workflow based on whether the condition is met or not.

A branching rule creates a true/false step. You can add a branching rule after any step (such as
identity verification). After the branching rule, you define the next action for
whether the rule is met (true) or not met (false).

As an example, you might add a branching rule after a Verify Someone's Identity step. If the participant
passes the identity verification, they continue to the next step in the agreement
process, such as a Send Documents for Signature step. If they fail the identity verification, you
end the workflow with a confirmation step explaining that they must retry or contact
you for assistance.

Branching rules support a variety of advanced conditionality options:

- Nest up to eight levels of branching rules. Use nested rules to test for multiple criteria,
  making your workflow more flexible and powerful.
- Add AND or OR conditions. Configure a single branching rule with up to five AND or OR
  conditions.
- Use advanced operators to create more precise conditional logic statements.

You can add branching rules to your workflow as you build it. You can also insert an
Add a Branching Rule step between two steps. If you do
this, you specify to which branch to move all steps that follow.

![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/gifinsertbranchingrule.gif?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4MzEsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.9gslAu5SY6FTguvcODeHj9n0zT-9VhWD5m37EpyqGJQ&_LANG=enus)

Similarly, if you delete a branching rule, you select whether to keep to True or
False path steps in your workflow. You can also choose to delete all steps from both
paths.

For more information about complex logic for branching rules, see [Configuration Details for Complex Branching Rules](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=jaf1716918359288.html&_LANG=enus "Read about nesting IF statements and using advanced operators in a branching rule workflow step.").

1. Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus "Learn the basics of how to create an agreement workflow. Process creators and account administrators can create workflows to automate the agreement process.").
2. Select Add a step >  Add a Branching Rule.

   - If you are adding the step between existing steps, select which path to
     move the steps that come after the new rule. Then select
     Configure.
   - If you are adding the step to the end of your workflow, select
     Configure.

   The configuration panel for the branching rule opens.
3. Select the variable for the rule to check, the operator for the rule, and the
   result for the rule. Then select Done.

   Example: Add a branching rule after a Verify Someone's Identity step to
   determine the next step based on whether the participant successfully validates
   their identity.
   - For what to check, select the identity step and the variable
     `Validated`.
   - For the operator, select `Is`.
   - For the result, select `True`.![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/branch_rule_example.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4MzEsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.9gslAu5SY6FTguvcODeHj9n0zT-9VhWD5m37EpyqGJQ&_LANG=enus)Example: Add a branching rule after a web forms step to determine the next
   step based on the data entered in the form.
   - For what to check, select the web forms step and the form variable to
     evaluate.
   - For the operator, select one of the comparisons, such as `Equal
     to`, `Contains`, or `Does not start
     with`.
   - For the result, enter the value to compare to.
4. (Optional) Select + AND or + OR
   to add conditions to your rule. You can configure up to five conditions.
5. (Optional) Select the option Merge true and false branches? if you want to combine the branches to continue the workflow. This optional setting is useful if you want to combine the paths to use a single step outcome.

   Important: Variables and files generated from steps inside of a merged branching rule will not be available to use in steps occurring after the merge. For more details, see [Merging branches](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=jaf1716918359288.html&anchor=jaf1716918359288__section-426&_LANG=enus).
6. Select Apply.

Your branching rule is configured. Add the next steps for your
branching rule, depending on whether the rule was met (True) or
not (False).
