# Automate Agreement Desk Requests with Workflow Builder

Learn how to streamline the agreement lifecycle by using Workflow Builder to triage, manage, and automate incoming Agreement Desk requests.

By combining Agreement Desk triggers and actions inside Workflow Builder, you can streamline your contract lifecycle from initial intake to final signature. Use these automated processes to manage incoming requests, route approvals, assign ownership, and generate supplemental documents.

This topic provides the basic process for creating a workflow to automate many Agreement Desk request actions. For detailed guidance, including configuration details for the different Agreement Desk steps, see our guide, [Automating Agreement Desk Requests with Docusign Workflow Builder](https://support.docusign.com/s/document-item?bundleId=map4317ef56-de36-43e2-9921-c76a445c3276&topicId=con27b1a75a-da95-4952-94a4-a20c498bf81c.html&_LANG=enus).

1. Define the Workflow Start Method.

   Configure your workflow to initiate automatically based on specific Agreement Desk events using the From an Agreement Desk request start method. This method includes triggers such as when a request is submitted, its type or status changes, an envelope is sent, or a signature is completed.![Screenshot showing the configuration options for the 'From an Agreement Desk request' start method in a workflow.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/start_method.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU2NzYsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.v6jcKvUUSalxp_JHDwxntTfkBYI7hk1-hRcrkc8GH1Y&_LANG=enus "Screenshot showing the configuration options for the 'From an Agreement Desk request' start method in a workflow.")
2. Add Agreement Desk Steps to your workflow.

   Incorporate a variety of Agreement Desk-specific steps into your workflow to manage requests efficiently. These steps allow you to:

   - Assign an owner to a request.
   - Add followers for visibility.
   - Change the request status.
   - Set a due date for the request.
   - Add permissions for users or groups.
   - Create internal approval requests.
3. Generate Documents for a Request.

   Use the **Prepare Document Template** step to generate customized documents within your workflow. These documents are then automatically added to the Agreement Desk request and can be sent for signature.
4. Collect a Request Document.

   Use the **Get Document from a Request** to collect the latest version of a request document to send for signature or archive to a system of records.
5. Implement Branching Rules.

   Use branching rules to introduce conditional logic, allowing your workflow to adapt dynamically based on outcomes such as approvals or the status of signature requests. This ensures that the process only moves forward when specific conditions are met.
6. Send documents for signature or to a system of record.

   Use standard Workflow Builder steps to send generated and collected documents for signature or store in Salesforce, Google Drive, or other systems of record. As an option, this step supports automated sending from the request submitter. See [Sender Impersonation and Consent for Agreement Desk Workflows](https://support.docusign.com/s/document-item?bundleId=map4317ef56-de36-43e2-9921-c76a445c3276&topicId=con85f610e6-2584-423e-840e-8823eb1ed065.html&_LANG=enus) for more information.

The workflow will run when the specified Agreement Desk event occurs with the conditions you selected. Your workflow steps automate key actions, streamlining and simplifying the request process.
