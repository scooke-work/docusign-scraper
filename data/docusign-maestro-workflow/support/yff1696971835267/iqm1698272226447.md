# Review and Publish a Workflow

As a process builder, publishing a workflow validates your steps are properly configured and makes the workflow available to your account members. Once published, preparers can start instances of your workflow.

Publishing your workflow validates that the steps are properly configured and makes the workflow available to members on your account to use. As a process builder, you can publish your workflow once you have it fully configured.

Publishing includes these actions:

Check for errors
:   Resolve any reported configuration issues before you can publish the workflow.

Update the workflow name
:   Optional. You can edit the workflow name.

Name instances
:   Optional. By default, instances will be named "[start date]\_[timestamp]\_[instance ID]". You can specify a unique and more easily identifiable name with workflow variables.
:   The default instance name is used in the Workflow Instances list until the instance completes successfully. In progress, canceled, or failed instances retain the default system name.
:   Once an instance is completed, the instance name is updated to use the variable data. See [General Limitations and Considerations for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=otm1696972807664.html&_LANG=enus "Review important account limits, supported regions, and functional considerations before building automated processes in Workflow Builder.") for the supported data types.
:   Note: If your workflow uses an API start method, the API payload can pass the `instanceName` value. This value will take precedence over the instance name definition configured in the publishing step as described in this topic.

    Note: If your workflow uses the Manually in Docusign start method, the preparer can override the instance name specified here. When the preparer starts the workflow, they get an option to enter an instance name. If they do not enter a value, then the naming you specify here will be used.

Authorize applications
:   Authorize applications to use your account information for the workflow.

Path End steps added
:   Automatic. When you successfully publish a workflow, a Path End step is added to the last step on all paths of your workflow. The Path End utility step has no effect on workflow function, its purpose is merely to help you structure your workflow design.

Important: There is a limit to how many published workflows you can have in your account. See [General Limitations and Considerations for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=otm1696972807664.html&_LANG=enus "Review important account limits, supported regions, and functional considerations before building automated processes in Workflow Builder.") for details.

1. Log in to your Docusign account and select Agreements >  Workflows.

   The Workflows list shows all workflows that you have access to.
2. Locate the workflow that you want to publish. Filter by Status: Unpublished and Created by: Me.
3. Select the workflow more actions menu and choose Publish  to start the publishing process.

   The workflow opens to the Review before publishing panel.
4. (Optional) Review the workflow name and edit the entry as desired.
5. If the panel shows errors, you must address the configuration errors before you can publish. Address all errors and select Publish to restart the publishing process.![](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/publishingerrors.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4NzUsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.1KAM_dZGWEQ4BjF7gVm6_ASLuBtqCWOI0cb8aaLbPHE&_LANG=enus)
6. If the panel shows that the workflow is ready to publish, select Next.

   Note:

   If your workflow includes a Verify Someone's Identity step but lacks a Store Files step, a note prompts you to create one. Adding a Store Files step after the Verify Someone's Identity step ensures that ID Evidence sends the identity verification data in a proof file into an archive folder. Read the [Configure a Verify Someone's Identity Step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=qht1698126463108.html&_LANG=enus "Learn how to configure a Verify Someone's Identity step in an agreement workflow. You use an identity verification step to verify a participant's identity using Docusign's ID Verification process.") page for more information.
7. To make workflow instances more identifiable, add workflow variables to build unique instance names. Then select Done >  Publish.

   ![Screenshot of the publishing step to name workflow instances.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/nameinstances.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4NzUsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.1KAM_dZGWEQ4BjF7gVm6_ASLuBtqCWOI0cb8aaLbPHE&_LANG=enus "Screenshot of the publishing step to name workflow instances.")
8. Depending on your workflow steps, you may be required to authorize applications. Select Authorize My Account >  Publish.

   ![Screenshot of the authorization window to publish a workflow.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/authorizeaccountworkflow.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4NzUsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.1KAM_dZGWEQ4BjF7gVm6_ASLuBtqCWOI0cb8aaLbPHE&_LANG=enus "Screenshot of the authorization window to publish a workflow.")

   Your workflow status updates to "Publishing". Refresh your browser after a few minutes to see the final status "Published". Once published, the workflow is ready for your process preparers to use.
