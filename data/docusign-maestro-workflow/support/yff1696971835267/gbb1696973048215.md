# Workflow Builder Roles and Permission Requirements

Read about the different roles involved with creating and using workflows with Docusign Workflow Builder. Each role requires specific user
permissions.

Using Workflow Builder to create workflows and automate your agreement processes involves
different roles and responsibilities for your organization. For you to participate in
creating or using workflows, you must have the appropriate user permissions assigned to
you by your account administrator. Administrators can find information on assigning
permissions in the [Permission Profiles](https://support.docusign.com/s/document-item?bundleId=pik1583277475390&topicId=pof1583277362435.html&_LANG=enus) article.

Docusign administrator
:   The administrator for a Docusign IAM account authorizes
    applications for Workflow Builder and enables users to create and run workflows by
    granting required user permissions. Account administrators are granted full
    workflow permissions by default. Minimum required permissions are listed for
    each user role.

Workflow administrator
:   The workflow administrator is a Docusign user who can view, edit, publish, trigger, export, and audit all workflows across the account, regardless of who originally created them. This role is designed for users who need centralized oversight and management of workflows without being the original workflow creator.

    All Docusign administrators inherit the Manage permission by default. Administrators can also assign this permission to other users or groups. With this role, non-administrators can manage all workflows across the account like a Docusign administrator, but without granting them full administrative privileges.
:   Required user permissions:

    - Workflows: Manage
    - Allow user to send envelopes: enabled
    - Web Forms: Create
    - Templates: Create

    Note: If a workflow step uses a extension app with a third-party connection that is private and the workflow administrator is not the connection owner, they cannot edit the step. In this case, the workflow administrator can still run the workflow or delete the step, but cannot view the connector configuration, modify the step, or change the connection. Process builder
:   The process builder is a Docusign user who creates, edits, and
    publishes their own workflows. They can also use any published workflow to start
    an agreement process. The builder role requires access to the Docusign applications and supporting features they will use to
    build workflow steps. They will authorize these applications to use their
    account to run workflow steps.
:   Required user permissions:

    - Workflows: Create
    - Allow user to send envelopes: enabled
    - Web Forms: Create
    - Templates: Create

Preparer
:   The preparer runs published workflows to start the agreement process and
    monitors the progress of running instances. The preparer has access to use any
    published workflow on the account to start an agreement process. The preparer
    needs only limited permissions as the agreement process gets required
    authorization and permissions from the process builder's account.
:   Required user permissions:

    - Workflows: Prepare

    Note: See the topic [Configure a DS Sender Profile for Workflow Builder Preparer Permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=tsk1906294d-5fb2-4f24-8764-7b59448b20f3.html&_LANG=enus "Learn how to configure a custom Docusign DS Sender permission profile to grant users Workflow Builder preparer access.") for how to add a user profile for the Preparer role.

Participant
:   Participants represent the end users who will participate in an agreement
    workflow. Participants do not need a Docusign account to receive and sign documents. However, if they do have an account, any envelopes generated in a workflow for their user credentials will register in that account.
:   Required user permissions: N/A – no Docusign account required.
