# Data Types for Workflow Variables

Look up details about the data types that you can use to pass data to workflow variables.

When initiating a process from an external application or API payload, you must define the correct data type for each starting variable. Mapping precise data types ensures that incoming strings, numbers, or JSON objects parse correctly across subsequent workflow steps.

There are different types of data that you can define for a starting variable. Each of these supports specific input values.

## General character limits

All variables have the following character limits:

- Variable name: 150 characters
- Hard-coded variable (fixed value entered in step configuration):
  - name: 150 characters
  - value: 4000 characters
- Input limit at workflow start: 4000 characters

## Supported data types and input specifications

Text
:   Supports alphanumeric and special characters. Text variables are suitable for a variety of strings, such as names, addresses, and other details with separators, such as dashes or currency symbols.

Email
:   Supports valid email formats. Example: marcus.chen@mail.com

Phone Number
:   Requires a country code and phone number when manually entering a fixed value during workflow step configuration. Extension is optional. If sending values by API from an external system, adhere to the following JSON structure:

    ```
    |  |  |
    | --- | --- |
    |  | { |
    |  | countryCode:'1', |
    |  | number: '1234567890', |
    |  | extension: '1234', |
    |  | } |
    ```

Date
:   When manually entering a fixed date during workflow step configuration, the required formats are locale-specific (defined by your account's regional Admin settings). If triggering the process from an external API payload, inputs must adhere to ISO 8601 standard patterns:

    - YYYY-MM-DD (defaults to midnight UTC)
    - YYYY-MM-DDTHH:mm:ssZ
    - YYYY-MM-DDTHH:mm:ss±HH:mm
    - YYYY-MM-DDTHH:mm:ss.SZ
    - YYYY-MM-DDTHH:mm:ss.SSZ
    - YYYY-MM-DDTHH:mm:ss.SSSZ
    - YYYY-MM-DDTHH:mm:ss.S±HH:mm
    - YYYY-MM-DDTHH:mm:ss.SS±HH:mm
    - YYYY-MM-DDTHH:mm:ss.SSS±HH:m

Currency
:   Supports a numeric amount and a corresponding ISO currency code. Use this for storing and reusing monetary values (for example, `1000.00 USD`).

    If sending values by API from an external system, adhere to the following format:

    ```
    |  |  |
    | --- | --- |
    |  | { |
    |  | currencyCode: "USD", |
    |  | doubleValue: 1234.56 |
    |  | } |
    ```

Numbers
:   Supports numbers up to 16 digits. Values can include localized group and decimal separators in addition to the digits. Examples: 1,234.5 in US English. 1 234,5 in French.

Object
:   Supports any valid JSON object (key-value pairs surrounded by curly braces and separated by commas). Use this data type to pull in complex, grouped data packets from external systems to distribute to downstream workflow steps.

    Note: This data type is commonly used to handle structured data outputs and custom header mappings when configuring a [Make a Web Request](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=task-8583.html&_LANG=enus "Use a Make a Web Request step to call an external or internal service from Workflow Builder to look up data or push an update without a dedicated app.") step.

Array
:   Supports any valid JSON array (ordered elements enclosed in square brackets). Use this data type when a workflow step needs to process a list of repeating items sequentially.

    Note: When using a [Make a Web Request](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=task-8583.html&_LANG=enus "Use a Make a Web Request step to call an external or internal service from Workflow Builder to look up data or push an update without a dedicated app.") step, you can use arrays to pass lists of repeating items to an external API or to parse multi-item lists returned by an external service.
