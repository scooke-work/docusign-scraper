# Configure an Add Parallel Branches Step

Use the Add Parallel Branches step to split a single workflow path into multiple branches that run at the same time.

The **Add Parallel Branches** step creates a point in your workflow where the path splits into multiple branches that run concurrently. Use this step when you want approvals, reviews, or other tasks to happen in parallel instead of one after another.

Parallel branches provide the following support:

- Create up to 15 branches from a single point in the workflow.
- Define distinct paths for different teams or tasks, such as Legal, Procurement, and Finance.
- Reduce overall cycle time by allowing independent work to proceed in parallel.

Tip: Use **Add Parallel Branches** when multiple paths should run at the same time. Use branching rule steps when you want a single path to be chosen based on specific conditions.

**Limitations for Parallel Branches**

- **Branch limit:** You can create a maximum of 15 branches within a single parallel step.
- **Plan availability:** This feature is available only for IAM Professional and above plans (including CX, Sales, Professional, and Enterprise).
- **Step limit:** You can add a maximum of 2 Add Parallel Branches steps per workflow.
- **Participant limit:** Only one participant can participate in a single branch of a parallel step.
- **Agreement Desk restrictions:** If the workflow starts with an Agreement Desk step followed by a parallel step, you cannot use the Change Status, Set Due Date, and Assign Owner steps in multiple parallel branches.
- **Retry restriction:** You cannot retry any step within a workflow if that workflow contains an Add Parallel Branches step.

**Before you begin**

- You have already created the workflow, as described in the basic workflow creation task.
- You have identified where the workflow should begin running steps in parallel. For example, after data collection or an initial document preparation step.

1. Open your workflow in edit mode and go to the workflow canvas.

   Locate the step after which the flow should split into parallel paths.
2. Add the Add Parallel Branches step.

   Select Add a step at the desired point in the workflow, choose Add Parallel Branches, and then select Edit.
3. Add additional branches as needed.

   Use the Add Branch control to create more parallel paths, up to a maximum of 15 branches for this step.
4. Apply the parallel branch configuration.

   When you finish adding branches, select Apply to save the configuration for the **Add Parallel Branches** step.
5. Configure the steps in each branch.

   For every branch, add the workflow steps you want to run in that path, such as sending documents for signature or calling third-party apps.
