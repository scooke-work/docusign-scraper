# Understand Manual and Automatic Sending and Sender Options

Learn about manual and automatic sending options for envelopes in your workflows, including how to choose who appears as the sender.

With the Send Documents for Signature step in certain workflows, process builders can specify two important features around how the envelope is sent. These settings determine **how the envelope is dispatched** and **whose name and email the recipient sees as the sender**.

Important: These sending options apply only to workflows that use either the Manually in Docusign or an API Call start method.

## Sending option details

For workflows that use either the Manually in Docusign or an API start method, the process builder chooses how to send the eSignature envelope:

Automatically
:   When the workflow reaches the Send Documents for Signature step, the envelope is sent automatically.

    By default, the sender is the person who last published the workflow. However, the process builder can specify that the person who starts the workflow is the sender. This provides flexibility for who is responsible for sending the document.

Manually
:   When the workflow reaches the Send Documents for Signature step, the envelope goes through a required review and send step. The person who starts the workflow can review and edit all envelope details and the envelope is sent from the user's Docusign account.

## Key benefits and use cases

Using these sending options provides significant advantages in personalization and error prevention:

| Benefit | Description |
| --- | --- |
| Trusted sender experience | The envelope arrives from a known contact (the initiating user), rather than an automated workflow system. This personalization is vital for closing deals faster, adding assurance that the document came from a trusted source. |
| Final quality check | Manual sending provides an opportunity for a crucial, hands-on checkpoint to ensure all recipient details and any custom fields are accurate immediately before the document is sent for signature. |
| User familiarity | This experience makes the Send Document for Signature step align more closely with the standard Docusign eSignature process that customers are already familiar with. |
