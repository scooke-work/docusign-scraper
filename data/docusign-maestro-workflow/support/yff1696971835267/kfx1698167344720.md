# Configure a Set Up Invite Step

Learn how to use a Set Up Invite step in your agreement workflow
to invite participants to join the workflow.

You use the Set Up Invite
step type to invite a participant to your workflow. This step can generate invitations
to the participant through email, SMS text message, or WhatsApp. The invitation includes a link to enter the workflow, allowing the participant to engage in a subsequent workflow step. The link is valid for one year and cannot be resent.

Note: Email invitations are issued from the sender "Docusign Workflows" at "workflows@docusign.net". Docusign applies your account's default signing brand to all notification steps—including Set up Invite, Send an Email, and Send Documents for Signature—as well as the signing session itself. If no default brand is defined, standard Docusign branding is used.

For all
invited participants, you must provide values as required for name, email address, and phone number. You can map workflow variables for the required participant information, or hardcode values. Once
defined, these variables are set at the workflow level and will be referenced when
mapping the same participant in any subsequent step.

Note: Account settings must allow for SMS or WhatsApp delivery to use these delivery methods. Your
account administrator manages these settings in Admin > Sending Settings > Fields and Properties To use both SMS text message and email delivery methods
together, your account administrator must also ensure the Allow SMS
delivery to recipients user permission is enabled for your user
profile. If you are unable to follow the steps in this topic, contact your
account administrator to enable these options. Read more about these methods in
the topic [Multi-Channel
Delivery for Notifications: SMS and WhatsApp](https://support.docusign.com/s/document-item?bundleId=gav1643676262430&topicId=kwk1607393003979.html&_LANG=enus).

Table 1. Workflow start methods and Set Up Invite step requirements

| Start Method | First Participant | Additional Participants |
| --- | --- | --- |
| From a Link | Enters workflow automatically | Needs an invitation step to enter the workflow |
| Manually in Docusign | Enters workflow automatically | Needs an invitation step to enter the workflow |
| From an API Call - Human | Enters workflow automatically | Needs an invitation step to enter the workflow |
| From an API Call - Automated process | Needs an invitation step | Needs an invitation step to enter the workflow |
| From Docusign CLM Workflow | Needs an invitation step | Needs an invitation step to enter the workflow |

1. Create a workflow.
2. Select Add a step >  Set Up Invite >  Configure. Select a participant and choose the desired delivery channel.
   This is the participant who will receive the invitation to your workflow.
3. (Optional) For email delivery, you can select the option Include
   custom message and input your own email message to the
   participant.
4. Select
   Apply.
5. Complete any missing participant information. Select the variables to use for
   the name, email address, and phone number from any prior workflow step, or enter fixed values manually.

   ![Screenshot of the configuration window for a Set Up Invite step.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/set_up_invite_config.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU4MjIsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.usR7EH4pph44QfBpnyF3nRhh_0SeIUfirKhPTudHUzE&_LANG=enus)

   The participant's information typically comes from any prior steps, such as
   a Collect Data with Web Forms or Collect Data
   From an ID step. For CLM and API workflow starts, these may be
   incoming variables from an external system.
6. Select Apply.

Your Set Up Invite step is configured. When the workflow reaches this step, an invitation is sent to the participant to join the workflow. The
invitation uses the selected delivery channel (for example, email or SMS text message).
The link in the invitation is valid for one year and cannot be resent. For email delivery, if you specified a custom message, it is included in the
communication.
