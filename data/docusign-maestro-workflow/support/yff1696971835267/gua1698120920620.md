# Configure a Collect Data with Web Forms Step

Learn how to configure a Collect Data with Web Forms step in an agreement workflow. You use a web forms step to gather information from participants. You can use any existing stand-alone web form on your account that you have access to for your step.

You use a Collect Data with Web Forms step to gather information for your business process. In subsequent workflow steps, you can use any information and documents gathered from your web form to pre-fill document fields, verify identity, and populate other data points.

Consider the following scenarios when you add a web form step to your workflow:

A user within your organization starts the workflow. Start method: Manually in Docusign
:   The preparer knows the identity of a subsequent participant. When they start the workflow, they complete the next participant's information. For this case, add a Collect Data with Web Forms step as the first step of the workflow. Assign it to the preparer participant who will launch the process manually in Docusign. When they start the workflow, they are directed to the web form to provide participant information, and then to any other assigned steps. Data gathered in the web form is available to map to fields in subsequent steps.

An anonymous participant initiates the workflow from a link. Start method: From a Link
:   The participant uses a published link to your workflow. In this self-service scenario (similar to a PowerForm), the workflow starts with a web form step. The step is not mapped to any data. Data gathered in the web form is available to map to fields in subsequent steps.

You can use any existing standalone web form on your account that you have access to. A standalone web form is not tied to a template but exists on its own. Use a standalone web form to gather data and supporting documents from participants for an agreement process. For more information about standalone web forms, see [Web Form Requirements and Data-Mapping Information](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=oha1716325069710.html&_LANG=enus "Review the configuration rules, standalone formatting options, and precise field-matching requirements needed to map data between web forms and agreement templates.").

Before you begin
:   - You must be an account administrator or have Workflows: Create permissions.
    - You must have access to an existing stand-alone web form to use for your workflow step.
    - Your stand-alone web form must be active.

1. Create a new workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus "Learn the basics of how to create an agreement workflow. Process creators and account administrators can create workflows to automate the agreement process.").
2. Select Add a step >  Collect Data with Web Forms.

   The configurator panel for your step opens.
3. Select the form to use and then select Next.

   Note: In the form selection box, at the end of the list of web forms, there is an option Create New Web Form. Use this option if you don't have an existing web form you want to use.

   ![Screenshot of the Web Forms configurator.](https://docusign-be-prod.zoominsoftware.io/api/bundle/yff1696971835267/page/images/webformsconfig.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MDU3MjIsInNoZWFmIjoieWZmMTY5Njk3MTgzNTI2NyJ9.YBfDeJtkT1vtF_a9V-qyHREWUlKHIXzJSphSlFCmZDc&_LANG=enus "Screenshot of the Web Forms configurator.")
4. Specify the participant for this step.

   Any pre-existing participants you configured are listed in the dropdown list. To add a new participant, select Add Participant. Depending on your workflow start method, you may also be able to select from these dynamic roles:
   - Workflow Initiator
   - User who last published the workflow
   - User who starts the workflow
5. (Optional) You can map variables to data fields in your web form. This pre-fills web form fields for your participant.
   1. Select Continue to map data fields.
   2. For any web form field, select the variable dropdown. Then select an appropriate variable to map to the web form field.

      Tip: Use the Map for Me option to generate recommended field mappings. Matches are based on field names that are the same or similar. (For example, the name of a starting variable that closely matches a field name on a form.) Review and apply the recommendations or undo the action to try again or use manual mapping.
6. Select Apply.

Your Collect Data with Web Forms step is configured with a web form and a participant. If you mapped data fields, the web form fields will be pre-filled with the relevant data when the participant accesses the web form.
