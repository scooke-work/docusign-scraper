# Success Rules for ID Verification

Read about how you can use branching rules to check the results of an identity
verification step.

If your workflow contains a Verify Someone's Identity step, you should include a true/false branching rule
to check whether the participant passed the identity verification. If they fail the
verification, you can use a branching rule to control what happens in the workflow.

If a participant fails the identity check, the workflow will still move to the next step.
To control the workflow progress, add a branching rule after the Verify Someone's Identity step to
check if they successfully validated their identity.

## Configure the branching rule

Add a branching rule after the Verify Someone's Identity step. Configure the rule as
follows:

- For what to check, select the identity step and the variable
  `Validated`.
- For the operator, select `Is`.
- For the result, select `True`.

## Add steps to the rule outcomes

Depending on how you want your workflow to proceed after an identity step, add steps
to the True and False outcomes of the
rule.

- Continue your workflow steps on the True branch of the
  rule. These additional steps define the workflow process after the participant
  successfully passes the identity check.
- Add a confirmation step to the False branch. Configure it
  to display custom message explaining that the participant did not pass the
  identity verification.
