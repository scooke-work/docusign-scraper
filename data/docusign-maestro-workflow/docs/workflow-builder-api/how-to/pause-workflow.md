# How to pause the creation of workflow instances

This topic demonstrates how to pause the creation of new workflow instances for a [Workflow Builder workflow](https://developers.docusign.com/docs/workflow-builder-api/workflow-builder101/workflows/).

**Important:** This how-to requires an active workflow. Run the [How to trigger a Workflow Builder workflow](https://developers.docusign.com/docs/workflow-builder-api/how-to/trigger-workflow/) example from our launcher projects to create, publish, and trigger a workflow instance before running this example.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PowerShell](https://github.com/docusign/code-examples-powershell).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at   https://<domain>  Where <domain> is https://api-d.docusign.com/v1 for the developer environment and https://api.docusign.com/v1 for production. |
| {WORKFLOW\_ID} | A GUID value that identifies your Workflow Builder workflow. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario. See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

Workflow Builder OAuth access tokens are the same ones used for our [eSignature REST API](https://developers.docusign.com/docs/esign-rest-api/esign101/auth/), although you must request one or more Workflow Builder scopes to call the Workflow Builder API. The endpoint calls made in this how-to require that you request the Workflow Builder-specific `aow_manage` scope when obtaining your access token.

See [Workflow Builder Authentication](https://developers.docusign.com/docs/workflow-builder-api/auth/) for details and the full list of Workflow Builder scopes.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid OAuth access token for `{ACCESS_TOKEN}`.

1

2

3

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}" \

'--header' "Accept: application/json" \

'--header' "Content-Type: application/json")

## Step 3. Call the Workflow Builder API

Now that you have constructed your API headers, make a `POST` request to the [Workflows:pauseNewWorkflowInstances](https://developers.docusign.com/docs/workflow-builder-api/reference/workflowbuilder/workflows/pausenewworkflowinstances/) endpoint. Here’s how you make the call, but make sure to substitute the base path for `{BASE_PATH}`, your API account ID for `{ACCOUNT_ID}` and the workflow ID for `{WORKFLOW_ID}`.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/Maestro/eg002PauseWorkflow.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/Maestro/Examples/PauseWorkflow.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/maestro/examples/Mae002PauseWorkflowController.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/maestro/examples/pauseWorkflow.js), or [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/Maestro/eg002PauseWorkflow.ps1).

response=$(mktemp /tmp/response-wftmp.XXXXXX)

Status=$(

curl -w '%{http\_code}' --request POST "$

{base\_path}/accounts/${ACCOUNT\_ID}/workflows/$

{workflow\_id}/actions/pause" \

"${Headers[@]}" \

--output ${response}

)

# If the status code returned is greater than 201

(OK / Accepted), display an error message with the

API response.

if [[ "$Status" -gt "201" ]]; then

echo ""

echo "Unable to pause workflow: ${workflow\_id}"

echo ""

cat $response

exit 0

fi

echo ""
