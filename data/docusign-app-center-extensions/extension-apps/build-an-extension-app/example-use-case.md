# Example extension app use cases

Explore some example extension app use cases and learn how extension apps can help streamline your agreement processes.

## Implementing use cases

For any of these use cases, create the appropriate extension app by specifying the corresponding extension in the `extension` section of the app’s manifest file. An extension is a group of actions that support a type of external functionality. See [Extensions and extension points](https://developers.docusign.com/extension-apps/extension-apps-101/concepts/extensions-and-extension-points/) for details. In an extension app's manifest file, the [connection object](https://developers.docusign.com/extension-apps/extension-app-reference/app-manifest-reference/connection/) defines the necessary parameters to access APIs of an external platform. Additionally, [actions](https://developers.docusign.com/extension-apps/extension-apps-101/concepts/actions/) and [capabilities](https://developers.docusign.com/extension-apps/extension-apps-101/concepts/capabilities/) represent the specific functions performed within your extension app, such as verifying an email address. These actions define the endpoints of the external platform that are called during your Docusign workflows.

To start developing an extension app based on one of these examples, you’ll need an extension app manifest. We’ve provided a sample manifest for each use case. Feel free to upload this manifest to the [Developer Console](https://devconsole.docusign.com) immediately to experience the ease of creating an extension app.

**Note:** Remember to replace the sample manifest elements with your own. View the [App manifest reference](https://developers.docusign.com/extension-apps/extension-app-reference/app-manifest-reference/) for a complete list of the elements in the manifest and their descriptions.
