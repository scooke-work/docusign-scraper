# File output system of record test: An event triggers the workflow (beta)

This procedure explains how to set up and execute a [functional test](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/) that invokes a [file output system of record extension](https://developers.docusign.com/extension-apps/extension-apps-101/supported-extensions/file-output-system-of-record/) from a [workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=dnx1696972415150.html). For this test, you configure and run a workflow whose instances start automatically based on an envelope event that occurs in the Docusign account.

The workflow stores files associated with the envelope that triggered the workflow instance. To test a scenario in which files to be exported are generated by a workflow's [Send Documents for Signature step](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html), follow the procedure in [File output system of record test: Workflow step creates envelopes](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-workflow-step-creates-envelopes/). See [Identify the file source](https://developers.docusign.com/extension-apps/extension-apps-101/supported-extensions/file-output-system-of-record/identify-file-source/) for information about the two options.

The high-level steps to create and run this test are:

- [Step 1. Identify or create an eSignature template](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-event-triggers-workflow/#step-1-identify-or-create-an-esignature-template)
- [Step 2. Create and publish the workflow](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-event-triggers-workflow/#step-2-create-and-publish-the-workflow)
- [Step 3. Trigger a workflow instance](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-event-triggers-workflow/#step-3-trigger-a-workflow-instance)

This is one of many possible approaches to setting up a workflow. For additional options, see the [workflow user documentation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=dnx1696972415150.html).

Make sure that you’ve completed all the requirements listed for functional tests in [Test prerequisites](https://developers.docusign.com/extension-apps/build-an-extension-app/test/#test-prerequisites).

## Step 1. Identify or create an eSignature template

Before you create the workflow, make sure that you have an eSignature [template](https://support.docusign.com/s/document-item?bundleId=xry1643227563338&topicId=dqj1578456412286.html&rsc_301) in your account that you can use to send envelopes. You can [use an existing eSignature template](https://support.docusign.com/s/document-item?bundleId=xry1643227563338&topicId=epp1578456404448.html) in your account, or [create a new template](https://support.docusign.com/s/document-item?bundleId=xry1643227563338&topicId=lsg1578456368117.html).

When selecting or creating a template:

- For a simpler workflow setup, the template should have a single recipient instead of multiple recipients.
- The recipient type must be **Needs to Sign**.
- Do not hard-code a recipient name or email in the template.
- Make sure that the template document includes, at minimum, a Signature tab.

## Step 2. Create and publish the workflow

To create a workflow that tests a file output system of record extension:

1. Log in to the [Workflow Designer](https://apps-d.docusign.com/send/workflows/).
2. Follow the steps in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html)﻿ to create and save a workflow. When creating a workflow, select **Blank Workflow**. Your workflow should include:
   - The workflow [start method](https://support.docusign.com/s/document-item?language=en_US&bundleId=yff1696971835267&topicId=ztb1727892686033.html) **From an event**. See [Workflow start method](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-event-triggers-workflow/#workflow-start-method) for configuration guidelines.
   - A step that invokes your extension app to write the signed document to the system of record. See [File output system of record extension step](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-event-triggers-workflow/#file-output-system-of-record-extension-step) for configuration guidelines.

     When youʼve finished configuring the start method and the step, the workflow should appear as shown.
3. Complete the steps in [Review and Publish a Workflow](https://support.docusign.com/s/document-item?language=en_US&bundleId=yff1696971835267&topicId=iqm1698272226447.html)﻿. Then you can [trigger instances of the workflow](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-event-triggers-workflow/#step-3-trigger-a-workflow-instance) to test your extension app.

### Workflow start method

When you define the workflow [start method](https://support.docusign.com/s/document-item?language=en_US&bundleId=yff1696971835267&topicId=ztb1727892686033.html), configure the settings as follows:

| Setting | Value |
| --- | --- |
| What starts this workflow? | Select the start method **From an Event**. |
| What event should happen? | Select the **Envelope Completed** event. |
| Set conditions for the selected event | Under **Add conditional fields for selected event?**, select **Yes**, and set these values:   - **What will this condition check?**: Template - **What is specific about this condition?**: Id - **What operator will this condition use?**: Is - **What value will this condition look for?**: Search for and select the template you created or identified for use with this test. |

### File output system of record extension step

This step will invoke your extension app.

To add this step, display the **Apps** tab on the **Add New Step** panel and select the extension thatʼs defined in your extension app registration. The **Add New Step** panel displays the values from the [extensions.name](https://developers.docusign.com/extension-apps/extension-app-reference/app-manifest-reference/extension/) and [extensions.description](https://developers.docusign.com/extension-apps/extension-app-reference/app-manifest-reference/extension/) properties in the app manifest.

![](data:image/svg+xml;charset=utf-8,%3Csvg height='248.00000000000003' width='400' xmlns='http://www.w3.org/2000/svg' version='1.1'%3E%3C/svg%3E)

![Add file output system of record step](https://images.ctfassets.net/aj9z008chlq0/5MloqkCCUwHruFQuwtMQ62/7ef7ae7439ea6f2cce911e2c9f1c2d46/WorkflowConfig_AddNewStep_smallnew.png?w=400&h=248&q=50&fm=png)

**Note:** The step for your extension will not appear in the **Connected Apps** list if the extension app is not [installed and connected](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ctg1698170340729.html) to your account.

When you configure this step, the [Get Type Names](https://developers.docusign.com/extension-apps/extension-app-reference/extension-contracts/file-output-system-of-record/#get-type-names) capability will be invoked to retrieve a list of objects in the system of record. When you execute an instance of the workflow, this step invokes the [Write File](https://developers.docusign.com/extension-apps/extension-app-reference/extension-contracts/file-output-system-of-record/#write-file) action.

When you add this step, configure the settings as follows:

| Setting | Value |
| --- | --- |
| What are you storing? | Select **Envelope Info** under **Variables from Workflow Start**.Under **How do you want to configure it?**, select one of these file formats:   - **Combine all documents into a single PDF**: All envelope files will be sent as a single Base64-encoded string in the [Write File](https://developers.docusign.com/extension-apps/extension-app-reference/extension-contracts/file-output-system-of-record/#write-file) request, and the request's `files` array will contain a single element. - **Store all documents individually**: Each file will be sent as a separate Base-64 encoded string, and each will be in a separate element in the `files` array. - **Combine all documents into a ZIP file**: All envelope files will be sent as a single Base64-encoded string, and the request's `files` array will contain a single element.   You can also select the option to include the [certificate of completion](https://support.docusign.com/s/document-item?bundleId=oeq1643226594604&topicId=gpa1578456339545.html) with the envelope files. |
| Where are you storing it? | **Select connection**: Select the default connection.  **Select record**: Select a workflow variable whose runtime value should be sent with the **Write File** request. This should be a value that the external system can use to uniquely identify the record to write the file to. The **Variables from Workflow Start** section in the dropdown includes envelope fields from the template you selected when defining the workflow start method. See [Workflow variables](https://developers.docusign.com/extension-apps/build-an-extension-app/test/functional-tests/file-output-system-of-record-event-triggers-workflow/#workflow-variables) for information about the variables that you can select.  **Object**: Select the system of record object to write files to. The list is populated with objects returned by the **Get Type Names** request to the external system. |
| What will you call it? | Define a naming convention for the file. The naming convention can include hardcoded text, workflow variables, or a combination of the two. |

#### Workflow variables

When defining the **File Output** step, you can include workflow variables for the record identifier and file naming convention. Variables will be populated with values at workflow instance run time.

The types of variables available for selection include:

- **Variables from workflow start.** This includes envelope and template fields associated with the account.
- **Workflow instance variables.** These are values associated with the workflow instance, such as the instance ID and the start date and time.

## Step 3. Trigger a workflow instance

After you've published your test workflow, you can send an envelope from your Docusign account and complete it to trigger an instance of the workflow.

To trigger a workflow instance:

1. Follow the procedure in [Use a Template to Create an Envelope](https://support.docusign.com/s/document-item?bundleId=xry1643227563338&topicId=nxv1578456289719.html)﻿ to generate an envelope from the template you created or identified for use in this test. When completing this procedure, supply an address where you can receive email.
2. When you receive the email, access the envelope via the link, and complete the signing process. See [Sign a document sent to you via email](https://support.docusign.com/s/articles/How-do-I-sign-a-DocuSign-document-Basic-Signing?language=en_US#Email) for detailed steps. On envelope completion, the workflow instance will be triggered, and the completed envelope files will be stored to your system of record.
3. Check the object and record to which the external service stores files, and confirm that the **File Output** step succeeded.

If your extension is not invoked successfully, run the Developer Console [connection and extension tests](https://developers.docusign.com/extension-apps/build-an-extension-app/test/integration-tests/) for the extension app to get detailed error messages and troubleshoot.
