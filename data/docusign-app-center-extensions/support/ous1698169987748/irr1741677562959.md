# Configure a Read From Dynamics 365 Step in Agreement Workflows﻿

Learn to add and configure a Read from Dynamics 365 step to import data from Dynamics
365 records.

Before you begin
:   - Required: Installation of the Dynamics 365 app
      from Docusign App Center. Review the [Installing Extension Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more information.
    - Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account administrator.
    - Required: Access to a suitable template to use for your workflow
      step. Review the [Template Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=xnw1709852160602.html&_LANG=enus) topic for more information.
    - Recommended: Open the Docusign template in another
      window to reference as you set up your step. This will help you
      correctly configure your Read from Dynamics 365 step.

You can use a Read from Dynamics 365 step to import data from
Dynamics 365 records to your Docusign workflow. Administrators
link workflow fields to Dynamics 365 fields in this step . When task
recipients open an envelope, they see Dynamics 365 data imported from the record.
Importing Dynamics 365 data improves efficiency and reduces errors by eliminating
manual data entry.

To learn about installing, configuring, and integrating Dynamics 365 with agreement
workflows, see the [Use Docusign
Workflow Builder with Microsoft Dynamics 365](https://support.docusign.com/s/document-item?bundleId=pic1741250291019&topicId=lkn1741250360427.html&_LANG=enus) article.

Example: You build a workflow with these steps:
Collect Data with Web Forms, Read from Dynamics
365, and Writeback to Dynamics 365 step. In the
writeback step, you link the Dynamics 365 opportunity Budget
Amount field to the web form's Budget Amount
field. A workflow recipient opens a contract and sees the opportunity's Budget
Amount field pre-filled. The recipient changes the budget amount and Docusign updates the Budget Amount field on the Dynamics 365
record.

Note: To read data from child or related records, first create an
Account-based step.  [Next, create
Read from Dynamics 365 steps based on child objects](https://support.docusign.com/s/document-item?bundleId=pic1741250291019&topicId=ahc1741673652580.html&_LANG=enus) .

Use the following procedure to create a Read from Dynamics 365
step in a Docusign workflow:

1. Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add steps like [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus), [web
   forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and more to gather recipients, documents, and merge fields for your workflow.
2. Select Add a step  >  Read from Dynamics 365  >  Configure.
3. Step Name: Keep or edit the name. It is helpful to add
   the name of the Dynamics 365 object, like Account - Read from
   Dynamics 365.
4. Where will you read from?
   1. Connection: Choose a Dynamics 365 organization
      you connected to Workflow Builder.
   2. Dynamics 365 object: Use the search field or
      dropdown menu to select an object.
   3. Select Next.
5. Which fields would you like to read?: Select workflow
   fields to map to Dynamics 365 fields. When workflow recipients open contracts,
   they see data Docusign imported from a record.
   1. Select Add or Remove Fields.
   2. Select Dynamics 365 fields that you'd like to use as workflow
      variables. You'll be able to use them in later steps.
   3. Select Save and Exit.

   The Select Fields dialog box closes. The fields you selected
   display.
6. Which record are you reading from?: Create up to five
   rules to read from Dynamics 365 records:
   1. Select Add rule.
   2. IF:

      - Any Rules: Select to choose a Dynamics
        365 record matching any rules.
      - All Rules: Only choose a record that
        matches all rules.
   3. Dynamics 365 field: Select a Dynamics 365 field
      to match with a Docusign workflow field.
   4. Workflow field: Select a Docusign workflow field to match with a Dynamics 365
      field.

      - Starting Variables: Find custom variables
        created outside of the workflow steps.
      - WORKFLOW STEPS: Find variables from other
        workflow steps.
   5. Select Save and Exit.

      ![A screen capture demonstrates adding a rule to a Read from Dynamics 365 workflow step.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/configure_a_read_step.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MDIsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.Tb9QiKlsWWWdQ3z5xiaGQpcqoxXvKB6JXuDvekF3oQI&_LANG=enus "A screen capture demonstrates adding a rule to a Read from Dynamics 365 workflow step.")
   6. Select Next.
7. Select Apply.

   The Read From Dynamics 365 column closes. The workflow page
   displays your new step.

You have completed the process. You have built a Read from
Dynamics 365 step in your workflow. When this step is triggered in a workflow,
it imports field data from the Dynamics 365 records that you mapped.
