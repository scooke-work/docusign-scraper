# Configure a Store Files in Dynamics 365 Step in an Agreement Workflow

Learn how to configure a Store Files in Dynamics 365 step in an Agreement Workflow.

## Before you begin

- Required: Installation of the **Dynamics 365** app from Docusign App Center. Review the [Installing Extension Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more information.
- Required: [Workflows Roles and Permission Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account administrator.
- Required: Access to a suitable template to use for your workflow step. Review the [Template Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=xnw1709852160602.html&_LANG=enus) topic for more information.
- Recommended: Open the template in another window to view as you set up your step. This helps you to correctly configure your Store Files in Dynamics 365 step.

The **Store Files in Dynamics 365** step adds signed agreements and certificates of completion to
Dynamics 365 entities. Administrators choose files to store, a location to store them, and a naming
convention for the files. The **Store Files in Dynamics 365** step decides the name and storage location of agreements sent from workflows.

**For Example**: You build a workflow with a **Store Files in Dynamics 365** step. Your workflow
sends an agreement to three recipients. When all three recipients sign the agreement, Docusign
uses **Store Files in Dynamics 365** variables to add the agreement to the correct record.

**Procedure**

1. Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add steps like [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus), [web forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and more to gather recipients, documents, and merge fields for your workflow.
2. (Optional): Add the Read from Dynamics 365 step to your workflow. See the [Configure a Read From Dynamics 365 Step in Agreement Workflows﻿﻿](https://support.docusign.com/s/document-item?language=en_US&bundleId=pic1741250291019&topicId=irr1741677562959.html) article for details.
3. Select Add a step > Store Files in Dynamics 365 > Configure.
4. Step Name: Keep or edit the name.
5. What are you storing?: Select a file or envelope from an earlier step in the workflow and
   select Next.
6. Where are you storing it?:
   1. Select connection: Choose a Dynamics 365 organization you connected to
      Workflow Builder.
   2. Select record: Select the Microsoft Dynamics 365 record ID (for example,
      Opportunity).
   3. Microsoft Dynamics 365 object: Select the appropriate Microsoft Dynamics 365 object from the dropdown list.

      Note: If you add the Read from Dynamics 365 step and select a record ID from the same step, the Dynamics 365 object is automatically populated. You can edit the Dynamics 365 object, if required.
   4. Select Next.

      ![A screenshot of the selected connection, record, and Dynamics 365 object for the workflow.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/iam-sales-dyn365-3.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE4ODgsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.ndu8AXtFDJS5UuEyqdJV9GJ1X5JtUE6ikqggUGc-ens&_LANG=enus "A screenshot of the selected connection, record, and Dynamics 365 object for the workflow.")
7. What will you call it?: Use variables and text to create a custom file name for your
   agreements:
   1. Add Variable: Use the search field or dropdown menu to select variables to add to
      the agreement name.
   2. (Optional) Add text: Enter text to add to the agreement name and select Add.

      Note: The File path field displays the naming convention. As you add variables and
      text, the name updates. **Example**: File path Microsoft Dynamics 365 1
      Shared/[Envelope ID]FontaraNDA.

      ![A screenshot of the variables and text used to create a custom file name for your agreements.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/ds-dyn365-app-store_file2.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE4ODgsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.ndu8AXtFDJS5UuEyqdJV9GJ1X5JtUE6ikqggUGc-ens&_LANG=enus "A screenshot of the variables and text used to create a custom file name for your agreements.")
   3. Select Apply.

      The Store Files in Dynamics 365 column closes. The workflow page
      displays your new step.

You have completed the process. You have added a **Store Files in Dynamics 365** step to your
workflow. When signers complete an agreement, Docusign adds the agreement and
certificate of completion to the specified record.
