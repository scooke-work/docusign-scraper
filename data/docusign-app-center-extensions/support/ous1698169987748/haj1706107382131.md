# Configure a SharePoint Store Files Step in an Agreement Workflow

Learn how to configure a SharePoint
store
files step in a Docusign agreement workflow.

Use the SharePoint
store
files step to save documents generated by Workflow Builder steps.
Generally, you position your archive step at the end of your workflow to archive
completed documents. Whatever state a document is in when the archive step generates
is what is saved to the specified file location.

Restriction: Do not use a SharePoint
store
files step if your workflow uses an eSignature step with multiple
participants and your first signer is designated as an embedded signer.
In this scenario, the first signer goes directly to sign from a prior workflow step.
Documents are archived directly after the embedded signer finishes signing and would
not include the actions of any subsequent participants.

Before you begin
:   - You must be an account administrator or have Workflows: Create user permissions as described in the [Workflow Builder Roles and Permission Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) article.
    - The extension app must be [installed and authorized](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ctg1698170340729.html&_LANG=enus) on your account.

1. Create a new workflow as described in the [Create
   a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus) article.
2. Select Add a step >  Store files with SharePoint >  Configure.
3. Select the file to archive and select Apply.

   ![A screenshot of the SharePoint app file selection dialog box.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/apspfile.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE4OTMsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.f0niZ7lCqYu_HElOp8D_f1vE9Yq4OmnqFoJplYihefI&_LANG=enus "A screenshot of the SharePoint app file selection dialog box.")
4. Specify where to archive the file:
   1. Select the connection.
   2. Select the site.
   3. Select a drive.
   4. (Optional) Select a folder on the drive by choosing Add
      Folder. All existing folders are listed, as well as an
      option to create a new folder.

   ![A screenshot of the SharePoint location selection dialog box.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/apspwhere.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE4OTMsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.f0niZ7lCqYu_HElOp8D_f1vE9Yq4OmnqFoJplYihefI&_LANG=enus "A screenshot of the SharePoint location selection dialog box.")
5. Specify a file name for your archive. Use the text and variable controls to
   build your file name.

   ![A screenshot of the SharePoint app archive name dialog box.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/apspnamearchive.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE4OTMsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.f0niZ7lCqYu_HElOp8D_f1vE9Yq4OmnqFoJplYihefI&_LANG=enus "A screenshot of the SharePoint app archive name dialog box.")
6. Review the completed file path and select Apply.

Your SharePoint
store
files step is configured.
