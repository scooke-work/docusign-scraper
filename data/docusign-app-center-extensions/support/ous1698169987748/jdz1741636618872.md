# Configure a Writeback to Dynamics 365 Step in an Agreement Workflow

Learn to add and configure a Writeback to Dynamics 365 step to update Dynamics 365 records.

Before you begin
:   - Required: Installation of the Dynamics 365 app from Docusign App Center. Review the [Installing Extension Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more information.
    - Required: [Workflows Create permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account administrator.
    - Required: Access to a suitable template to use for your workflow step. Review the [Template Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=xnw1709852160602.html&_LANG=enus) topic for more information.
    - Required: [Connect Docusign Workflow Builder with the Microsoft Dynamics 365 Account](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=och1741251029395.html&_LANG=enus "Learn how to establish a connection between Docusign Workflow Builder and the Microsoft Dynamics 365 account.").
    - Recommended: Open the template in another window to view as you set up your step. This helps you to correctly configure your Writeback to Dynamics 365 step.

You can use a Writeback to Dynamics 365 step to update Dynamics 365 records with information from your Docusign workflow. This step updates Dynamics 365 fields, creates new records, and more. Administrators link workflow fields to Dynamics 365 fields in this step. When task recipients enter data in workflow fields, Docusign updates the information in the linked Dynamics 365 record fields. Writeback improves efficiency and reduces errors by removing the need to manually update records.

To learn about installing, configuring, and integrating Dynamics 365 with agreement workflows, see the [Use Docusign Workflow Builder with Microsoft Dynamics 365](https://support.docusign.com/s/document-item?bundleId=pic1741250291019&topicId=lkn1741250360427.html&_LANG=enus) article.

Example: You can build a workflow that includes a Collect Data with Web Forms step and a Writeback to Dynamics 365 step. In the writeback step, you link the Dynamics 365 opportunity's Budget Amount field to the web form's Budget Amount field. A workflow recipient updates and signs a contract for an existing opportunity named Fontara. The recipient updates the Fontara contract's Amount field from $20,000 to $25,000. When the recipient completes the assignment, Docusign updates the Fontara record's Amount field to $25,000.

Note: Follow the required formatting for all date fields. See the [Format Date Fields on a Writeback to Dynamic](https://support.docusign.com/s/document-item?bundleId=pic1741250291019&topicId=dfz1742231023702.html&_LANG=enus) article for more information.

Use the following procedure to create a Writeback to Dynamics 365 step in a Docusign workflow:

1. Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add steps like [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus), [web forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and more to gather recipients, documents, and merge fields for your workflow.
2. Select Add a step  >  Writeback to Dynamics 365  >  Configure.
3. Step Name: Keep or edit the name. It is helpful to add the name of the Dynamics 365 object, like Account - Writeback to Dynamics 365.
4. Where will you write to?
   1. Connection: Choose a Dynamics 365 organization connected to Workflow Builder.
   2. Dynamics 365 object: Select an object from the dropdown menu.
   3. Write settings: Select an option from the list that opens:

      - Update or Create: Update a record or create a new record if none exists.
      - Update: Update an existing record (no required fields): Updating a record that exists
      - Create: Create a new record (required fields) New account sign-up flow or onboarding
   4. Select Next.
5. Which fields are you writing to?: Select workflow fields to map to Dynamics 365 fields. When the workflow recipients complete tasks, the data they enter updates field data on Dynamics 365 records.
   1. (Optional) Add or Remove Fields: Choose more fields and select Save and Exit.
   2. Select the Select field below a Dynamics 365 field name.
   3. Choose an existing workflow field or create a new field:

      - Starting Variables: Find custom variables created outside of the workflow steps.
      - WORKFLOW STEPS: Find variables from other workflow steps.![A screenshot displays matching workflow fields to Dynamics 365 fields.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/matching-workflow-fields-to-Dynamics-365-fields_writeback.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MDUsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.aYTwy4mO2L3VlMRlEmstBcr7F8RU09CeezMXHNQQClc&_LANG=enus "A screenshot displays matching workflow fields to Dynamics 365 fields.")
      - (Optional) Overwrite if field value exists: Select this check box if you want the data entered by a workflow participant to overwrite existing Dynamics 365 data.
6. Which record are you writing to? (optional):  [Build a rule to writeback to existing Dynamics 365 records](https://support.docusign.com/s/document-item?bundleId=pic1741250291019&topicId=ykg1741635819126.html&_LANG=enus). If you do not add a rule, the writeback step creates a new Dynamics 365 record.
7. Select Apply.

You have completed the process. You have built a Writeback to Dynamics 365 step in your workflow. When this step is triggered in a workflow, it updates field data on Dynamics 365 records you mapped. Users do not need to manually update records.
