# Configure a Writeback to HubSpot Step

Learn to add and configure a Writeback to HubSpot step to update HubSpot records.

Before you begin
:   - Required: Installation of the HubSpot app from
      Docusign App Center. Review the [Installing Extension
      Applications for Workflow Builder](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=bgb1697668738892.html&_LANG=enus) topic for more
      information.
    - Required: [Workflows Create
      permissions](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) or be an account
      administrator.
    - Required: Create a workflow as described in [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus). Add
      steps like [invitation](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=kfx1698167344720.html&_LANG=enus),
      [web forms](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gua1698120920620.html&_LANG=enus), [eSignature](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uww1698167288211.html&_LANG=enus) and
      more to gather recipients, documents, and merge fields for your
      workflow.

You can use a Writeback to HubSpot step to update HubSpot records with
information from your workflow. Administrators link workflow fields to
HubSpot fields in the Writeback to HubSpot step.

When task recipients enter data in workflow fields, Docusign
updates the information in the linked HubSpot record fields.

Writeback improves efficiency and reduces errors caused by manually updating
records.

Example: You can build a workflow that includes a Collect Data
with Web Forms step and a Writeback to HubSpot
step. In the writeback step, you link a HubSpot record's Amount
field to the web form's Amount field.

A workflow recipient updates and signs an agreement for an existing organization named
Fontara. The recipient updates the Fontara agreement's Amount field from $20,000 to
$25,000. When the recipient completes the assignment, Docusign
updates the Fontara record's Amount field to $25,000.

Note: Follow the required formatting for all fields. See the [Supported HubSpot Field Types](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ref1bcd0a62-7428-42fe-9760-709faa21b8ac.html&_LANG=enus "Learn about the HubSpot field types that you can use in Workflow Builder workflows.") topic for more information.

Use the following procedure to create a Writeback to HubSpot
step in a Docusign workflow:

1. Add a Writeback to HubSpot step as a Docusign administrator:
   1. Go to Agreements > Workflows and select Edit next to a workflow
      name.

      The workflow editor page displays.
   2. Select the Add a step
      ![A screenshot displays a plus sign inside of a square.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/Workflow_Builder_add_step_plus_sign.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.PhxSjcOhEGwVrdGAw2ONuwn_4wKGNFGxWFv_7X9KZTw&_LANG=enus "A screenshot displays a plus sign inside of a square.") icon.

      The Add New Step dialog box displays.
   3. Search for steps: Enter `Writeback to
      HubSpot` in the search field and select the
      Writeback to HubSpot step that displays.

      The workflow editor page displays with the Writeback to HubSpot
      step added.
2. Name your Writeback to HubSpot step:
   1. Select Configure on the Writeback to HubSpot
      step.

      The configuration dialog box displays.
   2. Step Name: Keep or edit the name. It is helpful
      to add the name of the HubSpot object, like `Company -
      Writeback to HubSpot`.
3. Where will you write to?
   1. Connection: Choose a HubSpot account that you
      connected to Docusign. Multiple options display if you connected more than one HubSpot
      account.
   2. (Optional) Add New Connection: Select this button if you want to [connect more than one HubSpot
      organization](https://support.docusign.com/s/document-item?bundleId=qwr1706590641805&topicId=ogk1719256147008.html&_LANG=enus).
   3. (Optional) Manage Connections: Select this
      button if you want to [refresh your HubSpot connection](https://support.docusign.com/s/document-item?bundleId=nta1727383289518&topicId=sqw1734725367259.html&_LANG=enus).
   4. HubSpot object: Search for a HubSpot object or
      choose one from the dropdown menu.
   5. Write settings: Select an option from the list
      that opens:

      - Update or Create: Update a record or
        create a new record if none exists.
      - Update: Update data on an existing
        record.
      - Create: Create a new record.
   6. Select Next.

   ![A screenshot displays a Writeback to HubSpot workflow step with the Connection, HubSpot object, and Write settings highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/Workflow_Builder_for_HubSpot_Writeback_Where_will_you_write_to_Update_or_create.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.PhxSjcOhEGwVrdGAw2ONuwn_4wKGNFGxWFv_7X9KZTw&_LANG=enus "A screenshot displays a Writeback to HubSpot workflow step with the Connection, HubSpot object, and Write settings highlighted.")
4. Which fields are you writing to?: Select workflow fields to
   map to HubSpot fields. When the workflow recipients complete tasks, the data
   they enter updates HubSpot records.

   Note: If you only want to [make an object
   association](https://support.docusign.com/s/document-item?topicId=ref2de68171-0b88-4a3c-a74f-c60b390974fd.html&_LANG=enus), skip to Are you making an association
   with another object?

   1. Select Add or Remove Fields.

      The Add or Remove Fields page displays.
   2. Select the check box next to each HubSpot field that you'd like to use
      as a workflow variable. You'll be able to use these fields in later
      steps.

      ![A screenshot displays the Add or Remove Fields page with HubSpot Owner Email, and HubSpot Owner Name fields selected.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/Workflow_Builder_Writeback_to_HubSpot_Add_or_Remove_Fields.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.PhxSjcOhEGwVrdGAw2ONuwn_4wKGNFGxWFv_7X9KZTw&_LANG=enus "A screenshot displays the Add or Remove Fields page with HubSpot Owner Email, and HubSpot Owner Name fields selected.")
   3. Select Save and Exit.

      The Add or Remove Fields dialog box closes. The fields that you
      selected display.
   4. Fields: Add a variable or step for each
      field:

      - Manual entry: Add text for your new
        variable and select Save.

        Note: The name that you enter must exactly
        match an existing field. For example, the rule fails if you
        enter CompanyName for the
        Company\_Name
        variable.
      - Select a field from the start step or another step in this workflow:

        Note: Steps like Read from
        Salesforce and Read from
        HubSpot display fields from up to five
        records. If more than five records are present, HubSpot
        logic decides which five to display. Selecting fields from
        multiple records helps you automate processes like mapping
        multiple HubSpot contacts to multiple Docusign participants.![A screenshot displays read step fields highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/Workflow_Builder_for_Salesforce_Read_Workflow_field_selections.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.PhxSjcOhEGwVrdGAw2ONuwn_4wKGNFGxWFv_7X9KZTw&_LANG=enus "A screenshot displays read step fields highlighted.")

        You can use the fields that you select here in later
        steps like Collect Data with
        Webforms or Send Documents
        for Signature.
   5. Repeat this process for each variable displayed.
   6. Select Next.

      ![A screenshot displays matching Docusign and HubSpot fields with field values highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/Workflow_Builder_Writeback_to_HubSpot_Which_fields_are_you_writing_to_Next.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.PhxSjcOhEGwVrdGAw2ONuwn_4wKGNFGxWFv_7X9KZTw&_LANG=enus "A screenshot displays matching Docusign and HubSpot fields with field values highlighted.")
5. Which record are you writing to?: Create a rule that
   matches each HubSpot field with an equivalent workflow field. This allows
   Workflow Builder to interact with HubSpot data.
   1. HubSpot field: Select a HubSpot field to match
      with a Docusign field.

      Note: When you select a
      (reference) type field, a new field opens
      below it. Choose a child object from the new field.
   2. Operator: Choose an option like Equal
      to, Less than, and so on.
   3. Workflow field: Select a Docusign field to match with your HubSpot field. The following
      options display:

      - Manual entry: Add text for your new
        variable and select Save.
      - Select a field from the start step or another workflow
        step.
   4. Select Done to save your rule.

      ![A screenshot displays a HubSpot field, an operator value of Greater than or equal to, and a workflow field all highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/Workflow_Builder_for_HubSpot_Writeback_Which_record_are_you_writing_to_rule.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.PhxSjcOhEGwVrdGAw2ONuwn_4wKGNFGxWFv_7X9KZTw&_LANG=enus "A screenshot displays a HubSpot field, an operator value of Greater than or equal to, and a workflow field all highlighted.")
6. (Optional) Add more rules:
   - +AND: Workflow Builder only imports HubSpot data that
     matches all of your rules.
   - +OR: Workflow Builder imports HubSpot data that matches
     any of your rules.
7. (Optional) Are you making an association with another object?: Use this procedure if you want to associate objects. If you don’t want to associate objects, select No and Apply:

   Note: Workflow Builder supports [creating primary label associations for HubSpot](https://knowledge.hubspot.com/object-settings/create-and-use-association-labels).
   HubSpot manages the primary label according to their [association rules](https://developers.hubspot.com/docs/guides/api/crm/associations/associations-v4).

   1. Select to continue: Select
      Yes.

      Object association fields open.
   2. Object Association (Parent): Choose a HubSpot
      object from the dropdown menu.
   3. Object ID (Parent): Enter an object ID, or
      choose a record ID output from a previous step.

      ![A screenshot displays Object Association fields highlighted connecting HubSpot contacts with a parent record ID.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/Workflow_Builder_for_HubSpot_Writeback_to_HubSpot_Are_you_making_an_association.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.PhxSjcOhEGwVrdGAw2ONuwn_4wKGNFGxWFv_7X9KZTw&_LANG=enus "A screenshot displays Object Association fields highlighted connecting HubSpot contacts with a parent record ID.")
   4. Select Apply.

   The Writeback to HubSpot dialog box closes. The Workflow Builder page
   displays your new step.

You have completed the process. You have built a Writeback to
HubSpot workflow step. When this step starts, it
updates field data on HubSpot records that you mapped.
