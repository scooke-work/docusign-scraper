# Install Apps in the Docusign App Center

Learn how to install Docusign extension apps in the Docusign App Center.

Your access level determines the types of apps you can install.

Docusign account administrators

You must be an account administrator to install and manage packaged and partner apps from l Docusign App Center. See the [Organization Administrators](https://support.docusign.com/s/document-item?bundleId=rrf1583359212854&topicId=god1583359152535.html&_LANG=enus) article to learn more about these permissions.

Non-admin users

If your organization has a Docusign plan that includes packaged apps, you can install these approved apps. See the [Use Approved Apps](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=aak1747679824594.html&_LANG=enus "Learn about Docusign approved extension apps and who can install and use.") article for detailed information.

You install apps at the account level. View and change accounts by selecting your profile picture and selecting Switch Account.

l Docusign workflows and eSignature templates are affected when you uninstall and reinstall apps and their connections.

Important:

- - If you uninstall and reinstall, an app, you must reinstall and reconfigure the eSignature template or the app step in the workflow.
  - If you delete an app's connection, you must create a new connection and reconfigure the app step with this new connection. See the [Create Docusign App Connections](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=zsb1714159452358.html&_LANG=enus "Learn how to create Docusign apps connections for different user credentials in Docusign agreement workflows. You can create connections that are shared by all account users or used privately by the user who created the app connection.") article for details.
  - You can refresh an installed app's connection without deleting without affecting a workflow or template. See the [Reconnect a Docusign App Connection](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=qkv1716320270335.html&_LANG=enus "Learn how to use the Docusign app Reconnect a connection feature to reestablish the Docusign app’s connection to workflows.") article for details.
  - Reinstall any apps that were previously installed from the Docusign App Center. Due to version issues, backward compatibility is not currently possible
  - Reconnect all Workflow Builder fields after upgrading.

You can install Docusign App Center apps:

- - On the Docusign App Center home page: Select the app tile and select Install App.
- - From an app's detail page: Select  Install App from any detail page.

1. Access  [Docusign App Center](https://apps.docusign.com/app-center) to begin using extension apps in your Docusign agreement workflows.

   Here is an example of the Docusign App Center page.

   ![A screenshot of the Docusign App Center dashboard.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/ac-newest-dash.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE3OTMsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.VyOQpM29oZ0pmdqVMCepra0tPIiih3HwJm2PMp-dSzI&_LANG=enus "A screenshot of the Docusign App Center dashboard.")
2. Select Login to log on to your account.

   - Note the following:

     - Some apps do not require authentication or an account connection.
     - The Install App  option is grayed out If you are not authorized to use an app. A message banner provides the reasoning. See the [Docusign App Center Banner Messages and App Labels](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=map1704218548543.html&_LANG=enus "Learn about the messages and availability app labels that you encounter when using Docusign App Center and the actions you should take.") to learn more.
3. Optionally refine your search as described in the [Locate Docusign Apps for Your Workflow Processes](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ljy1744998611944.html&_LANG=enus "Learn how to locate apps you can use in your workflow processes by using the Docusign Search feature and Works With and Industry filters.")  article.
4. Select an app to install.

   The app's detail page displays. From this page, you can:

   - Install and authorize the app to use with platform services.
   - Review the app's overview, requirements, and resources.
5. Select Install App.
6. At the prompt, select  Install and Authorize to connect the app.
7. Select Continue.
8. If the app requires authorization, select Connect Account to connect the app to your Docusign account.
9. If you have Administrator privileges, select Manage > Add New Connection and specify the App connection type. See the [Create Docusign App Connections](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=zsb1714159452358.html&_LANG=enus "Learn how to create Docusign apps connections for different user credentials in Docusign agreement workflows. You can create connections that are shared by all account users or used privately by the user who created the app connection.") article for detailed information.

   These connection types are available:

   - Shared: Created by an Admin. Any account user can view and use the connection.
   - Private: Created by an Admin or an account user. Only the account user who created the connection can view and use the connection.
10. Authorize the application for use in Docusign by entering your credentials for the app.
11. Select Continue.

    The app is now ready to use to customize and extend agreement processes across multiple Docusign product surfaces, including Docusign Workflow Builder, the workflow builder tool.
12. To use the app in a platform service, such as Workflow Builder, select Use this app.

    The platform service displays. For example, if you want to use the app in a workflow, the workflows page displays.

    To learn more about using workflows, see the [Workflow Builder Templates](https://support.docusign.com/s/document-item?topicId=irb1736981148403.html&_LANG=enus) article.
