# Configure a ServiceNow Writeback Step in an Agreement Workflow

Learn about using the ServiceNow app to run multiple business applications per instance
through Docusign agreement workflows.

ServiceNow app instances run multiple business applications per instance, with each application
sharing data. You can use ServiceNow through Docusign Workflow Builder
workflows.

You must be an account administrator or have Workflows: Create user permissions as
described in the [Workflow Builder Roles and Permission
Requirements](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=gbb1696973048215.html&_LANG=enus) article.

Configuring ServiceNow for Docusign involves:

- [Set Up the Docusign App Center ServiceNow Instance](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=qau1712843715884.html&_LANG=enus "Learn how to set up the ServiceNow instance for use in the Docusign App Center.")
- [Troubleshoot Installing the Docusign IAM for ServiceNow App](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=bur1727290839878.html&_LANG=enus "Learn how to troubleshoot issues with installing Docusign IAM for ServiceNow.")
- [Create the ServiceNow Writeback
  Step](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ejl1711393084084.html&_LANG=enus "Learn how to create the ServiceNow writeback step to use a ServiceNow instance in Docusign Workflow Builder workflows.")
