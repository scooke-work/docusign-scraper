# Configure a Stripe Writeback Step in an Agreement Workflow

Learn how to use the Stripe extension app to streamline automation for payments,
billing, and subscriptions through Docusign agreement
workflows.

## Before you begin

- Be an account administrator or have Workflows: Create user permissions as described in the
  [Workflow Builder Roles and Permission Requirements](https://support.docusign.com/s/document-item?language=en_US&bundleId=yff1696971835267&topicId=gbb1696973048215.html)
  article.
- The Stripe extension app must be [installed and authorized](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ctg1698170340729.html&_LANG=enus) on your account.
- Complete these steps:
  - [Set Up Docusign Workflow Builder for Stripe](https://support.docusign.com/s/document-item?bundleId=qzg1726786604706&topicId=ubn1727888317315.html&_LANG=enus)
  - [Install Docusign for Stripe](https://support.docusign.com/s/document-item?bundleId=qzg1726786604706&topicId=ols1724269694863.html&_LANG=enus)
- To create a step that writes an invoice or invoice item object to Stripe, first set up a product
  in Stripe.

The Stripe extension app lets you streamline automation for payments, billing, and
subscriptions through Docusign Workflow Builder agreement workflows.

1. Create a new workflow as described in the [Create a Basic Workflow](https://support.docusign.com/s/document-item?bundleId=yff1696971835267&topicId=uhm1697826856678.html&_LANG=enus) article.
2. Select Add a step  >  Writeback to Stripe  >  Configure.

   The step is added to the workflow.
3. (Optional) The default Step Name is Writeback
   to Stripe. Change this to a more meaningful name, such as
   Create Stripe Customer.
4. Where will you write to?
   1. Connection: Make sure that the
      Connection type is
      StripeOAuth.
   2. Stripe object: Select an object from the
      dropdown menu.
   3. Write settings: Select an option from the list
      that opens:

      - Update or Create: Update a record or
        create a new record if none exists.
      - Update: Update data on an existing
        record.
      - Create: Create a new record.

      ![A screenshot displays the Stripe Write settings options.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/write-to-options.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTksInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.DbuKLJbgFO5gfumUjhJD4tXZr-yCGym432Dntg4yAWM&_LANG=enus "A screenshot displays the Stripe Write settings options.")
   4. Select Next.
5. Where
   will you
   writeto?:
   Select
   a Stripe object. When the workflow recipients complete tasks,
   the data they enter updates field data on Stripe records.

   ![A screenshot displays sample Stripe Write fields.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/where-will-you-write-to.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTksInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.DbuKLJbgFO5gfumUjhJD4tXZr-yCGym432Dntg4yAWM&_LANG=enus "A screenshot displays sample Stripe Write fields.")

   1. Select Add or Remove Fields.

      The Add or Remove Fields page displays.
   2. Select Stripe fields that you'd like to use as workflow variables.
      You'll be able to use them in later steps.

      ![A screenshot displays field values for the Company object.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/add-or-remove-fields-for-stripe-writeback.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTksInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.DbuKLJbgFO5gfumUjhJD4tXZr-yCGym432Dntg4yAWM&_LANG=enus "A screenshot displays field values for the Company object.")
   3. Select Save and Exit.

      The Add or Remove Fields dialog box closes. The fields you
      selected display.
   4. Select the Select field below a Stripe field
      name.
   5. Choose an existing workflow field or create a new field:

      - Enter manually: Add text for your new
        variable and select Save.

        Note: The name you enter must exactly match
        an existing field. For example, the rule fails if you enter
        CompanyName and the variable you
        want is Company\_Name.
      - Starting Variables: Find custom variables
        created outside of the workflow steps.
      - WORKFLOW STEPS: Find variables from other
        workflow steps.
      - OTHER SOURCES: Choose
        Variables from Workflow Start,
        Instance details, or other
        types.
   6. Repeat this process for each variable displayed.

      ![A screenshot displays matching Workflow Builder and Stripe fields with field values highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/acstripewhichfieldsdb.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTksInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.DbuKLJbgFO5gfumUjhJD4tXZr-yCGym432Dntg4yAWM&_LANG=enus "A screenshot displays matching Workflow Builder and Stripe fields with field values highlighted.")
   7. Select Next.
6. Which record are you writing to?: Create a rule that
   matches each Stripe field with the equivalent workflow field. This allows
   Workflow Builder to read Stripe data. You can create as many rules as you want:
   1. Stripe field: Select a Stripe field to match
      with a Docusign workflow field.
   2. Operator: Keep Equal to.
   3. Workflow field: Select a Docusign workflow field to match with your Stripe
      field. The following options display:

      - Enter manually: Add text for your new
        variable and select Save.

        Note: The name you enter must exactly match
        an existing field. For example, the rule fails if you enter
        CompanyName and the variable you
        want is Company\_Name.
      - Starting Variables: Find custom variables
        created outside of the workflow steps.
      - WORKFLOW STEPS: Find variables from other
        workflow steps.
      - OTHER SOURCES: Choose
        Variables from Workflow Start,
        Instance details, or other
        types.
   4. Select Done to save your rule.

      ![A screenshot displays the Stripe Company Name field related to the Contact ID field, with the Done button highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/acstripewritetorecord.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MTksInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.DbuKLJbgFO5gfumUjhJD4tXZr-yCGym432Dntg4yAWM&_LANG=enus "A screenshot displays the Stripe Company Name field related to the Contact ID field, with the Done button highlighted.")
   5. (Optional) Add more rules:

      - +AND: Add a new rule. Workflow Builder only writes
        back to Stripe fields that match all of your rules.
      - +OR: Add a new rule. Workflow Builder writes back
        to Stripe fields that match any of your rules.
7. Select Apply.

   You have completed the process. You have built a Writeback to Stripe
   step in your Workflow Builder workflow. When this step is generated in a workflow, it
   updates field data on Stripe records you mapped.

You completed this task. You created and configured a
Writeback to Stripe step. You return to the workflow overview in the Workflow Builder
workflow.
