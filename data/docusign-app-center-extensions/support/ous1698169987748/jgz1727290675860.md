# Set Up the Docusign App Center ServiceNow Instance, Step 2: Generate a New Client Secret for OAuth

Learn how to create a new client secret for Docusign IAM for ServiceNow.

Exchange data between Docusign and ServiceNow with Docusign IAM for ServiceNow. Use the integration to kick off software and hardware procurement requests and other downstream workflows.

This topic shows how to create a new client secret for OAuth.

1. In App Engine Studio, select the All tab.

   A new panel with a search box opens.
2. Search for OAuth. Select Application Registry from the results.

   ![A screenshot of the All panel. It shows a search for Oath. The result named Application Registry is highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/servicenowoauth.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MzAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.eK4aE3frQYCBheLNuEdRuJWJtJ2Iw7SHv7t-WMZh-eQ&_LANG=enus "A screenshot of the All panel. It shows a search for Oath. The result named Application Registry is highlighted.")

   The Application Registry window displays.
3. Select Docusign IAM OAuth from the list.

   The details page for the application displays.
4. Unlock the field by selecting the lock icon, then replace the existing Client Secret with a new one. Once finished, lock the field.
5. Set the Redirect URL for your region.
   - Demo: https://demo.services.docusign.net/act-gateway/v1.0/oauth/callback

     Note: You can use the demo redirect URL irrespective of your region.
   - Production (U.S.): https://us.services.docusign.net/act-gateway/v1.0/oauth/callback
   - Production (CA): https://ca.services.docusign.net/act-gateway/v1.0/oauth/callback
   - Production (AU): https://au.services.docusign.net/act-gateway/v1.0/oauth/callback
   - Production (EU): https://eu.services.docusign.net/act-gateway/v1.0/oauth/callback
   - Production (JP): https://jp.services.docusign.net/act-gateway/v1.0/oauth/callback
6. Make a note of the Client ID, Client Secret, and ServiceNow instance name. You must enter them on the Docusign platform later. You can find the ServiceNow instance name at the beginning of the URL in your browser’s address bar. When you are done and you have the details that you need, select Update at the top right.

   ![A screenshot of the details page for the application. The Client Secret field is highlighted.](https://docusign-be-prod.zoominsoftware.io/api/bundle/ous1698169987748/page/images/servicenowregistrydetails.png?token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJjdXN0b21lciI6ImRvY3VzaWduX3Byb2R1Y3Rpb24iLCJleHAiOjE3ODE4MTE5MzAsInNoZWFmIjoib3VzMTY5ODE2OTk4Nzc0OCJ9.eK4aE3frQYCBheLNuEdRuJWJtJ2Iw7SHv7t-WMZh-eQ&_LANG=enus "A screenshot of the details page for the application. The Client Secret field is highlighted.")

You return to the Application Registry window. You now have the Client ID, Client Secret, and ServiceNow instance name. You need these to authorize the ServiceNow app when you [Create the ServiceNow App Writeback Step](https://support.docusign.com/s/document-item?bundleId=ous1698169987748&topicId=ejl1711393084084.html&_LANG=enus "Learn how to create the ServiceNow writeback step to use a ServiceNow instance in Docusign Workflow Builder workflows.").
