# Docusign Data Verification Field Types

Learn about the different types of Docusign Data Verification
Fields that you can use in your eSignature envelope preparation.

Here are the different types of Docusign Data Verification Fields that
you can use in your eSignature envelope preparation.

## Basic Data Verification Fields

Basic Data Verification Fields confirm the existence of data that doesn't reveal
identity or other sensitive information. For example, email address, phone number,
and postal address are basic Data Verification Fields.

Availability
:   Global

eSignature Add Fields page location
:   Standard fields: Global

Supported plans
:   - Business Pro
    - Enterprise
    - IAM Professional, Enterprise
    - Enhanced plans: IAM for Sales, IAM for CX

Basic fields
:   - Email address
    - Phone number
    - Postal Address verification
      - Address line 1
      - Address line 2
      - City
      - County or region
      - Postal Code
      - State

## Premium Data Verification Fields

Premium Data Verification Fields verify identity or require confirmation of sensitive
data. For example, bank account number, bank account ownership, and business entity
are premium data verification fields.

Availability
:   U.S.

eSignature Add Fields page location
:   Custom fields

Supported plans
:   Premium Data Verification add-on plan that also includes everything in Basic
    Data Verification.

Premium fields
:   - Bank account
      - Routing number
      - Account numbers
      - Account type (Savings or Checking)
    - Bank account owner
      - Routing number
      - Account numbers
      - Account type (Savings or Checking)
      - First and last (family) names
    - Business Identity
      - Federal Employer Identification Number (FEIN)
      - Organization Name
    - Social security number (SSN)
      - SSN
      - Date of birth
      - First and family names

## Custom Data Verification Fields

In addition to using prebuilt extension apps from the Docusign
[App Center](https://www.docusign.com/products/platform/app-center), you can now build your own extension apps.
Use these apps to verify data against your internal databases, or the web service of
your choice.

See the [Use Docusign Custom Data
Verification for your Organization](https://support.docusign.com/s/document-item?bundleId=qki1698158012210&topicId=qrx1741973884664.html&_LANG=enus "Learn about Docusign custom data verification fields that let you verify agreement data against your internal systems in real time.") article for details.

## Data verification app sources

Data Verification uses these types of extension apps:

| App type | Description | Basic Apps | Premium Apps |
| --- | --- | --- | --- |
| Packaged | Docusign-built apps using Docusign credentials with a partner system Requires no user authentication | Email (LexisNexis)  Phone (Vonage)  Address (Smarty | SSN (LexisNexis)  FEIN (LexisNexis) |
| Partner | Partner-built apps Requires user authentication with partner system | N/A | Bank account (LSEG)  Bank account owner (LSEG)  Business bank account owner (LSEG) |
