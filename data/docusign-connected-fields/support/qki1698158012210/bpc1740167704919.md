# Create a Private Docusign App

Learn how to create a private Custom private app for data verification , eSignature,
or other Docusign features. Private apps provides custom data verification
fields for your organization.

You a private Custom private app for data verification and for other Docusign features such as eSignature, or Workflow Builder. Private apps provides
custom fields for your organization.

Before you begin

Identify:

- Use cases
- Envelope templates
- Target system APIs, such as a verification service or an internal database

1. Create your extension app as described in the [Build an Extension App](https://developers.docusign.com/extension-apps/build-an-extension-app/) article.

   For information about using Connected fields extensions, see these
   articles:

   - [Connected Fields API](https://developers.docusign.com/docs/connected-fields-api/concepts/)
   - [Connected fields extension overview](https://developers.docusign.com/extension-apps/extension-apps-101/supported-extensions/connected-fields/)
2. Evaluate your API proxy requirements as described in the [API proxy](https://developers.docusign.com/extension-apps/build-an-extension-app/it-infrastructure/api-proxy/) article.

   Determine if you need to build a proxy layer to transform data exchange between
   your Custom Data Verification extension app to the target system.
3. Define your Custom data models.

   Define structured sets of fields that represent your agreements and
   data structures. Docusign uses the open source data
   modeling language [Concerto](https://concerto.accordproject.org/).

   A custom model can represent agreements such as documents or forms,
   or your external system’s object model, or both.
4. Integrate the extension app.
   1. Define a connection in the extension app.

      Configure parameters to obtain authorization from your target system
      as described in the [Connections](https://developers.docusign.com/extension-apps/extension-apps-101/concepts/connections/) article.
   2. Implement actions as described in the [Actions](https://developers.docusign.com/extension-apps/extension-apps-101/concepts/actions/) article.
5. Submit and test your app.

   Submit the app for review as described in the [Publish an extension app to the Docusign App Center](https://developers.docusign.com/extension-apps/build-an-extension-app/publish/) article.
6. Publish your app.

   Privately publish the app for select accounts as described in the [Choosing private distribution instead of
   public](https://developers.docusign.com/extension-apps/extension-apps-101/choosing-private-distribution/) article.
