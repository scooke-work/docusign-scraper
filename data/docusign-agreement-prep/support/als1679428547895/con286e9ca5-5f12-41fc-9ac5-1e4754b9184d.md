# About the Template Assistant

Read about the Word Template Assistant. The Microsoft Word add-in helps you create, test, and upload dynamic agreement templates to your Docusign account using fields, rules, code, and preview tools.

The Docusign Word Template Assistant is a Microsoft Word add-in that helps you create, test, and upload dynamic agreement templates to your Docusign account.

## Fields

The Fields tab allows you browse and search for specific data fields that will be populated when an agreement is generated.

- **Search icon (Magnifying glass):** Quickly find a specific field by name.
- **Agreement type (Edit icon):** Change the agreement type.
- **Fields/Tables toggle:** Insert fields relevant to the agreement type or a dynamic table.

Note: Dynamic tables in the Word Template Assistant support up to 20 columns. If your table requires more than 20 columns, consider splitting it into multiple tables.

## Rules

The Rules tab makes your documents dynamic. Instead of having multiple versions of an agreement, you can have one document that changes its content automatically.

- **Visibility:**

  - **Show content:** The selected text in your template will only display if the rule is met.
  - **Hide content:** The selected text will disappear if the rule is met.
- **Field name:** Select the "trigger" field (for example, "State" or "Total Amount").
- **Operator:** Define the logic of the comparison. Common operators include:

  - **Matches (=):** The value must be exactly the same.
  - **Doesn't match (≠):** The value must be different.
  - **Greater than or equal to/Less than or equal to:** Used for numerical values.
- **Value:** The specific "answer" to trigger the rule (for example, if "State" matches "California").

## Code

The Code tab allows you view and edit the JSON data structure used for document generation.

Insert JSON: Paste or edit the JSON representation of your fields.

## Account

The Account tab displays the user and organization information connected to the Word add-in. This is vital for ensuring that the templates you build are being saved to the correct Docusign environment.

- **Name and email:** The identity of the user currently logged into the add-in.
- **Account number and ID:** The unique numerical and alphanumeric identifiers for your Docusign account. These are often required for troubleshooting with Docusign Support. When signing in to WTA, the system will always log in to your user's default account, which is set under "My preferences".
- **Company:** The name of the organization or "Sandbox" environment you are currently accessing.
- **Environment:** Displays Demo (for testing) or Production (for live documents).
- Disable Classic Mode: Blocks access to classic mode.
- Switch Account: If the logged-in user has multiple accounts, they can switch between them without logging out.
- Logout: Logs out of the current account session.

## Classic Switcher

In classic mode, you use the same core tabs as the new experience: Fields, Tables, Rules, Code, and Account.

You can browse and insert fields, configure tables, build rules, manage JSON, and verify account details without changing your workflow.

In classic mode, fields are reusable only for the individual user and are not shared at the account level. In the new experience, fields are reusable across all users in the account.

You can browse and insert data fields, configure dynamic tables, build conditional logic, review or paste JSON, and confirm account details without changing your existing workflow.

Important: When you switch to the new experience, existing classic-mode fields are not carried over. You must create new fields in the new experience. In classic mode, fields are reusable only for the individual user and are not shared at the account level. In the new experience, fields are reusable across all users in the account.

## Preview

This is the final step in the agreement preparation process, where you can test your work before uploading it to Docusign.

**Preview:** The standard testing tool that generates a mock version of your Word document, showing how fields and rules display after data is merged. Use it to catch formatting errors or broken logic early.

**Preview with AI:** An advanced testing feature that uses Docusign's AI to analyze the document for risks, inconsistencies, and missing data. It helps ensure that your agreement is legally robust and logically sound by simulating how the AI will interpret the final document.

## Upload document

Once you have finished adding your fields, setting your rules, and testing with the Preview tool, the Upload document button syncs your Word document to your Docusign account.

- **Primary function:** It uploads the current DOCX file as a new or updated template in your Docusign account.
- **Automation:** It carries over the technical syntax and field mapping that you created in the assistant, so you don't have to manually re-create those fields in the Docusign web app.
