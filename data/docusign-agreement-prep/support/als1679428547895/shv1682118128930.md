# Error: Invalid Field Name

Learn how to resolve a document generation syntax error that occurs when a data field
name is not formatted correctly.

An invalid field name will cause an error when you upload a document generation .docx
file.

Sample error:

```
|  |  |
| --- | --- |
|  |  |
|  | {"tagIdentifier": "{{First Name}}", |
|  | "errorCode": "Undefined", |
|  | "message": "There was an error parsing this merge tag as json (check spacing, quotes, json syntax): Invalid character after parsing property name. Expected ':' but got: N. Path '', line 1, position 7."} |
|  |  |
```

Document generation data field names must not have any spaces.

## Ensure all data field names are formatted correctly

Template creator

1. Edit your .docx file to correct the syntax errors.
2. Locate the inserted data field name that is identified in the error
   message for improper formatting and correct as needed.

   Correct syntax: `{{First_Name}}`

   Incorrect syntax: `{{First Name}}`
3. Upload the corrected .docx file to your eSignature template.
