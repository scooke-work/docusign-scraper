# Error: Data Field Is Not Included on Data-Entry Form

Troubleshoot a merge error for an unrecognized field.

If a field is incorrectly formatted such that it is not recognized as a field, the syntax will
remain in the document as is. Since it is not a valid data field, it won't be
included on the data-entry form for the sender. No error will be returned because it
is not recognized as a field. It is up to the sender to notice the field is not
included in the data-entry form, and to spot the faulty syntax when reviewing the
final generated document.

Example: A data field is missing one curly bracket `{residential_address}}`.
The field is incorrectly formatted and unrecognizable, so it is ignored in the
data merge.

Sender and Template owner

1. Sender: Review the final generated document before sending and locate
   the invalid syntax.
2. Template owner: Open the source .docx file for the document generation
   template you are using. Correct the invalid syntax to conform to
   document generation syntax rules.
3. Template owner: Edit the template and replace the .docx document with
   the corrected file.
