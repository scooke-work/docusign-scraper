# Error: Invalid Conditional Statement

Troubleshoot a document generation syntax error that occurs when a conditional statement
is not formatted correctly.

A conditional statement with improper formatting will cause an error when you upload
a document generation .docx file.

`{{if statement}}` does not have a valid
`{{endif}}`. Example:

```
|  |  |
| --- | --- |
|  | "errorCode": "Undefined", |
|  | "message": "There was an error parsing this merge tag as json (check spacing, quotes, json syntax): Invalid character after parsing property name. Expected ':' but got: N. Path '', line 1, position 7."} |
```

Template creator

Ensure that there are no formatting
issues with the `{{endif}}`, such as a missing
bracket.

`{{if statement}}` is not valid. Common examples:

```
|  |  |
| --- | --- |
|  | {“tagIdentifier": "{\"type\":\"EndConditional\"}", |
|  | "errorCode": "EndTagWithoutStartTag", |
|  | "message": "End tag without matching start tag. -- Error: EndConditional without matching Conditional"} |
```

Template creator

Ensure that there are no formatting
issues with the `{{if statement}}` .
