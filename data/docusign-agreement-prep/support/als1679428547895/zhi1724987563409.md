# Dynamic Tables for Document Generation

Read about dynamic tables. Dynamic tables are designed for agreements that include
tables with a variable number of rows, depending on the data provided.

## What is a dynamic table?

Many agreements require tables to present data. Creating and formatting tables is
time-consuming and prone to manual errors. This is especially true when the number
of rows and information in a table changes to customize each agreement. With agreement preparation, you can create dynamic tables that adjust to fit
the data provided, adding as many rows as needed.

Data for the table is specified when a document is generated. For Docusign IAM customers, table data can come from a third-party system record (such as Salesforce) or from an uploaded CSV file. Docusign eSignature customers can upload a CSV file only.

Note: ATB also provides an option to insert **static tables**. Static tables are defined in the template with a fixed number of rows and columns. Static tables are useful when the number of rows and columns is predetermined and remains constant across different documents. This makes static tables suitable for standardized reports, fixed-format documents, or any scenario where the data structure is stable and predictable.

## Use cases

Here are some common use case examples for dynamic tables:

Purchase Orders and Invoices
:   The number of rows in the table varies depending on the number of items in
    the order.

Grant Applications
:   The number of rows in the table varies depending on the number of
    applications.

Project Status Report
:   The number of rows depends on the size of the project and what the
    milestones are.

## How do you create dynamic tables?

You can create dynamic tables in your templates with either the Agreement Template Builder or
the Docusign Template Assistant.

See the following help topics for detailed instructions:

- [Insert a Dynamic Table](https://support.docusign.com/s/document-item?bundleId=als1679428547895&topicId=kbb1723763558691.html&_LANG=enus "Learn how to add a dynamic table with the agreement template builder. Dynamic tables can be populated with as many rows as necessary when the agreement is generated.")
- [Create a Dynamic Table in the Template Assistant](https://support.docusign.com/s/document-item?bundleId=als1679428547895&topicId=nhd1712957935412.html&_LANG=enus "Learn how to create dynamic tables using the DocusignTemplate Assistant for Word, which automatically adjusts to the data provided.")

Note: For Docusign eSignature customers, sending documents with dynamic tables requires purchasing the
Document Generation for eSignature add-on. Users without the add-on can send up to
five trial envelopes that include Document Generation add-on features. This feature is included with Docusign IAM plans.

## Data accuracy

Dynamic tables offer improved data accuracy over building tables manually. You can
use API endpoints to transfer data from systems of record directly into agreements
to generate tables. Or if you are using the eSignature web application, you upload a
CSV file with the data to populate the table. Transmitting table data to your
agreement in either way reduces the potential for human error.

Note: For developer information on how to use the
API to transfer data from external systems of record, see [this API documentation](https://developers.docusign.com/docs/esign-rest-api/esign101/concepts/documents/document-generation/#dynamic-tables).

## Limitations

There are a few limitations to consider when building dynamic tables.

- You can define the data for dynamic tables with either Agreement Fields (standard and custom) or third-party fields (such as Salesforce fields). Data for the table can come from only one source.
- In ATB, tables can contain up to 10 columns from either Agreement Fields or third-party fields.
- In WTA, tables can contain up to 20 columns from either Agreement Fields or third-party fields.
