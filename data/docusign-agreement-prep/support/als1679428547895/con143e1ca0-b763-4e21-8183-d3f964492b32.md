# Agreement Type Detection

ATB uses AI to detect automatically the agreement type of the document you upload when building a
template. The agreement type classifies your document and surfaces relevant data fields.
You can apply these fields to your template to insert data placeholders for senders to
complete when generating agreements.

The agreement type is also stored with completed documents in Docusign Agreement Manager. Agreement Manager helps you unlock data usage and visibility. Your account administrator manages agreement types and fields in Agreement Manager and can remove and add new types.
Administrators with the necessary permissions can access these controls from Agreement Manager > Manage Agreement Types. For details, see [this help topic](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=ypm1727282359686.html&_LANG=enus).

## How it works

When you create a template with ATB and upload your document, the content is analyzed. There are several pre-defined agreement types which should cover the majority of agreements. You can select a type or use AI detection to choose which type to apply to your document. There's also a "Miscellaneous" type if you don't find a suitable match.

You have the option to change the agreement type selected by the AI detection. While the AI is designed to provide high-quality detection and classification, there may be instances where manual adjustments are beneficial. This flexibility ensures the most accurate classification of the document, tailored to your specific needs or preferences.

Agreement fields are the fields associated with the agreement type. You can use these dynamic
fields to customize generated agreements. Agreement fields are assigned to the
Sender role and act as placeholders for data that will be provided by the preparer.
You can add new custom fields to an agreement type. You can also select any custom
field created on the account by you or other account users. Adding or selecting
other custom fields creates an association with the current agreement type. This
association means the fields will be included by default the next time you use that
type.

Note: For a list of the standard agreement types provided by Docusign and
their associated fields, see [this help topic](https://support.docusign.com/s/document-item?bundleId=pqz1702943441912&topicId=smm1727282189861.html&_LANG=enus).
