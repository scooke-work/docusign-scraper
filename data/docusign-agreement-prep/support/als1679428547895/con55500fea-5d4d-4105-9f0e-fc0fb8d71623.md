# Use Data from Third-Party Systems

In this section, learn about how Docusign IAM customers can use data from third-party systems, such as Salesforce or Microsoft Dynamics, to generate dynamic content in generated documents.

## Install and connect a third-party account

Before you can build templates that use data from your third-party account, your account administrator must install and authorize the relevant extension app. Supported extension apps are available in the Docusign App Center.

Note: The availability of specific integrations depends on your Docusign IAM plan. For information about which third-party systems are supported for your account, contact your Docusign Sales representative.

For basic information on how to install apps, see the help topic [Install Apps in the Docusign App Center](https://support.docusign.com/s/document-item?topicId=ctg1698170340729.html&_LANG=enus).

For detailed information on how to install and connect your Docusign account with your Salesforce organization, see the help topic [Install the Salesforce Application From the Docusign App Center﻿](https://support.docusign.com/s/document-item?language=en_US&bundleId=qwr1706590641805&topicId=jxo1706295810553.html&_LANG=enus).

## Create templates with third-party sender fields

Once the necessary extension app is installed, you can create templates with data fields from your third-party system of record.

This involves first selecting one **source object** from your connected system (for example, a
Salesforce object). The source object will serve as the primary source of data for the
agreement. You can select one source object only per template.

Once the source object is selected, you can insert fields from that object directly into your template. These fields act as dynamic placeholders that are automatically populated with data from a specific record in your third-party system when an agreement is generated.

You can also include fields from **related objects** to pull in a wider range of necessary
data. These are objects that have a direct relationship to your chosen source object.

When agreements are generated from the template, they are customized with accurate and up-to-date information from your system of record.

## Generating documents with third-party data

While you can add third-party fields to a template using the Agreement Template Builder (ATB), generating the final document with live data requires an **orchestration layer** or an **embedded integration**.

Templates with third-party fields (for example, Salesforce) cannot be generated directly from the
Docusign web app. Because there is no manual "look up" for this data, you must use one of
the following:

- **Workflow Builder Integration:** Use Docusign Workflow Builder to build an automated process. You define the connection
  and use **workflow variables** to identify the source record. The workflow uses the
  connection to fetch data automatically during runtime.
- **Embedded Scenarios:** Use Docusign APIs to build a custom integration that programmatically passes record IDs and authentication tokens.

Note: If you attempt to generate an agreement directly from a template containing third-party fields, a warning banner will explain that the connection ID is missing. ## Limitations

- You can select one source object per document. (You can access additional source object fields by using related objects.)
- If you change the source object after inserting third-party fields, those fields will be removed from your document.
- Actual field values from your third-party account are not available in the ATB. You can preview
  your document with sample data that you enter manually. To pull in actual values, you
  must generate an agreement.
- Documents with third-party fields require a supported data connection (like Workflow Builder or API). Attempting to use these templates directly in the Docusign web app will trigger an error banner.
