# Error: Value Provided Does Not Pass Validation

Troubleshoot a merge error for a validation issue.

Currently, there is no upfront validation for document generation; senders can enter any value
for a data field. If a value does not meet the expected validation for a field, it
will result in a merge error.

Note: In the future, we plan to build error handling into the
form-filling experience so the sender is prevented from proceeding if there are
issues with the data provided.

Example: A document has a conditional statement that expects a number `{{if
$number(count) = 30}}`. The sender enters a non-numeric value for
the data field "count", resulting in a merge error.

Sender

Review the document preview and
locate the highlighted merge error. These errors display in red text with a
yellow highlight. If needed, download the .JSON error file for more information
about the merge errors. Navigate back to the data-entry form and correct the
values that triggered the error.
