# PHP SDK

The PHP eSignature SDK is provided by Docusign to help PHP developers use the [Docusign eSignature REST API](https://developers.docusign.com/docs/esign-rest-api/). It is provided as both a [Composer package](https://packagist.org/packages/docusign/esign-client/) and an [open source repo on GitHub](https://github.com/docusign/docusign-php-client). The PHP eSignature SDK is also used when developers opt to get started with the PHP [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/), which generates ready-to-use, personalized PHP code.

## Compatibility

- The PHP SDK package is compatible with PHP 7.4 and later.

## SDK versions

New versions of the SDK are released periodically to use the latest version of the [eSignature REST API](https://github.com/docusign/OpenAPI-Specifications/blob/master/esignature.rest.swagger-v2.1.json) . SDK versions use the standard period-separated [semantic versioning](https://semver.org/) scheme. General release versions are of the form x.y.z, and release candidate (prerelease) versions are of the form x.y.z-rc.

The [SDK’s changelog file](https://github.com/docusign/docusign-php-client/blob/master/CHANGELOG.md) lists each SDK update and its notable changes from the prior version.

Developers are not required to update their applications to new versions of the SDK unless there’s a specific reason to do so:

- Access to a new eSignature API feature or method
- An SDK bug fix including security-related issues
- To support an updated version of one of the SDK's dependencies

## SDK Issues

If you find any issues with the SDKs, you can open an issue on the [GitHub repo](https://github.com/docusign/docusign-php-client/issues) or reach out to [Support](https://support.docusign.com/s/contactSupport).
