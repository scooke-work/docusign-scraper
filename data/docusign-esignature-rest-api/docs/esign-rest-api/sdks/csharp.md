# C# SDK

The C# eSignature SDK is provided by Docusign to help C# developers use the [Docusign eSignature REST API](https://developers.docusign.com/docs/esign-rest-api/). It is provided as both a [NuGet package](https://www.nuget.org/packages/DocuSign.eSign.dll) and an [open source repo on GitHub](https://github.com/docusign/docusign-esign-csharp-client). The C# eSignature SDK is also used when developers opt to get started with the C# [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/), which generates ready-to-use, personalized C# code.

## Compatibility

- The C# eSignature SDK is compatible with .NET Framework 4.6.2 and later.
- The C# eSignature SDK is also compatible with .NET Standard 2.0 and later.
- The .NET package restore process will automatically include the correct SDK DLLs for the target .NET version your project uses.
- The C# SDK package is dependent on other NuGet packages (DLLs) for both .NET Core and the .NET Framework. The list of dependencies can be found on the [DocuSign.eSign.dll NuGet](https://www.nuget.org/packages/DocuSign.eSign.dll) page. Next to each dependency is the minimum version required for that package.

## SDK versions

New versions of the SDK are released periodically to use the latest version of the eSignature REST API. SDK versions use the standard period-separated [semantic versioning](https://learn.microsoft.com/en-us/nuget/concepts/package-versioning?tabs=semver20sort) scheme. General release versions are of the form x.y.z, and release candidate (prerelease) versions are of the form x.y.z-rc.

All SDK versions prior to 4.0.0 are targeted at the older [eSignature REST API v2.0](https://developers.docusign.com/docs/esign-rest-api/v2/reference/).

The [SDK changelog](https://github.com/docusign/docusign-esign-csharp-client/blob/master/CHANGELOG.md) lists each SDK update and its notable changes from the prior version.

Developers are not required to update their applications to new versions of the SDK, unless there’s a specific reason to do so:

- Access to a new eSignature API feature or method.
- An SDK bug fix including security-related issues.
- To support an updated version of one of the SDK's dependencies.

## SDK issues

If you find any issues with the SDKs, you can open an issue on the [GitHub repo](https://github.com/docusign/docusign-esign-csharp-client/issues), or reach out to [Support](https://support.docusign.com).
