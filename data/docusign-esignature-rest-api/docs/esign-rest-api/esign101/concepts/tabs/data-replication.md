# Data replication

You can use *data replication* to share a single value between multiple tabs. This is useful for speeding up the signing process for your customers and reducing potential mistakes or typos.To use data replication, each tab to share a value must have the same type and `tabLabel` value. Any update to one tab value will also be automatically applied to all other tabs with the same type and `tabLabel`.

To use data replication in operations involving the eSignature API, the `tabLabel` value must not contain any spaces.

For example, you could create an envelope with a two-page document and add the following tabs for a given recipient:

```
"tabs": {
    "textTabs": [{
        "tabLabel": "SharedDataField",
        ...
        "documentId": "1",
        "pageNumber": "1"
    },
    {
        "tabLabel": "SharedDataField",
         ...
        "documentId": "1",
        "pageNumber": "2"
    }]
}
```

Even though there are two individual tabs, each added to a different page of the document, because both of these tabs share the same `tabLabel`, both will be updated simultaneously when the recipient enters text into either one of them.

**Note:** Read-only fields cannot be populated by data replication.

## Supported tab types

Not every tab type supports data replication. Currently, you can only use data replication to share tab values for following tab types:

- Check box
- Company
- Text
- Drop Down
- Name
- Note
- Title

## Document or envelope scope

By default, data replication can share values between any tabs within a single document. You can customize your account settings to expand the scope of data replication to all tabs within the documents contained in an envelope. To set the scope for data replication:

1. Log in to your Docusign account
2. To open eSignature Admin, select **Admin**
3. Under Signing and Sending, select **Sending Settings**
4. Choose a value for the **Automatically replicate information in fields with the same Data Label setting**
