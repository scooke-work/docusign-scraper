# How to send an envelope via your app

eSignature REST API 2.1 only

This topic demonstrates how to embed the Docusign Sender UI within your web app. For the best user experience, Docusign recommends using the API to construct draft envelopes and add recipients and tabs to those envelopes. After envelopes have been created, the Sender View enables your app’s users to edit the tabs, documents, recipients, and other envelope settings within your app before sending them out for approval. **Important**: This example requires that you have created an envelope with `created` status or implemented the code from the [embedded signing how-to](https://developers.docusign.com/docs/esign-rest-api/how-to/request-signature-in-app-embedded/), to generate a new envelope with `created` status.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {CC\_RECIPIENT\_EMAIL} | A string value for the email address of the carbon copy recipient. |
| {CC\_RECIPIENT\_NAME} | A string value for the name of the carbon copy recipient. |
| Return URL | A URL to which the user will be forwarded after signing is complete. |
| {SIGNER\_EMAIL} | A string value for the email address where the signer will receive a notification of the signing request. |
| {SIGNER\_NAME} | A string value for the full name of the signer. |
| {STARTING\_VIEW} | The starting view value determines where the web app will open. You can set the `starting_view` to a specific envelope ID to open the web app to view that envelope or with the embedded console, you can choose to route a user to a specific envelope ID, if available, or to the front page of Docusign eSignature by default. |

## Step 1. Obtain your OAuth token

To make the API calls shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

## Step 2. Create the envelope

Use the following steps to create the envelope:

- Instantiate the Docusign API client.
- Set the following default request header to add your access token.
  `"Authorization" : "Bearer {ACCESS_TOKEN}"`
- Initialize the Envelopes API.
- Create a new draft envelope and set the `status` property to `created`.

> **Important:** This how-to draws on the code defined in the embedded signing how-to (which is part of the same code example launcher) to generate a new envelope with `created` status.

## Step 3. Create the sender view

To create the sender view, first create an `EnvelopeViewRequest` object. This object contains the return URL to which the sender will be directed after they have sent the envelope. The `viewAccess` property is also required for this object and should be set to `"envelope"`. The `EnvelopeSettingsView` object contains UX settings for the sender view. See [EnvelopeViews:createSender](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/envelopeviews/createsender/) for details on all of these settings.

Next, call the [EnvelopeViews:createSender](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/envelopeviews/createsender/) method with your API account ID, the ID of the draft envelope, and the `EnvelopeViewRequest` object. The code will return the URL for the embedded sending session at your requested starting view.
