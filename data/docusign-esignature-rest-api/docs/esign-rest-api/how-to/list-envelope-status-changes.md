# How to list envelope status changes

This topic demonstrates how to get a set of status history data for all of your envelopes updated within a specified range of dates.

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {FROM\_DATE} | A filter for the query in the format YYYY-MM-DD. Only envelopes whose status has changed beginning with the date specified by `from_date` will be returned. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

## Step 2. List envelope status

Call the `envelopes.listStatusChanges` method to get a list of envelopes and their statuses during a specified range of time. To get the list, you must specify either a from date or provide a list of envelope IDs or transaction IDs in the request. Otherwise, an error will be returned. This example uses the from date to get an array of envelopes created within the last 30 days. Use the following steps to make this method call and obtain the list of envelopes and statuses:

1. Instantiate the Docusign API client.
2. Set the following default request header to add your access token.
   `"Authorization" : "Bearer {ACCESS_TOKEN}"`
3. Initialize the Envelopes API.
4. Create a filter that specifies the set of envelope statuses that you want to return. Your request must include one or more of the following filter parameters: `from_date`, `envelope_ids`, or `transaction_ids`. Each of these parameters can accept a comma-delimited list.

   The example documented here uses the `from_date` filter, which returns all envelope status changes made since the specified date. Docusign recommends using ISO 8601 format dates with an explicit time zone offset.

   **Note:**The `status` and `from_to_status` options are very useful for search filtering. The former enables filtering by way of a comma-delimited list of statuses, whereas the latter will enable you to filter by a specific status for a given `from_date` and `to_date` time range. You can learn about more optional query filtering parameters by consulting [Envelopes:listStatusChanges](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/envelopes/liststatuschanges/) in the API Reference.
5. After calling the `envelopes.listStatusChanges` method to obtain the envelope status data, you will receive both data on the status of each document within the envelope (`sent` vs `draft`, for instance) and status change dates as well as resource locators such as the documents combined URI, the attachments URI, and the recipients URI.

View the source code for this example on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/eSignature/eg003ListEnvelopes.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/eSignature/Examples/ListAccountEnvelopes.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/eSignature/examples/EG003ControllerListEnvelopes.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/eSignature/examples/listEnvelopes.js), [PHP](https://github.com/docusign/code-examples-php/blob/master/src/Services/Examples/eSignature/ListEnvelopesService.php), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/eSignature/eg003ListEnvelopes.ps1), [Python](https://github.com/docusign/code-examples-python/blob/master/app/eSignature/examples/eg003_list_envelopes.py), or [Ruby](https://github.com/docusign/code-examples-ruby/blob/main/app/services/e_sign/eg003_list_envelopes_service.rb).
