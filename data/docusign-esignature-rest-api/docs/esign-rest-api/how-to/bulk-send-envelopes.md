# How to bulk send envelopes

Expanded by eSignature REST API 2.1

This topic demonstrates how to send envelopes in bulk to multiple recipients with the eSignature API. To accomplish the task, you will first create a bulk send recipients list, then create an envelope. From there, you will combine the envelope and bulk list to initiate bulk send.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {BULK\_LIST\_ID} | A [GUID](http://guid.one/) value that identifies the bulk list whose data will be retrieved. This value is automatically generated by Docusign when the bulk list is created. |
| Bulk sending enabled for your account | Your user role needs the **bulkSendingEnabled** permission set to **true** in order to send bulk envelopes. To enable this feature, contact your Account Manager or Business Development representative. |
| Document to be signed | You must provide at least one document to be signed by the signers of this bulk send operation. For this example, you will be sending a base-64 encoded string representing an NDA form. |
| Document Visibility features set to **Off** | This how-to requires that [Document Visibility](https://support.docusign.com/s/document-item?language=en_US&bundleId=gbo1643332197980&topicId=tmi1578456411160.html&_LANG=enus) be disabled. Disable Document Visibility in the [Sending Settings](https://admindemo.docusign.com/authenticate?goTo=sending) page. Select the Document Visibility drop-down menu to change it to **Off,** and then click **Save.** |
| {ENVELOPE\_ID} | A [GUID](http://guid.one/) value that identifies a Docusign [envelope](https://developers.docusign.com/docs/esign-rest-api/esign101/concepts/envelopes/). This value is automatically generated by Docusign when the envelope is created. |

## Step 1. Obtain your OAuth token

To make the API calls shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

## Step 3. Submit a bulk list

Your bulk send recipients list begins as a file you create in comma-separated values (CSV) format, holding information for each recipient. To prepare the list for bulk sending, you need to create a JSON representation of your CSV data and add it to your request body before sending it to the API as follows:

1. Create a request body specifying your emails, names, and role names (`roleName`) for each recipient.
2. With your constructed API headers, send the current request body to the eSignature REST API, but make sure to substitute your API account ID for {API\_ACCOUNT\_ID}.
3. Extract the `listId` from the response as the {BULK\_LIST\_ID}. You will use it again in future steps.

## Step 7. Confirm successful batch send

Now that your bulk send operation has been initiated, check the progress of the operation by sending a request to the eSignature REST API, furnishing the `batchId` that was generated during the previous step. The sample code as configured here will wait for 10 seconds to give the API enough time to process the bulk envelopes from step 6.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/eSignature/eg031BulkSending.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/eSignature/Examples/SendBulkEnvelopes.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/eSignature/examples/EG031ControllerBulkSendEnvelopes.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/eSignature/examples/bulkSendEnvelope.js), [PHP](https://github.com/docusign/code-examples-php/blob/master/src/Services/Examples/eSignature/BulkSendEnvelopesService.php), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/eSignature/eg031BulkSending.ps1), [Python](https://github.com/docusign/code-examples-python/blob/master/app/eSignature/examples/eg031_bulk_send.py), or [Ruby](https://github.com/docusign/code-examples-ruby/blob/main/app/services/e_sign/eg031_bulk_sending_envelopes_service.rb).
