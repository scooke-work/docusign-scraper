# How to send a request for payment

This topic demonstrates how to use [Docusign Payments](https://www.docusign.com/products/payments) to collect payments by envelope. It walks you through the process of configuring a payment gateway that will accept and process the payments from signers. Then it shows you how to collect online credit card payments through an order form that you send in an envelope. You can test this example using one of the [test credit cards](https://stripe.com/docs/testing#cards) provided by Stripe.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. | {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {CC\_RECIPIENT\_EMAIL} | A string value for the email address of the carbon copy recipient. |
| {CC\_RECIPIENT\_NAME} | A string value for the name of the carbon copy recipient. |
| {PAYMENT\_GATEWAY\_ACCOUNT\_ID} | Your payment gateway account ID. |
| {PAYMENT\_GATEWAY\_DISPLAY\_NAME} | The display name for your payment gateway. |
| {PAYMENT\_GATEWAY\_NAME} | The name of your payment gateway. |
| Redirect URI | A URL to which the user will be forwarded after authentication. The value for this URL must exactly match one of those pre-registered for the [integration key](https://developers.docusign.com/platform/configure-app/#integration-key) in your Docusign account. | {SIGNER\_EMAIL} | A string value for the email address where the signer will receive a notification of the signing request. |
| {SIGNER\_NAME} | A string value for the full name of the signer. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

## Step 2. Configure a payment gateway

Before you can use Docusign Payments, an account administrator (for this example, you, the developer) must set up one or more payment gateways for your account by following the steps in [Managing Payment Gateways](https://support.docusign.com/s/document-item?language=en_US&rsc_301&bundleId=juu1573854950452&topicId=knc1573854895499.html&).

Docusign offers built-in connections to multiple payment gateways. This example uses a demo account and the Stripe payment gateway service.

Follow these steps:

1. Select the Stripe button on the [Payments](https://admindemo.docusign.com/authenticate?goTo=payments) page.
2. For development, you can skip the Stripe account application by using the **Skip this account form** link at the top of the page. An enabled Stripe payment gateway account is now associated with your developer account and is shown under **Payment Gateway**.
3. Configure your code with the **Gateway Account ID** that displays in Settings. For this example, you will need to include the gateway account ID, gateway name, and gateway display name in the envelope arguments.

## Step 3. Create the envelope definition

Next, you create a new envelope definition. The envelope in this example uses an HTML document that contains an order form, formula fields, and multiple tags. The example also sets constants for items related to the order form. (In your integrations, you would pull these items from a product database.) Define a `makeEnvelope` method to create the envelope definition:

1. Read the HTML file for the order form. This example pulls from the `order_form.html` file in the source repository’s `documents` folder.
2. In the form, replace the placeholders for `signerName`, `signerEmail`, `ccName`, and `ccEmail` with the actual values.
3. Create the envelope definition.
4. Add the file containing the order form to the envelope as `doc1`. Create the document as a base64-encoded variable, then use that to create the document object.
5. Create a signer recipient using the signer’s name and email address. The example gives the signer a `signingOrder` of `1` so that they’re the first recipient to receive it, ensuring that the CC recipient receives a signed copy.
6. Create a CC recipient using the CC user’s name and email. In the example, the CC recipient has a `routingOrder` of `2`.
7. Create tabs for signature and quantities. The tabs in this example use [anchor positioning](https://developers.docusign.com/docs/esign-rest-api/esign101/concepts/tabs/auto-place/).
   1. The first tab is a signHere tab anchored to `/sn1/` in the `order_form.html` file.
   2. The next two tabs are list tabs, both of which use the same list item definitions. They are anchored to `/l1q/` and `/l2q/` in the order form.
8. Create tabs for amounts. These are formula tabs anchored to `/l1e/`, `/l2e/` and `/l3t/` in the order form. The first two calculate values based on the values from the list tabs, multiplied by the item prices. The third calculates the total based on the first two.
9. Create objects for Docusign Payments. These include `PaymentLineItem` objects for each line item, plus a `PaymentDetails` object for capturing the transaction details and gateway information.
10. Create a formula tab for the payment itself. This tab is hidden (note that `x_position` and `y_position` are both `0`).
11. Associate the signer’s tabs with the signer recipient object.
12. Add the recipients to the envelope definition.
13. Set the envelope’s `status`. By default, the `status` is set to `sent`, which indicates that the system will immediately send the envelope. To create the envelope as a draft, set the `status` to `created` instead.

## Step 4. Call the eSignature REST API

Next, create the envelope from the definition that you created in the previous step.

1. Instantiate the Docusign API client.
2. Set the following default request header to add your access token.
   `"Authorization" : "Bearer {ACCESS_TOKEN}"`
3. Initialize the eSignature REST API.
4. Create an envelope by passing in the generated envelope definition and your API account ID.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/eSignature/eg014CollectPayment.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/eSignature/Examples/CollectPaymentInEnvelope.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/eSignature/services/CollectPaymentService.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/eSignature/examples/collectPayment.js), [PHP](https://github.com/docusign/code-examples-php/blob/master/src/Services/Examples/eSignature/CollectPaymentService.php), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/eSignature/eg014CollectPayment.ps1), [Python](https://github.com/docusign/code-examples-python/blob/master/app/eSignature/examples/eg014_collect_payment.py), or [Ruby](https://github.com/docusign/code-examples-ruby/blob/main/app/services/e_sign/eg014_collect_payment_service.rb).
