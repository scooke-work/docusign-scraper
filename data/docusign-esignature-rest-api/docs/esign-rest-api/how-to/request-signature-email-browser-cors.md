# How to request a signature by email using CORS

This topic demonstrates how to send a signature request via an email using JavaScript through your [Cross-Origin Resource Sharing](https://developer.mozilla.org/en-US/docs/Web/HTTP/CORS/) (CORS) app within your developer account.

CORS is a technology standard that allows restricted resources such as scripts, forms, or external web pages to be requested from a domain that's different from the server originating the request. Access to each resource requested via CORS is controlled by CORS access policies set on the server hosting that resource. 

CORS enables you to build browser-based applications that can directly call the Docusign eSignature REST API. Your app can use React, Vue, or other browser-based frameworks, or just JavaScript. With CORS, no back-end server is needed.

## Prerequisites

To send CORS requests:

- Since the sender needs Docusign credentials to call the eSignature REST API, the sender needs their own login to the Docusign service. If the sender will also be signing the envelope, or if the signer is physically present with the sender, then this example is applicable.

  If the signer does not have a Docusign login, then a back-end service (using JWT authentication) should create an access token and use it to call the eSignature REST API.
- The system administrator for the application’s integration key (client ID) must enable the application to use CORS from the origin(s) the application will be loaded from. For example, if the application’s URL is `https://apps.mycompany.com/docusign`, the origin `https://apps.mycompany.com` must be enabled for CORS. See [Enable or disable CORS for specific apps](https://developers.docusign.com/platform/single-page-applications-cors/enable-disable-cors/#update-app-specific-cors-settings) for details.
- The account administrator for every user of your application must also enable CORS access by your application. While the default is to enable CORS access, your users’ administrators can change the default. For instance, suppose your application has 20 users scattered across four companies and five individual eSignature accounts (one of the companies has multiple accounts). In this scenario, five different account administrators must enable CORS access by your application or leave the default setting to **CORS enabled by default for all new apps**. See [Requirements for an app to use CORS](https://developers.docusign.com/platform/single-page-applications-cors/#requirements-for-an-app-to-use-cors) for details.

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| Documents to be signed | You must provide at least one document to be signed. Document used in this how-to:  - [PDF file](https://docusign.github.io/examples/docs/anchorfields.pdf) |
| Redirect URI | A URL to which the user will be forwarded after authentication. The value for this URL must exactly match one of those pre-registered for the [integration key](https://developers.docusign.com/platform/configure-app/#integration-key) in your Docusign account. See [redirect URI](https://developers.docusign.com/platform/configure-app/#redirect-uri) for details. |
| {SIGNER\_EMAIL} | A string value for the email address where the signer will receive a notification of the signing request. |
| {SIGNER\_NAME} | A string value for the full name of the signer. |

## Step 1. Obtain your OAuth token

To make the API calls shown in this how-to, you need a valid OAuth access token. CORS-enabled apps should use OAuth Authorization Code Grant, without a secret, with [Proof Key for Code Exchange (PKCE)](https://developers.docusign.com/platform/auth/authcode/#public-authorization-code-grant) to obtain an access token. A special `cors` [scope](https://developers.docusign.com/platform/auth/reference/scopes/) is required along with `signature` when generating your OAuth token request.
