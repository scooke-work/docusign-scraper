# How to request a signature by SMS or WhatsApp delivery

eSignature REST API 2.1 only

This topic demonstrates how to send a signature request via an [SMS or WhatsApp message](https://developers.docusign.com/docs/esign-rest-api/esign101/concepts/sms-delivery/). Recipients receive a notification from SMS or WhatsApp with a signing link that they can use to open and electronically sign a document using the Docusign mobile app or the Docusign website.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

Running the code in this how-to requires this data:

| {CC\_PHONE\_NUMBER} | The phone number of the carbon copy recipient to whom the SMS or WhatsApp notification will be sent. |
| {CC\_RECIPIENT\_NAME} | A string value for the name of the carbon copy recipient. |
| {COUNTRY\_CODE} | The country code corresponding to the PHONE\_NUMBER. |
| {DELIVERY\_METHOD} | The delivery method specified for a given recipient. One of:`Email`, `fax`, `SMS`, `WhatsApp`, or `offline`The SMS and WhatsApp delivery methods are limited to signer, carbonCopy, and certifiedDelivery recipients. |
| Documents to be signed | You must provide at least one document to be signed. Documents used in this how-to:  - [DOCX file](https://github.com/docusign/code-examples-java/blob/master/src/main/resources/World_Wide_Corp_Battle_Plan_Trafalgar.docx) - [PDF file](https://github.com/docusign/code-examples-java/blob/master/src/main/resources/World_Wide_Corp_lorem.pdf) - A third document in HTML is constructed dynamically by the sample code |

| {SIGNER\_NAME} | A string value for the full name of the signer. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

## Step 2. Create the envelope definition

To request a signature, you need to create a new envelope definition. The accompanying code shows how to create an individual envelope by defining a method named `makeEnvelope` that creates the envelope definition. The `makeEnvelope` method:

- Creates and attaches multiple documents to the envelope.
- Sets signer and CC recipient data.
- Creates a routing order for the documents.
- Creates and assigns signable fields to specific signers within the documents.

To send an SMS or WhatsApp notification, you need to edit the signer's `phnoneNumber` field to include values for `number` and `countryCode`. You also have to set the `deliveryMethod` field to either "`SMS"` or `"WhatsApp"`. You should make these changes for each recipient you want to notify via SMS or WhatsApp. The accompanying code also demonstrates how to notify a carbon copy recipient via SMS or WhatsApp by editing the same fields in the `carbonCopy` object.

## Step 3. Create and send the envelope

With the envelope definition and the attributes defined for the signer, document, and custom fields, you can now use the [Envelopes:create](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/envelopes/create/) API method to create an envelope
 and send it, adding your access token and base path to the request header.

## Expected response

If you run this code example from our Quickstart or launcher projects, you should see a JSON response similar to the code shown.

In addition, the signer receives an SMS or WhatsApp message containing a signing link that they can use to open and electronically sign a document using the Docusign mobile app or the Docusign website. After signing is complete, the recipient receives a carbon copy (CC) of the email that includes all of the completed and signed documents.
