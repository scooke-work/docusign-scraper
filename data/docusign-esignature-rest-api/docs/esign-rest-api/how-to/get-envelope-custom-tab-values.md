# How to get envelope custom tab values

This topic demonstrates how to retrieve the values of custom tabs (also known as *tags* or *fields*) in an envelope using the Docusign eSignature REST API.

Run this code example from our [Quickstart](https://developers.docusign.com/docs/esign-rest-api/quickstart/) or launcher projects: [Bash](https://github.com/docusign/code-examples-bash), [C#](https://github.com/docusign/code-examples-csharp), [Java](https://github.com/docusign/code-examples-java), [Node.js](https://github.com/docusign/code-examples-node), [PHP](https://github.com/docusign/code-examples-php), [PowerShell](https://github.com/docusign/code-examples-powershell), [Python](https://github.com/docusign/code-examples-python), or [Ruby](https://github.com/docusign/code-examples-ruby).

## Required data

Running the code in this how-to requires this data:

| Data element | Description |
| --- | --- |
| {API\_ACCOUNT\_ID} | A [GUID](http://guid.one/) value that identifies your account. This value is automatically generated by Docusign for any account you create. Copy the value from the **API Account ID** field on the [Apps and Keys](https://admindemo.docusign.com/authenticate?goTo=appsAndKeys) page. |
| {BASE\_PATH} | A string value that forms the root of the URL required to make API calls. The base path value in the example code is set to target the developer environment at  https://<domain>/restapi  Where <domain> is demo.docusign.net for the developer environment and is the result of the GetUserInfo call in production. |
| {ENVELOPE\_ID} | A [GUID](http://guid.one/) value that identifies a Docusign [envelope](https://developers.docusign.com/docs/esign-rest-api/esign101/concepts/envelopes/). This value is automatically generated by Docusign when the envelope is created. |

## Step 1. Obtain your OAuth token

To make the API call shown in this how-to, you need a valid OAuth access token. Docusign supports access tokens for three different recommended OAuth grant types: Public Authorization Code Grant, Confidential Authorization Code Grant, and JSON Web Token (JWT) Grant. Choose the OAuth grant type appropriate for your scenario.

See [Confidential Authorization Code Grant](https://developers.docusign.com/platform/auth/authcode/#confidential-authorization-code-grant) for guidance on your options to get an Confidential Authorization Code Grant OAuth token.

## Step 2. Construct your API headers

After you have a valid OAuth access token, you are ready to construct the API headers that you will pass to the API call. Construct your headers like this, but make sure to substitute your valid OAuth access token for {ACCESS\_TOKEN}.

declare -a Headers=('--header' "Authorization:

Bearer ${ACCESS\_TOKEN}" \

'--header' "Accept: application/

json" \

'--header' "Content-Type:

application/json")

## Step 3. Call the eSignature REST API

Now that you have constructed your API headers, make a request to the eSignature REST API [EnvelopeCustomFields:list](https://developers.docusign.com/docs/esign-rest-api/reference/envelopes/envelopecustomfields/list/) endpoint to retrieve envelope tab data. Here’s how you make the call, but make sure to substitute your API account ID for {ACCOUNT\_ID} and your envelope ID for {ENVELOPE\_ID}.

View the source code for this how-to on GitHub: [Bash](https://github.com/docusign/code-examples-bash/blob/master/examples/eSignature/eg018EnvelopeCustomFieldData.sh), [C#](https://github.com/docusign/code-examples-csharp/blob/master/launcher-csharp/eSignature/Examples/GetCustomFields.cs), [Java](https://github.com/docusign/code-examples-java/blob/master/src/main/java/com/docusign/controller/eSignature/examples/EG018ControllerEnvelopeCustomFieldValues.java), [Node.js](https://github.com/docusign/code-examples-node/blob/master/lib/eSignature/examples/envelopeCustomFieldData.js), [PHP](https://github.com/docusign/code-examples-php/blob/master/src/Services/Examples/eSignature/EnvelopeCustomFieldDataService.php), [PowerShell](https://github.com/docusign/code-examples-powershell/blob/master/examples/eSignature/eg018EnvelopeCustomFieldData.ps1), [Python](https://github.com/docusign/code-examples-python/blob/master/app/eSignature/examples/eg018_envelope_custom_field_data.py), or [Ruby](https://github.com/docusign/code-examples-ruby/blob/main/app/services/e_sign/eg018_get_envelope_custom_field_data_service.rb).
