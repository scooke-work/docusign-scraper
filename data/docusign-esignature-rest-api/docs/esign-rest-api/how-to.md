# How-to guides overview

The how-to guides in this section demonstrate common [Docusign eSignature functionality](https://developers.docusign.com/docs/esign-rest-api/esign101/) and how you can implement it into your own apps.

All API how-to guides assume you are developing with a Docusign developer account. A developer account is not specific to the [eSignature REST API](https://developers.docusign.com/docs/esign-rest-api/). If you have a developer account that you are using with our eSignature REST API, it will also work with other [Docusign APIs](https://developers.docusign.com/docs/). If you need to create a free developer account, select **Developer Account** from the menu at the top-right of the page, then **Create Account** or use the button below.

[Create account](https://www.docusign.com/developers/sandbox)

The eSignature REST API how-to categories and guides are listed below:

- [Requesting a signature](https://developers.docusign.com/docs/esign-rest-api/how-to/#requesting-a-signature)
- [Working with envelopes and templates](https://developers.docusign.com/docs/esign-rest-api/how-to/#working-with-envelopes-and-templates)
- [Working with advanced recipient routing](https://developers.docusign.com/docs/esign-rest-api/how-to/#working-with-advanced-recipient-routing)
- [Working with documents](https://developers.docusign.com/docs/esign-rest-api/how-to/#working-with-documents)
- [Working with tabs](https://developers.docusign.com/docs/esign-rest-api/how-to/#working-with-tabs)
- [Working with brands](https://developers.docusign.com/docs/esign-rest-api/how-to/#working-with-brands)
- [Working with permissions](https://developers.docusign.com/docs/esign-rest-api/how-to/#working-with-permissions)
- [Implementing multi-factor recipient (signer) authentication](https://developers.docusign.com/docs/esign-rest-api/how-to/#implementing-multi-factor-recipient-(signer)-authentication)
