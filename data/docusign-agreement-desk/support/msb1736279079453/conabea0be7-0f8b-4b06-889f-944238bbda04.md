# Comments in Request Documents

Learn about the Agreement Desk commenting features that let users discuss specific terms, request input from internal stakeholders, and optionally create approvals directly from a comment.

Use Comments in request documents to highlight text to start a discussion,
instantly pulling in collaborators who get immediate access directly in the thread.

This feature centralizes internal discussions directly in the draft to eliminate messy email chains and ensure total stakeholder alignment all in one place.

With Agreement Desk request comments, you can:

- Use comments on documents to highlight text to start a discussion,
  instantly pulling in collaborators who get immediate access directly in the
  thread.
- Use the enhanced comments sidebar to:

  - Always access document comments from the sidebar, even when no
    comments exist yet.
  - Quickly filter comments to focus on open items or resolved
    threads.
  - See timestamps that show when each comment was added or
    updated.
  - Reply to, resolve, and reopen comments directly from the
    sidebar without scrolling through the document.
  - Share direct links to specific comments so collaborators can
    jump straight to the exact place in the document.
- Manage comments: Add, edit, delete, respond to, and resolve to comments in an Agreement Desk request document.

## Use cases

Here are some sample use case for Comments in documents.

Legal
:   Turn complex negotiations into quick resolutions by using @mention for stakeholders directly on the relevant clause.

Sales and Procurement
:   Provide the sales teams with instant, in-context negotiation boundaries by flagging difficult terms.

## How comments in request documents work

Mentions and permissions
:   - Using @mention for a user who does not already have access to the request,immediately adds that user to the permissions list.
    - Users who are @mentioned receive an email notification with basic request and comment details, and an in-document link to the comment.
